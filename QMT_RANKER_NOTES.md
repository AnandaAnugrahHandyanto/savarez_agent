# QMT 候选池读取与打分脚本

## 文件
- `qmt_candidate_ranker.py`
- `qmt_export_candidates_vm_patch.py`

## 新口径
当前候选池已切换为更贴近主人的正式策略：
1. 主板非ST全量
2. 竞价硬筛
3. 动态题材簇优先
4. 龙头优先
5. 次日溢价导向排序

## 候选池入口规则
### 全量范围
- 仅保留主板：`600/601/603/605/000/001/002`
- 排除：`300/301/688/689/8/4`
- 排除名称中含 `ST` / `*ST`

### 竞价硬筛
至少满足：
- 成交额 >= 5亿
- 竞价挂单金额 >= 10万（当前为近似口径，基于 bid1/ask1 与挂单量估算）
- 高开不低于 -1.5%
- 高开不高于 6.5%
- bid_ask_ratio 不低于 0.8
- 不接受一字顶死
- 不接受交易题材不清晰（泛市场/未知题材）
- 对明显权重白马且无法归入交易题材者，直接过滤

## 动态题材簇
当前已新增“动态题材簇强度”近似指标，综合：
- 题材内涨停家数
- 最高板
- 候选池成交额
- 个股涨幅/高开/承接/板数等代理强度

说明：
- 这仍不是完整盘中联动引擎
- 但已优于纯静态行业/概念标签

## 最强票三层口径
- `盘中最强`：当前模型视角下的综合最强样本
- `题材最强`：当前动态题材簇中的最强核心
- `可执行最强`：当前策略边界下最接近可交易的最强票

## 排序逻辑
硬筛通过后，再做：
- 动态题材簇识别
- 题材内龙头/前排排序
- 环境判断（强势/分歧/退潮）
- 次日溢价导向打分
- 输出 `primary / backup / observe / avoid`

## 说明
- 当前“候选池题材聚合”只是候选池样本聚合，不是全市场成交额。
- 当前分歧日口径已收紧：低开票默认不进备选，避免把弱势竞价误判成可执行机会。
- 当前题材识别已引入动态题材簇代理强度，但仍依赖现有静态样本字段；若要进一步逼近真实盘中主线，需要新增更细粒度盘中联动数据。
