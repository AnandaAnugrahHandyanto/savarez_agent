# Генерация изображений

Hermes Agent генерирует изображения из текстовых подсказок через FAL.ai. Поддерживается одиннадцать моделей «из коробки», каждая со своими компромиссами скорости, качества и стоимости. Активную модель можно настроить через `hermes tools`, она сохраняется в `config.yaml`.

|---|---|---|---|

## Настройка

:::tip Nous Subscribers
Если у тебя есть платная подписка на [Nous Portal](https://portal.nousresearch.com), ты можешь использовать генерацию изображений через **[Tool Gateway](tool-gateway.md)** без ключа API FAL. Выбор модели сохраняется для обоих путей. Новые установки могут выполнить `hermes setup --portal` для входа и одновременного включения всех инструментов шлюза; существующие установки могут выбрать **Nous Subscription** в качестве бэкенда генерации изображений через `hermes tools`.

Если управляемый шлюз возвращает `HTTP 4xx` для конкретной модели, значит эта модель ещё не проксируется со стороны портала — агент сообщит об этом и предложит шаги исправления (установить `FAL_KEY` для прямого доступа или выбрать другую модель).
:::

### Получить ключ API FAL

1. Зарегистрируйся на [fal.ai](https://fal.ai/)
2. Сгенерируй ключ API в своей панели управления

### Настроить и выбрать модель

Выполни команду tools:

```bash
hermes tools
```

Перейди в **🎨 Image Generation**, выбери бэкенд (Nous Subscription или FAL.ai), после чего появится таблица со всеми поддерживаемыми моделями в виде колонок — используй стрелки для навигации, `Enter` — для выбора:

```
  Model                          Speed    Strengths                    Price
  fal-ai/flux-2/klein/9b         <1s      Fast, crisp text             $0.006/MP   ← currently in use
  fal-ai/flux-2-pro              ~6s      Studio photorealism          $0.03/MP
  fal-ai/z-image/turbo           ~2s      Bilingual EN/CN, 6B          $0.005/MP
  ...
```

Твой выбор сохраняется в `config.yaml`:

```yaml
image_gen:
  model: fal-ai/flux-2/klein/9b
  use_gateway: false            # true if using Nous Subscription
```

### Качество GPT‑Image

Запросы `fal-ai/gpt-image-1.5` и `fal-ai/gpt-image-2` фиксированы на уровне `medium` (~$0.034–$0.06/image при 1024×1024). Мы не выставляем пользователю варианты `low` / `high`, чтобы биллинг в Nous Portal оставался предсказуемым для всех — разница в стоимости между уровнями 3–22×. Если нужен более дешёвый вариант, выбирай Klein 9B или Z‑Image Turbo; если требуется более высокое качество — Nano Banana Pro или Recraft V4 Pro.

## Использование

Схема, видимая агенту, преднамеренно минимальна — модель использует всё, что ты настроил:

```
Generate an image of a serene mountain landscape with cherry blossoms
```

```
Create a square portrait of a wise old owl — use the typography model
```

```
Make me a futuristic cityscape, landscape orientation
```

## Соотношения сторон

Все модели принимают одинаковые три соотношения сторон с точки зрения агента. Внутри каждая модель автоматически подставляет свой нативный размер:

| Ввод агента | image_size (flux/z-image/qwen/recraft/ideogram) | aspect_ratio (nano-banana-pro) | image_size (gpt-image-1.5) | image_size (gpt-image-2) |
|---|---|---|---|---|
| `landscape` | `landscape_16_9` | `16:9` | `1536x1024` | `landscape_4_3` (1024×768) |
| `square` | `square_hd` | `1:1` | `1024x1024` | `square_hd` (1024×1024) |
| `portrait` | `portrait_16_9` | `9:16` | `1024x1536` | `portrait_4_3` (768×1024) |

GPT Image 2 использует пресеты 4:3 вместо 16:9, потому что его минимальное количество пикселей — 655 360; пресет `landscape_16_9` (1024×576 = 589 824) был бы отклонён.

Эта трансляция происходит в `_build_fal_payload()` — код агента не должен знать о различиях схемы между моделями.

## Автоматическое увеличение разрешения

Увеличение через **Clarity Upscaler** от FAL включено по модели:

| Модель | Увеличивать? | Почему |
|---|---|---|
| `fal-ai/flux-2-pro` | ✓ | Обратная совместимость (был предустановленным до выбора) |
| Все остальные | ✗ | Быстрые модели потеряли бы своё преимущество субсекундной скорости; модели высокого разрешения его не нуждаются |

При выполнении увеличения используются такие параметры:

| Параметр | Значение |
|---|---|
| Коэффициент увеличения | 2× |
| Креативность | 0.35 |
| Сходство | 0.6 |
| Масштаб руководства | 4 |
| Шагов вывода | 18 |

Если увеличение не удалось (проблема сети, ограничение запросов), автоматически возвращается оригинальное изображение.

## Как это работает внутри

1. **Разрешение модели** — `_resolve_fal_model()` читает `image_gen.model` из `config.yaml`, при отсутствии берёт переменную окружения `FAL_IMAGE_MODEL`, затем падает к `fal-ai/flux-2/klein/9b`.
2. **Сбор полезной нагрузки** — `_build_fal_payload()` переводит твой `aspect_ratio` в нативный формат модели (перечисление пресетов, перечисление соотношений или литерал GPT), объединяет параметры модели по умолчанию, применяет любые переопределения вызывающего и фильтрует их по whitelist `supports`, чтобы неподдерживаемые ключи никогда не отправлялись.
3. **Отправка** — `_submit_fal_request()` маршрутизирует запрос через прямые учётные данные FAL или управляемый шлюз Nous.
4. **Увеличение** — выполняется только если у модели в метаданных указано `upscale: True`.
5. **Доставка** — финальный URL изображения возвращается агенту, который выдаёт тег `MEDIA:<url>`, а адаптеры платформ преобразуют его в нативный медиа‑контент.

## Отладка

Включи журнал отладки:

```bash
export IMAGE_TOOLS_DEBUG=true
```

Логи отладки пишутся в `./logs/image_tools_debug_<session_id>.json` с деталями каждого вызова (модель, параметры, время, ошибки).

## Ограничения

- **Требуются учётные данные FAL** (прямой `FAL_KEY` или подписка Nous)
- **Только текст‑в‑изображение** — нет inpainting, img2img или редактирования через этот инструмент
- **Временные URL** — FAL возвращает хост‑URL, которые истекают через часы/дни; при необходимости сохраняй локально
- **Ограничения по модели** — некоторые модели не поддерживают `seed`, `num_inference_steps` и т.п. Фильтр `supports` тихо отбрасывает неподдерживаемые параметры; это ожидаемое поведение
