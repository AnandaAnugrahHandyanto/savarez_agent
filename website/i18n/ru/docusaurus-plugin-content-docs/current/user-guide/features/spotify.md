# Spotify

Hermes может управлять Spotify напрямую — воспроизведение, очередь, поиск, плейлисты, сохранённые треки/альбомы и история прослушиваний — используя официальное Web API Spotify с PKCE OAuth. Токены хранятся в `~/.hermes/auth.json` и автоматически обновляются при 401; вход требуется выполнить только один раз на машине.

В отличие от встроенных OAuth‑интеграций Hermes (Google, GitHub Copilot, Codex), Spotify требует, чтобы каждый пользователь зарегистрировал своё собственное лёгковесное приложение‑разработчика. Spotify не позволяет сторонним сервисам распространять публичное OAuth‑приложение, которым может пользоваться любой. Процесс занимает около двух минут, а `hermes auth spotify` проведёт тебя через него.
## Предварительные требования

- Учётная запись Spotify. **Free** подходит для инструментов поиска, плейлистов, библиотеки и активности. **Premium** требуется для управления воспроизведением (play, pause, skip, seek, volume, queue add, transfer).
- Hermes Agent установлен и запущен.
- Для инструментов воспроизведения: **активное устройство Spotify Connect** — приложение Spotify должно быть открыто хотя бы на одном устройстве (телефон, компьютер, веб‑плеер, колонка), чтобы Web API имело что контролировать. Если ни одно устройство не активно, ты получишь `403 Forbidden` с сообщением «no active device»; открой Spotify на любом устройстве и повтори запрос.
## Настройка

### Одноразовый запуск: `hermes tools` или первоначальная настройка

Самый быстрый путь. Выполни:

```bash
hermes tools
```

Прокрути до `🎵 Spotify`, нажми пробел, чтобы включить, затем `s` — чтобы сохранить. Та же кнопка переключения доступна и во время первого запуска `hermes setup` / `hermes setup tools`. Spotify остаётся включённым, поэтому включение его здесь запускает ту же конфигурацию, учитывающую провайдера, что и `hermes tools`.

Hermes сразу переводит тебя в OAuth‑процесс — если у тебя ещё нет приложения Spotify, он проведёт тебя через его создание прямо в терминале. После завершения набор инструментов включён **и** аутентифицирован за один проход.

Если ты предпочитаешь выполнять шаги отдельно (или переаутентифицируешься позже), используй двухшаговый процесс ниже.

### Двухшаговый процесс

#### 1. Включить набор инструментов

```bash
hermes tools
```

Включи `🎵 Spotify`, сохрани, и когда откроется встроенный мастер, закрой его (Ctrl+C). Набор инструментов останется включённым; только шаг аутентификации откладывается.

#### 2. Запустить мастер входа

```bash
hermes auth spotify
```

7 инструментов Spotify появляются в наборе инструментов агента только после шага 1 — по умолчанию они выключены, чтобы пользователи, которым они не нужны, не передавали лишние схемы инструментов при каждом вызове API.

Если переменная `HERMES_SPOTIFY_CLIENT_ID` не задана, Hermes проведёт тебя через регистрацию приложения прямо в терминале:

1. Открывает `https://developer.spotify.com/dashboard` в браузере
2. Выводит точные значения, которые нужно вставить в форму «Create app» Spotify
3. Запрашивает у тебя Client ID, полученный в результате
4. Сохраняет его в `~/.hermes/.env`, чтобы будущие запуски пропускали этот шаг
5. Сразу переходит к процессу согласия OAuth

После подтверждения токены записываются в `providers.spotify` в файле `~/.hermes/auth.json`. Активный провайдер вывода НЕ меняется — аутентификация Spotify независима от твоего провайдера LLM.

### Создание приложения Spotify (что запрашивает мастер)

Когда открывается панель управления, нажми **Create app** и заполни:

| Поле | Значение |
|------|----------|
| App name | любое (например, `hermes-agent`) |
| App description | любое (например, `personal Hermes integration`) |
| Website | оставь пустым |
| Redirect URI | `http://127.0.0.1:43827/spotify/callback` |
| Which API/SDKs? | отметь **Web API** |

Согласись с условиями и нажми **Save**. На следующей странице нажми **Settings** → скопируй **Client ID** и вставь его в запрос Hermes. Это единственное значение, которое нужен Hermes — PKCE не использует client secret.

### Запуск через SSH / в безголовом окружении

Если установлена переменная `SSH_CLIENT` или `SSH_TTY`, Hermes пропускает автоматическое открытие браузера как в мастере, так и на этапе OAuth. Скопируй URL панели управления и URL авторизации, которые выводит Hermes, открой их в браузере на своей локальной машине и продолжи обычным образом — локальный HTTP‑слушатель всё равно работает на удалённом хосте на порту `43827`. Браузер твоего ноутбука не сможет достучаться до удалённого loopback без локального перенаправления SSH:

```bash
ssh -N -L 43827:127.0.0.1:43827 user@remote-host
```

Для настроек jump‑box / bastion и других подводных камней (mosh, tmux, конфликты портов) смотри [OAuth over SSH / Remote Hosts](../../guides/oauth-over-ssh.md).
## Проверка

```bash
hermes auth status spotify
```

Показывает, присутствуют ли токены и когда истекает срок действия **access‑token**. Обновление происходит автоматически: когда любой вызов Spotify API возвращает 401, клиент использует **refresh‑token** и повторяет запрос один раз. **Refresh‑токены** сохраняются между перезапусками Hermes, поэтому тебе нужно повторно аутентифицироваться только в случае, если ты отзовёшь приложение в настройках своей учётной записи Spotify или выполнишь `hermes auth logout spotify`.
## Использование

После входа в систему агент получает доступ к 7 инструментам Spotify. Ты разговариваешь с агентом естественно — он выбирает нужный инструмент и действие. Для лучшего поведения агент загружает сопутствующий навык, который обучает каноничным шаблонам использования (single-search-then-play, когда не следует предварительно вызывать `get_state` и т.д.).

```
> play some miles davis
> what am I listening to
> add this track to my Late Night Jazz playlist
> skip to the next song
> make a new playlist called "Focus 2026" and add the last three songs I played
> which of my saved albums are by Radiohead
> search for acoustic covers of Blackbird
> transfer playback to my kitchen speaker
```

### Справочник по инструментам

Все действия, изменяющие воспроизведение, принимают необязательный параметр `device_id` для указания конкретного устройства. Если параметр опущен, Spotify использует текущее активное устройство.

#### `spotify_playback`
Управление и просмотр воспроизведения, а также получение истории недавно проигранных треков.

| Действие | Назначение | Premium? |
|----------|------------|----------|
| `get_state` | Полное состояние воспроизведения (трек, устройство, прогресс, перемешивание/повтор) | No |
| `get_currently_playing` | Только текущий трек (возвращает пустой ответ при 204 — см. ниже) | No |
| `play` | Запуск/возобновление воспроизведения. Опционально: `context_uri`, `uris`, `offset`, `position_ms` | Yes |
| `pause` | Пауза воспроизведения | Yes |
| `next` / `previous` | Переход к следующему/предыдущему треку | Yes |
| `seek` | Перейти к `position_ms` | Yes |
| `set_repeat` | `state` = `track` / `context` / `off` | Yes |
| `set_shuffle` | `state` = `true` / `false` | Yes |
| `set_volume` | `volume_percent` = 0‑100 | Yes |
| `recently_played` | Последние проигранные треки. Опционально `limit`, `before`, `after` (Unix ms) | No |

#### `spotify_devices`
| Действие | Назначение |
|----------|------------|
| `list` | Все устройства Spotify Connect, видимые в твоём аккаунте |
| `transfer` | Переместить воспроизведение на `device_id`. Опционально `play: true` запускает воспроизведение при переносе |

### Спикеры, управляемые Home Assistant

Если Home Assistant управляет колонками, которые уже поддерживают Spotify Connect (например, Sonos, Echo, Nest или другие устройства с поддержкой Connect), они автоматически появляются в `spotify_devices list`, как только Spotify их видит. Hermes не требует мост‑соединения Home Assistant ↔ Spotify для этого пути — Spotify самостоятельно обрабатывает маршрутизацию устройств.

Попроси Hermes перенести воспроизведение по отображаемому имени колонки (например, «transfer Spotify to the kitchen speaker»), либо вызови `spotify_devices list` и передай точный `device_id` в `spotify_devices transfer` при написании скриптов. Если колонка отсутствует, открой приложение Spotify или интеграцию колонки в Spotify один раз, чтобы Spotify зарегистрировал её как активную цель Connect.

#### `spotify_queue`
| Действие | Назначение | Premium? |
|----------|------------|----------|
| `get` | Текущие треки в очереди | No |
| `add` | Добавить `uri` в очередь | Yes |

#### `spotify_search`
Поиск по каталогу. Параметр `query` обязателен. Опционально: `types` (массив `track` / `album` / `artist` / `playlist` / `show` / `episode`), `limit`, `offset`, `market`.

#### `spotify_playlists`
| Действие | Назначение | Требуемые аргументы |
|----------|------------|----------------------|
| `list` | Плейлисты пользователя | — |
| `get` | Один плейлист + треки | `playlist_id` |
| `create` | Новый плейлист | `name` (+ опционально `description`, `public`, `collaborative`) |
| `add_items` | Добавить треки | `playlist_id`, `uris` (+ опционально `position`) |
| `remove_items` | Удалить треки | `playlist_id`, `uris` (+ опционально `snapshot_id`) |
| `update_details` | Переименовать / изменить | `playlist_id` + любой из `name`, `description`, `public`, `collaborative` |

#### `spotify_albums`
| Действие | Назначение | Требуемые аргументы |
|----------|------------|----------------------|
| `get` | Метаданные альбома | `album_id` |
| `tracks` | Список треков альбома | `album_id` |

#### `spotify_library`
Единый доступ к сохранённым трекам и альбомам. Выбирай коллекцию с помощью аргумента `kind`.

| Действие | Назначение |
|----------|------------|
| `list` | Постраничный список библиотеки |
| `save` | Добавить `ids` / `uris` в библиотеку |
| `remove` | Удалить `ids` / `uris` из библиотеки |

Требуется: `kind` = `tracks` или `albums`, плюс `action`.

### Матрица функций: Free vs Premium

Инструменты только для чтения работают с бесплатными аккаунтами. Всё, что изменяет воспроизведение или очередь, требует Premium.

| Работает в Free | Требуется Premium |
|------------------|--------------------|
| `spotify_search` (все) | `spotify_playback` — play, pause, next, previous, seek, set_repeat, set_shuffle, set_volume |
| `spotify_playback` — get_state, get_currently_playing, recently_played | `spotify_queue` — add |
| `spotify_devices` — list | `spotify_devices` — transfer |
| `spotify_queue` — get | |
| `spotify_playlists` (все) | |
| `spotify_albums` (все) | |
| `spotify_library` (все) | |
## Планирование: Spotify + cron

Поскольку инструменты Spotify являются обычными инструментами Hermes, cron‑задача, запущенная в сессии Hermes, может запускать воспроизведение по любому расписанию. Новый код не требуется.

### Плейлист для утреннего пробуждения

```bash
hermes cron add \
  --name "morning-commute" \
  "0 7 * * 1-5" \
  "Transfer playback to my kitchen speaker and start my 'Morning Commute' playlist. Volume to 40. Shuffle on."
```

Что происходит в 7 утра каждый будний день:
1. Cron поднимает безголовую сессию Hermes.
2. Агент читает подсказку, вызывает `spotify_devices list`, чтобы найти «kitchen speaker» по имени, затем `spotify_devices transfer` → `spotify_playback set_volume` → `spotify_playback set_shuffle` → `spotify_search` + `spotify_playback play`.
3. Музыка начинается на целевом динамике. Общая стоимость: одна сессия, несколько вызовов инструментов, без вмешательства человека.

### Завершение дня

```bash
hermes cron add \
  --name "wind-down" \
  "30 22 * * *" \
  "Pause Spotify. Then set volume to 20 so it's quiet when I start it again tomorrow."
```

### Подводные камни

- **Во время срабатывания cron должен быть активен девайс.** Если клиент Spotify не запущен (телефон/десктоп/Connect‑динамик), действия воспроизведения возвращают `403 no active device`. Для утренних плейлистов трюк — нацеливаться на устройство, которое всегда включено (Sonos, Echo, умный динамик), а не на телефон.
- **Для любых действий, изменяющих воспроизведение, требуется Premium** — воспроизведение, пауза, пропуск, регулировка громкости, передача. Cron‑задачи только для чтения (например, «отправить мне по email недавно проигранные треки») работают на Free.
- **Cron‑агент наследует твои активные наборы инструментов.** Spotify должен быть включён в `hermes tools`, чтобы cron‑сессия видела инструменты Spotify.
- **Cron‑задачи запускаются с `skip_memory=True`**, поэтому они не записывают данные в хранилище памяти.

Полная справка по cron: [Cron Jobs](./cron).
## Выйти

```bash
hermes auth logout spotify
```

Удаляет токены из `~/.hermes/auth.json`. Чтобы также очистить конфигурацию приложения, удалите `HERMES_SPOTIFY_CLIENT_ID` (и `HERMES_SPOTIFY_REDIRECT_URI`, если вы его задавали) из `~/.hermes/.env` или запустите мастер заново.

Чтобы отозвать доступ приложения на стороне Spotify, перейдите в [Apps connected to your account](https://www.spotify.com/account/apps/) и нажмите **REMOVE ACCESS**.
## Устранение неполадок

**`403 Forbidden — Player command failed: No active device found`** — Нужно, чтобы Spotify был запущен хотя бы на одном устройстве. Открой приложение Spotify на телефоне, компьютере или в веб‑плеере, запусти любой трек на секунду, чтобы он зарегистрировался, и повтори попытку. `spotify_devices list` показывает, какие устройства сейчас видны.

**`403 Forbidden — Premium required`** — Ты используешь бесплатный аккаунт и пытаешься выполнить действие, изменяющее воспроизведение. Смотри матрицу функций выше.

**`204 No Content` на `get_currently_playing`** — На любом устройстве сейчас ничего не воспроизводится. Это обычный ответ Spotify, а не ошибка; Hermes отображает его как пояснительный пустой результат (`is_playing: false`).

**`INVALID_CLIENT: Invalid redirect URI`** — URI перенаправления в настройках твоего приложения Spotify не совпадает с тем, что использует Hermes. По умолчанию это `http://127.0.0.1:43827/spotify/callback`. Добавь его в список разрешённых URI в приложении или задай `HERMES_SPOTIFY_REDIRECT_URI` в `~/.hermes/.env` на значение, которое ты зарегистрировал.

**`429 Too Many Requests`** — Ограничение частоты запросов Spotify. Hermes возвращает дружелюбную ошибку; подожди минуту и попробуй снова. Если это продолжается, скорее всего, ты запускаешь слишком быстрый цикл в скрипте — квота Spotify сбрасывается примерно каждые 30 секунд.

**`401 Unauthorized` продолжает появляться** — Твой токен обновления был отозван (обычно потому, что ты удалил приложение из своего аккаунта или приложение было удалено). Выполни `hermes auth spotify` ещё раз.

**Мастер не открывает браузер** — Если ты подключён по SSH или находишься в контейнере без дисплея, Hermes это обнаруживает и пропускает автоматическое открытие. Скопируй URL панели управления, который он выводит, и открой его вручную.
## Расширенный: пользовательские области доступа

По умолчанию Hermes запрашивает области доступа, необходимые для каждого поставляемого инструмента. Переопредели их, если хочешь ограничить доступ:

```bash
hermes auth spotify --scope "user-read-playback-state user-modify-playback-state playlist-read-private"
```

Справочник областей доступа: [Spotify Web API scopes](https://developer.spotify.com/documentation/web-api/concepts/scopes). Если запросить меньше областей доступа, чем требуется инструменту, вызовы этого инструмента завершатся ошибкой 403.
## Advanced: custom client ID / redirect URI

```bash
hermes auth spotify --client-id <id> --redirect-uri http://localhost:3000/callback
```

Or set them permanently in `~/.hermes/.env`:

```
HERMES_SPOTIFY_CLIENT_ID=<your_id>
HERMES_SPOTIFY_REDIRECT_URI=http://localhost:3000/callback
```

The redirect URI must be allow-listed in your Spotify app's settings. The default works for almost everyone — only change it if port 43827 is taken.
