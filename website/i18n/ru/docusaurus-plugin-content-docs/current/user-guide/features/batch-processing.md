# Пакетная обработка

Пакетная обработка позволяет запускать Hermes Agent на сотнях или тысячах запросов параллельно, генерируя структурированные данные траекторий. Это в основном используется для **генерации обучающих данных** — создания траекторий в формате ShareGPT со статистикой использования инструментов, которые можно применять для дообучения или оценки.

## Обзор

Пакетный раннер (`batch_runner.py`) обрабатывает набор данных в формате JSONL, запуская каждый запрос в полной сессии агента с доступом к инструментам. Каждый запрос получает собственное изолированное окружение. Выходные данные — структурированные траектории с полной историей диалога, статистикой вызовов инструментов и метриками охвата рассуждений.

## Быстрый старт

```bash
# Basic batch run
python batch_runner.py \
    --dataset_file=data/prompts.jsonl \
    --batch_size=10 \
    --run_name=my_first_run \
    --model=anthropic/claude-sonnet-4.6 \
    --num_workers=4

# Resume an interrupted run
python batch_runner.py \
    --dataset_file=data/prompts.jsonl \
    --batch_size=10 \
    --run_name=my_first_run \
    --resume

# List available toolset distributions
python batch_runner.py --list_distributions
```

:::tip Predictable cost at scale
Пакетные запуски создают множество одновременных сессий агента, каждая из которых делает вызовы модели и инструменты. Подписка [Nous Portal](/user-guide/features/tool-gateway) объединяет доступ к модели, веб‑поиску, генерации изображений, TTS и облачным браузерам в один счёт — удобно, когда нужен стабильный расход на траекторию без постоянного переключения лимитов между пятью аккаунтами поставщиков. Настраивается командой `hermes setup --portal`, затем указывай `--model` на модель Nous.
:::

## Формат набора данных

Входной набор данных — файл JSONL (по одному JSON‑объекту на строку). Каждая запись должна содержать поле `prompt`:

```jsonl
{"prompt": "Write a Python function that finds the longest palindromic substring"}
{"prompt": "Create a REST API endpoint for user authentication using Flask"}
{"prompt": "Debug this error: TypeError: cannot unpack non-iterable NoneType object"}
```

Опционально записи могут включать:
- `image` или `docker_image`: образ контейнера, используемый для песочницы этого запроса (работает с Docker, Modal и бэкендами Singularity)
- `cwd`: переопределение рабочей директории для терминальной сессии задачи

## Параметры конфигурации

| Параметр | По умолчанию | Описание |
|-----------|--------------|----------|
| `--dataset_file` | (обязательно) | Путь к JSONL‑набору данных |
| `--batch_size` | (обязательно) | Количество запросов в пакете |
| `--run_name` | (обязательно) | Имя текущего запуска (используется для каталога вывода и контрольных точек) |
| `--distribution` | `"default"` | Распределение наборов инструментов для выборки |
| `--model` | `claude-sonnet-4.6` | Модель для использования |
| `--base_url` | `https://openrouter.ai/api/v1` | Базовый URL API |
| `--api_key` | (переменная окружения) | API‑ключ для модели |
| `--max_turns` | `10` | Максимальное число итераций вызова инструментов на запрос |
| `--num_workers` | `4` | Количество параллельных рабочих процессов |
| `--resume` | `false` | Возобновить с контрольной точки |
| `--verbose` | `false` | Включить подробный лог |
| `--max_samples` | all | Обрабатывать только первые N образцов из набора |
| `--max_tokens` | model default | Максимальное число токенов в ответе модели |

### Маршрутизация провайдеров (OpenRouter)

| Параметр | Описание |
|-----------|----------|
| `--providers_allowed` | Список провайдеров, разрешённых через запятую (например, `"anthropic,openai"`) |
| `--providers_ignored` | Список провайдеров, игнорируемых через запятую (например, `"together,deepinfra"`) |
| `--providers_order` | Предпочтительный порядок провайдеров через запятую |
| `--provider_sort` | Сортировать по `"price"`, `"throughput"` или `"latency"` |

### Управление рассуждениями

| Параметр | Описание |
|-----------|----------|
| `--reasoning_effort` | Уровень усилий: `none`, `minimal`, `low`, `medium`, `high`, `xhigh` |
| `--reasoning_disabled` | Полностью отключить токены рассуждений/мышления |

### Расширенные параметры

| Параметр | Описание |
|-----------|----------|
| `--ephemeral_system_prompt` | Системный запрос, используемый во время выполнения, но НЕ сохраняемый в траектории |
| `--log_prefix_chars` | Количество символов для предварительного просмотра в логе (по умолчанию: 100) |
| `--prefill_messages_file` | Путь к JSON‑файлу с предварительными сообщениями для few‑shot‑приминга |

## Распределения наборов инструментов

Каждому запросу случайным образом назначается набор инструментов из **распределения**. Это гарантирует, что обучающие данные охватывают разнообразные комбинации инструментов. Команда `--list_distributions` выводит все доступные распределения.

В текущей реализации распределения задают вероятность для **каждого отдельного инструмента**. Сэмплер независимо решает включить каждый инструмент, затем гарантирует, что хотя бы один инструмент включён. Это отличается от вручную составленных таблиц готовых комбинаций.

## Формат вывода

Весь вывод сохраняется в `data/<run_name>/`:

```text
data/my_run/
├── trajectories.jsonl    # Combined final output (all batches merged)
├── batch_0.jsonl         # Individual batch results
├── batch_1.jsonl
├── ...
├── checkpoint.json       # Resume checkpoint
└── statistics.json       # Aggregate tool usage stats
```

### Формат траектории

Каждая строка в `trajectories.jsonl` — это JSON‑объект:

```json
{
  "prompt_index": 42,
  "conversations": [
    {"from": "human", "value": "Write a function..."},
    {"from": "gpt", "value": "I'll create that function...",
     "tool_calls": [...]},
    {"from": "tool", "value": "..."},
    {"from": "gpt", "value": "Here's the completed function..."}
  ],
  "metadata": {
    "batch_num": 2,
    "timestamp": "2026-01-15T10:30:00",
    "model": "anthropic/claude-sonnet-4.6"
  },
  "completed": true,
  "partial": false,
  "api_calls": 3,
  "toolsets_used": ["terminal", "file"],
  "tool_stats": {
    "terminal": {"count": 2, "success": 2, "failure": 0},
    "read_file": {"count": 1, "success": 1, "failure": 0}
  },
  "tool_error_counts": {
    "terminal": 0,
    "read_file": 0
  }
}
```

Поле `conversations` использует формат, похожий на ShareGPT, с полями `from` и `value`. Статистика инструментов нормализована: включены все возможные инструменты с нулевыми значениями по умолчанию, что обеспечивает согласованную схему для совместимости с наборами данных HuggingFace.

## Контрольные точки

Пакетный раннер обладает надёжным механизмом контрольных точек для отказоустойчивости:

- **Файл контрольной точки:** Сохраняется после завершения каждого пакета, фиксируя, какие индексы запросов выполнены.
- **Восстановление по содержимому:** При `--resume` раннер сканирует существующие файлы пакетов и сопоставляет выполненные запросы по их реальному текстовому содержимому (а не только по индексам), позволяя восстановиться даже при изменении порядка в наборе данных.
- **Неудавшиеся запросы:** Помечаются как невыполненные — они будут повторно обработаны при возобновлении.
- **Объединение пакетов:** По завершении все файлы пакетов (включая из предыдущих запусков) объединяются в один `trajectories.jsonl`.

### Как работает возобновление

1. Сканировать все файлы `batch_*.jsonl` на предмет завершённых запросов (по совпадению содержимого).
2. Отфильтровать набор данных, исключив уже выполненные запросы.
3. Пересобрать оставшиеся запросы в новые пакеты.
4. Обработать только оставшиеся запросы.
5. Объединить все файлы пакетов (старые + новые) в финальный вывод.

## Фильтрация качества

Пакетный раннер автоматически фильтрует низкокачественные образцы:

- **Фильтр без рассуждений:** Образцы, в которых ни один ход ассистента не содержит рассуждений (нет `<REASONING_SCRATCHPAD>` или токенов мышления), отбрасываются.
- **Фильтр повреждённых записей:** Записи с вымышленными названиями инструментов (отсутствующими в списке допустимых) отфильтровываются при финальном объединении.
- **Статистика рассуждений:** Отслеживает процент ходов с/без рассуждений за весь запуск.

## Статистика

По завершении раннер выводит полную статистику:

- **Использование инструментов:** Количество вызовов, процент успешных/неуспешных для каждого инструмента.
- **Охват рассуждений:** Процент ходов ассистента с рассуждениями.
- **Отброшенные образцы:** Число образцов, отфильтрованных из‑за отсутствия рассуждений.
- **Продолжительность:** Общее время обработки.

Статистика также сохраняется в `statistics.json` для программного анализа.

## Сценарии использования

### Генерация обучающих данных

Создание разнообразных траекторий использования инструментов для дообучения:

```bash
python batch_runner.py \
    --dataset_file=data/coding_prompts.jsonl \
    --batch_size=20 \
    --run_name=coding_v1 \
    --model=anthropic/claude-sonnet-4.6 \
    --num_workers=8 \
    --distribution=default \
    --max_turns=15
```

### Оценка модели

Оценка того, насколько хорошо модель использует инструменты на стандартизированных запросах:

```bash
python batch_runner.py \
    --dataset_file=data/eval_suite.jsonl \
    --batch_size=10 \
    --run_name=eval_gpt4 \
    --model=openai/gpt-4o \
    --num_workers=4 \
    --max_turns=10
```

### Образы контейнеров per‑prompt

Для бенчмарков, требующих специфических окружений, каждый запрос может указывать собственный образ контейнера:

```jsonl
{"prompt": "Install numpy and compute eigenvalues of a 3x3 matrix", "image": "python:3.11-slim"}
{"prompt": "Compile this Rust program and run it", "image": "rust:1.75"}
{"prompt": "Set up a Node.js Express server", "image": "node:20-alpine", "cwd": "/app"}
```

Пакетный раннер проверяет доступность Docker‑образов перед запуском каждого запроса.
