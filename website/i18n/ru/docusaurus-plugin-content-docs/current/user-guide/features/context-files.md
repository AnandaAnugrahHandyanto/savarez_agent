# Файлы контекста

Hermes Agent автоматически обнаруживает и загружает файлы контекста, которые формируют его поведение. Некоторые из них локальны для проекта и находятся в рабочем каталоге. `SOUL.md` теперь глобален для экземпляра Hermes и загружается только из `HERMES_HOME`.

## Поддерживаемые файлы контекста

| Файл | Назначение | Обнаружение |
|------|------------|--------------|
| **.hermes.md** / **HERMES.md** | Инструкции проекта (наивысший приоритет) | Переход к корню git |
| **AGENTS.md** | Инструкции проекта, конвенции, архитектура | Текущий каталог при старте + подкаталоги последовательно |
| **CLAUDE.md** | Файлы контекста Claude Code (также обнаруживаются) | Текущий каталог при старте + подкаталоги последовательно |
| **SOUL.md** | Глобальная персонализация личности и тона для этого экземпляра Hermes | Только `HERMES_HOME/SOUL.md` |
| **.cursorrules** | Конвенции кодинга Cursor IDE | Только текущий каталог |
| **.cursor/rules/*.mdc** | Модули правил Cursor IDE | Только текущий каталог |

:::info Приоритетная система
Загружается **только один** тип контекстного файла проекта за сессию (первое совпадение выигрывает): `.hermes.md` → `AGENTS.md` → `CLAUDE.md` → `.cursorrules`. **SOUL.md** всегда загружается независимо как идентичность агента (слот #1).
:::

## AGENTS.md

`AGENTS.md` — основной файл контекста проекта. Он сообщает агенту, как структурирован твой проект, какие конвенции следует соблюдать и любые специальные инструкции.

### Прогрессивное обнаружение подкаталогов

При начале сессии Hermes загружает `AGENTS.md` из твоего рабочего каталога в системный запрос. Когда агент перемещается в подкаталоги во время сессии (через `read_file`, `terminal`, `search_files` и т.п.), он **прогрессивно обнаруживает** файлы контекста в этих каталогах и вставляет их в разговор в тот момент, когда они становятся релевантными.

```
my-project/
├── AGENTS.md              ← Loaded at startup (system prompt)
├── frontend/
│   └── AGENTS.md          ← Discovered when agent reads frontend/ files
├── backend/
│   └── AGENTS.md          ← Discovered when agent reads backend/ files
└── shared/
    └── AGENTS.md          ← Discovered when agent reads shared/ files
```

Такой подход имеет два преимущества перед загрузкой всего сразу:
- **Отсутствие раздувания системного запроса** — подсказки подкаталогов появляются только при необходимости;
- **Сохранение кэша запроса** — системный запрос остаётся стабильным между ходами.

Каждый подкаталог проверяется не более одного раза за сессию. Обнаружение также поднимается по родительским каталогам, поэтому чтение `backend/src/main.py` обнаружит `backend/AGENTS.md`, даже если в `backend/src/` собственного файла контекста нет.

:::info
Файлы контекста подкаталогов проходят тот же [скан безопасности](#security-prompt-injection-protection), что и файлы контекста при старте. Вредоносные файлы блокируются.
:::

### Пример AGENTS.md

```markdown
# Project Context

This is a Next.js 14 web application with a Python FastAPI backend.

## Architecture
- Frontend: Next.js 14 with App Router in `/frontend`
- Backend: FastAPI in `/backend`, uses SQLAlchemy ORM
- Database: PostgreSQL 16
- Deployment: Docker Compose on a Hetzner VPS

## Conventions
- Use TypeScript strict mode for all frontend code
- Python code follows PEP 8, use type hints everywhere
- All API endpoints return JSON with `{data, error, meta}` shape
- Tests go in `__tests__/` directories (frontend) or `tests/` (backend)

## Important Notes
- Never modify migration files directly — use Alembic commands
- The `.env.local` file has real API keys, don't commit it
- Frontend port is 3000, backend is 8000, DB is 5432
```

## SOUL.md

`SOUL.md` управляет личностью, тоном и стилем общения агента. См. страницу [Personality](/user-guide/features/personality) для полной информации.

**Расположение:**

- `~/.hermes/SOUL.md`
- или `$HERMES_HOME/SOUL.md`, если ты запускаешь Hermes с пользовательским домашним каталогом

Важные детали:

- Hermes автоматически создаёт файл `SOUL.md` по умолчанию, если он ещё не существует;
- Hermes загружает `SOUL.md` только из `HERMES_HOME`;
- Hermes не ищет `SOUL.md` в рабочем каталоге;
- Если файл пуст, ничего из `SOUL.md` не добавляется в запрос;
- Если файл содержит данные, они вставляются дословно после сканирования и усечения.

## .cursorrules

Hermes совместим с файлом `.cursorrules` IDE Cursor и модулями правил `.cursor/rules/*.mdc`. Если эти файлы находятся в корне проекта и не найден файл контекста более высокого приоритета (`.hermes.md`, `AGENTS.md` или `CLAUDE.md`), они загружаются как контекст проекта.

Это значит, что твои существующие конвенции Cursor автоматически применяются при работе с Hermes.

## Как загружаются файлы контекста

### При старте (системный запрос)

Файлы контекста загружаются функцией `build_context_files_prompt()` в `agent/prompt_builder.py`:

1. **Сканирование рабочего каталога** — проверяется наличие `.hermes.md` → `AGENTS.md` → `CLAUDE.md` → `.cursorrules` (первое совпадение выигрывает);
2. **Чтение содержимого** — каждый файл читается как UTF‑8‑текст;
3. **Сканирование безопасности** — проверка на паттерны внедрения запросов;
4. **Усечение** — файлы более 20 000 символов усекаются: 70 % начала, 20 % конца, с маркером посередине;
5. **Сборка** — все секции объединяются под заголовком `# Project Context`;
6. **Вставка** — собранное содержимое добавляется в системный запрос.

### Во время сессии (прогрессивное обнаружение)

`SubdirectoryHintTracker` в `agent/subdirectory_hints.py` отслеживает аргументы вызовов инструментов для путей к файлам:

1. **Извлечение путей** — после каждого вызова инструмента пути извлекаются из аргументов (`path`, `workdir`, команды оболочки);
2. **Подъём по предкам** — проверяется каталог и до 5 родительских каталогов (остановка на уже посещённых каталогах);
3. **Загрузка подсказки** — если найден `AGENTS.md`, `CLAUDE.md` или `.cursorrules`, он загружается (первое совпадение в каталоге);
4. **Сканирование безопасности** — тот же скан на внедрение запросов, что и для файлов при старте;
5. **Усечение** — ограничение 8 000 символов на файл;
6. **Вставка** — добавляется к результату инструмента, так модель видит его в контексте естественно.

Итоговый раздел запроса выглядит примерно так:

```text
# Project Context

The following project context files have been loaded and should be followed:

## AGENTS.md

[Your AGENTS.md content here]

## .cursorrules

[Your .cursorrules content here]

[Your SOUL.md content here]
```

Обрати внимание, что содержимое `SOUL.md` вставляется напрямую, без дополнительного обёрточного текста.

## Безопасность: защита от внедрения запросов

Все файлы контекста сканируются на потенциальные попытки внедрения запросов перед включением. Сканер проверяет наличие:

- **Попыток переопределения инструкций**: «ignore previous instructions», «disregard your rules»;
- **Обманных паттернов**: «do not tell the user»;
- **Переопределения системного запроса**: «system prompt override»;
- **Скрытых HTML‑комментариев**: `<!-- ignore instructions -->`;
- **Скрытых элементов `div`**: `<div style="display:none">`;
- **Эксфильтрации учётных данных**: `curl ... $API_KEY`;
- **Доступа к секретным файлам**: `cat .env`, `cat credentials`;
- **Невидимых символов**: нулевые ширины, би‑дирекциональные переопределения, соединители слов.

Если обнаружен любой из паттернов угроз, файл блокируется:

```
[BLOCKED: AGENTS.md contained potential prompt injection (prompt_injection). Content not loaded.]
```

:::warning
Этот сканер защищает от распространённых паттернов внедрения, но не заменяет проверку файлов контекста в общих репозиториях. Всегда проверяй содержимое `AGENTS.md` в проектах, которые ты не создавал.
:::

## Ограничения по размеру

| Ограничение | Значение |
|------------|----------|
| Макс. символов в файле | 20 000 (~7 000 токенов) |
| Коэффициент усечения начала | 70 % |
| Коэффициент усечения конца | 20 % |
| Маркер усечения | 10 % (показывает количество символов и предлагает использовать файловые инструменты) |

Когда файл превышает 20 000 символов, сообщение об усечении выглядит так:

```
[...truncated AGENTS.md: kept 14000+4000 of 25000 chars. Use file tools to read the full file.]
```

## Советы по эффективным файлам контекста

:::tip Лучшие практики для AGENTS.md
1. **Делай их лаконичными** — держи объём значительно ниже 20 К символов; агент читает их каждый ход;
2. **Структурируй заголовками** — используй секции `##` для архитектуры, конвенций, важных заметок;
3. **Включай конкретные примеры** — показывай предпочтительные шаблоны кода, формы API, правила именования;
4. **Указывай, чего НЕ делать** — «never modify migration files directly»;
5. **Перечисляй ключевые пути и порты** — агент использует их для команд терминала;
6. **Обновляй по мере развития проекта** — устаревший контекст хуже, чем отсутствие контекста.
:::

### Контекст per‑subdirectory

Для монорепозиториев размещай инструкции, специфичные для подкаталогов, во вложенных файлах `AGENTS.md`:

```markdown
<!-- frontend/AGENTS.md -->
# Frontend Context

- Use `pnpm` not `npm` for package management
- Components go in `src/components/`, pages in `src/app/`
- Use Tailwind CSS, never inline styles
- Run tests with `pnpm test`
```

```markdown
<!-- backend/AGENTS.md -->
# Backend Context

- Use `poetry` for dependency management
- Run the dev server with `poetry run uvicorn main:app --reload`
- All endpoints need OpenAPI docstrings
- Database models are in `models/`, schemas in `schemas/`
```
