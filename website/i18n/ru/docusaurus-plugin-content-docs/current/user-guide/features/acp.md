# Интеграция ACP Editor

Hermes Agent может работать как сервер ACP, позволяя редакторам, совместимым с ACP, взаимодействовать с Hermes через stdio и отображать:

- сообщения чата
- активность инструмента
- диффы файлов
- команды терминала
- запросы на одобрение
- потоковые фрагменты размышлений и ответов

ACP хорошо подходит, когда ты хочешь, чтобы Hermes вёл себя как агент, встроенный в редактор, а не как отдельный CLI или бот для обмена сообщениями.
## Что Hermes предоставляет в режиме ACP

Hermes работает с отобранным набором инструментов `hermes‑acp`, предназначенным для рабочих процессов редактора. Он включает:

- инструменты работы с файлами: `read_file`, `write_file`, `patch`, `search_files`
- инструменты терминала: `terminal`, `process`
- инструменты веб/браузера
- поиск по памяти, todo, сессии
- `skills`
- `execute_code` и `delegate_task`
- `vision`

Он намеренно исключает элементы, не вписывающиеся в типичный UX редактора, такие как доставка сообщений и управление cron‑задачами.
## Установка

Установи Hermes обычным способом, затем добавь дополнительный пакет ACP:

```bash
pip install -e '.[acp]'
```

Это установит зависимость `agent-client-protocol` и включит:

- `hermes acp`
- `hermes-acp`
- `python -m acp_adapter`

Для установок из реестра Zed Zed запускает Hermes через официальную запись ACP Registry. Эта запись использует дистрибутив `uvx`, который выполняет:

```bash
uvx --from 'hermes-agent[acp]==<version>' hermes-acp
```

Убедись, что `uv` доступен в `PATH` перед использованием пути установки из реестра.
## Запуск сервера ACP

Любая из следующих команд запускает Hermes в режиме ACP:

```bash
hermes acp
```

```bash
hermes-acp
```

```bash
python -m acp_adapter
```

Hermes пишет логи в `stderr`, поэтому `stdout` остаётся зарезервированным для трафика ACP JSON‑RPC.

Для неинтерактивных проверок:

```bash
hermes acp --version
hermes acp --check
```

### Инструменты браузера (необязательно)

Инструменты браузера (`browser_navigate`, `browser_click` и т.д.) зависят от npm‑пакета `agent-browser` и Chromium, которые не входят в Python‑wheel. Установи их командой:

```bash
hermes acp --setup-browser           # interactive (prompts before ~400 MB download)
hermes acp --setup-browser --yes     # accept the download non-interactively
```

Это самостоятельная команда. Поток аутентификации терминала реестра Zed (`hermes acp --setup`) также предлагает загрузку браузера как последующий вопрос после выбора модели, поэтому большинству пользователей не требуется запускать `--setup-browser` напрямую.

Что делает команда:

- Устанавливает Node.js 22 LTS в `~/.hermes/node/`, если его нет
- Выполняет `npm install -g agent-browser @askjo/camofox-browser` в этом префиксе (без `sudo` — `npm`‑опция `--prefix` указывает на пользовательскую директорию, управляемую Hermes)
- Устанавливает Chromium через Playwright или использует обнаруженный системный Chrome/Chromium, если он доступен

Bootstrap идемпотентен — повторный запуск выполняется быстро и пропускает уже выполненные действия.
## Настройка редактора

### VS Code

Установи расширение [ACP Client](https://marketplace.visualstudio.com/items?itemName=formulahendry.acp-client).

Чтобы подключиться:

1. Открой панель ACP Client на панели активности.
2. Выбери **Hermes Agent** из встроенного списка агентов.
3. Подключись и начни общение.

Если хочешь задать Hermes вручную, добавь его через настройки VS Code в `acp.agents`:

```json
{
  "acp.agents": {
    "Hermes Agent": {
      "command": "hermes",
      "args": ["acp"]
    }
  }
}
```

### Zed

Zed v0.221.x и новее устанавливают внешние агенты через официальный ACP Registry.

1. Открой панель Agent.
2. Нажми **Add Agent** или выполни команду `zed: acp registry`.
3. Найди **Hermes Agent**.
4. Установи его и запусти новый поток внешнего агента Hermes.

Требования:

- Сначала настрой учётные данные провайдера Hermes с помощью `hermes model`, либо укажи их в `~/.hermes/.env` / `~/.hermes/config.yaml`.
- Установи `uv`, чтобы запускатель реестра мог выполнить `uvx --from 'hermes-agent[acp]==<version>' hermes-acp`.

Для локальной разработки, пока запись в реестре недоступна, используй пользовательский сервер агента в настройках Zed:

```json
{
  "agent_servers": {
    "hermes-agent": {
      "type": "custom",
      "command": "hermes",
      "args": ["acp"]
    }
  }
}
```

### JetBrains

Используй совместимый с ACP плагин и укажи ему:

```text
/path/to/hermes-agent/acp_registry
```
## Манифест реестра

Исходная копия официальных метаданных ACP Registry от Hermes находится по адресу:

```text
acp_registry/agent.json
acp_registry/icon.svg
```

PR в upstream‑реестре копирует эти файлы в корневую директорию `hermes-agent/` внутри `agentclientprotocol/registry`.

Запись реестра использует дистрибутив `uvx`, который указывает напрямую на выпуск `hermes-agent` в PyPI:

```text
uvx --from 'hermes-agent[acp]==<version>' hermes-acp
```

CI реестра проверяет, что закреплённая версия существует в PyPI, поэтому `version` в манифесте и закрепление `package` в `uvx` всегда должны совпадать с `pyproject.toml`. `scripts/release.py` поддерживает их в тесной синхронности автоматически.
## Конфигурация и учётные данные

ACP‑режим использует ту же конфигурацию Hermes, что и CLI:

- `~/.hermes/.env`
- `~/.hermes/config.yaml`
- `~/.hermes/skills/`
- `~/.hermes/state.db`

Разрешение провайдера использует обычный рантайм‑резолвер Hermes, поэтому ACP наследует текущие настроенные провайдер и учётные данные. Hermes также предлагает метод аутентификации в терминале (`--setup`) для клиентов реестра при первом запуске; он открывает интерактивную настройку модели/провайдера Hermes.
## Поведение сессии

Сессии ACP отслеживаются менеджером сессий в памяти адаптера ACP, пока сервер работает.

Каждая сессия хранит:

- ID сессии
- рабочий каталог
- выбранную модель
- текущую историю диалога
- событие отмены

Базовый `AIAgent` по‑прежнему использует обычные пути постоянного хранения/логирования Hermes, но команды ACP `list/load/resume/fork` ограничены текущим процессом работающего сервера ACP.
## Поведение рабочей директории

Сеансы ACP привязывают текущий рабочий каталог редактора (cwd) к идентификатору задачи Hermes, поэтому инструменты работы с файлами и терминалом выполняются относительно рабочего пространства редактора, а не текущего рабочего каталога процесса сервера.
## Одобрения

Опасные команды терминала могут быть перенаправлены обратно в редактор в виде запросов на одобрение. Параметры одобрения ACP проще, чем поток CLI:

- разрешить один раз
- разрешить всегда
- отклонить

При истечении времени ожидания или ошибке мост одобрения отклоняет запрос.

### Автоодобрение в рамках сессии

ACP добавляет третий уровень между *разрешить один раз* и *разрешить всегда*: **Разрешить для сессии**. Выбор этого варианта в запросе разрешения редактора сохраняет одобрение только внутри текущей сессии ACP — каждая последующая совпадающая команда в этой сессии выполняется без запроса, но новая сессия ACP (или перезапуск редактора) сбрасывает состояние и запрашивает подтверждение в первый раз.

| Option | Editor label | Scope | Persisted across restarts |
|---|---|---|---|
| `allow_once` | Allow once | This one tool call | No |
| `allow_session` | Allow for session | All matching calls in this ACP session | No — cleared when the session ends |
| `allow_always` | Allow always | All future sessions | Yes (written to the Hermes permanent allowlist) |
| `deny` | Deny | This one tool call | No |

`allow_session` — это правильный вариант по умолчанию для рабочего процесса редактора, когда ты доверяешь агенту на время выполнения задачи, но не хочешь создавать долговременную запись в allowlist. Компромисс в безопасности прост: чем шире область действия, тем реже редактор будет тебя прерывать, и тем больше вреда может нанести некорректно работающий агент (или инъекция подсказки), прежде чем ты заметишь. Начинай с `allow_once` для незнакомых команд; переходи к `allow_session`, когда увидишь, что агент несколько раз правильно выполняет тот же шаблон; резервируй `allow_always` для действительно идемпотентных команд, которым ты доверяешь навсегда (например, `git status`).

Мост ACP отображает эти варианты на внутреннюю семантику одобрения Hermes — `allow_always` записывает постоянную запись в allowlist так же, как делает это CLI, тогда как `allow_session` влияет только на кэш одобрений в процессе для текущей сессии ACP.
## Устранение неполадок

### ACP‑агент не появляется в редакторе

Проверь:

- В Zed открой реестр ACP с помощью `zed: acp registry` и найди **Hermes Agent**.
- Для ручной/локальной разработки убедись, что кастомная команда `agent_servers` указывает на `hermes acp`.
- Hermes установлен и находится в твоём PATH.
- Дополнение ACP установлено (`pip install -e '.[acp]'`).
- `uv` установлен, если запуск происходит из официального пункта реестра Zed.

### ACP запускается, но сразу выдаёт ошибку

Попробуй выполнить следующие проверки:

```bash
hermes acp --version
hermes acp --check
hermes doctor
hermes status
```

### Отсутствуют учётные данные

Режим ACP использует существующую настройку провайдера Hermes. Настрой учётные данные с помощью:

```bash
hermes model
```

или отредактировав `~/.hermes/.env`. Клиенты реестра также могут инициировать поток терминальной аутентификации Hermes, который запускает тот же интерактивный процесс настройки провайдера/модели.

### Запускатель реестра Zed не может найти uv

Установи `uv` согласно официальной документации по установке uv, затем повтори попытку запуска потока Hermes Agent из Zed.
