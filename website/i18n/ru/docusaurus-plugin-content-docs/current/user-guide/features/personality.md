# Личность и ДУША.md

Персональность Hermes Agent полностью настраиваемая. `SOUL.md` — **основная идентичность** — это первое, что попадает в системный запрос, и определяет, кто такой агент.

- `SOUL.md` — долговременный файл персонажа, который находится в `HERMES_HOME` и служит идентичностью агента (слот № 1 в системном запросе)
- встроенные или пользовательские пресеты `/personality` — наложения системного запроса уровня сессии

Если хочешь изменить, кем является Hermes — или заменить его полностью другой персоной — отредактируй `SOUL.md`.

## Как сейчас работает `SOUL.md`

Hermes теперь автоматически создаёт файл `SOUL.md` по умолчанию в:

```text
~/.hermes/SOUL.md
```

Точнее, используется текущий `HERMES_HOME` экземпляра, поэтому если запустить Hermes с пользовательским домашним каталогом, будет использовано:

```text
$HERMES_HOME/SOUL.md
```

### Важное поведение

- **`SOUL.md` — основная идентичность агента.** Он занимает слот № 1 в системном запросе, заменяя жёстко заданную идентичность по умолчанию.
- Hermes автоматически создаёт стартовый `SOUL.md`, если он ещё не существует.
- Существующие пользовательские файлы `SOUL.md` никогда не перезаписываются.
- Hermes загружает `SOUL.md` только из `HERMES_HOME`.
- Hermes не ищет `SOUL.md` в текущем рабочем каталоге.
- Если `SOUL.md` существует, но пустой, или его нельзя загрузить, Hermes переходит к встроенной идентичности по умолчанию (запасной вариант).
- Если в `SOUL.md` есть содержимое, оно вставляется дословно после сканирования на безопасность и усечения.
- `SOUL.md` **не** дублируется в разделе файлов контекста — он появляется только один раз, как идентичность.

Это делает `SOUL.md` истинной персоной для каждого пользователя или экземпляра, а не просто добавочным слоем.

## Почему так задумано

Так персональность становится предсказуемой.

Если бы Hermes загружал `SOUL.md` из любого каталога, из которого его запустили, персональность могла бы неожиданно меняться между проектами. Загрузка только из `HERMES_HOME` привязывает её к самому экземпляру Hermes.

Это также упрощает обучение пользователей:
- «Отредактируй `~/.hermes/SOUL.md`, чтобы изменить персональность Hermes по умолчанию.»

## Где его редактировать

Для большинства пользователей:

```bash
~/.hermes/SOUL.md
```

Если используешь пользовательский домашний каталог:

```bash
$HERMES_HOME/SOUL.md
```

## Что должно находиться в `SOUL.md`?

Используй его для долговременных рекомендаций по голосу и персональности, например:
- тон
- стиль общения
- степень прямоты
- стиль взаимодействия по умолчанию
- чего следует избегать стилистически
- как Hermes должен обращаться с неопределённостью, разногласиями или неоднозначностью

Используй реже для:
- одноразовых инструкций проекта
- путей к файлам
- конвенций репозитория
- временных деталей рабочего процесса

Эти вещи лучше помещать в `AGENTS.md`, а не в `SOUL.md`.

## Хорошее содержание `SOUL.md`

Хороший файл `SOUL.md`:
- стабилен в разных контекстах
- достаточно широк, чтобы применяться в многих беседах
- достаточно конкретен, чтобы реально формировать голос
- сосредоточен на коммуникации и идентичности, а не на инструкциях по задачам

### Пример

```markdown
# Personality

You are a pragmatic senior engineer with strong taste.
You optimize for truth, clarity, and usefulness over politeness theater.

## Style
- Be direct without being cold
- Prefer substance over filler
- Push back when something is a bad idea
- Admit uncertainty plainly
- Keep explanations compact unless depth is useful

## What to avoid
- Sycophancy
- Hype language
- Repeating the user's framing if it's wrong
- Overexplaining obvious things

## Technical posture
- Prefer simple systems over clever systems
- Care about operational reality, not idealized architecture
- Treat edge cases as part of the design, not cleanup
```

## Что Hermes вставляет в запрос

Содержимое `SOUL.md` попадает напрямую в слот № 1 системного запроса — позицию идентичности агента. Обёртка вокруг него не добавляется.

Содержимое проходит:
- сканирование на инъекции запросов
- усечение, если оно слишком велико

Если файл пуст, содержит только пробелы или не может быть прочитан, Hermes переходит к встроенной идентичности по умолчанию («You are Hermes Agent, an intelligent AI assistant created by Nous Research…»). Этот запасной вариант также срабатывает, когда установлен `skip_context_files` (например, в контекстах subagent/delegation).

## Сканирование безопасности

`SOUL.md` сканируется, как и другие файлы контекста, на наличие паттернов инъекции запросов перед включением.

Поэтому следует оставлять его сфокусированным на персонаже/голосе, а не пытаться прятать странные мета‑инструкции.

## `SOUL.md` vs `AGENTS.md`

Это самое важное различие.

### `SOUL.md`
Используй для:
- идентичности
- тона
- стиля
- настроек коммуникации
- поведения на уровне персональности

### `AGENTS.md`
Используй для:
- архитектуры проекта
- конвенций кодирования
- предпочтений инструментов
- рабочих процессов, специфичных для репозитория
- команд, портов, путей, заметок по развертыванию

Полезное правило:
- если должно следовать за тобой везде, помещай в `SOUL.md`
- если относится к проекту, помещай в `AGENTS.md`

## `SOUL.md` vs `/personality`

`SOUL.md` — твоя долговременная персональность по умолчанию.
`/personality` — наложение уровня сессии, которое меняет или дополняет текущий системный запрос.

Итого:
- `SOUL.md` = базовый голос
- `/personality` = временный переключатель режима

Примеры:
- держи прагматичный `SOUL` по умолчанию, а затем используй `/personality teacher` для репетиторского диалога
- держи лаконичный `SOUL`, а затем используй `/personality creative` для мозгового штурма

## Встроенные персональности

Hermes поставляется с набором встроенных персональностей, которые можно переключать с помощью `/personality`.

| Name | Description |
|------|-------------|
| **helpful** | Дружелюбный, универсальный помощник |
| **concise** | Краткие, по‑сути ответы |
| **technical** | Подробный, точный технический эксперт |
| **creative** | Инновационное, нестандартное мышление |
| **teacher** | Терпеливый наставник с ясными примерами |
| **kawaii** | Милые выражения, блестки и энтузиазм ★ |
| **catgirl** | Неко‑чан с кошачьими выражениями, ня~ |
| **pirate** | Капитан Hermes, техничный пират |
| **shakespeare** | Бардовская проза с драматическим размахом |
| **surfer** | Совершенно расслабленные «бро» вайбы |
| **noir** | Жёсткое детективное повествование |
| **uwu** | Максимальная милота с uwu‑речью |
| **philosopher** | Глубокие размышления над каждым запросом |
| **hype** | МАКСИМАЛЬНАЯ ЭНЕРГИЯ И ЭНТУЗИАЗМ!!! |

## Переключение персональностей командами

### CLI

```text
/personality
/personality concise
/personality technical
```

### Платформы обмена сообщениями

```text
/personality teacher
```

Это удобные наложения, но твой глобальный `SOUL.md` всё равно задаёт постоянную персональность Hermes, если только наложение не меняет её существенно.

## Пользовательские персональности в конфигурации

Можно также определить именованные пользовательские персональности в `~/.hermes/config.yaml` в разделе `agent.personalities`.

```yaml
agent:
  personalities:
    codereviewer: >
      You are a meticulous code reviewer. Identify bugs, security issues,
      performance concerns, and unclear design choices. Be precise and constructive.
```

Затем переключиться на неё с помощью:

```text
/personality codereviewer
```

## Рекомендованный рабочий процесс

Надёжная базовая настройка:

1. Храни продуманный глобальный `SOUL.md` в `~/.hermes/SOUL.md`
2. Инструкции проекта помещай в `AGENTS.md`
3. Используй `/personality` только когда нужен временный режим

Это даёт:
- стабильный голос
- проектно‑специфическое поведение там, где оно нужно
- временный контроль при необходимости

## Как персональность взаимодействует с полным запросом

На высоком уровне стек запросов выглядит так:
1. **`SOUL.md`** (идентичность агента — или встроенный запасной вариант, если `SOUL.md` недоступен)
2. руководство по поведению, учитывающее инструменты
3. память/контекст пользователя
4. руководство по навыкам
5. файлы контекста (`AGENTS.md`, `.cursorrules`)
6. метка времени
7. подсказки по форматированию, специфичные для платформы
8. опциональные наложения системного запроса, такие как `/personality`

`SOUL.md` — фундамент, на котором строится всё остальное.

## Вид CLI vs разговорная персональность

Разговорная персональность и внешний вид CLI раздельны:

- `SOUL.md`, `agent.system_prompt` и `/personality` влияют на то, как говорит Hermes
- `display.skin` и `/skin` влияют на внешний вид Hermes в терминале

Для внешнего вида терминала смотри [Skins & Themes](./skins.md).
