# Режим результата

Когда Hermes Agent работает внутри шлюза обмена сообщениями (Slack, Discord, Telegram, WhatsApp, Signal и т.д.), он может доставлять сгенерированные файлы напрямую в чат — не в виде путей, которые пользователь должен копировать, а в виде нативных вложений.

Диаграмма отображается как встроенное изображение. PDF‑отчёт появляется как файл для скачивания. Таблица загружается как `.xlsx`. Агенту не нужно писать тег `MEDIA:` или делать что‑то особенное — он просто генерирует файл и упоминает его абсолютный путь в ответе. Шлюз извлекает путь из текста, удаляет его из видимого сообщения и загружает файл нативно.

## Как это работает

Три части работают вместе:

1. **У агента есть инструменты, которые создают файлы.** `execute_code` — для диаграмм через matplotlib, навык `latex-pdf-report` — для PDF, навык `powerpoint` — для презентаций, `image_generate` — для изображений, `text_to_speech` — для аудио и т.д.

2. **Шлюз сканирует ответы агента в поисках путей к файлам.** Любой абсолютный путь (`/tmp/...`) или путь относительно домашней директории (`~/...`), оканчивающийся поддерживаемым расширением, извлекается. Пути внутри блоков кода и встроенного кода игнорируются, чтобы образцы кода никогда не были искажены.

3. **Шлюз распределяет файлы по типу.** Изображения встраиваются inline там, где платформа это поддерживает; видео — inline; аудио — в виде голосовых/аудио‑вложений; всё остальное — загружается как файловое вложение.

## Поддерживаемые расширения файлов

| Категория      | Расширения                                                            | Доставка                              |
|----------------|-----------------------------------------------------------------------|---------------------------------------|
| Изображения    | `.png .jpg .jpeg .gif .webp .bmp .tiff .svg`                         | Встроенное отображение                |
| Видео          | `.mp4 .mov .avi .mkv .webm`                                          | Встроенное отображение (где поддерживается) |
| Аудио          | `.mp3 .wav .ogg .m4a .flac`                                          | Голосовое / аудио вложение            |
| Документы      | `.pdf .docx .doc .odt .rtf .txt .md`                                 | Загрузка файла                        |
| Данные         | `.xlsx .xls .csv .tsv .json .xml .yaml .yml`                         | Загрузка файла                        |
| Презентации    | `.pptx .ppt .odp`                                                    | Загрузка файла                        |
| Архивы         | `.zip .tar .gz .tgz .bz2 .7z`                                        | Загрузка файла                        |
| Веб            | `.html .htm`                                                          | Загрузка файла                        |

`.py`, `.log` и другие расширения исходных файлов намеренно исключены, чтобы агент не отправлял произвольные исходные файлы; если нужно отправить код пользователю, используй блок кода.

## Как заставить агента создавать артефакты

По умолчанию агент не генерирует артефакты — ему нужно явно знать, что делать. Два способа подтолкнуть его:

**Per‑session:** явно попроси («отправь мне сравнение в виде диаграммы», «верни данные в CSV») или создай собственные пользовательские инструкции / запись личности, которые склоняют ответы к артефактному стилю на платформах обмена сообщениями.

**Project‑level:** добавь смещение в `AGENTS.md` / `CLAUDE.md` / `.cursorrules` проекта, из которого работает агент, в глобальную персону в `~/.hermes/SOUL.md` или как именованный пресет в `agent.personalities` в `~/.hermes/config.yaml` (переключаемый per session через `/personality`).

Механизм, который агент использует, прост: сохранить файл по абсолютному пути (например `/tmp/q3-revenue.png`) и упомянуть этот путь как обычный текст в ответе. Шлюз делает остальное. Пути внутри ограждённых блоков кода или обратных кавычек игнорируются, чтобы образцы кода не были искажены.

## Kanban: артефакты в уведомлениях о завершении

Если ты используешь многопользовательский workflow Kanban от Hermes, работники могут прикреплять файлы‑результаты к своему вызову `kanban_complete`:

```python
kanban_complete(
    summary="rendered Q3 revenue chart and report",
    artifacts=[
        "/tmp/q3-revenue.png",
        "/tmp/q3-report.pdf",
    ],
)
```

Когда шлюз‑уведомитель доставляет сообщение «задача выполнена» тем, кто подписан на задачу в Slack/Telegram и т.п., он также загружает каждый артефакт как нативное вложение в этот чат. Человек получает результат и резюме в одном месте.

Файлы, которые не существуют на диске в момент выполнения уведомления, пропускаются без предупреждения.

## Подключение дополнительных сервисов через MCP

Помимо конвейера доставки артефактов, агент может обращаться к другим сервисам через MCP (Model Context Protocol). Экосистема MCP поставляет серверы сообществ для большинства популярных инструментов — установи те, которые нужны:

| Сервис                | Что открывает                                                                      |
|-----------------------|------------------------------------------------------------------------------------|
| **Notion**            | Чтение/запись страниц Notion, баз данных, запросы к рабочему пространству          |
| **GitHub**            | Issues, PR, комментарии, поиск репозиториев помимо CLI `gh`                         |
| **Linear**            | Тикеты, проекты, циклы                                                             |
| **Slack**             | Поиск по всему рабочему пространству, чтение других каналов                        |
| **Gmail**             | Триаж входящих, отправка писем, управление метками                                 |
| **Salesforce**        | Лиды, возможности, данные аккаунтов                                                |
| **Snowflake / BigQuery** | SQL‑запросы к хранилищам данных                                                    |
| **Google Drive**      | Поиск файлов, их содержимое, управление доступом                                   |

Устанавливай серверы MCP через `~/.hermes/config.yaml` в разделе `mcp_servers`. Смотри [MCP integration](./mcp.md) для полного руководства по настройке.

## Сравнение с Perplexity Computer в Slack

Интеграция Perplexity Computer для Slack построена на той же идее: агент генерирует результат (диаграмму, PDF, набор слайдов) и публикует его в ветке как нативное вложение. Режим результата Hermes Agent предоставляет тот же пользовательский паттерн локально:

- Генерация происходит в собственном `venv` / песочнице пользователя (без удалённого арендатора).
- Файлы попадают в чат через тот же API Slack `files.uploadV2`.
- Широта коннекторов обеспечивается MCP, а не курируемым каталогом из 400 хостированных интеграций — устанавливай только те, что действительно используешь.

OAuth‑токены остаются на машине пользователя в `auth.json` / `.env`. Нет хостинга токенов. Нет многопользовательского microVM. Итоговый результат тот же.
