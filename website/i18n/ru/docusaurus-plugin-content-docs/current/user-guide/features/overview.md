# Обзор возможностей

Hermes Agent включает богатый набор функций, выходящих далеко за рамки базового чата. От постоянной памяти и контекста, учитывающего файлы, до автоматизации браузера и голосовых бесед — эти возможности работают вместе, делая Hermes мощным автономным помощником.

:::tip Не знаешь, с чего начать?
`hermes setup --portal` покрывает провайдера модели плюс все четыре инструмента шлюза инструментов (веб‑поиск, генерация изображений, TTS, браузер) одной командой. Смотри [Nous Portal](/integrations/nous-portal).
:::

## Core

- **[Инструменты и наборы инструментов](tools.md)** — Инструменты — это функции, расширяющие возможности агента. Они организованы в логические наборы, которые можно включать или отключать для каждой платформы, охватывая веб‑поиск, выполнение терминала, редактирование файлов, память, делегирование и многое другое.
- **[Система навыков](skills.md)** — Документы знаний по запросу, которые агент может загрузить при необходимости. Навыки следуют принципу прогрессивного раскрытия, чтобы минимизировать использование токенов, и совместимы с открытым стандартом [agentskills.io](https://agentskills.io/specification).
- **[Постоянная память](memory.md)** — Ограниченная, курируемая память, сохраняющаяся между сессиями. Hermes запоминает твои предпочтения, проекты, окружение и то, чему он научился через `MEMORY.md` и `USER.md`.
- **[Контекстные файлы](context-files.md)** — Hermes автоматически обнаруживает и загружает файлы контекста проекта (`.hermes.md`, `AGENTS.md`, `CLAUDE.md`, `SOUL.md`, `.cursorrules`), формирующие его поведение в твоём проекте.
- **[Контекстные ссылки](context-references.md)** — Введи `@` и ссылку, чтобы вставить файлы, папки, git‑diff'ы и URL непосредственно в сообщения. Hermes разворачивает ссылку inline и автоматически добавляет содержимое.
- **[Контрольные точки](../checkpoints-and-rollback.md)** — Hermes автоматически делает снимки твоего рабочего каталога перед изменением файлов, предоставляя сетку безопасности для отката с помощью `/rollback`, если что‑то пошло не так.

## Автоматизация

- **[Запланированные задачи (Cron)](cron.md)** — Планируй задачи для автоматического выполнения с помощью естественного языка или cron‑выражений. Задания могут прикреплять навыки, доставлять результаты на любую платформу и поддерживают операции пауза/возобновление/редактирование.
- **[Делегирование субагентов](delegation.md)** — Инструмент `delegate_task` создаёт дочерние экземпляры агента с изолированным контекстом, ограниченными наборами инструментов и собственными терминальными сессиями. По умолчанию запускается 3 параллельных субагента (настраиваемо) для параллельных рабочих потоков.
- **[Выполнение кода](code-execution.md)** — Инструмент `execute_code` позволяет агенту писать Python‑скрипты, вызывающие инструменты Hermes программно, сворачивая многошаговые рабочие процессы в один ход LLM через изолированное RPC‑выполнение.
- **[Хуки событий](hooks.md)** — Выполняй пользовательский код в ключевых точках жизненного цикла. Хуки шлюза обрабатывают логирование, оповещения и вебхуки; хуки плагинов — перехват инструментов, метрики и защитные механизмы.
- **[Пакетная обработка](batch-processing.md)** — Запускай агент Hermes на сотнях или тысячах запросов параллельно, генерируя структурированные данные траекторий в формате ShareGPT для создания обучающих наборов или оценки.

## Медиа и веб

- **[Голосовой режим](voice-mode.md)** — Полное голосовое взаимодействие в CLI и мессенджерах. Говори агенту через микрофон, слушай озвученные ответы и веди живые голосовые беседы в голосовых каналах Discord.
- **[Автоматизация браузера](browser.md)** — Полная автоматизация браузера с несколькими бекэндами: облако Browserbase, облако Browser Use, локальный Chrome/Brave/Chromium/Edge через CDP или локальный Chromium. Навигируй по сайтам, заполняй формы и извлекай информацию.
- **[Зрение и вставка изображений](vision.md)** — Поддержка мультимодального зрения. Вставляй изображения из буфера обмена в CLI и проси агента анализировать, описывать или работать с ними, используя любую модель с поддержкой зрения.
- **[Генерация изображений](image-generation.md)** — Генерируй изображения из текстовых подсказок с помощью FAL.ai. Поддерживается девять моделей (FLUX 2 Klein/Pro, GPT‑Image 1.5/2, Nano Banana Pro, Ideogram V3, Recraft V4 Pro, Qwen, Z‑Image Turbo); выбирай их через `hermes tools`.
- **[Голос и TTS](tts.md)** — Вывод текста в речь и транскрипция голосовых сообщений на всех платформах обмена, с десятью нативными провайдерами: Edge TTS (бесплатно), ElevenLabs, OpenAI TTS, MiniMax, Mistral Voxtral, Google Gemini, xAI, NeuTTS, KittenTTS и Piper — плюс пользовательские провайдеры команд для любого локального TTS CLI.

## Интеграции

- **[Интеграция MCP](mcp.md)** — Подключайся к любому серверу MCP через stdio или HTTP‑транспорт. Доступ к внешним инструментам из GitHub, баз данных, файловых систем и внутренних API без написания нативных инструментов Hermes. Включает фильтрацию инструментов и поддержку сэмплинга на уровне сервера.
- **[Маршрутизация провайдеров](provider-routing.md)** — Тонкая настройка того, какие AI‑провайдеры обрабатывают запросы. Оптимизируй по стоимости, скорости или качеству с помощью сортировки, белых/чёрных списков и приоритетного порядка.
- **[Запасные (вариант) провайдеры](fallback-providers.md)** — Автоматический фоллбэк к резервным LLM‑провайдерам, когда основной модельный провайдер выдаёт ошибку, включая независимый фоллбэк для вспомогательных задач, таких как зрение и сжатие.
- **[Пулы учётных данных](credential-pools.md)** — Распределяй API‑вызовы по нескольким ключам одного провайдера. Автоматическая ротация при ограничениях по частоте или сбоях.
- **[Кеширование подсказок](../configuration#prompt-caching)** — Встроенный кросс‑сессионный кеш префикса на 1 час для Claude на нативном Anthropic, OpenRouter и Nous Portal. Всегда включён; конфигурация не требуется.
- **[Провайдеры памяти](memory-providers.md)** — Подключай внешние бекэнды памяти (Honcho, OpenViking, Mem0, Hindsight, Holographic, RetainDB, ByteRover, Supermemory) для кросс‑сессионного моделирования пользователей и персонализации за пределами встроенной системы памяти.
- **[API‑сервер](api-server.md)** — Выставляй Hermes как совместимую с OpenAI HTTP‑точку. Подключай любой фронтенд, поддерживающий формат OpenAI — Open WebUI, LobeChat, LibreChat и др.
- **[Интеграция IDE (ACP)](acp.md)** — Используй Hermes внутри редакторов, совместимых с ACP, таких как VS Code, Zed и JetBrains. Чат, активность инструментов, диффы файлов и команды терминала отображаются внутри редактора.
- **[Пакетная обработка](batch-processing.md)** — Запускай агента на множестве запросов или задач параллельно из CLI, получая структурированные выводы и захват траекторий, подходящие для оценок или downstream‑тренировочных конвейеров.

## Настройка

- **[Личность и SOUL.md](personality.md)** — Полностью настраиваемая личность агента. `SOUL.md` — основной файл идентичности, первое, что попадает в системный запрос, и ты можешь менять встроенные или пользовательские пресеты `/personality` для каждой сессии.
- **[Скины и темы](skins.md)** — Настраивай визуальное оформление CLI: цвета баннера, лица и глаголы спиннеров, метки в окне ответа, брендовый текст и префикс активности инструментов.
- **[Плагины](plugins.md)** — Добавляй пользовательские инструменты, хуки и интеграции без изменения ядра. Три типа плагинов: общие плагины (инструменты/хуки), провайдеры памяти (кросс‑сессионные знания) и движки контекста (альтернативное управление контекстом). Управляются через единый интерактивный UI `hermes plugins`.
