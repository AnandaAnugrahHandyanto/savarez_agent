# Ссылки на контекст

Введи `@`, а затем ссылку, чтобы напрямую вставить содержимое в своё сообщение. Hermes разворачивает ссылку inline и добавляет содержимое в раздел `--- Attached Context ---`.

## Поддерживаемые ссылки

| Синтаксис | Описание |
|-----------|----------|
| `@file:path/to/file.py` | Вставить содержимое файла |
| `@file:path/to/file.py:10-25` | Вставить конкретный диапазон строк (нумерация с 1, включительно) |
| `@folder:path/to/dir` | Вставить дерево каталогов с метаданными файлов |
| `@diff` | Вставить `git diff` (непроиндексированные изменения) |
| `@staged` | Вставить `git diff --staged` (проиндексированные изменения) |
| `@git:5` | Вставить последние N коммитов с патчами (максимум 10) |
| `@url:https://example.com` | Получить и вставить содержимое веб‑страницы |

## Примеры использования

```text
Review @file:src/main.py and suggest improvements

What changed? @diff

Compare @file:old_config.yaml and @file:new_config.yaml

What's in @folder:src/components?

Summarize this article @url:https://arxiv.org/abs/2301.00001
```

Несколько ссылок могут использоваться в одном сообщении:

```text
Check @file:main.py, and also @file:test.py.
```

Конечные знаки пунктуации (`,`, `.`, `;`, `!`, `?`) автоматически удаляются из значений ссылки.

## Автодополнение в CLI

В интерактивном CLI ввод `@` вызывает автодополнение:

- `@` — показать все типы ссылок (`@diff`, `@staged`, `@file:`, `@folder:`, `@git:`, `@url:`)
- `@file:` и `@folder:` — автодополнение путей файловой системы с указанием размера файлов
- Обычный `@`, за которым следует часть текста, — показать совпадающие файлы и каталоги из текущего каталога

## Диапазоны строк

Ссылка `@file:` поддерживает диапазоны строк для точного внедрения содержимого:

```text
@file:src/main.py:42        # Single line 42
@file:src/main.py:10-25     # Lines 10 through 25 (inclusive)
```

Нумерация строк начинается с 1. Неправильные диапазоны игнорируются без предупреждения (возвращается весь файл).

## Ограничения по размеру

Ссылки на контекст ограничены, чтобы не перегрузить окно контекста модели:

| Порог | Значение | Поведение |
|------|----------|-----------|
| Мягкий лимит | 25 % от длины контекста | Добавляется предупреждение, расширение продолжается |
| Жёсткий лимит | 50 % от длины контекста | Расширение отклоняется, оригинальное сообщение возвращается без изменений |
| Записи каталога | максимум 200 файлов | Избыточные записи заменяются на `- ...` |
| Коммиты Git | максимум 10 | `@git:N` ограничивается диапазоном [1, 10] |

## Безопасность

### Блокировка чувствительных путей

Эти пути всегда блокируются для ссылок `@file:` во избежание утечки учётных данных:

- SSH‑ключи и конфиги: `~/.ssh/id_rsa`, `~/.ssh/id_ed25519`, `~/.ssh/authorized_keys`, `~/.ssh/config`
- Профили оболочки: `~/.bashrc`, `~/.zshrc`, `~/.profile`, `~/.bash_profile`, `~/.zprofile`
- Файлы учётных данных: `~/.netrc`, `~/.pgpass`, `~/.npmrc`, `~/.pypirc`
- Переменные Hermes: `$HERMES_HOME/.env`

Эти каталоги полностью блокируются (любой файл внутри):

- `~/.ssh/`, `~/.aws/`, `~/.gnupg/`, `~/.kube/`, `$HERMES_HOME/skills/.hub/`

### Защита от обхода пути

Все пути разрешаются относительно рабочей директории. Ссылки, которые выходят за пределы корня разрешённого рабочего пространства, отклоняются.

### Обнаружение бинарных файлов

Бинарные файлы определяются по MIME‑типу и сканированию на нулевые байты. Известные текстовые расширения (`.py`, `.md`, `.json`, `.yaml`, `.toml`, `.js`, `.ts` и др.) обходят проверку MIME. Бинарные файлы отклоняются с предупреждением.

## Доступность по платформам

Ссылки на контекст в основном являются **фичей CLI**. Они работают в интерактивном CLI, где `@` вызывает автодополнение, а ссылки разворачиваются до отправки сообщения агенту.

На **мессенджерах** (Telegram, Discord и др.) синтаксис `@` не разворачивается шлюзом — сообщения передаются как есть. Сам агент всё равно может обращаться к файлам через инструменты `read_file`, `search_files` и `web_extract`.

## Взаимодействие с компрессией контекста

Когда контекст беседы сжимается, расширенное содержимое ссылки включается в сводку компрессии. Это означает:

- Большие файлы, вставленные через `@file:`, учитываются в использовании контекста
- При последующей компрессии содержимое файла суммируется (не сохраняется дословно)
- Для очень больших файлов рекомендуется использовать диапазоны строк (`@file:main.py:100-200`), чтобы вставлять только нужные части

## Распространённые шаблоны

```text
# Code review workflow
Review @diff and check for security issues

# Debug with context
This test is failing. Here's the test @file:tests/test_auth.py
and the implementation @file:src/auth.py:50-80

# Project exploration
What does this project do? @folder:src @file:README.md

# Research
Compare the approaches in @url:https://arxiv.org/abs/2301.00001
and @url:https://arxiv.org/abs/2301.00002
```

## Обработка ошибок

Неправильные ссылки вызывают встроенные предупреждения, а не ошибки выполнения:

| Условие | Поведение |
|--------|-----------|
| Файл не найден | Предупреждение: «file not found» |
| Бинарный файл | Предупреждение: «binary files are not supported» |
| Папка не найдена | Предупреждение: «folder not found» |
| Ошибка Git‑команды | Предупреждение с выводом `git stderr` |
| URL не вернул содержимое | Предупреждение: «no content extracted» |
| Чувствительный путь | Предупреждение: «path is a sensitive credential file» |
| Путь вне рабочего пространства | Предупреждение: «path is outside the allowed workspace» |
