# Секреты

Hermes может получать API‑ключи из внешних менеджеров секретов при запуске процесса, вместо того чтобы хранить их в `~/.hermes/.env`. Bootstrap‑токен для менеджера секретов находится в `.env`; все остальные ключи провайдеров (OpenAI, Anthropic, OpenRouter и т.д.) могут оставаться в менеджере и централизованно ротироваться.

Поддерживаются:

- [Bitwarden Secrets Manager](./bitwarden) — `bws` CLI, устанавливается по требованию, бесплатный тариф работает.

Другие бекэнды (Vault, AWS Secrets Manager, 1Password CLI) легко добавить через тот же интерфейс — достаточно добавить модуль в `agent/secret_sources/` и обработчик CLI. Создай запрос, если у тебя есть конкретный вариант в виду.
