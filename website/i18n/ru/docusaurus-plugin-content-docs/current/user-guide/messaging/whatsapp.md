# Настройка WhatsApp

Hermes подключается к WhatsApp через встроенный мост, основанный на **Baileys**. Это работает за счёт эмуляции сессии WhatsApp Web — **не** через официальное WhatsApp Business API. Учётная запись разработчика Meta или проверка Business не требуются.

> Запусти `hermes gateway setup` и выбери **WhatsApp** для пошагового мастера.

:::warning Неофициальный API — риск блокировки
WhatsApp **не** официально поддерживает сторонних ботов вне Business API. Использование стороннего моста несёт небольшой риск ограничения аккаунта. Чтобы минимизировать риск:
- **Используй отдельный номер телефона** для бота (не свой личный номер)
- **Не отправляй массовые/спам‑сообщения** — держи общение в разговорном формате
- **Не автоматизируй исходящие сообщения** людям, которые не писали первым
:::

:::warning Обновления протокола WhatsApp Web
WhatsApp периодически обновляет свой Web‑протокол, что может временно нарушить совместимость со сторонними мостами. Когда это происходит, Hermes обновит зависимость моста. Если бот перестаёт работать после обновления WhatsApp, скачай последнюю версию Hermes и выполни повторное сопряжение.
:::
## Два режима

| Режим | Как работает | Оптимально для |
|------|--------------|----------------|
| **Отдельный номер бота** (рекомендовано) | Выделить отдельный номер телефона для бота. Пользователи пишут сообщения напрямую на этот номер. | Чистый UX, несколько пользователей, меньший риск блокировки |
| **Личный чат с собой** | Использовать свой WhatsApp и писать себе, чтобы общаться с агентом. | Быстрая настройка, один пользователь, тестирование |

---
## Предварительные требования

- **Node.js v18+** и **npm** — мост WhatsApp работает как процесс Node.js
- **Телефон с установленным WhatsApp** (для сканирования QR‑кода)

В отличие от старых браузерных мостов, текущий мост на основе Baileys **не** требует локального стека зависимостей Chromium или Puppeteer.
## Шаг 1: Запусти мастер настройки

```bash
hermes whatsapp
```

Мастер выполнит следующее:

1. Спросит, какой режим ты хочешь (**bot** или **self-chat**)
2. Установит зависимости моста, если это необходимо
3. Отобразит **QR‑код** в твоём терминале
4. Подождёт, пока ты его отсканируешь

**Как отсканировать QR‑код:**

1. Открой WhatsApp на телефоне
2. Перейди в **Settings → Linked Devices**
3. Нажми **Link a Device**
4. Направь камеру на QR‑код в терминале

После сопряжения мастер подтвердит соединение и завершит работу. Твоя сессия сохраняется автоматически.

:::tip
Если QR‑код выглядит искажённым, убедись, что твой терминал шириной минимум 60 колонок и поддерживает Unicode. Можно также попробовать другой эмулятор терминала.
:::

---
## Шаг 2: Получение второго номера телефона (режим бота)

Для режима бота нужен номер телефона, который ещё не зарегистрирован в WhatsApp. Три варианта:

| Option | Cost | Notes |
|--------|------|-------|
| **Google Voice** | Free | Только США. Получи номер на [voice.google.com](https://voice.google.com). Подтверди WhatsApp через SMS в приложении Google Voice. |
| **Prepaid SIM** | $5–15 однократно | Любой оператор. Активируй, подтвердите WhatsApp, затем SIM‑карту можно хранить в ящике. Номер должен оставаться активным (делай звонок каждые 90 дней). |
| **VoIP services** | Free–$5/month | TextNow, TextFree или аналогичные. Некоторые VoIP‑номера блокируются WhatsApp — попробуй несколько, если первый не сработает. |

После получения номера:

1. Установи WhatsApp на телефон (или используй приложение WhatsApp Business с двойной SIM)
2. Зарегистрируй новый номер в WhatsApp
3. Запусти `hermes whatsapp` и отсканируй QR‑код из этой учётной записи WhatsApp

---
## Шаг 3: Настройка Hermes

Добавь следующее в файл `~/.hermes/.env`:

```bash
# Required
WHATSAPP_ENABLED=true
WHATSAPP_MODE=bot                          # "bot" or "self-chat"

# Access control — pick ONE of these options:
WHATSAPP_ALLOWED_USERS=15551234567         # Comma-separated phone numbers (with country code, no +)
# WHATSAPP_ALLOWED_USERS=*                 # OR use * to allow everyone
# WHATSAPP_ALLOW_ALL_USERS=true            # OR set this flag instead (same effect as *)
```

:::tip Сокращение для разрешения всем
Установка `WHATSAPP_ALLOWED_USERS=*` разрешает **всем** отправителям (эквивалентно `WHATSAPP_ALLOW_ALL_USERS=true`).
Это согласуется с [Signal group allowlists](/reference/environment-variables).
Чтобы вместо этого использовать процесс сопряжения, удали обе переменные и полагайся на
[DM pairing system](/user-guide/security#dm-pairing-system).
:::

Опциональные настройки поведения в `~/.hermes/config.yaml`:

```yaml
unauthorized_dm_behavior: pair

whatsapp:
  unauthorized_dm_behavior: ignore
```

- `unauthorized_dm_behavior: pair` — глобальное значение по умолчанию. Неизвестные отправители DM получают код сопряжения.
- `whatsapp.unauthorized_dm_behavior: ignore` заставляет WhatsApp молчать при неавторизованных DM, что обычно лучше для личного номера.

Затем запусти gateway:

```bash
hermes gateway              # Foreground
hermes gateway install      # Install as a user service
sudo hermes gateway install --system   # Linux only: boot-time system service
```

Gateway автоматически запускает мост WhatsApp, используя сохранённую сессию.

---
## Сохранение сессии

Мост Baileys сохраняет свою сессию в `~/.hermes/platforms/whatsapp/session`. Это означает:

- **Сессии сохраняются при перезапуске** — тебе не нужно каждый раз сканировать QR‑код
- Данные сессии включают ключи шифрования и учётные данные устройства
- **Не публикуй и не коммить эту директорию сессии** — она предоставляет полный доступ к аккаунту WhatsApp

---
## Повторное сопряжение

Если сессия прерывается (перезагрузка телефона, обновление WhatsApp, ручное отключение), в журналах **gateway** появятся ошибки соединения. Чтобы исправить это:

```bash
hermes whatsapp
```

Это генерирует новый QR‑код. Сканируй его снова, и сессия будет повторно установлена. **Gateway** автоматически обрабатывает **временные** разрывы соединения (короткие сбои сети, кратковременное отключение телефона) с помощью логики повторного подключения.
## Голосовые сообщения

Hermes поддерживает голос в WhatsApp:

- **Входящие:** Голосовые сообщения (`.ogg` opus) автоматически транскрибируются с помощью настроенного провайдера STT: локальный `faster-whisper`, Groq Whisper (`GROQ_API_KEY`) или OpenAI Whisper (`VOICE_TOOLS_OPENAI_KEY`)
- **Исходящие:** Ответы TTS отправляются в виде вложений аудиофайлов MP3
- Ответы агента по умолчанию начинаются с «⚕ **Hermes Agent**». Ты можешь изменить или отключить это в `config.yaml`:

```yaml
# ~/.hermes/config.yaml
whatsapp:
  reply_prefix: ""                          # Empty string disables the header
  # reply_prefix: "🤖 *My Bot*\n──────\n"  # Custom prefix (supports \n for newlines)
```

## Форматирование сообщений и доставка

WhatsApp поддерживает **потоковые (прогрессивные) ответы** — бот редактирует своё сообщение в реальном времени по мере генерации текста ИИ, так же как в Discord и Telegram. Внутри WhatsApp классифицируется как платформа уровня `TIER_MEDIUM` по возможностям доставки.

### Фрагментация

Длинные ответы автоматически разбиваются на несколько сообщений по **4 096 символов** на фрагмент (практический предел отображения в WhatsApp). Настраивать ничего не нужно — `gateway` обрабатывает разбиение и отправляет фрагменты последовательно.

### Совместимый с WhatsApp Markdown

Стандартный Markdown в ответах ИИ автоматически преобразуется в нативное форматирование WhatsApp:

| Markdown | WhatsApp | Отображается как |
|----------|----------|------------------|
| `**bold**` | `*bold*` | **bold** |
| `~~strikethrough~~` | `~strikethrough~` | ~~strikethrough~~ |
| `# Heading` | `*Heading*` | Жирный текст (заголовки не поддерживаются) |
| `[link text](url)` | `link text (url)` | Встроенный URL |

Блоки кода и встроенный код сохраняются без изменений, так как WhatsApp нативно поддерживает форматирование тройными обратными кавычками.

### Прогресс инструментов

Когда агент вызывает инструменты (веб‑поиск, операции с файлами и т.п.), WhatsApp отображает индикаторы прогресса в реальном времени, показывающие, какой инструмент запущен. Это включено по умолчанию — настройка не требуется.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| **QR‑код не сканируется** | Убедись, что терминал достаточно широк (60 + столбцов). Попробуй другой терминал. Проверь, что сканируешь QR‑код из правильного аккаунта WhatsApp (номер бота, а не личный). |
| **QR‑код истёк** | QR‑коды обновляются примерно каждые 20 секунд. Если время истекло, перезапусти `hermes whatsapp`. |
| **Сессия не сохраняется** | Проверь, что каталог `~/.hermes/platforms/whatsapp/session` существует и доступен для записи. При работе в контейнере смонтируй его как постоянный том. |
| **Неожиданный выход из системы** | WhatsApp отключает устройства после длительного простоя. Держи телефон включённым и подключённым к сети, при необходимости повторно привяжи его с помощью `hermes whatsapp`. |
| **Сбой моста или бесконечные попытки переподключения** | Перезапусти шлюз, обнови Hermes и повторно привяжи, если сессия была аннулирована изменением протокола WhatsApp. |
| **Бот перестаёт работать после обновления WhatsApp** | Обнови Hermes, чтобы получить последнюю версию моста, затем повторно привяжи. |
| **macOS: «Node.js не установлен», но node работает в терминале** | Службы launchd не наследуют PATH твоего шелла. Выполни `hermes gateway install`, чтобы сохранить текущий PATH в plist, затем `hermes gateway start`. Подробнее смотри в [документации службы шлюза](./index.md#macos-launchd). |
| **Сообщения не приходят** | Убедись, что `WHATSAPP_ALLOWED_USERS` содержит номер отправителя (с кодом страны, без `+` и пробелов), или установи `*`, чтобы разрешить всех. Задай `WHATSAPP_DEBUG=true` в `.env` и перезапусти шлюз, чтобы увидеть сырые события сообщений в `bridge.log`. |
| **Бот отвечает незнакомцам кодом сопряжения** | Добавь `whatsapp.unauthorized_dm_behavior: ignore` в `~/.hermes/config.yaml`, если хочешь, чтобы неавторизованные личные сообщения игнорировались. |
## Безопасность

:::warning
**Настрой контроль доступа** перед запуском в продакшн. Укажи `WHATSAPP_ALLOWED_USERS` с конкретными
номерами телефонов (включая код страны, без `+`), используй `*`, чтобы разрешить всем, или задай
`WHATSAPP_ALLOW_ALL_USERS=true`. Если ничего из этого не указано, шлюз **отклонит все входящие
сообщения** как меру предосторожности.
:::

По умолчанию неавторизованные личные сообщения всё равно получают ответ с кодом сопряжения. Если ты хочешь, чтобы приватный номер WhatsApp полностью молчал для посторонних, задай:

```yaml
whatsapp:
  unauthorized_dm_behavior: ignore
```

- В каталоге `~/.hermes/platforms/whatsapp/session` хранятся полные учётные данные сессии — защищай её как пароль
- Установи права доступа к директории: `chmod 700 ~/.hermes/platforms/whatsapp/session`
- Используй **выделенный номер телефона** для бота, чтобы изолировать риск от твоего личного аккаунта
- Если подозреваешь компрометацию, отвяжи устройство от WhatsApp → Settings → Linked Devices
- Номера телефонов в журналах частично скрыты, но проверь политику хранения логов
