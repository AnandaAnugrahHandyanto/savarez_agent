# Настройка Email

Hermes может получать и отвечать на письма, используя стандартные протоколы IMAP и SMTP. Отправь письмо на адрес агента, и он ответит в той же цепочке — без необходимости в специальном клиенте или API бота. Работает с Gmail, Outlook, Yahoo, Fastmail и любым провайдером, поддерживающим IMAP/SMTP.

:::info Gateway adapter only: no external dependencies
Эта страница описывает адаптер шлюза Email, который использует встроенные модули Python `imaplib`, `smtplib` и `email`. Для этого пути шлюза не требуются дополнительные пакеты или внешние сервисы.
:::

Это отдельный от встроенного [Himalaya email skill](/docs/user-guide/skills/bundled/email/email-himalaya), который позволяет агенту управлять почтой через терминальные команды и требует внешнего CLI `himalaya` и конфигурационного файла Himalaya.

| Сценарий использования | Что нужно настроить | Внешняя зависимость |
|---|---|---|
| Позволить людям писать агенту Hermes по email и получать ответы | Адаптер шлюза Email на этой странице | Нет, кроме учётной записи IMAP/SMTP |
| Позволить агенту просматривать, создавать, перемещать и управлять сообщениями из терминальных инструментов | Himalaya email skill | CLI `himalaya` и файл `~/.config/himalaya/config.toml` |

## Предварительные требования

- **Отдельная учётная запись email** для твоего агента Hermes (не используй личный email)
- **IMAP включён** в учётной записи
- **Пароль приложения**, если используешь Gmail или другого провайдера с 2FA

### Настройка Gmail

1. Включи двухфакторную аутентификацию в своём аккаунте Google
2. Перейди к [App Passwords](https://myaccount.google.com/apppasswords)
3. Создай новый пароль приложения (выбери «Mail» или «Other»)
4. Скопируй 16‑символьный пароль — он будет использоваться вместо обычного пароля

### Outlook / Microsoft 365

1. Перейди к [Security Settings](https://account.microsoft.com/security)
2. Включи 2FA, если ещё не активировано
3. Создай пароль приложения в разделе «Additional security options»
4. IMAP‑хост: `outlook.office365.com`, SMTP‑хост: `smtp.office365.com`

### Другие провайдеры

Большинство провайдеров поддерживают IMAP/SMTP. См. их документацию для получения:
- IMAP‑хост и порт (обычно порт 993 с SSL)
- SMTP‑хост и порт (обычно порт 587 с STARTTLS)
- Требуются ли пароли приложений

## Шаг 1: Настройка Hermes

Самый простой способ:

```bash
hermes gateway setup
```

Выбери **Email** в меню платформы. Мастер запросит адрес электронной почты, пароль, хосты IMAP/SMTP и разрешённых отправителей.

### Ручная настройка

Добавь в `~/.hermes/.env`:

```bash
# Required
EMAIL_ADDRESS=hermes@gmail.com
EMAIL_PASSWORD=abcd efgh ijkl mnop    # App password (not your regular password)
EMAIL_IMAP_HOST=imap.gmail.com
EMAIL_SMTP_HOST=smtp.gmail.com

# Security (recommended)
EMAIL_ALLOWED_USERS=your@email.com,colleague@work.com

# Optional
EMAIL_IMAP_PORT=993                    # Default: 993 (IMAP SSL)
EMAIL_SMTP_PORT=587                    # Default: 587 (SMTP STARTTLS)
EMAIL_POLL_INTERVAL=15                 # Seconds between inbox checks (default: 15)
EMAIL_HOME_ADDRESS=your@email.com      # Default delivery target for cron jobs
```

---

## Шаг 2: Запуск шлюза

```bash
hermes gateway              # Run in foreground
hermes gateway install      # Install as a user service
sudo hermes gateway install --system   # Linux only: boot-time system service
```

При старте адаптер:
1. Проверяет соединения IMAP и SMTP
2. Помечает все существующие сообщения во входящих как «прочитанные» (обрабатывает только новые письма)
3. Начинает опрашивать новые сообщения

## Как это работает

### Приём сообщений

Адаптер опрашивает папку INBOX IMAP на предмет **непрочитанных** сообщений с настраиваемым интервалом (по умолчанию — 15 сек). Для каждого нового письма:

- **Тема** включается в контекст (например, `[Subject: Deploy to production]`)
- **Ответные письма** (тема начинается с `Re:`) пропускают префикс темы — контекст уже установлен
- **Вложения** кэшируются локально:
  - Изображения (JPEG, PNG, GIF, WebP) → доступны инструменту зрения
  - Документы (PDF, ZIP и др.) → доступны для доступа к файлам
- **Только HTML‑письма** очищаются от тегов для получения чистого текста
- **Сообщения от самого агента** отфильтровываются, чтобы избежать циклов ответов
- **Автоматические / noreply‑отправители** игнорируются без уведомления — `noreply@`, `mailer-daemon@`, `bounce@`, `no-reply@` и письма с заголовками `Auto-Submitted`, `Precedence: bulk` или `List-Unsubscribe`

### Отправка ответов

Ответы отправляются через SMTP с правильным построением цепочки:

- Заголовки **In-Reply-To** и **References** сохраняют ветку
- Тема сохраняется с префиксом `Re:` (без двойного `Re: Re:`)
- **Message-ID** генерируется с доменом агента
- Ответы отправляются в виде обычного текста (UTF‑8)

### Вложенные файлы

Агент может отправлять вложения в ответах. Добавь `MEDIA:/path/to/file` в ответ, и файл будет прикреплён к исходящему письму.

### Пропуск вложений

Чтобы игнорировать все входящие вложения (защита от вредоносного кода или экономия трафика), добавь в `config.yaml`:

```yaml
platforms:
  email:
    skip_attachments: true
```

При включении вложения и встроенные части пропускаются до декодирования полезной нагрузки. Текст тела письма по‑прежнему обрабатывается.

## Управление доступом

Доступ по email следует той же схеме, что и остальные платформы Hermes:

1. **`EMAIL_ALLOWED_USERS` установлен** → обрабатываются только письма от указанных адресов
2. **Белый список не задан** → неизвестные отправители получают код сопряжения
3. **`EMAIL_ALLOW_ALL_USERS=true`** → принимаются письма от любого отправителя (используй с осторожностью)

:::warning
**Всегда настраивай `EMAIL_ALLOWED_USERS`.** Без этого любой, кто знает адрес агента, сможет отправлять команды. По умолчанию агент имеет доступ к терминалу.
:::

## Устранение неполадок

| Проблема | Решение |
|---------|----------|
| **"IMAP connection failed"** при старте | Проверь `EMAIL_IMAP_HOST` и `EMAIL_IMAP_PORT`. Убедись, что IMAP включён в учётной записи. Для Gmail включи его в Settings → Forwarding and POP/IMAP. |
| **"SMTP connection failed"** при старте | Проверь `EMAIL_SMTP_HOST` и `EMAIL_SMTP_PORT`. Убедись, что пароль верный (используй пароль приложения для Gmail). |
| **Письма не приходят** | Убедись, что `EMAIL_ALLOWED_USERS` содержит адрес отправителя. Проверь папку спама — некоторые провайдеры помечают автоматические ответы. |
| **"Authentication failed"** | Для Gmail необходимо использовать пароль приложения, а не обычный пароль. Сначала включи 2FA. |
| **Дублирующиеся ответы** | Убедись, что запущен только один экземпляр шлюза. Проверь `hermes gateway status`. |
| **Медленный отклик** | Интервал опроса по умолчанию — 15 сек. Уменьши его, задав `EMAIL_POLL_INTERVAL=5` для более быстрого ответа (но будет больше соединений IMAP). |
| **Ответы не образуют цепочку** | Адаптер использует заголовки In-Reply‑To. Некоторые клиенты (особенно веб‑версии) могут некорректно формировать цепочку для автоматических сообщений. |

## Безопасность

:::warning
**Используй отдельную учётную запись email.** Не используй личный почтовый ящик — пароль хранится в `.env`, а агент имеет полный доступ к ящику через IMAP.
:::

- Используй **пароли приложений** вместо основного пароля (обязательно для Gmail с 2FA)
- Задай `EMAIL_ALLOWED_USERS`, чтобы ограничить круг взаимодействующих пользователей
- Пароль хранится в `~/.hermes/.env` — защити файл (`chmod 600`)
- По умолчанию IMAP использует SSL (порт 993), а SMTP — STARTTLS (порт 587) — соединения зашифрованы

## Справочник переменных окружения

| Переменная | Обязательно | По умолчанию | Описание |
|----------|------------|--------------|----------|
| `EMAIL_ADDRESS` | Да | — | Адрес email агента |
| `EMAIL_PASSWORD` | Да | — | Пароль email или пароль приложения |
| `EMAIL_IMAP_HOST` | Да | — | Хост IMAP‑сервера (например, `imap.gmail.com`) |
| `EMAIL_SMTP_HOST` | Да | — | Хост SMTP‑сервера (например, `smtp.gmail.com`) |
| `EMAIL_IMAP_PORT` | Нет | `993` | Порт IMAP‑сервера |
| `EMAIL_SMTP_PORT` | Нет | `587` | Порт SMTP‑сервера |
| `EMAIL_POLL_INTERVAL` | Нет | `15` | Секунд между проверками входящих |
| `EMAIL_ALLOWED_USERS` | Нет | — | Список разрешённых адресов отправителей через запятую |
| `EMAIL_HOME_ADDRESS` | Нет | — | Адрес получателя по умолчанию для cron‑задач |
| `EMAIL_ALLOW_ALL_USERS` | Нет | `false` | Принимать письма от всех отправителей (не рекомендуется) |
