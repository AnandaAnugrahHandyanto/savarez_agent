# Настройка Mattermost

Hermes Agent интегрируется с Mattermost в виде бота, позволяя общаться с твоим AI‑ассистентом через личные сообщения или каналы команды. Mattermost — саморазмещаемая, открытая альтернатива Slack, которую ты запускаешь на своей инфраструктуре, полностью контролируя свои данные. Бот подключается к REST API Mattermost (v4) и WebSocket для событий в реальном времени, обрабатывает сообщения через конвейер Hermes Agent (включая использование инструментов, память и рассуждения) и отвечает мгновенно. Поддерживаются текст, вложения файлов, изображения и слеш‑команды.

Внешняя библиотека Mattermost не требуется — адаптер использует `aiohttp`, который уже является зависимостью Hermes.

Перед настройкой, вот та часть, которую интересует большинство людей: как Hermes ведёт себя, когда запущен в твоём экземпляре Mattermost.
## Как себя ведёт Hermes

| Context | Behavior |
|---------|----------|
| **DMs** | Hermes отвечает на каждое сообщение. `@mention` не требуется. Каждый DM имеет свою сессию. |
| **Public/private channels** | Hermes отвечает, когда ты `@mention` его. Без упоминания Hermes игнорирует сообщение. |
| **Threads** | Если `MATTERMOST_REPLY_MODE=thread`, Hermes отвечает в ветке под твоим сообщением. Контекст ветки остаётся изолированным от родительского канала. |
| **Shared channels with multiple users** | По умолчанию Hermes изолирует историю сессии для каждого пользователя внутри канала. Два человека, общающиеся в одном канале, не делятся одной транскрипцией, если ты явно не отключишь это. |

:::tip
Если ты хочешь, чтобы Hermes отвечал в виде ветвящихся разговоров (вложенных под твоим оригинальным сообщением), установи `MATTERMOST_REPLY_MODE=thread`. По умолчанию значение `off`, что отправляет плоские сообщения в канал.
:::

### Модель сессий в Mattermost

По умолчанию:

- каждый DM получает свою сессию
- каждая ветка получает своё пространство имён сессии
- каждый пользователь в общем канале получает свою сессию внутри этого канала

Это контролируется файлом `config.yaml`:

```yaml
group_sessions_per_user: true
```

Установи значение `false` только если ты явно хочешь одну общую беседу для всего канала:

```yaml
group_sessions_per_user: false
```

Общие сессии могут быть полезны для совместного канала, но они также означают:

- пользователи делятся ростом контекста и затратами токенов
- длительная задача одного человека, требующая большого количества инструментов, может раздувать контекст всех остальных
- запущенный процесс одного человека может прервать последующее действие другого человека в том же канале

Это руководство проведёт тебя через весь процесс настройки — от создания бота в Mattermost до отправки первого сообщения.
## Шаг 1: Включение учётных записей ботов

Учётные записи ботов должны быть включены на твоём сервере Matterlong, прежде чем ты сможешь создать их.

1. Войди в Mattermost как **System Admin**.
2. Перейди в **System Console** → **Integrations** → **Bot Accounts**.
3. Установи **Enable Bot Account Creation** в **true**.
4. Нажми **Save**.

:::info
Если у тебя нет доступа **System Admin**, попроси администратора Mattermost включить учётные записи ботов и создать их для тебя.
:::
## Шаг 2: Создать учётную запись бота

1. В Mattermost нажми меню **☰** (вверху слева) → **Integrations** → **Bot Accounts**.
2. Нажми **Add Bot Account**.
3. Заполни данные:
   - **Username**: например, `hermes`
   - **Display Name**: например, `Hermes Agent`
   - **Description**: по желанию
   - **Role**: `Member` достаточно
4. Нажми **Create Bot Account**.
5. Mattermost отобразит **bot token**. **Сразу скопируй его**.

:::warning[Token shown only once]
Токен бота отображается только один раз при создании учётной записи бота. Если ты его потеряешь, придётся генерировать заново в настройках учётной записи бота. Никогда не публикуй токен и не добавляй его в репозиторий — любой, кто получит токен, получит полный контроль над ботом.
:::

Сохрани токен в безопасном месте (например, в менеджере паролей). Он понадобится в Шаге 5.

:::tip
Можно также использовать **personal access token** вместо учётной записи бота. Перейди в **Profile** → **Security** → **Personal Access Tokens** → **Create Token**. Это удобно, если ты хочешь, чтобы Hermes публиковал сообщения от твоего собственного пользователя, а не от отдельного бота.
:::
## Шаг 3: Добавить бота в каналы

Бот должен быть участником любого канала, в котором ты хочешь, чтобы он отвечал:

1. Открой канал, в котором ты хочешь добавить бота.
2. Нажми на название канала → **Add Members**.
3. Найди имя пользователя своего бота (например, `hermes`) и добавь его.

Для личных сообщений просто открой прямое сообщение с ботом — он сразу сможет отвечать.
## Шаг 4: Найти свой Matter most User ID

Hermes Agent использует твой Mattermost User ID, чтобы контролировать, кто может взаимодействовать с ботом. Чтобы найти его:

1. Нажми на свой **аватар** (в левом верхнем углу) → **Профиль**.
2. Твой User ID отображается в диалоговом окне профиля — нажми на него, чтобы скопировать.

Твой User ID — это 26‑символьная буквенно‑цифровая строка, например `3uo8dkh1p7g1mfk49ear5fzs5c`.

:::warning
Твой User ID **не** является твоим именем пользователя. Имя пользователя отображается после `@` (например, `@alice`). User ID — это длинный буквенно‑цифровой идентификатор, который Mattermost использует внутренне.
:::

**Альтернативный способ**: ты также можешь получить свой User ID через API:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-mattermost-server/api/v4/users/me | jq .id
```

:::tip
Чтобы получить **Channel ID**: нажми на название канала → **View Info**. Channel ID будет показан в информационной панели. Он понадобится, если ты захочешь вручную задать домашний канал.
:::
## Шаг 5: Настройка Hermes Agent

### Вариант A: Интерактивная настройка (рекомендовано)

Запусти команду интерактивной настройки:

⟦HOLD_3⟩

Выбери **Mattermost**, когда будет предложено, затем вставь URL сервера, токен бота и ID пользователя, когда будет запрошено.

### Вариант B: Ручная конфигурация

Добавь следующее в файл `~/.hermes/.env`:

⟦HOLD_4⟩

Опциональные настройки поведения в `~/.hermes/config.yaml`:

⟦HOLD_5⟩

- `group_sessions_per_user: true` — обеспечивает изоляцию контекста каждого участника в общих каналах и ветках

### Запуск gateway

После настройки запусти Mattermost gateway:

⟦HOLD_6⟩

Бот подключится к твоему серверу Mattermost в течение нескольких секунд. Отправь ему сообщение — в личных сообщениях или в канале, где он добавлен — чтобы проверить.

:::tip
Ты можешь запустить `hermes gateway` в фоне или как службу systemd для постоянной работы. См. документацию по развертыванию для деталей.
:::
## Домашний канал

Ты можешь назначить «домашний канал», в который бот будет отправлять проактивные сообщения (например, вывод cron‑задач, напоминания и уведомления). Есть два способа установить его:

### С помощью слеш‑команды

Введи `/sethome` в любом канале Mattermost, где присутствует бот. Этот канал станет домашним каналом.

### Ручная настройка

Добавь следующее в свой `~/.hermes/.env`:

```bash
MATTERMOST_HOME_CHANNEL=abc123def456ghi789jkl012mn
```

Замени ID на фактический идентификатор канала (кликни по названию канала → View Info → скопируй ID).
## Режим ответа

Настройка `MATTERMOST_REPLY_MODE` управляет тем, как Hermes публикует ответы:

| Mode | Behavior |
|------|----------|
| `off` (default) | Hermes публикует обычные сообщения в канале, как обычный пользователь. |
| `thread` | Hermes отвечает в ветке под твоим исходным сообщением. Сохраняет каналы чистыми, когда происходит много обменов. |

Укажи её в файле `~/.hermes/.env`:

```bash
MATTERMOST_REPLY_MODE=thread
```
## Поведение упоминания

По умолчанию бот отвечает в каналах только при `@mentioned`. Ты можешь изменить это:

| Variable | Default | Description |
|----------|---------|-------------|
| `MATTERMOST_REQUIRE_MENTION` | `true` | Установи `false`, чтобы отвечать на все сообщения в каналах (личные сообщения всегда работают). |
| `MATTERMOST_FREE_RESPONSE_CHANNELS` | _(none)_ | Список ID каналов через запятую, в которых бот отвечает без `@mention`, даже если `MATTERMOST_REQUIRE_MENTION` включён. |

Чтобы найти ID канала в Mattermost: открой канал, нажми на заголовок названия канала и посмотри ID в URL или в деталях канала.

Когда бот `@mentioned`, упоминание автоматически удаляется из сообщения перед обработкой.
## Белый список каналов (`allowed_channels`)

Ограничивает бота фиксированным набором каналов Mattermost. Когда параметр установлен, бот **отвечает только** в каналах, чьи ID присутствуют в списке — сообщения из любых других каналов игнорируются без уведомления, даже если бот `@mentioned`.

**Личные сообщения** исключаются из этого фильтра, поэтому уполномоченные пользователи всегда могут связаться с ботом в личных сообщениях.

```yaml
mattermost:
  allowed_channels:
    - "abc123def456ghi789jkl012mno"   # #ops
    - "xyz987uvw654rst321opq098nml"   # #incident-response
```

Или через переменную окружения (через запятую):

```bash
MATTERMOST_ALLOWED_CHANNELS="abc123def456ghi789jkl012mno,xyz987uvw654rst321opq098nml"
```

Поведение:

- Пустой / не установлен → ограничений нет (полная обратная совместимость).
- Не пустой → ID канала должен быть в списке, иначе сообщение отбрасывается до выполнения любых других проверок (требование упоминания, `MATTERMOST_FREE_RESPONSE_CHANNELS` и т.п.).
- Найти ID канала можно в интерфейсе Mattermost → заголовок канала → «View Info», либо прочитать его из URL канала.

См. также: [разделение slash‑команд админа/пользователя](../../reference/slash-commands.md#permissions-and-adminuser-split).
## Устранение неполадок

### Бот не отвечает на сообщения

**Причина**: Бот не является участником канала, либо `MATTERMOST_ALLOWED_USERS` не содержит ваш User ID.

**Решение**: Добавьте бота в канал (название канала → **Add Members** → поиск бота). Убедитесь, что ваш User ID присутствует в `MATTERMOST_ALLOWED_USERS`. Перезапустите шлюз.

### Ошибки 403 Forbidden

**Причина**: Токен бота недействителен, либо у бота нет прав публиковать сообщения в канале.

**Решение**: Проверьте, что `MATTERMOST_TOKEN` в вашем файле `.env` указан правильно. Убедитесь, что учётная запись бота не была деактивирована. Проверьте, что бот добавлен в канал. Если используется персональный токен доступа, убедитесь, что у вашей учётной записи есть необходимые разрешения.

### Отключения WebSocket / циклы переподключения

**Причина**: Нестабильность сети, перезапуски сервера Mattermost или проблемы с брандмауэром/прокси, влияющие на соединения WebSocket.

**Решение**: Адаптер автоматически переподключается с экспоненциальным увеличением интервала (2 s → 60 s). Проверьте конфигурацию WebSocket на вашем сервере — обратные прокси (nginx, Apache) должны иметь настроенные заголовки обновления WebSocket. Убедитесь, что брандмауэр не блокирует соединения WebSocket на вашем сервере Mattermost.

Для nginx убедитесь, что ваша конфигурация содержит:

```nginx
location /api/v4/websocket {
    proxy_pass http://mattermost-backend;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_read_timeout 600s;
}
```

### «Failed to authenticate» при запуске

**Причина**: Токен или URL сервера указаны неверно.

**Решение**: Убедитесь, что `MATTERMOST_URL` указывает на ваш сервер Mattermost (включайте `https://`, без завершающего слеша). Проверьте, что `MATTERMOST_TOKEN` действителен — попробуйте запросить его с помощью `curl`:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-server/api/v4/users/me
```

Если в ответ вы получаете информацию о пользователе бота, токен действителен. Если возвращается ошибка, сгенерируйте токен заново.

### Бот офлайн

**Причина**: Шлюз Hermes не запущен, либо не удалось установить соединение.

**Решение**: Убедитесь, что запущен `hermes gateway`. Посмотрите вывод в терминале на предмет сообщений об ошибках. Частые причины: неверный URL, истёкший токен, недоступность сервера Mattermost.

### «User not allowed» / Бот игнорирует тебя

**Причина**: Твой User ID отсутствует в `MATTERMOST_ALLOWED_USERS`.

**Решение**: Добавь свой User ID в `MATTERMOST_ALLOWED_USERS` в файле `~/.hermes/.env` и перезапусти шлюз. Помни: User ID — это 26‑символьная буквенно‑цифровая строка, а не твой `@username`.
## Промпты для отдельных каналов

Назначай эфемерные системные промпты конкретным каналам Mattermost. Промпт внедряется во время выполнения на каждом шаге — никогда не сохраняется в историю транскрипта — поэтому изменения вступают в силу сразу.

```yaml
mattermost:
  channel_prompts:
    "channel_id_abc123": |
      You are a research assistant. Focus on academic sources,
      citations, and concise synthesis.
    "channel_id_def456": |
      Code review mode. Be precise about edge cases and
      performance implications.
```

Ключи — идентификаторы каналов Mattermost (их можно найти в URL канала или через API). Все сообщения в соответствующем канале получают промпт, внедрённый как эфемерную системную инструкцию.
## Безопасность

:::warning
Всегда задавай `MATTERMOST_ALLOWED_USERS`, чтобы ограничить, кто может взаимодействовать с ботом. Без этого шлюз отклоняет всех пользователей по умолчанию как меру предосторожности. Добавляй только ID пользователей, которым ты доверяешь — авторизованные пользователи получают полный доступ к возможностям агента, включая использование инструментов и доступ к системе.
:::

Для получения дополнительной информации о защите развертывания Hermes Agent смотри [Security Guide](../security.md).
## Примечания

- **Self-hosted friendly**: Работает с любой самостоятельной установкой Mattermost. Учётная запись Mattermost Cloud или подписка не требуются.
- **No extra dependencies**: Адаптер использует `aiohttp` для HTTP и WebSocket, который уже включён в Hermes Agent.
- **Team Edition compatible**: Совместим с Mattermost Team Edition (бесплатной) и Enterprise Edition.
