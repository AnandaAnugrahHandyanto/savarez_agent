# Weixin (WeChat)

Подключи Hermes к [WeChat](https://weixin.qq.com/) (微信), персональной платформе обмена сообщениями от Tencent. Адаптер использует **iLink Bot API** Tencent для персональных аккаунтов WeChat — это отличается от WeCom (Enterprise WeChat). Сообщения доставляются через long‑polling, поэтому публичный эндпоинт или webhook не требуются.

:::info
Этот адаптер предназначен для **персональных аккаунтов WeChat** (微信). Если нужен корпоративный вариант WeChat, смотри [адаптер WeCom](./wecom.md).
:::

:::warning iLink bot identity — обычные группы WeChat могут не работать
QR‑логин подключает Hermes к **iLink bot identity** (например, `a5ace6fd482e@im.bot`), **а не** к полностью скриптируемому обычному персональному аккаунту WeChat. Последствия:

- Идентификация iLink bot обычно **не может быть приглашена в обычные группы WeChat** так же, как обычный контакт.
- iLink, как правило, **не передаёт события обычных групп WeChat** (включая `@`‑упоминания персонального аккаунта, использованного для QR‑логина) в шлюз для большинства bot‑типов аккаунтов.
- `@`‑упоминание персонального аккаунта WeChat, использованного для сканирования QR‑кода, **не то же самое**, что `@`‑упоминание iLink bot — бот имеет отдельную идентичность.
- Настройки `WEIXIN_GROUP_POLICY` / `WEIXIN_GROUP_ALLOWED_USERS`, указанные ниже, действуют только тогда, когда iLink действительно возвращает события групп для вашего типа аккаунта. Если нет, сообщения групп никогда не дойдут до Hermes независимо от политики.

На практике большинство развертываний получают только личные сообщения (DM) к iLink bot надёжно. Если доставка в группы не работает после настройки, ограничение находится на стороне iLink, а не Hermes. Шлюз выводит `WARNING` при запуске каждый раз, когда `WEIXIN_GROUP_POLICY` установлен не в `disabled`.
:::
## Предварительные требования

- Личный аккаунт WeChat
- Пакеты Python: `aiohttp` и `cryptography`
- Отображение QR‑кода в терминале включено, если Hermes установлен с дополнительным модулем `messaging`

Установи необходимые зависимости:

```bash
pip install aiohttp cryptography
# Optional: for terminal QR code display
pip install hermes-agent[messaging]
```
## Настройка

### 1. Запусти мастер настройки

Самый простой способ подключить свой аккаунт WeChat — через интерактивный мастер настройки:

```bash
hermes gateway setup
```

Выбери **Weixin**, когда будет предложено. Мастер выполнит:

1. Запросит QR‑код у API iLink Bot
2. Отобразит QR‑код в твоём терминале (или предоставит URL)
3. Ожидет, пока ты отсканируешь QR‑код приложением WeChat на мобильном устройстве
4. Попросит подтвердить вход на телефоне
5. Сохранит учётные данные аккаунта автоматически в `~/.hermes/weixin/accounts/`

После подтверждения ты увидишь сообщение вроде:

```
微信连接成功，account_id=your-account-id
```

Мастер сохраняет `account_id`, `token` и `base_url`, чтобы их не приходилось настраивать вручную.

### 2. Настрой переменные окружения

После первого входа через QR‑код укажи как минимум ID аккаунта в `~/.hermes/.env`:

```bash
WEIXIN_ACCOUNT_ID=your-account-id

# Optional: override the token (normally auto-saved from QR login)
# WEIXIN_TOKEN=your-bot-token

# Optional: restrict access
WEIXIN_DM_POLICY=open
WEIXIN_ALLOWED_USERS=user_id_1,user_id_2

# Optional: restore legacy multiline splitting behavior
# WEIXIN_SPLIT_MULTILINE_MESSAGES=true

# Optional: home channel for cron/notifications
WEIXIN_HOME_CHANNEL=chat_id
WEIXIN_HOME_CHANNEL_NAME=Home
```

### 3. Запусти шлюз

```bash
hermes gateway
```

Адаптер восстановит сохранённые учётные данные, подключится к API iLink и начнёт длительный опрос сообщений.
## Возможности

- **Long-poll transport** — не требуется публичный эндпоинт, webhook или WebSocket
- **QR code login** — настройка подключения сканированием через `hermes gateway setup`
- **DM messaging** — настраиваемые политики доступа; групповое обмен сообщениями зависит от того, действительно ли iLink доставляет групповые события для подключённой идентичности (часто это не так для учётных записей iLink‑ботов — см. предупреждение выше)
- **Media support** — изображения, видео, файлы и голосовые сообщения
- **AES-128-ECB encrypted CDN** — автоматическое шифрование/расшифровка всех медиа‑пересылок
- **Context token persistence** — сохранение токенов контекста на диске для непрерывности ответов после перезапусков
- **Markdown formatting** — сохраняет Markdown, включая заголовки, таблицы и блоки кода, так что клиенты WeChat, поддерживающие Markdown, могут отображать его нативно
- **Smart message chunking** — сообщения остаются в одном «пузыре», пока находятся в пределах лимита; только слишком большие полезные нагрузки разбиваются на логические части
- **Typing indicators** — показывает статус «typing…» в клиенте WeChat, пока агент обрабатывает запрос
- **SSRF protection** — URL‑адреса внешних медиа проверяются перед загрузкой
- **Message deduplication** — скользящее окно в 5 минут предотвращает двойную обработку
- **Automatic retry with backoff** — восстанавливается после временных ошибок API
## Параметры конфигурации

Устанавливай их в `config.yaml` в разделе `platforms.weixin.extra`:

| Ключ | По умолчанию | Описание |
|-----|--------------|----------|
| `account_id` | — | ID аккаунта iLink Bot (обязательно) |
| `token` | — | Токен iLink Bot (обязательно, автоматически сохраняется после входа по QR) |
| `base_url` | `https://ilinkai.weixin.qq.com` | Базовый URL API iLink |
| `cdn_base_url` | `https://novac2c.cdn.weixin.qq.com/c2c` | Базовый URL CDN для передачи медиа |
| `dm_policy` | `open` | Доступ к личным сообщениям: `open`, `allowlist`, `disabled`, `pairing` |
| `group_policy` | `disabled` | Доступ к группам: `open`, `allowlist`, `disabled` |
| `allow_from` | `[]` | ID пользователей, которым разрешены личные сообщения (когда `dm_policy=allowlist`) |
| `group_allow_from` | `[]` | ID групп, которым разрешён доступ (когда `group_policy=allowlist`) |
| `split_multiline_messages` | `false` | При `true` разбивать многострочные ответы на несколько сообщений чата (устаревшее поведение). При `false` сохранять многострочные ответы в одном сообщении, если только они не превышают ограничение по длине. |
## Политики доступа

### Политика DM

Определяет, кто может отправлять прямые сообщения боту:

| Value | Behavior |
|-------|----------|
| `open` | Любой может отправлять DM боту (по умолчанию) |
| `allowlist` | Только пользовательские ID, указанные в `allow_from`, могут отправлять DM |
| `disabled` | Все DM игнорируются |
| `pairing` | Режим сопряжения (для первоначальной настройки) |

```bash
WEIXIN_DM_POLICY=allowlist
WEIXIN_ALLOWED_USERS=user_id_1,user_id_2
```

`WEIXIN_ALLOWED_USERS` — это **входящий фильтр**, а не система приглашений. QR‑вход соединяет одну учётную запись iLink‑бота с Hermes. Другие пользователи не сканируют QR‑код Hermes своими аккаунтами; они должны писать подключённому iLink‑боту/контакту через WeChat, и Hermes обработает DM только если пользовательский ID Weixin отправителя присутствует в `WEIXIN_ALLOWED_USERS`.

Практический порядок настройки:

1. Один раз сопрячьте Hermes с помощью `hermes gateway setup` и запомните подключённую учётную запись iLink‑бота.
2. Пусть каждый разрешённый пользователь отправит прямое сообщение этому боту/контакту.
3. Считайте ID отправителя из журналов шлюза или полезной нагрузки входящего события.
4. Добавьте эти ID в `WEIXIN_ALLOWED_USERS`, затем перезапустите шлюз.

Если только аккаунт, отсканировавший QR‑код, может общаться с Hermes, убедитесь, что остальные пользователи пишут непосредственно iLink‑боту, а не в личный аккаунт WeChat, который выполнил QR‑вход. iLink‑бот — отдельная идентичность, и обычная маршрутизация контактов/групп WeChat может быть ограничена поведением iLink от Tencent.

### Политика групп

Определяет, в каких группах бот отвечает **когда iLink доставляет групповые события для подключённой идентичности**. Для iLink‑ботов, вошедших через QR‑вход (например, `...@im.bot`), групповые события обычно вообще не доставляются, поэтому эта политика может не иметь эффекта — см. предупреждение о ограничениях iLink‑бота в начале страницы.

| Value | Behavior |
|-------|----------|
| `open` | Бот отвечает во всех группах (если события доставляются) |
| `allowlist` | Бот отвечает только в группах, указанных в `group_allow_from` (если события доставляются) |
| `disabled` | Все групповые сообщения игнорируются (по умолчанию) |

```bash
WEIXIN_GROUP_POLICY=allowlist
# NOTE: this is a comma-separated list of group chat IDs, NOT member user IDs,
# despite the variable name containing "USERS". Keep this in mind when configuring.
WEIXIN_GROUP_ALLOWED_USERS=group_id_1,group_id_2
```

:::note
Политика групп по умолчанию — `disabled` для Weixin (в отличие от WeCom, где по умолчанию `open`). Это сделано намеренно — личные аккаунты WeChat могут находиться во множестве групп, а iLink‑боты обычно не могут получать обычные групповые сообщения WeChat. Шлюз выводит `WARNING` при запуске, если установить `WEIXIN_GROUP_POLICY` в значение, отличное от `disabled`.
:::
## Поддержка медиа

### Входящие (получение)

Адаптер получает медиа‑вложения от пользователей, скачивает их с CDN WeChat, расшифровывает AES‑128‑ECB и кэширует локально для обработки агентом:

| Тип | Как обрабатывается |
|------|---------------------|
| **Изображения** | Скачиваются, расшифровываются AES‑128‑ECB и кэшируются как JPEG. |
| **Видео** | Скачиваются, расшифровываются AES‑128‑ECB и кэшируются как MP4. |
| **Файлы** | Скачиваются, расшифровываются AES‑128‑ECB и кэшируются. Оригинальное имя файла сохраняется. |
| **Голос** | Если доступна текстовая транскрипция, она извлекается как текст. Иначе аудио (формат SILK) скачивается и кэшируется. |

**Цитируемые сообщения:** Медиа из цитируемых (на которые отвечают) сообщений также извлекается, чтобы агент имел контекст того, на что пользователь отвечает.

### AES‑128‑ECB зашифрованный CDN

Медиа‑файлы WeChat передаются через зашифрованный CDN. Адаптер обрабатывает это прозрачно:

- **Входящие:** Зашифрованные медиа‑файлы скачиваются из CDN по URL‑адресам `encrypted_query_param`, затем расшифровываются AES‑128‑ECB с использованием ключа, предоставленного в полезной нагрузке сообщения.
- **Исходящие:** Файлы зашифровываются локально случайным ключом AES‑128‑ECB, загружаются в CDN, и зашифрованная ссылка включается в исходящее сообщение.
- Ключ AES — 16 байт (128 бит). Ключи могут приходить в виде обычного base64 или в hex‑кодировке — адаптер поддерживает оба формата.
- Для этого требуется пакет Python `cryptography`.

Настройка не требуется — шифрование и расшифровка происходят автоматически.

### Исходящие (отправка)

| Метод | Что отправляется |
|------|-------------------|
| `send` | Текстовые сообщения с форматированием Markdown |
| `send_image` / `send_image_file` | Нативные сообщения‑изображения (через загрузку в CDN) |
| `send_document` | Вложения‑файлы (через загрузку в CDN) |
| `send_video` | Видео‑сообщения (через загрузку в CDN) |

Все исходящие медиа проходят через процесс загрузки в зашифрованный CDN:

1. Генерировать случайный ключ AES‑128.
2. Зашифровать файл с помощью AES‑128‑ECB + паддинг PKCS#7.
3. Запросить URL для загрузки у API iLink (`getuploadurl`).
4. Загрузить зашифрованный контент в CDN.
5. Отправить сообщение с зашифрованной ссылкой на медиа.
## Сохранение токена контекста

iLink Bot API требует, чтобы `context_token` возвращался вместе с каждым исходящим сообщением для данного собеседника. Адаптер поддерживает дисковое хранилище токенов контекста:

- Токены сохраняются для каждой комбинации account+peer в `~/.hermes/weixin/accounts/<account_id>.context-tokens.json`
- При запуске ранее сохранённые токены восстанавливаются
- Каждое входящее сообщение обновляет сохранённый токен для отправителя
- Исходящие сообщения автоматически включают последний токен контекста

Это обеспечивает непрерывность ответов даже после перезапуска шлюза.
## Форматирование Markdown

WeChat‑клиенты, подключённые через iLink Bot API, могут напрямую отображать Markdown, поэтому адаптер сохраняет Markdown, а не переписывает его:

- **Headers** остаются заголовками Markdown (`#`, `##`, …)
- **Tables** остаются таблицами Markdown
- **Code fences** остаются блоками кода с ограничителями
- **Excessive blank lines** сворачиваются до двойных переводов строки вне блоков кода
## Разбиение сообщений

Сообщения доставляются как единое чат‑сообщение, если они помещаются в лимит платформы. Только слишком большие полезные нагрузки разбиваются для доставки:

- Максимальная длина сообщения: **4000 символов**
- Сообщения, укладывающиеся в лимит, остаются целыми, даже если содержат несколько абзацев или разрывов строк
- Слишком большие сообщения разбиваются на логических границах (абзацы, пустые строки, блоки кода)
- Блоки кода сохраняются целыми, насколько это возможно (никогда не разбиваются посередине блока, если только сам блок не превышает лимит)
- Слишком большие отдельные блоки используют базовую логику усечения адаптера
- Задержка **0.3 s** между частями предотвращает падения из‑за ограничения скорости WeChat при отправке нескольких частей
## Индикаторы набора текста

Адаптер отображает статус набора текста в клиенте WeChat:

1. Когда приходит сообщение, адаптер получает `typing_ticket` через API `getconfig`.
2. Билеты набора кэшируются на 10 минут для каждого пользователя.
3. `send_typing` отправляет сигнал начала набора; `stop_typing` — сигнал окончания набора.
4. Шлюз автоматически активирует индикаторы набора, пока агент обрабатывает сообщение.
## Длинный опрос (Long-Poll Connection)

Адаптер использует HTTP long‑polling (не WebSocket) для получения сообщений:

### Как это работает

1. **Connect:** Проверяет учётные данные и запускает цикл опроса
2. **Poll:** Вызывает `getupdates` с тайм‑аутом 35 секунд; сервер удерживает запрос, пока не поступят сообщения или не истечёт тайм‑аут
3. **Dispatch:** Входящие сообщения распределяются одновременно с помощью `asyncio.create_task`
4. **Sync buffer:** Постоянный курсор синхронизации (`get_updates_buf`) сохраняется на диск, чтобы адаптер возобновил работу с правильной позиции после перезапуска

### Поведение при повторных попытках

При ошибках API адаптер использует простую стратегию повторных попыток:

| Условие | Поведение |
|-----------|----------|
| Временная ошибка (1‑я–2‑я) | Повторить через 2 секунды |
| Повторяющиеся ошибки (3 и более) | Сделать паузу 30 секунд, затем сбросить счётчик |
| Сессия истекла (`errcode=-14`) | Пауза 10 минут (может потребоваться повторный вход) |
| Тайм‑аут | Немедленно выполнить новый опрос (обычное поведение long‑poll) |

### Дедупликация

Входящие сообщения дедуплицируются по идентификаторам сообщений в течение 5‑минутного окна. Это предотвращает двойную обработку при сетевых сбоях или перекрывающихся ответах опроса.

### Блокировка токена

Только один экземпляр **gateway** Weixin может использовать данный токен одновременно. Адаптер захватывает scoped‑lock при запуске и освобождает его при завершении. Если другой **gateway** уже использует тот же токен, запуск завершается информативным сообщением об ошибке.

## Troubleshooting

| Проблема | Решение |
|---------|-----|
| `Weixin startup failed: aiohttp and cryptography are required` | Установи оба пакета: `pip install aiohttp cryptography` |
| `Weixin startup failed: WEIXIN_TOKEN is required` | Запусти `hermes gateway setup` для завершения QR‑входа, либо задай `WEIXIN_TOKEN` вручную |
| `Weixin startup failed: WEIXIN_ACCOUNT_ID is required` | Укажи `WEIXIN_ACCOUNT_ID` в файле `.env` или запусти `hermes gateway setup` |
| `Another local Hermes gateway is already using this Weixin token` | Сначала останови другой экземпляр gateway — для одного токена разрешён только один poller |
| Session expired (`errcode=-14`) | Твоя сессия входа истекла. Запусти `hermes gateway setup` заново, чтобы отсканировать новый QR‑код |
| QR code expired during setup | QR‑код автоматически обновляется до 3 раз. Если он продолжает истекать, проверь сетевое соединение |
| Bot doesn't respond to DMs | Проверь `WEIXIN_DM_POLICY` — если он установлен в `allowlist`, отправитель должен быть в `WEIXIN_ALLOWED_USERS` |
| Bot ignores group messages | Политика групп по умолчанию `disabled`. Установи `WEIXIN_GROUP_POLICY=open` или `allowlist` — но учти, что идентичности iLink‑ботов с QR‑логином (`...@im.bot`) обычно не могут получать обычные сообщения WeChat в группах. Если в логах gateway нет сырых входящих событий для групповых сообщений, ограничение находится на стороне iLink, а не в Hermes |
| Media download/upload fails | Убедись, что установлен пакет `cryptography`. Проверь сетевой доступ к `novac2c.cdn.weixin.qq.com` |
| `Blocked unsafe URL (SSRF protection)` | URL медиа‑ресурса указывает на частный/внутренний адрес. Разрешены только публичные URL |
| Voice messages show as text | Если WeChat предоставляет транскрипцию, адаптер использует её как текст. Это ожидаемое поведение |
| Messages appear duplicated | Адаптер удаляет дубликаты по ID сообщения. Если видишь дубли, проверь, не запущено ли несколько экземпляров gateway |
| `iLink POST ... HTTP 4xx/5xx` | Ошибка API сервиса iLink. Проверь валидность токена и сетевое соединение |
| Terminal QR code doesn't render | Переустанови с extra `messaging`: `pip install hermes-agent[messaging]`. Либо открой URL, выведенный над QR‑кодом |
