# Настройка Signal

Hermes подключается к Signal через демон [signal-cli](https://github.com/AsamK/signal-cli), работающий в режиме HTTP. Адаптер передаёт сообщения в реальном времени через SSE (Server‑Sent Events) и отправляет ответы через JSON‑RPC.

Signal — самый конфиденциальный массовый мессенджер: сквозное шифрование включено по умолчанию, открытый протокол, минимальный сбор метаданных. Это делает его идеальным для рабочих процессов агентов, чувствительных к безопасности.

:::info Нет новых зависимостей Python
Адаптер Signal использует `httpx` (уже является основной зависимостью Hermes) для всей коммуникации. Дополнительные пакеты Python не требуются. Нужно лишь установить signal-cli внешне.
:::

---
## Предварительные требования

- **signal-cli** — клиент Signal на Java ([GitHub](https://github.com/AsamK/signal-cli))
- **Java 17+** runtime — требуется для signal-cli
- **Номер телефона** с установленным Signal (для привязки в качестве вторичного устройства)

### Установка signal-cli

```bash
# macOS
brew install signal-cli

# Linux (download latest release)
VERSION=$(curl -Ls -o /dev/null -w %{url_effective} \
  https://github.com/AsamK/signal-cli/releases/latest | sed 's/^.*\/v//')
curl -L -O "https://github.com/AsamK/signal-cli/releases/download/v${VERSION}/signal-cli-${VERSION}.tar.gz"
sudo tar xf "signal-cli-${VERSION}.tar.gz" -C /opt
sudo ln -sf "/opt/signal-cli-${VERSION}/bin/signal-cli" /usr/local/bin/
```

:::caution
signal-cli **не** находится в репозиториях apt или snap. Установка в Linux выше загружает напрямую из [GitHub releases](https://github.com/AsamK/signal-cli/releases).
:::

## Шаг 1: Подключи свой аккаунт Signal

Signal-cli работает как **связанное устройство** — как WhatsApp Web, но для Signal. Твой телефон остаётся основным устройством.

```bash
# Generate a linking URI (displays a QR code or link)
signal-cli link -n "HermesAgent"
```

1. Открой **Signal** на своём телефоне
2. Перейди в **Settings → Linked Devices**
3. Нажми **Link New Device**
4. Отсканируй QR‑код или введи URI

---
## Шаг 2: Запусти демон signal-cli

```bash
# Replace +1234567890 with your Signal phone number (E.164 format)
signal-cli --account +1234567890 daemon --http 127.0.0.1:8080
```

:::tip
Держи его запущенным в фоновом режиме. Можно использовать `systemd`, `tmux`, `screen` или запустить как сервис.
:::

Проверь, что он работает:

```bash
curl http://127.0.0.1:8080/api/v1/check
# Should return: {"versions":{"signal-cli":...}}
```

## Шаг 3: Настройка Hermes

Самый простой способ:

```bash
hermes gateway setup
```

Выбери **Signal** из меню платформы. Мастер выполнит:

1. Проверит, установлен ли `signal-cli`
2. Запросит HTTP‑URL (по умолчанию: `http://127.0.0.1:8080`)
3. Проверит соединение с демоном
4. Спросит номер телефона твоего аккаунта
5. Настроит разрешённых пользователей и политики доступа

### Ручная настройка

Добавь в `~/.hermes/.env`:

```bash
# Required
SIGNAL_HTTP_URL=http://127.0.0.1:8080
SIGNAL_ACCOUNT=+1234567890

# Security (recommended)
SIGNAL_ALLOWED_USERS=+1234567890,+0987654321    # Comma-separated E.164 numbers or UUIDs

# Optional
SIGNAL_GROUP_ALLOWED_USERS=groupId1,groupId2     # Enable groups (omit to disable, * for all)
SIGNAL_HOME_CHANNEL=+1234567890                  # Default delivery target for cron jobs
```

Затем запусти `gateway`:

```bash
hermes gateway              # Foreground
hermes gateway install      # Install as a user service
sudo hermes gateway install --system   # Linux only: boot-time system service
```

## Управление доступом

### Доступ к личным сообщениям (DM)

Доступ к DM следует той же схеме, что и на всех остальных платформах Hermes:

1. **`SIGNAL_ALLOWED_USERS` задан** → только эти пользователи могут отправлять сообщения
2. **Allowlist не задан** → неизвестные пользователи получают код сопряжения DM (одобрить командой `hermes pairing approve signal CODE`)
3. **`SIGNAL_ALLOW_ALL_USERS=true`** → любой может отправлять сообщения (использовать с осторожностью)

### Доступ к группам

Доступ к группам контролируется переменной окружения `SIGNAL_GROUP_ALLOWED_USERS`:

| Конфигурация | Поведение |
|--------------|-----------|
| Не задано (по умолчанию) | Все групповые сообщения игнорируются. Бот отвечает только на DM. |
| Задано идентификаторами групп | Мониторятся только указанные группы (например, `groupId1,groupId2`). |
| Задано `*` | Бот отвечает в любой группе, членом которой он является. |
## Возможности

### Вложения

Адаптер поддерживает отправку и получение медиа в обоих направлениях.

**Входящие** (user → agent):

- **Изображения** — PNG, JPEG, GIF, WebP (автоматически определяются по magic‑bytes)
- **Аудио** — MP3, OGG, WAV, M4A (голосовые сообщения транскрибируются, если настроен Whisper)
- **Документы** — PDF, ZIP и другие типы файлов

**Исходящие** (agent → user):

Агент может отправлять медиа‑файлы через теги `MEDIA:` в ответах. Поддерживаются следующие методы доставки:

- **Изображения** — `send_multiple_images` и `send_image_file` отправляют PNG, JPEG, GIF, WebP как нативные вложения Signal
- **Голос** — `send_voice` отправляет аудиофайлы (OGG, MP3, WAV, M4A, AAC) как вложения
- **Видео** — `send_video` отправляет MP4‑видеофайлы
- **Документы** — `send_document` отправляет любые типы файлов (PDF, ZIP и т.д.)

Все исходящие медиа проходят через стандартный API вложений Signal. В отличие от некоторых платформ, Signal не различает голосовые сообщения и файловые вложения на уровне протокола.

Ограничение размера вложения: **100 МБ** (в обе стороны).

:::warning
**Серверы Signal будут ограничивать частоту загрузки вложений**, адаптер использует планировщик для отправки нескольких изображений, группируя их по 32 и регулируя загрузки в соответствии с политикой серверов Signal.
:::

### Нативное форматирование, цитаты‑ответы и реакции

Сообщения Signal отображаются с **нативным форматированием** вместо буквальных символов markdown. Адаптер преобразует markdown (`**bold**`, `*italic*`, `` `code` ``, `~~strike~~`, `||spoiler||`, заголовки) в `bodyRanges` Signal, поэтому текст отображается с реальным стилем в клиенте получателя, а не как видимые `**` / `` ` `` символы.

**Цитаты‑ответы.** Когда Hermes отвечает на конкретное сообщение, он теперь публикует нативный ответ, цитирующий оригинал — тот же UI‑элемент, который видят пользователи Signal, используя «Reply». Это происходит автоматически для ответов, сгенерированных в ответ на входящее сообщение.

**Реакции.** Агент может реагировать на сообщения через стандартный API реакций; реакции отображаются в Signal как эмодзи‑реакции к соответствующему сообщению, а не как дополнительный текст.

Ничего дополнительного настраивать не требуется — функция включена по умолчанию в последних сборках signal-cli. Если версия `signal-cli` слишком старая, Hermes переходит к доставке простым текстом и выводит одноразовое предупреждение.

### Индикаторы набора текста

Бот отправляет индикаторы набора текста во время обработки сообщений, обновляя их каждые 8 секунд.

### Маскирование телефонных номеров

Все телефонные номера автоматически маскируются в логах:
- `+15551234567` → `+155****4567`
- Это применяется как к логам **gateway** Hermes, так и к глобальной системе маскирования.

### Заметка себе (один номер)

Если ты запускаешь signal-cli как **привязанное вторичное устройство** на своём собственном номере (а не отдельный номер бота), можешь взаимодействовать с Hermes через функцию Signal «Note to Self».

Просто отправь сообщение себе со своего телефона — signal-cli получит его, и Hermes ответит в том же разговоре.

**Как это работает:**
- Сообщения «Note to Self» приходят как конверты `syncMessage.sentMessage`
- Адаптер определяет, что они адресованы собственному аккаунту бота, и обрабатывает их как обычные входящие сообщения
- Защита от эхо‑ответов (отслеживание timestamps отправки) предотвращает бесконечные циклы — собственные ответы бота автоматически фильтруются

**Дополнительная конфигурация не требуется.** Это работает автоматически, пока `SIGNAL_ACCOUNT` совпадает с твоим номером телефона.

### Мониторинг состояния

Адаптер отслеживает SSE‑соединение и автоматически переподключается, если:
- Соединение прерывается (с экспоненциальным откатом: 2 s → 60 s)
- В течение 120 секунд не обнаруживается активности (отправляется ping к `signal-cli` для проверки)
## Устранение неполадок

| Проблема | Решение |
|---------|----------|
| **"Cannot reach signal-cli"** во время настройки | Убедись, что демон signal-cli запущен: `signal-cli --account +YOUR_NUMBER daemon --http 127.0.0.1:8080` |
| **Сообщения не приходят** | Проверь, что `SIGNAL_ALLOWED_USERS` содержит номер отправителя в формате E.164 (с префиксом `+`) |
| **"signal-cli not found on PATH"** | Установи signal-cli и убедись, что он находится в `PATH`, либо используй Docker |
| **Connection keeps dropping** | Проверь логи signal-cli на наличие ошибок. Убедись, что установлен Java 17+. |
| **Group messages ignored** | Настрой `SIGNAL_GROUP_ALLOWED_USERS` с конкретными ID групп, либо `*`, чтобы разрешить все группы. |
| **Bot responds to no one** | Настрой `SIGNAL_ALLOWED_USERS`, используй сопряжение в личных сообщениях или явно разреши всех пользователей через политику шлюза, если нужен более широкий доступ. |
| **Duplicate messages** | Убедись, что только один экземпляр signal-cli прослушивает твой номер телефона |
## Безопасность

:::warning
**Всегда настраивай контроль доступа.** Бот имеет доступ к терминалу по умолчанию. Без `SIGNAL_ALLOWED_USERS` или парного DM шлюз отклоняет все входящие сообщения в качестве меры предосторожности.
:::

- Номера телефонов скрыты во всех выводах журналов
- Используй парное DM или явные списки разрешённых пользователей для безопасного подключения новых пользователей
- Отключай группы, если только тебе не нужна поддержка групп, либо разрешай только те группы, которым доверяешь
- Сквозное (end‑to‑end) шифрование Signal защищает содержимое сообщений при передаче
- Данные сессии `signal-cli` в `~/.local/share/signal-cli/` содержат учётные данные аккаунта — защищай их как пароль

## Справочник переменных окружения

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `SIGNAL_HTTP_URL` | Yes | — | HTTP‑endpoint signal‑cli |
| `SIGNAL_ACCOUNT` | Yes | — | Номер телефона бота (E.164) |
| `SIGNAL_ALLOWED_USERS` | No | — | Номера телефонов/UUID, разделённые запятыми |
| `SIGNAL_GROUP_ALLOWED_USERS` | No | — | Идентификаторы групп для мониторинга, или `*` для всех (опустите, чтобы отключить группы) |
| `SIGNAL_ALLOW_ALL_USERS` | No | `false` | Разрешить взаимодействие любому пользователю (пропустить список разрешённых) |
| `SIGNAL_HOME_CHANNEL` | No | — | Целевой канал доставки по умолчанию для cron‑задач |
