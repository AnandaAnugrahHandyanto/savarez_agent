# CLI Interface

CLI Hermes Agent — это полноценный терминальный пользовательский интерфейс (TUI), а не веб‑интерфейс. Он поддерживает многострочное редактирование, автодополнение слеш‑команд, историю диалогов, прерывание и перенаправление, а также потоковый вывод инструментов. Создан для тех, кто живёт в терминале.

:::tip First-time setup
Одна команда — `hermes setup --portal` — и ты готов к `hermes chat`. См. [Nous Portal](/integrations/nous-portal).
:::

:::tip
Hermes также поставляется с современным TUI, включающим модальные оверлеи, выделение мышью и неблокирующий ввод. Запусти его с помощью `hermes --tui` — см. руководство по [TUI](tui.md).
:::
## Запуск CLI

```bash
# Start an interactive session (default)
hermes

# Single query mode (non-interactive)
hermes chat -q "Hello"

# With a specific model
hermes chat --model "anthropic/claude-sonnet-4"

# With a specific provider
hermes chat --provider nous        # Use Nous Portal
hermes chat --provider openrouter  # Force OpenRouter

# With specific toolsets
hermes chat --toolsets "web,terminal,skills"

# Start with one or more skills preloaded
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -q "open a draft PR"

# Resume previous sessions
hermes --continue             # Resume the most recent CLI session (-c)
hermes --resume <session_id>  # Resume a specific session by ID (-r)

# Verbose mode (debug output)
hermes chat --verbose

# Isolated git worktree (for running multiple agents in parallel)
hermes -w                         # Interactive mode in worktree
hermes -w -q "Fix issue #123"     # Single query in worktree
```

### Строка состояния

Постоянная строка состояния находится над областью ввода и обновляется в реальном времени:

```
 ⚕ claude-sonnet-4-20250514 │ 12.4K/200K │ [██████░░░░] 6% │ $0.06 │ 15m
```

| Элемент | Описание |
|---------|----------|
| Имя модели | Текущая модель (усекается, если длиннее 26 символов) |
| Кол‑во токенов | Токены контекста, использованные / максимальный размер окна контекста |
| Полоса контекста | Визуальный индикатор заполнения с порогами, закодированными цветом |
| Стоимость | Оценочная стоимость сессии (или `n/a` для неизвестных/бесплатных моделей) |
| 🗜️ N | **Количество сжатий контекста** — сколько раз текущая сессия была автоматически сжата. Появляется после первого сжатия. |
| ▶ N | **Активные фоновые задачи** — сколько запросов `/background` всё ещё выполняются в текущей сессии. Появляется, когда хотя бы одна задача в полёте. |
| Длительность | Прошедшее время сессии |
| ⚠ YOLO | **Предупреждение режима YOLO** — показывается, когда включён `HERMES_YOLO_MODE` (либо `hermes --yolo` при запуске, либо `/yolo` переключено в середине сессии). Отражает предупреждение в строке баннера, чтобы ты не забыл, что находишься в режиме автоодобрения. |

Строка адаптируется к ширине терминала — полный макет при ≥ 76 столбцах, компактный при 52–75, минимальный (модель + длительность, плюс значок YOLO при активности) ниже 52.

**Цветовое кодирование контекста:**

| Цвет | Порог | Значение |
|------|------|----------|
| Green | < 50 % | Много места |
| Yellow | 50–80 % | Заполняется |
| Orange | 80–95 % | Приближается к пределу |
| Red | ≥ 95 % | Близко к переполнению — рассмотрите `/compress` |

Используй `/usage` для детального разбора, включая затраты по категориям (входные vs выходные токены).

### Отображение возобновления сессии

При возобновлении предыдущей сессии (`hermes -c` или `hermes --resume <id>`), между баннером и полем ввода появляется панель «Previous Conversation», показывающая компактный обзор истории беседы. См. [Sessions — Conversation Recap on Resume](sessions.md#conversation-recap-on-resume) для деталей и настройки.
## Клавиши

| Клавиша | Действие |
|-----|--------|
| `Enter` | Отправить сообщение |
| `Alt+Enter`, `Ctrl+J` или `Shift+Enter` | Новая строка (многострочный ввод). `Shift+Enter` требует терминала, который различает её от `Enter` — см. ниже. В Windows Terminal `Alt+Enter` перехватывается терминалом (переключение полноэкранного режима); используйте `Ctrl+Enter` или `Ctrl+J`. |
| `Alt+V` | Вставить изображение из буфера обмена, если терминал поддерживает эту функцию |
| `Ctrl+V` | Вставить текст и при возможности прикрепить изображения из буфера обмена |
| `Ctrl+B` | Начать/остановить запись голоса, когда включён голосовой режим (`voice.record_key`, по умолчанию: `ctrl+b`) |
| `Ctrl+G` | Открыть текущий буфер ввода в `$EDITOR` (vim/nvim/nano/VS Code и т.д.). Сохранить и выйти, чтобы отправить отредактированный текст как следующий запрос — идеально для длинных, многоабзацных запросов. |
| `Ctrl+X Ctrl+E` | Альтернативная привязка в стиле Emacs для внешнего редактора (поведение такое же, как у `Ctrl+G`). |
| `Ctrl+C` | Прервать работу агента (двойное нажатие в течение 2 с для принудительного выхода) |
| `Ctrl+D` | Выход |
| `Ctrl+Z` | Приостановить Hermes в фоне (только Unix). Выполни `fg` в оболочке, чтобы возобновить. |
| `Tab` | Принять автодополнение (ghost text) или автодополнение slash‑команд |

**Предпросмотр вставки многострочного текста.** При вставке многострочного блока CLI выводит компактный однострочный предварительный просмотр (`[pasted: 47 lines, 1,842 chars — press Enter to send]`) вместо того, чтобы выводить весь payload в историю. Полное содержимое всё равно отправляется; это лишь визуальное улучшение.

**Удаление Markdown в окончательных ответах.** CLI удаляет самые громоздкие markdown‑ограничители и обёртки `**bold**` / `*italic*` из *окончательных* ответов агента, чтобы они отображались как читаемый текст в терминале, а не как сырой исходник. Блоки кода и списки сохраняются. Это не влияет на платформы‑шлюзы или результаты инструментов — они сохраняют свой markdown для нативного рендеринга.
## Slash Commands

Введи `/`, чтобы увидеть выпадающий список автодополнения. Hermes поддерживает большой набор CLI slash‑команд, динамических команд навыков и пользовательских быстрых команд.

Типичные примеры:

| Command | Description |
|---------|-------------|
| `/help` | Показать справку по командам |
| `/model` | Показать или изменить текущую модель |
| `/tools` | Вывести список доступных в данный момент инструментов |
| `/skills browse` | Просмотреть хаб навыков и официальные опциональные навыки |
| `/background <prompt>` | Выполнить prompt в отдельной фоновой сессии |
| `/skin` | Показать или переключить активный CLI‑скин |
| `/voice on` | Включить голосовой режим CLI (нажми `Ctrl+B` для записи) |
| `/voice tts` | Переключить озвучивание ответов Hermes |
| `/reasoning high` | Увеличить уровень рассуждения |
| `/title My Session` | Задать имя текущей сессии |
| `/status` | Показать информацию о сессии — модель/профиль/токены/длительность — затем блок локального **Session recap** (количество последних ходов, топ‑инструменты, затронутые файлы, последний запрос пользователя + ответ ассистента). Чисто локальные вычисления; без вызова LLM. |
| `/sessions` | Открыть интерактивный выбор сессий прямо в классическом CLI (тот же интерфейс, что использует TUI). Вводи для фильтрации, стрелками перемещайся, Enter — возобновить. |

Для полного списка встроенных CLI и сообщений смотри [Slash Commands Reference](../reference/slash-commands.md).

Для настройки, провайдеров, настройки тишины и использования голосового режима и Discord смотри [Voice Mode](features/voice-mode.md).

:::tip
Команды нечувствительны к регистру — `/HELP` работает так же, как `/help`. Установленные навыки также автоматически становятся slash‑командами.
:::
## Быстрые команды

Ты можешь определить пользовательские команды, которые мгновенно выполняют shell‑команды без обращения к LLM. Они работают как в CLI, так и в платформах обмена сообщениями (Telegram, Discord и т.д.).

```yaml
# ~/.hermes/config.yaml
quick_commands:
  status:
    type: exec
    command: systemctl status hermes-agent
  gpu:
    type: exec
    command: nvidia-smi --query-gpu=utilization.gpu,memory.used --format=csv,noheader
  restart:
    type: alias
    target: /gateway restart
```

Затем введи `/status`, `/gpu` или `/restart` в любом чате. Смотри [Руководство по конфигурации](/user-guide/configuration#quick-commands) для дополнительных примеров.
## Предзагрузка навыков при запуске

Если ты уже знаешь, какие навыки нужно активировать для сессии, передай их при запуске:

```bash
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -s github-auth
```

Hermes загружает каждый указанный навык в промпт сессии до первого хода. Тот же флаг работает в интерактивном режиме и в режиме одиночного запроса.
## Слеш‑команды навыков

Каждый установленный навык в `~/.hermes/skills/` автоматически регистрируется как слеш‑команда. Имя навыка становится командой:

```
/gif-search funny cats
/axolotl help me fine-tune Llama 3 on my dataset
/github-pr-workflow create a PR for the auth refactor

# Just the skill name loads it and lets the agent ask what you need:
/excalidraw
```
## Личности

Установи предопределённую личность, чтобы изменить тон агента:

```
/personality pirate
/personality kawaii
/personality concise
```

Встроенные личности включают: `helpful`, `concise`, `technical`, `creative`, `teacher`, `kawaii`, `catgirl`, `pirate`, `shakespeare`, `surfer`, `noir`, `uwu`, `philosopher`, `hype`.

Ты также можешь определить собственные личности в `~/.hermes/config.yaml`:

```yaml
personalities:
  helpful: "You are a helpful, friendly AI assistant."
  kawaii: "You are a kawaii assistant! Use cute expressions..."
  pirate: "Arrr! Ye be talkin' to Captain Hermes..."
  # Add your own!
```
## Ввод многострочного текста

Существует два способа вводить многострочные сообщения:

1. **`Alt+Enter`, `Ctrl+J` или `Shift+Enter`** — вставляет новую строку
2. **Продолжение с помощью обратного слеша** — завершите строку символом `\`, чтобы продолжить:

```
❯ Write a function that:\
  1. Takes a list of numbers\
  2. Returns the sum
```

:::info
Вставка многострочного текста поддерживается — используй любую из указанных клавиш для перехода на новую строку или просто вставь содержимое напрямую.
:::

### Совместимость Shift+Enter

Большинство терминалов по умолчанию отправляют одинаковую последовательность байтов для `Enter` и `Shift+Enter`, поэтому приложения не могут их различать. Hermes распознаёт `Shift+Enter` только когда терминал отправляет отдельную последовательность через [протокол клавиатуры Kitty](https://sw.kovidgoyal.net/kitty/keyboard-protocol/) или режим `modifyOtherKeys` в xterm.

| Терминал | Статус |
|---|---|
| Kitty, foot, WezTerm, Ghostty | Отдельный `Shift+Enter` включён по умолчанию |
| iTerm2 (недавние версии), Alacritty, терминал VS Code, Warp | Поддерживается после включения протокола Kitty в настройках |
| Windows Terminal Preview 1.25+ | Поддерживается после включения протокола Kitty в настройках |
| macOS Terminal.app, штатный Windows Terminal (stable) | Не поддерживается — `Shift+Enter` неотличим от `Enter` |

Если терминал не может их различать, `Alt+Enter` и `Ctrl+J` продолжают работать везде. **В Windows Terminal конкретно `Alt+Enter` перехватывается терминалом (переключает полноэкранный режим) и никогда не попадает в Hermes — используй `Ctrl+Enter` (передаётся как `Ctrl+J`) или напрямую `Ctrl+J` для новой строки.**
## Прерывание агента

Ты можешь прервать агента в любой момент:

- **Набери новое сообщение + Enter**, пока агент работает — он прерывается и обрабатывает твои новые инструкции
- **`Ctrl+C`** — прервать текущую операцию (нажми дважды в течение 2 с, чтобы принудительно выйти)
- Выполняющиеся в терминале команды завершаются немедленно (SIGTERM, затем SIGKILL через 1 с)
- Несколько сообщений, набранных во время прерывания, объединяются в один запрос

### Режим ввода при занятости

Ключ конфигурации `display.busy_input_mode` определяет, что происходит, когда ты нажимаешь **Enter**, пока агент работает:

| Mode | Behavior |
|------|----------|
| `"interrupt"` (по умолчанию) | Твоё сообщение прерывает текущую операцию и обрабатывается сразу |
| `"queue"` | Твоё сообщение тихо ставится в очередь и отправляется как следующий ход после завершения агента |
| `"steer"` | Твоё сообщение внедряется в текущий запуск через `/steer`, попадая к агенту после следующего вызова инструмента — без прерывания, без нового хода |

```yaml
# ~/.hermes/config.yaml
display:
  busy_input_mode: "steer"   # or "queue" or "interrupt" (default)
```

Режим `"queue"` полезен, когда нужно подготовить последующие сообщения, не отменяя случайно уже запущенную работу. Режим `"steer"` полезен, когда нужно перенаправить агента в середине задачи без прерывания — например, «кстати, тоже проверь тесты», пока он ещё редактирует код. Неизвестные значения возвращаются к `"interrupt"`.

У `"steer"` есть два автоматических запасных варианта: если агент ещё не начал работу, или если к сообщению прикреплены изображения, сообщение переходит в поведение `"queue"`, чтобы ничего не потерять.

Ты также можешь изменить это внутри CLI:

```text
/busy queue
/busy steer
/busy interrupt
/busy status
```

:::tip Подсказка при первом использовании
Самый первый раз, когда ты нажимаешь **Enter**, пока Hermes работает, Hermes выводит однострочное напоминание о переключателе `/busy` (`"(tip) Your message interrupted the current run…"`) . Оно срабатывает только один раз за установку — флаг в `config.yaml` под `onboarding.seen.busy_input_prompt` фиксирует это. Удали этот ключ, чтобы увидеть подсказку снова.
:::

### Приостановка в фоне

В системах Unix нажми **`Ctrl+Z`**, чтобы приостановить Hermes в фоне — так же, как любой процесс в терминале. Оболочка выводит подтверждение:

```
Hermes Agent has been suspended. Run `fg` to bring Hermes Agent back.
```

Введи `fg` в оболочке, чтобы возобновить сессию точно там, где ты остановился. Это не поддерживается в Windows.
## Отображение прогресса инструмента

CLI показывает анимированную обратную связь, пока агент работает:

**Анимация размышления** (во время API‑вызовов):
```
  ◜ (｡•́︿•̀｡) pondering... (1.2s)
  ◠ (⊙_⊙) contemplating... (2.4s)
  ✧٩(ˊᗜˋ*)و✧ got it! (3.1s)
```

**Лента выполнения инструмента:**
```
  ┊ 💻 terminal `ls -la` (0.3s)
  ┊ 🔍 web_search (1.2s)
  ┊ 📄 web_extract (2.1s)
```

Переключай режимы отображения с помощью `/verbose`: `off → new → all → verbose`. Эта команда также может быть включена для платформ обмена сообщениями — см. [configuration](/user-guide/configuration#display-settings).

### Длина предварительного просмотра инструмента

Ключ конфигурации `display.tool_preview_length` управляет максимальным количеством символов, показываемых в строках предварительного просмотра вызовов инструмента (например, пути к файлам, команды терминала). По умолчанию `0`, что означает отсутствие ограничения — показываются полные пути и команды.

```yaml
# ~/.hermes/config.yaml
display:
  tool_preview_length: 80   # Truncate tool previews to 80 chars (0 = no limit)
```

Это полезно в узких терминалах или когда аргументы инструмента содержат очень длинные пути к файлам.
## Управление сессиями

### Возобновление сессий

При выходе из CLI‑сессии выводится команда `resume`:

```
Resume this session with:
  hermes --resume 20260225_143052_a1b2c3

Session:        20260225_143052_a1b2c3
Duration:       12m 34s
Messages:       28 (5 user, 18 tool calls)
```

Опции возобновления:

```bash
hermes --continue                          # Resume the most recent CLI session
hermes -c                                  # Short form
hermes -c "my project"                     # Resume a named session (latest in lineage)
hermes --resume 20260225_143052_a1b2c3     # Resume a specific session by ID
hermes --resume "refactoring auth"         # Resume by title
hermes -r 20260225_143052_a1b2c3           # Short form
```

Возобновление восстанавливает полную историю разговора из SQLite. Агент видит все предыдущие сообщения, вызовы инструментов и ответы — так же, как если бы ты никогда не выходил.

Используй `/title My Session Name` внутри чата, чтобы задать название текущей сессии, или `hermes sessions rename <id> <title>` из командной строки. Команда `hermes sessions list` позволяет просматривать прошлые сессии.

### Хранение сессий

CLI‑сессии сохраняются в базе данных SQLite состояния Hermes по пути `~/.hermes/state.db`. База данных содержит:

- метаданные сессии (ID, название, метки времени, счётчики токенов)
- историю сообщений
- родословную между сжатыми/возобновлёнными сессиями
- индексы полнотекстового поиска, используемые `session_search`

Некоторые адаптеры обмена сообщениями также сохраняют файлы транскриптов для каждой платформы рядом с базой данных, но сам CLI возобновляет работу из хранилища сессий SQLite.

### Сжатие контекста

Длинные разговоры автоматически суммируются при приближении к ограничениям контекста:

```yaml
# In ~/.hermes/config.yaml
compression:
  enabled: true
  threshold: 0.50    # Compress at 50% of context limit by default

# Summarization model configured under auxiliary:
auxiliary:
  compression:
    model: ""  # Leave empty to use the main chat model (default). Or pin a cheap fast model, e.g. "google/gemini-3-flash-preview".
```

Когда срабатывает сжатие, средние реплики суммируются, а первые 3 и последние 20 реплик всегда сохраняются.
## Фоновые сессии

Запусти промпт в отдельной фоновой сессии, продолжая использовать CLI для другой работы:

```
/background Analyze the logs in /var/log and summarize any errors from today
```

Hermes сразу подтверждает задачу и возвращает тебе промпт:

```
🔄 Background task #1 started: "Analyze the logs in /var/log and summarize..."
   Task ID: bg_143022_a1b2c3
```

### Как это работает

Каждый промпт `/background` создаёт **полностью отдельную сессию агента** в daemon‑потоке:

- **Изолированный диалог** — у фонового агента нет доступа к истории текущей сессии. Он получает только предоставленный тобой промпт.
- **Та же конфигурация** — фоновый агент наследует твою модель, провайдера, наборы инструментов, настройки рассуждений и запасную модель из текущей сессии.
- **Неблокирующий** — твоя основная сессия остаётся полностью интерактивной. Ты можешь общаться, выполнять команды или даже запускать новые фоновые задачи.
- **Несколько задач** — можно запускать несколько фоновых задач одновременно. Каждая получает нумерованный идентификатор.

### Результаты

Когда фоновая задача завершается, результат появляется в виде панели в терминале:

```
╭─ ⚕ Hermes (background #1) ──────────────────────────────────╮
│ Found 3 errors in syslog from today:                         │
│ 1. OOM killer invoked at 03:22 — killed process nginx        │
│ 2. Disk I/O error on /dev/sda1 at 07:15                      │
│ 3. Failed SSH login attempts from 192.168.1.50 at 14:30      │
╰──────────────────────────────────────────────────────────────╯
```

Если задача завершилась с ошибкой, вместо этого будет показано уведомление об ошибке. Если в конфигурации включён `display.bell_on_complete`, при завершении задачи прозвенит звуковой сигнал терминала.

### Сценарии использования

- **Длительные исследования** — `"/background research the latest developments in quantum error correction"` пока ты работаешь над кодом
- **Обработка файлов** — `"/background analyze all Python files in this repo and list any security issues"` пока продолжаешь диалог
- **Параллельные расследования** — запускай несколько фоновых задач, чтобы одновременно исследовать разные аспекты

:::info
Фоновые сессии не отображаются в истории основной беседы. Это отдельные сессии со своим идентификатором задачи (например, `bg_143022_a1b2c3`).
:::
## Тихий режим

По умолчанию CLI работает в тихом режиме, который:
- Подавляет подробный вывод журналов инструментов
- Включает анимированную обратную связь в стиле «кавайи»
- Сохраняет вывод чистым и удобным для пользователя

Для вывода отладочной информации:
```bash
hermes chat --verbose
```
