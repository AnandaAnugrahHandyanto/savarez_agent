# TUI

TUI — это современный фронтенд для Hermes — терминальный UI, работающий на том же Python‑рантайме, что и [Classic CLI](cli.md). Тот же агент, те же сессии, те же слеш‑команды; более чистый и отзывчивый интерфейс для взаимодействия с ними.

Это рекомендуемый способ интерактивного запуска Hermes.
## Запуск

```bash
# Launch the TUI
hermes --tui

# Resume the latest TUI session (falls back to the latest classic session)
hermes --tui -c
hermes --tui --continue

# Resume a specific session by ID or title
hermes --tui -r 20260409_000000_aa11bb
hermes --tui --resume "my t0p session"

# Run source directly — skips the prebuild step (for TUI contributors)
hermes --tui --dev
```

Ты также можешь включить его через переменную окружения:

```bash
export HERMES_TUI=1
hermes          # now uses the TUI
hermes chat     # same
```

Классический CLI остаётся доступным по умолчанию. Всё, что задокументировано в [CLI Interface](cli.md) — slash‑команды, быстрые команды, предзагрузка навыков, персоны, ввод нескольких строк, прерывания — работает в TUI точно так же.
## Почему TUI

- **Instant first frame** — баннер отрисовывается до того, как приложение полностью загрузилось, поэтому терминал никогда не кажется «зависшим», пока Hermes запускается.
- **Non-blocking input** — вводи и ставь в очередь сообщения, пока сессия ещё не готова. Твой первый запрос отправляется в тот момент, когда агент выходит в онлайн.
- **Rich overlays** — выбор модели, выбор сессии, запросы подтверждения и уточнения отображаются в виде модальных панелей, а не как встроенные потоки.
- **Live session panel** — инструменты и навыки заполняются постепенно по мере инициализации.
- **Mouse-friendly selection** — выделяй, перетаскивая мышью, с однородным фоном вместо обратного SGR. Копируй обычным способом копирования в твоём терминале.
- **Alternate-screen rendering** — дифференциальные обновления устраняют мерцание при потоковой передаче и не оставляют лишних записей в скроллбэке после выхода.
- **Composer affordances** — встроенное сворачивание вставок для длинных фрагментов, `Cmd+V` / `Ctrl+V` вставка текста с резервным вариантом изображения из буфера, безопасность при вставке в скобках и нормализация вложений изображений/путей к файлам.

Те же [скины](features/skins.md) и [персоны](features/personality.md) работают. Переключай их в середине сессии с помощью `/skin ares`, `/personality pirate`, и интерфейс перерисовывается в реальном времени. Смотри [Skins & Themes](features/skins.md) для полного списка настраиваемых клавиш и того, какие из них применяются к классическому режиму и к TUI — TUI учитывает палитру баннера, цвета UI, глиф/цвет подсказки, отображение сессии, меню автодополнения, фон выделения, `tool_prefix` и `help_header`.

### Сворачиваемые секции баннера

Баннер при запуске TUI группирует информацию о среде выполнения в четыре сворачиваемые секции, каждая из которых отображается с chevron `▸` / `▾` рядом с заголовком секции:

| Section | Default state |
|---------|---------------|
| Tools | Open |
| Skills | Collapsed |
| System Prompt | Collapsed |
| MCP Servers | Collapsed |

Кликни в любой части заголовка секции (или её chevron), чтобы переключить её состояние. Список **Tools** открывается по умолчанию, потому что это самая часто проверяемая секция при старте сессии; **Skills**, **System Prompt** и **MCP Servers** свернуты по умолчанию, чтобы баннер оставался компактным даже при установленном десятках навыков или подключённых множестве MCP‑серверов. Состояние хранится локально в экземпляре баннера, поэтому при следующем запуске оно сбрасывается к значениям по умолчанию.
## Требования

- **Node.js** ≥ 20 — TUI запускается как подпроцесс, запущенный из Python CLI. `hermes doctor` проверяет это.
- **TTY** — как и в классическом CLI, передача stdin через конвейер или запуск в неинтерактивных средах приводит к переходу в режим одиночного запроса.

При первом запуске Hermes устанавливает Node‑зависимости TUI в `ui-tui/node_modules` (один раз, несколько секунд). Последующие запуски происходят быстро. Если ты обновляешь Hermes до новой версии, bundle TUI пересобирается автоматически, когда исходники новее, чем `dist`.

### External prebuild

Дистрибутивы, поставляющие предсобранный bundle (Nix, системные пакеты), могут указать Hermes путь к нему:

```bash
export HERMES_TUI_DIR=/path/to/prebuilt/ui-tui
hermes --tui
```

Каталог должен содержать `dist/entry.js`.
## Комбинации клавиш

Комбинации клавиш полностью соответствуют [Classic CLI](cli.md#keybindings). Единственные различия в поведении:

- **Перетаскивание мышью** выделяет текст с одинаковым фоном выделения.
- **`Cmd+V` / `Ctrl+V`** сначала пытаются обычную вставку текста, затем используют чтение буфера обмена через OSC52/нативный буфер, и в конце прикрепляют изображение, если буфер обмена или вставляемые данные представляют собой изображение.
- **`/terminal-setup`** устанавливает локальные привязки терминала VS Code / Cursor / Windsurf для лучшего `Cmd+Enter` и одинакового поведения undo/redo на macOS.
- **Автодополнение со слешем** открывается во всплывающей панели с описаниями, а не в виде встроенного выпадающего списка.
- **`Ctrl+X`** открывает переключатель живой сессии. Когда выделено отложенное сообщение (отправленное, пока агент ещё работал), оно всё равно удаляется. **`Esc`** отменяет редактирование и снимает выделение без удаления.
- **`Ctrl+G` / `Ctrl+X Ctrl+E`** — открыть текущий буфер ввода в `$EDITOR` для составления многострочного / длинного запроса; сохранение и выход отправляют содержимое обратно как запрос.
## Slash commands

Все slash‑команды работают без изменений. Некоторые из них принадлежат TUI — они выводят более насыщенный результат или отображаются как оверлей, а не как встроенные панели:

| Command | TUI behavior |
|---------|--------------|
| `/help` | Оверлей с категоризированными командами, навигация клавишами‑стрелками |
| `/sessions` (alias `/switch`) | Живой переключатель сессий — список открытых TUI‑сессий, переключение между ними, закрытие их или запуск новой |
| `/model` | Модальное окно выбора модели, сгруппированное по провайдеру, с подсказками о стоимости |
| `/skin` | Живой предпросмотр — изменение темы применяется сразу при просмотре |
| `/details` | Переключение подробных деталей вызова инструмента (глобально или для отдельного раздела) |
| `/usage` | Информативная панель токенов / стоимости / контекста |
| `/agents` (alias `/tasks`) | Оверлей наблюдаемости — живое дерево субагентов с элементами управления «kill/pause», сводками стоимости / токенов / файлов по ветвям, история ходов |
| `/reload` | Перечитывает `~/.hermes/.env` в работающий процесс TUI, чтобы недавно добавленные API‑ключи вступили в силу без перезапуска |
| `/mouse [on\|off\|toggle\|wheel\|buttons\|all]` | Выбор пресета отслеживания мыши во время работы (также сохраняется в `display.mouse_tracking` в `config.yaml`). `wheel` (1000+1006) сохраняет прокрутку колесика без событий наведения, которые заставляют tmux спамить «No image in clipboard» над строкой подсказки; `buttons` добавляет перетаскивание для выделения; `all` — значение по умолчанию с UI, управляемым наведением. |

Все остальные slash‑команды (включая установленные skills, быстрые команды и переключатели персональностей) работают так же, как в классическом CLI. См. [Slash Commands Reference](../reference/slash-commands.md).
## Переключатель живых сессий

Используй переключатель живых сессий, когда нужен один терминал в роли диспетчера для нескольких TUI‑сессий. Он отображает только те сессии, которые сейчас активны в этом процессе TUI; закрытые сессии остаются сохранёнными транскриптами и их всё ещё можно открыть с помощью `/resume` или `hermes --tui --resume <id-or-title>`.

Открой его любым из способов:

- `Ctrl+X` из TUI.
- `/sessions` или `/switch`.
- `/sessions new` — создать новую живую сессию сразу.
- Кликни по счётчику `N live sessions` в строке состояния.

Внутри переключателя:

- `↑` / `↓` — перемещают выделение; клики мышью тоже выбирают строки.
- `Enter` — переключает на выбранную живую сессию.
- `Ctrl+D` — закрывает выбранную живую сессию.
- `Ctrl+N` — запускает пустую живую сессию.
- `Ctrl+R` — обновляет список живых сессий.
- `Esc` — закрывает переключатель.
- Выбери `+new`, введи запрос и нажми `Enter`, чтобы запустить новую живую сессию. Сначала нажми `Tab`, если хочешь выбрать модель только для этой новой сессии.
## Рендеринг LaTeX‑математики

Markdown‑конвейер TUI рендерит LaTeX‑математику **inline**: `$E = mc^2$` и `$$\frac{a}{b}$$` выводятся в виде Unicode‑форматированной формулы вместо исходного TeX‑кода. Работает как для встроенной, так и для блочной математики; неподдерживаемый синтаксис переходит в **запасной вариант** — отображается буквальный TeX, обёрнутый в `code span`, чтобы его можно было скопировать.

Это всегда включено — настройки не требуются. Классический CLI оставляет сырой TeX.
## Обнаружение светлого терминала

TUI автоматически определяет светлые терминалы и переключается на светлую тему соответственно. Обнаружение работает в трёх уровнях:

1. `HERMES_TUI_THEME` — переменная окружения с наивысшим приоритетом. Значения: `light`, `dark` или сырой 6‑символьный hex‑код фона (например, `ffffff`, `1a1a2e`).
2. `COLORFGBG` — переменная окружения, классическая подсказка «какой у меня цвет фона?», используемая терминалами, основанными на xterm.
3. Проверка фона терминала через OSC 11 — работает в современных терминалах (Ghostty, Warp, iTerm2, WezTerm, Kitty), которые не задают `COLORFGBG`.

Если ты хочешь использовать светлую тему постоянно, независимо от терминала:

```bash
export HERMES_TUI_THEME=light
```
## Стили индикатора занятости

Индикатор занятости в строке состояния — подключаемый: по умолчанию каждые 2,5 секунды во время работы агента вращает палитру «kawaii‑лиц» Hermes. Выбери другой стиль через конфигурацию или слеш‑команду `/indicator`:

```yaml
display:
  tui_status_indicator: kaomoji   # kaomoji | emoji | unicode | ascii
```

Или в текущей сессии: `/indicator emoji` (и т.д.). Стили поставляются с одинаковой шириной глифов, поэтому остальная часть строки состояния не дрожит при вращении.
## Автоматическое возобновление

По умолчанию `hermes --tui` запускает новую сессию при каждом старте. Чтобы автоматически повторно подключаться к последней TUI‑сессии (это полезно, когда терминал или SSH‑соединение неожиданно обрывается), включи её:

```bash
export HERMES_TUI_RESUME=1          # most-recent TUI session
# or:
export HERMES_TUI_RESUME=<session-id>   # specific session
```

Сними значение переменной или явно передай `--resume <id>`, чтобы переопределить поведение при отдельном запуске.
## Строка состояния

Строка состояния TUI отслеживает состояние агента в реальном времени:

| Статус | Значение |
|--------|----------|
| `starting agent…` | ID сессии активен; инструменты и навыки ещё подключаются. Ты можешь печатать — сообщения ставятся в очередь и отправляются, когда будешь готов. |
| `ready` | Агент находится в режиме ожидания, принимает ввод. |
| `thinking…` / `running…` | Агент рассуждает или запускает инструмент. |
| `interrupted` | Текущий ход был отменён; нажми **Enter**, чтобы отправить снова. |
| `forging session…` / `resuming…` | Первоначальное подключение или рукопожатие `--resume`. |

Цвета и пороги строки состояния для каждой темы совместно используются с классическим CLI — смотри [Skins](features/skins.md) для настройки.

Строка состояния также показывает:

- **Текущий каталог с веткой git** — `~/projects/hermes-agent (docs/two-week-gap-sweep)`. Суффикс ветки обновляется при `git checkout` в боковом терминале (кешировано по времени изменения), поэтому TUI отражает фактическую активную ветку, а не ту, что была при запуске.
- **Время, прошедшее с момента подсказки** — `⏱ 12s/3m 45s` пока ход выполняется (в реальном времени), замораживается в `⏲ 32s / 3m 45s` после завершения хода. Первое число — время с последнего сообщения пользователя; второе — общая продолжительность сессии. Сбрасывается при каждой новой подсказке.
- **`🗜️ N`** — количество автоматических сжатий текущей сессии. Появляется после первого сжатия.
- **`▶ N`** — количество задач `/background`, запущенных в этой сессии. Появляется, когда хотя бы одна задача находится в процессе выполнения.
- **`⚠ YOLO`** — видимое предупреждение, когда включён режим YOLO (`hermes --yolo`, `/yolo` или `HERMES_YOLO_MODE=1`). Та же метка отображается в стартовом баннере, поэтому нельзя запустить сессию с автоматическим одобрением, не заметив этого.
## Конфигурация

TUI учитывает всю стандартную конфигурацию Hermes: `~/.hermes/config.yaml`, профили, personalities, скины, быстрые команды, пул учётных данных, провайдеры памяти, включение инструментов/навыков. Специфичного файла конфигурации для TUI не существует.

Небольшой набор ключей настраивает поверхность TUI специально:

```yaml
display:
  skin: default              # any built-in or custom skin
  personality: helpful
  details_mode: collapsed    # hidden | collapsed | expanded — global accordion default
  sections:                  # optional: per-section overrides (any subset)
    thinking: expanded       # always open
    tools: expanded          # always open
    activity: collapsed      # opt back IN to the activity panel (hidden by default)
  mouse_tracking: all        # off | wheel | buttons | all (or true/false for back-compat).
                             #   wheel   — 1000+1006 (scroll + click; no drag, no hover —
                             #             recommended inside tmux to silence the prompt-row
                             #             "No image in clipboard" spam from hover events)
                             #   buttons — adds 1002 for terminal-side drag selection
                             #   all     — adds 1003 for hover (scrollbar paginate-on-hover,
                             #             link mouseenter, etc.)
```

Переключатели во время выполнения:

- `/details [hidden|collapsed|expanded|cycle]` — установить глобальный режим
- `/details <section> [hidden|collapsed|expanded|reset]` — переопределить один раздел
  (разделы: `thinking`, `tools`, `subagents`, `activity`)

**Видимость по умолчанию**

TUI поставляется с продуманными значениями по умолчанию для каждого раздела, которые делают ход диалога живой транскрипцией вместо стены стрелок:

- `thinking` — **expanded**. Поток рассуждений выводится встроенно, пока модель его генерирует.
- `tools` — **expanded**. Вызовы инструментов и их результаты отображаются открыто.
- `subagents` — наследует глобальный `details_mode` (по умолчанию **collapsed** под стрелкой — остается тихим, пока делегирование действительно не произойдёт).
- `activity` — **hidden**. Фоновая мета‑информация (подсказки шлюза, подсказки терминала, фоновые уведомления) является шумом для большинства ежедневных задач. Ошибки инструментов всё равно выводятся встроенно в строке с ошибкой; фоновые ошибки/предупреждения появляются через плавающий alert, когда все панели скрыты.

Переопределения для отдельных разделов имеют приоритет над значением по умолчанию раздела и над глобальным `details_mode`. Чтобы изменить раскладку:

- `display.sections.thinking: collapsed` — вернуть рассуждения под стрелку
- `display.sections.tools: collapsed` — вернуть вызовы инструментов под стрелку
- `display.sections.activity: collapsed` — включить панель активности
- `/details <section> <mode>` во время выполнения

Любое явное значение в `display.sections` переходит над значениями по умолчанию, поэтому существующие конфигурации продолжают работать без изменений.
## Сессии

Сессии общие для TUI и классического CLI — оба пишут в один и тот же `~/.hermes/state.db`. Можно запустить сессию в одном интерфейсе, а продолжить её в другом. Выборщик сессий отображает сессии из обоих источников, помечая их тегом источника.

См. [Сессии](sessions.md) для сведения о жизненном цикле, поиске, сжатии и экспорте.
## Подключение к запущенному gateway

По умолчанию TUI запускает собственный in‑process gateway, поэтому каждый экземпляр TUI автономен. Если у тебя уже работает длительно живущий gateway (например, `hermes gateway run` в tmux или сервис systemd / launchd), можешь направить TUI к этому gateway — тогда TUI становится тонким клиентом и делит состояние со всеми другими surface (messaging‑платформами, web‑dashboard, другими TUI‑сессиями), подключёнными к тому же gateway.

Установи URL websocket через переменную окружения перед запуском:

```bash
export HERMES_TUI_GATEWAY_URL="ws://localhost:8765/api/ws?token=<auth-token>"
hermes --tui
```

Токен берётся из конфигурации API‑auth gateway (см. [API Server](features/api-server.md)). Когда переменная окружения задана, TUI:

- Полностью пропускает запуск локального gateway — никаких дублирующих адаптеров платформ, конфликтов портов.
- Маршрутизирует каждое действие (slash‑команды, прикрепление изображений, прогресс браузера, голосовые события, …) через websocket к общему gateway.
- Автоматически переподключается, если URL gateway меняется (новый токен) между запросами.

Это тот же канал, который использует встроенный в web‑dashboard TUI (см. [Web Dashboard](features/web-dashboard.md#chat)) — один gateway, множество клиентов.
## Возврат к классическому CLI

Запуск `hermes` (без `--tui`) остаётся в классическом CLI. Чтобы заставить систему предпочитать TUI, установи `HERMES_TUI=1` в профиле оболочки. Чтобы вернуться назад, удали эту переменную.

Если TUI не удаётся запустить (нет Node, отсутствует bundle, проблема с TTY), Hermes выводит диагностику и переходит к запасному варианту — вместо того чтобы оставить тебя в подвешенном состоянии.
