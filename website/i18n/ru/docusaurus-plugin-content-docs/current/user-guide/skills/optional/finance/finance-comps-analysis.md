Результатом является структурированная Excel‑таблица, позволяющая принимать обоснованные инвестиционные решения через сравнение с аналогами.

**ВСЕГДА задавай себе вопросы сначала:**

3. **«Какой ключевой вопрос?»** (оценка, анализ роста, конкурентное позиционирование, эффективность)
4. **«Каков контекст?»** (оценка M&A, инвестиционное решение, отраслевой бенчмарк, обзор результатов)

**Адаптируй в зависимости от конкретики:**
- **Отраслевой контекст**: крупные технологические мега‑капитализации требуют иных метрик, чем развивающиеся SaaS‑стартапы

- **Знакомство с компанией**: известные компании могут требовать меньше фона, больше фокуса на анализе отклонений
- **Тип решения**: M&A требует иной акцент, чем текущий мониторинг портфеля

**Основной принцип:** используй принципы шаблона (четкая структура, статистическая строгость, прозрачные формулы), но варьируй исполнение в зависимости от контекста. Цель — анализ институционального качества, а не шаблоны, выглядящие институционально.

## ⚠️ КРИТИЧНО: Формулы вместо хардкодов + пошаговая проверка

**Формулы, а не хардкоды:**
- Каждое производное значение (margin, multiple, statistic) ДОЛЖНО быть формулой Excel, ссылающейся на ячейки ввода — никогда не вставляй заранее вычисленное число
- При использовании Python/openpyxl для построения листа: записывай `cell.value = "=E7/C7"` (строка‑формула), а НЕ `cell.value = 0.687` (вычисленный результат)
- Единственными хардкоженными значениями должны быть сырые входные данные (revenue, EBITDA, share price и т.д.) — и каждому из них должна сопутствовать ячейка‑комментарий с указанием источника
- Почему: модель должна обновляться автоматически при изменении входных данных. Хардкоженный margin — это скрытая ошибка, готовая проявиться.

**Проверяй пошагово вместе с пользователем:**
- После настройки структуры → покажи пользователю макет заголовков перед заполнением данных
- После ввода сырых входных данных → покажи пользователю блок ввода и подтвердите источники/периоды перед построением формул
- После построения формул операционных метрик → покажи рассчитанные margins и проведи sanity‑check с пользователем перед переходом к оценке
- После построения valuation multiples → покажи multiples и убедись, что они выглядят разумно, прежде чем добавлять статистику
- НЕ строй весь лист от начала до конца, а затем представляй его — выявляй ошибки рано, подтверждая каждый раздел.

- **Жирный текст**: заголовки разделов, названия компаний, подписи статистики

**Цвета и заливка по умолчанию — профессиональная синяя/серая палитра (чем меньше, тем лучше):**
- **Сдержанность** — только синие и серые оттенки. Не добавляй зелёный, оранжевый, красный или несколько акцентных цветов. Чистый лист компоновки использует 3‑4 цвета в сумме.
- **Заголовки разделов** (например, «OPERATING STATISTICS & FINANCIAL METRICS»):
  - Тёмно‑синий фон (`#1F4E79` или `#17365D` navy)
  - Белый жирный текст
  - Полная заливка строки по всем столбцам
- **Заголовки столбцов** (например, «Company», «Revenue», «Margin»):
  - Светло‑синий фон (`#D9E1F2` или аналогичный бледный синий)
  - Чёрный жирный текст
  - Выравнивание по центру
- **Строки с данными**:
  - Белый фон для данных компаний
  - Чёрный текст для формул; синий текст для жёстко заданных вводов
- **Строки со статистикой** (Maximum, 75th Percentile и т.п.):
  - Светло‑серый фон (`#F2F2F2`)
  - Чёрный текст, подписи выровнены по левому краю
- **Это вся палитра**: тёмно‑синий + светло‑синий + светло‑серый + белый. Ничего другого, если только шаблон пользователя не требует иначе.

**Рекомендуемые правила форматирования:**
- **Точность отображения:**
  - Проценты — 1 знак после запятой (12.3 %)
  - Мультипликаторы — 1 знак после запятой (13.5x)
  - Суммы в долларах — без десятичных, разделитель тысяч (69,632)
  - Маржа, отображаемая в процентах — 1 знак после запятой (68.7 %)
- **Границы**: без границ (чистый, минималистичный вид)
- **Выравнивание**: все метрики центрированы для аккуратного, однородного вида
- **Размеры ячеек**: ширина всех столбцов должна быть одинаковой/равномерной, высота всех строк — согласованной (создаёт чистую, профессиональную сетку)

**Примечание:** Если пользователь предоставляет файл‑шаблон или указывает другое форматирование, используй его.
## Раздел 2: Операционные показатели и финансовые метрики

### Основные столбцы (начни с них)
1. **Company** – названия с единообразным форматированием
2. **Revenue** – метрика объёма (может быть LTM, квартальная или годовая в зависимости от контекста)
3. **Revenue Growth** – годовое процентное изменение
4. **Gross Profit** – выручка минус стоимость проданных товаров
5. **Gross Margin** – GP/Revenue (базовая рентабельность)
6. **EBITDA** – прибыль до вычета процентов, налогов, износа и амортизации
7. **EBITDA Margin** – EBITDA/Revenue (операционная эффективность)

### Дополнительные столбцы (выбирай в зависимости от отрасли/цели)
- **Quarterly vs LTM** – добавляй оба, если важна сезонность
- **Free Cash Flow** – для капиталоёмких или SaaS‑бизнесов
- **FCF Margin** – FCF/Revenue (эффективность генерации наличности)
- **Net Income** – для зрелых, прибыльных компаний
- **Operating Income** – для компаний с различными D&A
- **CapEx metrics** – для отраслей с тяжёлыми активами
- **Rule of 40** – специфично для SaaS (Growth % + Margin %)
- **FCF Conversion** – для анализа качества прибыли (продвинутая метрика)

### Примеры формул (используя строку 7 как пример)
```excel
// Core ratios - these are always calculated
Gross Margin (F7): =E7/C7
EBITDA Margin (H7): =G7/C7

// Optional ratios - include if relevant
FCF Margin: =[FCF]/[Revenue]
Net Margin: =[Net Income]/[Revenue]
Rule of 40: =[Growth %]+[FCF Margin %]
```

**Золотое правило:** каждый коэффициент должен быть [Что‑то] / [Revenue] или [Что‑то] / [Что‑то из этой таблицы]. Делай всё просто.

### Блок статистики (после данных компаний)

**CRITICAL: добавь формулы статистики для всех сопоставимых метрик (коэффициенты, маржи, темпы роста, мультипликаторы).**
```
[Leave one blank row for visual separation]
- Maximum: =MAX(B7:B9)
- 75th Percentile: =QUARTILE(B7:B9,3)
- Median: =MEDIAN(B7:B9)
- 25th Percentile: =QUARTILE(B7:B9,1)
- Minimum: =MIN(B7:B9)
```

**Столбцы, которым нужна статистика (сопоставимые метрики):**
- Revenue Growth %
- Gross Margin %
- EBITDA Margin %
- EPS
- EV/Revenue
- EV/EBITDA
- P/E
- Dividend Yield %
- Beta

**Столбцы, которым не нужна статистика (метрики размера):**
- Revenue
- EBITDA
- Net Income (абсолютный размер варьируется в зависимости от масштаба компании)
- Market Cap
- Enterprise Value (не сопоставимы между компаниями разного размера)

**Примечание:** добавь одну пустую строку между данными компаний и строками статистики для визуального разделения. Не добавляй строку‑заголовок «SECTOR STATISTICS» или «VALUATION STATISTICS».

**Почему важны квартильные показатели:** они показывают распределение, а не только среднее. 75‑й процентиль мультипликатора говорит, по какой «премии» торгуются компании.
## Раздел 3: Оценочные мультипликаторы и инвестиционные метрики

### Основные столбцы оценки (начни с них)
1. **Company** – тот же порядок, что и в разделе операционных данных
2. **Market Cap** – текущая рыночная капитализация
3. **Enterprise Value** – Market Cap ± Net Debt/Cash
4. **EV/Revenue** – сколько рынок платит за доллар продаж
5. **EV/EBITDA** – сколько рынок платит за доллар EBITDA
6. **P/E Ratio** – цена относительно чистой прибыли

### Дополнительные оценочные метрики (выбирай в зависимости от контекста)
- **FCF Yield** – FCF/Market Cap (для анализа, ориентированного на денежные потоки)
- **PEG Ratio** – P/E/Growth Rate (для компаний роста)
- **Price/Book** – рыночная стоимость vs. балансовая стоимость (для капиталоёмких бизнесов)
- **ROE/ROA** – метрики доходности (для сравнения прибыльности)
- **Revenue/EBITDA CAGR** – исторические темпы роста (для анализа трендов)
- **Asset Turnover** – Revenue/Assets (для операционной эффективности)
- **Debt/Equity** – финансовый рычаг (для анализа структуры капитала)

**Ключевой принцип:** включай 3‑5 основных мультипликаторов, которые важны для твоей отрасли. Не включай каждую возможную метрику только потому, что можешь.

### Примеры формул
```excel
// Core multiples - always include these
EV/Revenue: =[Enterprise Value]/[LTM Revenue]
EV/EBITDA: =[Enterprise Value]/[LTM EBITDA]
P/E Ratio: =[Market Cap]/[Net Income]

// Optional multiples - include if data available
FCF Yield: =[LTM FCF]/[Market Cap]
PEG Ratio: =[P/E]/[Growth Rate %]
```

### Правило перекрёстных ссылок
**CRITICAL:** оценочные мультипликаторы ДОЛЖНЫ ссылаться на раздел операционных метрик. Никогда не вводи одни и те же сырые данные дважды. Если выручка находится в C7, то формула EV/Revenue должна ссылаться на C7.

### Блок статистики
Та же структура, что и в разделе операционных данных: Max, 75th, Median, 25th, Min для каждой метрики. Добавь одну пустую строку для визуального разделения между данными компаний и статистикой. НЕ добавляй строку‑заголовок «VALUATION STATISTICS».

## Раздел 5: Выбор правильных метрик (рамки принятия решений)

### Начни с вопроса «На какой вопрос я отвечаю?»

**«Какая компания недооценена?»**
→ **Фокус:** EV/Revenue, EV/EBITDA, P/E, Market Cap
→ **Пропустить:** операционные детали, метрики роста

**«Какая компания наиболее эффективна?»**
→ **Фокус:** Gross Margin, EBITDA Margin, FCF Margin, Asset Turnover
→ **Пропустить:** метрики размера, абсолютные денежные суммы

**«Какая компания растёт быстрее всего?»**
→ **Фокус:** Revenue Growth %, EBITDA CAGR, User/Customer Growth
→ **Пропустить:** метрики маржи, коэффициенты финансового рычага

**«Какая компания генерирует наибольший денежный поток?»**
→ **Фокус:** FCF, FCF Margin, FCF Conversion, CapEx intensity
→ **Пропустить:** EBITDA, P/E ratios

### Выбор метрик по отраслям

**Software/SaaS:**
**Обязательно:** Revenue Growth, Gross Margin, Rule of 40
**Опционально:** ARR, Net Dollar Retention, CAC Payback
**Пропустить:** Asset Turnover, Inventory metrics

**Manufacturing/Industrials:**
**Обязательно:** EBITDA Margin, Asset Turnover, CapEx/Revenue
**Опционально:** ROA, Inventory Turns, Backlog
**Пропустить:** Rule of 40, SaaS‑metrics

**Financial Services:**
**Обязательно:** ROE, ROA, Efficiency Ratio, P/E
**Опционально:** Net Interest Margin, Loan Loss Reserves
**Пропустить:** Gross Margin, EBITDA (не имеет смысла для банков)

**Retail/E‑commerce:**
**Обязательно:** Revenue Growth, Gross Margin, Inventory Turnover
**Опционально:** Same-Store Sales, Customer Acquisition Cost
**Пропустить:** тяжёлые R&D‑ или CapEx‑метрики

### «Правило 5‑10»

**5 операционных метрик** — Revenue, Growth, 2‑3 маржи/метрики эффективности
**5 оценочных метрик** — Market Cap, EV, 3 мультипликатора
**= 10 столбцов в сумме** — достаточно, чтобы рассказать историю, но не настолько много, чтобы потерять нить

Если у тебя более 15 метрик, скорее всего, в наборе есть шум. Редактируй безжалостно.
## Раздел 6: Лучшие практики и проверки качества

### Перед началом
1. **Определить группу сравнения** – компании должны быть действительно сопоставимыми (аналогичная бизнес‑модель, масштаб, география)
2. **Выбрать правильный период** – LTM сглаживает сезонность; квартальные данные показывают тренды
3. **Стандартизировать единицы измерения заранее** – выбор «миллионы» или «миллиарды» влияет на всё остальное
4. **Составить карту источников данных** – знать, откуда берётся каждое число

### При построении
1. **Сначала ввести все необработанные данные** – заполнить синий текст перед написанием формул
2. **Добавлять комментарии к ВСЕМ жёстко заданным вводам** – щелкни правой кнопкой → *Insert Comment* → укажи источник ИЛИ предположение

   **Для полученных данных указывай точный источник:**
   - Пример: “Bloomberg Terminal – MSFT Equity DES, accessed 2024‑10‑02”
   - Пример: “Q4 2024 10‑K filing, page 42, line item ‘Total Revenue’”
   - Пример: “FactSet consensus estimate as of 2024‑10‑02”
   - **По возможности добавляй гиперссылки**: щелкни правой → *Link* → вставь URL к SEC filing, источнику данных или отчёту

   **Для предположений объясняй логику:**
   - Пример: “Assumed 15 % EBITDA margin based on peer median, company does not disclose”
   - Пример: “Estimated Enterprise Value as Market Cap + $50M net debt (from Q3 balance sheet, Q4 not yet available)”
   - Пример: “Forward P/E based on street consensus EPS of $3.45 (average of 12 analyst estimates)”

   **Почему это важно**: обеспечивает трассировку аудита, проверку данных, прозрачность предположений и упрощает будущие обновления
3. **Строить формулы строка за строкой** – проверять каждый расчёт перед переходом к следующему
4. **Использовать абсолютные ссылки для заголовков** – `$C$6` фиксирует строку заголовка
5. **Форматировать последовательно** – проценты отображать как проценты, а не как десятичные дроби
6. **Добавлять условное форматирование** – автоматически выделять выбросы

### Проверки на здравый смысл
- **Тест маржи**: Gross margin > EBITDA margin > Net margin (всегда верно по определению)
- **Обоснованность мультипликаторов**:
  - EV/Revenue: обычно 0.5‑20× (значительно варьируется по отраслям)
  - EV/EBITDA: обычно 8‑25× (относительно стабильно в разных отраслях)
  - P/E: обычно 10‑50× (зависит от темпов роста)
- **Корреляция рост‑мультипликатор**: более высокий рост обычно сопровождается более высокими мультипликаторами
- **Компромисс размер‑эффективность**: крупные компании часто имеют лучшие маржи (выгода от масштаба)

### Распространённые ошибки, которых следует избегать
❌ Смешивание market cap и enterprise value в формулах
❌ Использование разных периодов для числителя и знаменателя (LTM vs quarterly)
❌ Жёсткое кодирование чисел в формулах вместо ссылок на ячейки
❌ **Жёстко заданные вводы без комментариев ячеек, указывающих источник ИЛИ объясняющих предположение**
❌ Отсутствие гиперссылок к SEC filing или другим источникам данных, когда они доступны
❌ Слишком большое количество метрик без чёткой цели
❌ Включение некоррелируемых компаний (разные бизнес‑модели)
❌ Использование устаревших данных без указания этого факта
❌ Неправильный расчёт средних значений процентов (должно быть медианой)

## Раздел 11: Справочник формул

### Основные формулы Excel
```excel
// Statistical Functions
=AVERAGE(range)          // Simple mean
=MEDIAN(range)           // Middle value
=QUARTILE(range, 1)      // 25th percentile
=QUARTILE(range, 3)      // 75th percentile
=MAX(range)              // Maximum value
=MIN(range)              // Minimum value
=STDEV.P(range)          // Standard deviation

// Financial Calculations
=B7/C7                   // Simple ratio (Margin)
=SUM(B7:B9)/3            // Average of multiple companies
=IF(B7>0, C7/B7, "N/A")  // Conditional calculation
=IFERROR(C7/D7, 0)       // Handle divide by zero

// Cross-Sheet References
='Sheet1'!B7             // Reference another sheet
=VLOOKUP(A7, Table1, 2)  // Lookup from data table
=INDEX(MATCH())          // Advanced lookup

// Formatting
=TEXT(B7, "0.0%")        // Format as percentage
=TEXT(C7, "#,##0")       // Thousands separator
```

### Формулы общих отношений
```excel
Gross Margin = Gross Profit / Revenue
EBITDA Margin = EBITDA / Revenue
FCF Margin = Free Cash Flow / Revenue
FCF Conversion = FCF / Operating Cash Flow
ROE = Net Income / Shareholders' Equity
ROA = Net Income / Total Assets
Asset Turnover = Revenue / Total Assets
Debt/Equity = Total Debt / Shareholders' Equity
```

## Краткое резюме ключевых принципов

1. **Структура порождает инсайты** — правильные заголовки стимулируют правильное мышление
2. **Меньше — лучше** — 5‑10 значимых метрик важнее, чем 20 несущественных
3. **Выбирай метрики под свой вопрос** — оценочный анализ ≠ анализ эффективности
4. **Статистика показывает шаблоны** — медиана/квартиль раскрывают больше, чем среднее
5. **Прозрачность превосходит сложность** — простые формулы, понятные всем
6. **Сравнимость — главный критерий** — лучше исключить, чем принуждать к плохому сравнению
7. **Документируй свои выборы** — объясни, какие метрики и почему, в разделе «Заметки»

- [ ] Данные относятся к согласованным периодам времени

- [ ] **Гиперссылки добавлены там, где это уместно** (SEC EDGAR filings, Bloomberg pages, research reports)

- [ ] Аудит формул не показывает ошибок (#DIV/0!, #REF!, #N/A)
## Непрерывное улучшение

После завершения анализа конкурентов задай себе вопросы:
1. Выявила ли статистика неожиданные инсайты?
2. Были ли пробелы в данных, ограничившие анализ?
3. Запрашивали ли заинтересованные стороны метрики, которые ты не включил?

5. Что сделает его более полезным в следующий раз?

Много ниже встречается фраза «использовать S&P Kensho MCP / Daloopa MCP / FactSet MCP». Это коммерческие финансовые MCP из оригинального контекста плагина Cowork. В Hermes:

- **Если у тебя настроен любой структурированный финансовый MCP** (Hermes поддерживает MCP — см. навык `native-mcp`), используй его в первую очередь для точечных сравнений, прецедентных сделок и документов.
- **Иначе** переходи к запасному варианту:
  - `web_search` / `web_extract` против SEC EDGAR (`https://www.sec.gov/cgi-bin/browse-edgar`) для американских документов
  - Страницы IR компаний для пресс‑релизов, презентаций earnings
  - `browser_navigate` для интерактивных порталов данных
  - Данные, предоставленные пользователем (спрашивай явно, когда контекст их не содержит)
- **Никогда не выдумывай**. Если множитель, прецедент или номер документа нельзя получить из источника, пометь ячейку как `[UNSOURCED]` и покажи её пользователю.
