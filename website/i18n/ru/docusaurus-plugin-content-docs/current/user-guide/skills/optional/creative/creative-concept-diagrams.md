# Диаграммы концепций

Генерируй SVG‑диаграммы производственного качества с унифицированной плоской минималистичной системой дизайна. Выводом является один самодостаточный HTML‑файл, который отображается одинаково во всех современных браузерах, с автоматическим переключением светлой/тёмной темы.
## Область применения

**Наиболее подходит для:**
- Физические установки, химические механизмы, математические кривые, биология
- Физические объекты (самолёты, турбины, смартфоны, механические часы, клетки)
- Анатомия, поперечные срезы, развернутые виды слоёв
- Планы этажей, архитектурные преобразования
- Нарративные путешествия (жизненный цикл X, процесс Y)
- Интеграции системы «центр‑спица» (умный город, сети IoT, электросети)
- Образовательные визуализации в стиле учебника в любой области
- Количественные диаграммы (группированные столбцы, энергетические профили)

**Сначала ищи в другом месте:**
- Специализированное программное обеспечение / облачная инфраструктура с темной технологической эстетикой (рассмотри `architecture-diagram`, если доступно)
- Ручные наброски на доске (рассмотри `excalidraw`, если доступно)
- Анимированные объяснительные ролики или видеовывод (рассмотри навык анимации)

Если существует более специализированный навык для темы, предпочтительно использовать его. Если подходящего нет, этот навык может выступать в качестве общего SVG‑диаграммного запасного варианта — вывод будет иметь чистую образовательную эстетику, описанную ниже, что является разумным по умолчанию для почти любой темы.
## Рабочий процесс

1. Выбери тип диаграммы (см. раздел «Типы диаграмм» ниже).
2. Размести компоненты согласно правилам **Design System**.
3. Напиши полную HTML‑страницу, используя `templates/template.html` в качестве оболочки — вставь свой SVG туда, где шаблон содержит `<!-- PASTE SVG HERE -->`.
4. Сохрани как отдельный файл `.html` (например `~/my-diagram.html` или `./my-diagram.html`).
5. Пользователь открывает его напрямую в браузере — без сервера, без зависимостей.

Опционально: если пользователь хочет просматривать галерею из нескольких диаграмм, смотри раздел «Local Preview Server» внизу.

Загрузить HTML‑шаблон:
```
skill_view(name="concept-diagrams", file_path="templates/template.html")
```

Шаблон включает полный CSS **Design System** (`c-*` классы цветов, классы текста, переменные light/dark, стили маркеров стрелок). Твой SVG полагается на наличие этих классов на странице‑хосте.
## Design System

### Philosophy

- **Flat**: без градиентов, теней, размытия, свечения или неоновых эффектов.
- **Minimal**: показывай только самое необходимое. Нет декоративных иконок внутри блоков.
- **Consistent**: одинаковые цвета, отступы, типографика и ширина линий во всех диаграммах.
- **Dark-mode ready**: все цвета автоматически адаптируются через CSS‑классы — без отдельного SVG для каждого режима.

#### Color Assignment Rules

Цвет передаёт **значение**, а не порядок. Не перебирай цвета, как радугу.

- Группируй узлы по **категории** — все узлы одного типа используют один цвет.
- Для нейтральных/структурных узлов (start, end, общие шаги, пользователи) используй `c-gray`.
- Используй **2‑3 цвета на диаграмму**, а не 6+.
- Предпочитай `c-purple`, `c-teal`, `c-coral`, `c-pink` для общих категорий.
- Резервируй `c-blue`, `c-green`, `c-amber`, `c-red` для семантического значения (info, success, warning, error).

Отображение светлой/тёмной ступени (обрабатывается шаблонным CSS — просто используй класс):
- Светлый режим: 50 fill + 600 stroke + 800 title / 600 subtitle
- Тёмный режим: 800 fill + 200 stroke + 100 title / 200 subtitle

### Typography

Только два размера шрифта. Без исключений.

| Class | Size | Weight | Use |
|-------|------|--------|-----|
| `th`  | 14px | 500    | Заголовки узлов, подписи регионов |
| `ts`  | 12px | 400    | Подзаголовки, описания, подписи стрелок |
| `t`   | 14px | 400    | Обычный текст |

- **Всегда sentence case**. Никакого Title Case и ALL CAPS.
- Каждый `<text>` ДОЛЖЕН иметь класс (`t`, `ts` или `th`). Текст без класса недопустим.
- `dominant-baseline="central"` у всех текстов внутри блоков.
- `text-anchor="middle"` для центрированного текста в блоках.

**Оценка ширины (примерно):**
- 14 px, weight 500: ~8 px на символ
- 12 px, weight 400: ~6.5 px на символ
- Всегда проверяй: `box_width >= (char_count × px_per_char) + 48` (по 24 px отступа с каждой стороны)

### Spacing & Layout

- **ViewBox**: `viewBox="0 0 680 H"`, где H = высота контента + 40 px буфера.
- **Safe area**: x=40 → x=640, y=40 → y=(H‑40).
- **Между блоками**: минимум 60 px.
- **Внутри блоков**: 24 px горизонтального отступа, 12 px вертикального.
- **Отступ стрелки от края**: 10 px.
- **Однострочный блок**: высота 44 px.
- **Двухстрочный блок**: высота 56 px, 18 px между базовыми линиями заголовка и подзаголовка.
- **Отступ контейнера**: минимум 20 px внутри каждого контейнера.
- **Максимальная вложенность**: 2‑3 уровня. Глубже становится нечитаемо при ширине 680 px.

### Stroke & Shape

- **Ширина линии**: 0.5 px у всех границ узлов. Не 1 px, не 2 px.
- **Скругление прямоугольников**: `rx="8"` для узлов, `rx="12"` для внутренних контейнеров, `rx="16"`‑`rx="20"` для внешних контейнеров.
- **Контурные пути**: ДОЛЖНЫ иметь `fill="none"`. Иначе SVG по умолчанию задаёт `fill: black`.

### Arrow Marker

Вставь этот блок `<defs>` в начало **каждого** SVG:

```xml
<defs>
  <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
          markerWidth="6" markerHeight="6" orient="auto-start-reverse">
    <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
          stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
  </marker>
</defs>
```

Используй `marker-end="url(#arrow)"` у линий. Стрелка наследует цвет линии через `context-stroke`.

- Текст: `.t`, `.ts`, `.th`
- Нейтральные: `.box`, `.arr`, `.leader`, `.node`
- Цветовые градации: `.c-purple`, `.c-teal`, `.c-coral`, `.c-pink`, `.c-gray`, `.c-blue`, `.c-green`, `.c-amber`, `.c-red` (все с автоматическим переключением свет/тёмный режим)

Эти классы **не нужно** переопределять — просто применяй их в своём SVG. Полные определения CSS находятся в файле шаблона.
## Шаблон SVG

Каждый SVG на странице шаблона начинается со следующей структуры:

```xml
<svg width="100%" viewBox="0 0 680 {HEIGHT}" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
            markerWidth="6" markerHeight="6" orient="auto-start-reverse">
      <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
            stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
    </marker>
  </defs>

  <!-- Diagram content here -->

</svg>
```

Замените `{HEIGHT}` на реально вычисленную высоту (нижний край последнего элемента + 40 px).

## Типы диаграмм

Выбери макет, соответствующий теме:

1. **Flowchart** — CI/CD‑конвейеры, жизненный цикл запросов, процессы согласования, обработка данных. Поток в одном направлении (сверху‑вниз или слева‑направо). Максимум 4‑5 узлов в строке.
2. **Structural / Containment** — Вложенность облачной инфраструктуры, системная архитектура со слоями. Большие внешние контейнеры с внутренними областями. Пунктирные прямоугольники для логических группировок.
3. **API / Endpoint Map** — REST‑маршруты, схемы GraphQL. Дерево от корня, ветвление к группам ресурсов, каждая из которых содержит узлы конечных точек.
4. **Microservice Topology** — Сервисная сетка, событийно‑ориентированные системы. Сервисы как узлы, стрелки для схем коммуникации, очереди сообщений между ними.
5. **Data Flow** — ETL‑конвейеры, потоковые архитектуры. Поток слева‑направо от источников через обработку к приёмникам.
6. **Physical / Structural** — Транспортные средства, здания, оборудование, анатомия. Используй формы, соответствующие физическому объекту — `<path>` для изогнутых корпусов, `<polygon>` для сужающихся форм, `<ellipse>`/`<circle>` для цилиндрических частей, вложенные `<rect>` для отсеков. См. `references/physical-shape-cookbook.md`.
7. **Infrastructure / Systems Integration** — Умные города, сети IoT, многодоменные системы. Макет «звезда‑спица» с центральной платформой, соединяющей подсистемы. Семантические стили линий (`.data-line`, `.power-line`, `.water-pipe`, `.road`). См. `references/infrastructure-patterns.md`.
8. **UI / Dashboard Mockups** — Административные панели, мониторинговые дашборды. Рамка экрана с вложенными элементами графиков/индикаторов/счётчиков. См. `references/dashboard-patterns.md`.

Для физических, инфраструктурных и дашборд‑диаграмм загрузите соответствующий файл‑справочник перед генерацией — каждый из них предоставляет готовые CSS‑классы и примитивы форм.
## Чек‑лист проверки

Перед окончательной подготовкой любого SVG убедись, что выполнены **ВСЕ** пункты:

1. Каждый `<text>` имеет класс `t`, `ts` или `th`.
2. Каждый `<text>` внутри блока имеет `dominant-baseline="central"`.
3. Каждый `<path>` или `<line>`, используемый как стрелка, имеет `fill="none"`.
4. Ни одна линия‑стрелка не пересекает блок, к которому не относится.
5. `box_width >= (longest_label_chars × 8) + 48` для текста 14 px.
6. `box_width >= (longest_label_chars × 6.5) + 48` для текста 12 px.
7. Высота `viewBox` = нижний элемент + 40 px.
8. Всё содержимое находится в диапазоне от `x=40` до `x=640`.
9. Классы цветов (`c-*`) применяются к `<g>` или к элементам‑форм, **не** к соединительным `<path>`.
10. Блок `<defs>` со стрелками присутствует.
11. Нет градиентов, теней, размытия или свечения.
12. Толщина линии границы узла = 0.5 px.
## Вывод и предварительный просмотр

### По умолчанию: автономный HTML‑файл

Создай один файл `.html`, который пользователь может открыть напрямую. Без сервера, без зависимостей, работает офлайн. Шаблон:

```python
template = skill_view("concept-diagrams", "templates/template.html")

html = template.replace(
    "<!-- DIAGRAM TITLE HERE -->", "SN2 reaction mechanism"
).replace(
    "<!-- OPTIONAL SUBTITLE HERE -->", "Bimolecular nucleophilic substitution"
).replace(
    "<!-- PASTE SVG HERE -->", svg_content
)

# 3. Write to a user-chosen path (or ./ by default)
write_file("./sn2-mechanism.html", html)
```

Сообщи пользователю, как его открыть:

```
# macOS
open ./sn2-mechanism.html
# Linux
xdg-open ./sn2-mechanism.html
```

### По желанию: локальный сервер предварительного просмотра (галерея нескольких диаграмм)

Используй только в том случае, если пользователь явно хочет просматривать галерею из нескольких диаграмм.

**Правила:**
- Привязывайся только к `127.0.0.1`. Никогда к `0.0.0.0`. Открытие диаграмм на всех сетевых интерфейсах представляет угрозу безопасности в общих сетях.
- Выбирай свободный порт (не указывай фиксированный) и сообщи пользователю выбранный URL.
- Сервер является опциональным и включается только по запросу — сначала предпочтительно использовать автономный HTML‑файл.

Рекомендуемый шаблон (позволяет ОС выбрать свободный эфемерный порт):

```bash
# Put each diagram in its own folder under .diagrams/
mkdir -p .diagrams/sn2-mechanism
# ...write .diagrams/sn2-mechanism/index.html...

# Serve on loopback only, free port
cd .diagrams && python3 -c "
import http.server, socketserver
with socketserver.TCPServer(('127.0.0.1', 0), http.server.SimpleHTTPRequestHandler) as s:
    print(f'Serving at http://127.0.0.1:{s.server_address[1]}/')
    s.serve_forever()
" &
```

Если пользователь настаивает на фиксированном порте, используй `127.0.0.1:<port>` — по‑прежнему никогда `0.0.0.0`. Опиши, как остановить сервер (`kill %1` или `pkill -f "http.server"`).
