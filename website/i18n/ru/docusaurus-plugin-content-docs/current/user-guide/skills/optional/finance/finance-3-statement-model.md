# Завершение шаблона финансовой модели «3‑Statement»

Заполни и заверши шаблоны интегрированной финансовой модели с правильными связями между отчётом о прибылях и убытках, балансом и отчётом о движении денежных средств.
## ⚠️ КРИТИЧЕСКИЕ ПРИНЦИПЫ — прочитай перед заполнением любого шаблона

**Формулы вместо жёстких значений (не обсуждается):**
- Каждая ячейка проекции, roll-forward, связь и подытог **ДОЛЖНЫ** быть формулой Excel — никогда не предвычисленным значением
- При работе с Python/openpyxl: записывайте строки формул (`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`), а **НЕ** вычисленные результаты (`ws["D15"] = 12500`)
- **Единственные** ячейки, где допускаются жёстко заданные числа: (1) исторические фактические данные, (2) драйверы предположений на листе **Assumptions**
- Если ты находишь себя вычисляющим значение в Python и записывающим результат в ячейку — **ОСТАНОВИСЬ**. Запиши вместо этого формулу.
- Почему: модель должна гибко реагировать на переключение сценариев или изменение предположений. Жёсткие значения тихо ломают все последующие проверки целостности.

**Проверяй пошагово вместе с пользователем:**
1. **После сопоставления шаблона** → покажи пользователю, какие вкладки/разделы ты определил, и получи подтверждение перед изменением любых ячеек
2. **После заполнения исторических данных** → покажи пользователю блок исторических данных и убедись, что значения/периоды соответствуют исходным данным
3. **После построения прогноза IS** → запусти проверки подытогов, покажи пользователю спроектированный IS и получи подтверждение перед переходом к BS
4. **После построения BS** → покажи пользователю проверку баланса (Assets = L+E) для каждого периода и получи подтверждение перед переходом к CF
5. **После построения CF** → покажи пользователю сверку наличных (CF ending cash = BS cash) и получи подтверждение перед финализацией
6. **НЕ заполняй всю модель от начала до конца и не представляй её полностью** — остановись после каждого шага, показывай проделанную работу, выявляй ошибки на ранних этапах

Перед заполнением шаблона ознакомься с его текущим расположением, чтобы вводить данные в правильные места и не нарушать формулы.

**Определение структуры строк**
- Найди заголовок модели в верхней части каждой вкладки.
- Определи заголовки разделов и их визуальное разделение.
- Найди строку с единицами измерения, указывающую $ млн, %, x и т.п.
- Обрати внимание на заголовки столбцов, различающие периоды фактических и прогнозных данных.
- Подтверди метки периодов (например, FY2024A, FY2025E).
- Определи ячейки ввода и ячейки с формулами (обычно различаются цветом шрифта).

**Определение структуры столбцов**
- Подтверди метки статей в самом левом столбце.
- Убедись, что исторические годы идут перед прогнозными годами.
- Обрати внимание на визуальную границу, отделяющую исторические данные от прогнозных.
- Проверь согласованность порядка столбцов во всех вкладках.

**Работа с именованными диапазонами**
Шаблоны часто используют именованные диапазоны для ключевых входов и выходов. Перед вводом данных:
- Просмотри существующие именованные диапазоны в шаблоне (`Formulas → Name Manager` в Excel).
- Типичные именованные диапазоны включают: темпы роста выручки, процентные доли расходов, ключевые выходы (Net Income, EBITDA, Total Debt, Cash), ячейку выбора сценария.
- Убедись, что вводимые данные помещаются в ячейки, которые попадают в эти именованные диапазоны.

### Прогнозный период
- Шаблоны обычно прогнозируют 5 лет вперёд от последнего исторического года.
- Убедись, что столбцы исторических (A) и прогнозных (E) данных чётко разделены.
- Подтверди, что столбцы используют обозначение финансового года (например, FY2024A, FY2025E).
## Анализ маржи

**Примечание: Следующий анализ маржи следует выполнять только по запросу пользователя или если шаблон явно требует этого. Если запрос не получен, пропусти этот раздел.**

Рассчитай и отобрази маржи прибыльности на вкладке **Income Statement (IS)**, чтобы отслеживать операционную эффективность и позволять сравнение с аналогами.

### Основные маржи для включения

| Маржа | Формула | Что измеряется |
|--------|---------|------------------|
| Валовая маржа | Gross Profit / Revenue | Ценовая политика, эффективность производства |
| Маржа EBITDA | EBITDA / Revenue | Основная операционная прибыльность |
| Маржа EBIT | EBIT / Revenue | Операционная прибыльность после амортизации и износа |
| Маржа чистой прибыли | Net Income / Revenue | Прибыль после всех расходов |

### Макет отчёта о прибылях и убытках с маржами

Отображай процентные значения маржи непосредственно под каждой строкой прибыли:
- % валовой маржи под **Gross Profit**
- % маржи EBIT под **EBIT**
- % маржи EBITDA под **EBITDA**
- % маржи чистой прибыли под **Net Income**
## Кредитные метрики

**Примечание: Анализ кредитных показателей следует выполнять только по запросу пользователя или если шаблон явно требует этого. Если запрос не получен, пропусти этот раздел.**

Вычисли и отобрази кредитные/левередж‑метрики на вкладке **Balance Sheet (BS)** для оценки финансового здоровья, долговой ёмкости и соблюдения ковенант.

### Основные кредитные метрики

| Метрика | Формула | Что измеряется |
|--------|---------|----------------|
| Total Debt / EBITDA | Total Debt / LTM EBITDA | Коэффициент финансового рычага |
| Net Debt / EBITDA | (Total Debt - Cash) / LTM EBITDA | Левередж за вычетом наличных |
| Interest Coverage | EBITDA / Interest Expense | Способность обслуживать долг |
| Debt / Total Cap | Total Debt / (Total Debt + Equity) | Структура капитала |
| Debt / Equity | Total Debt / Total Equity | Финансовый левередж |
| Current Ratio | Current Assets / Current Liabilities | Краткосрочная ликвидность |
| Quick Ratio | (Current Assets - Inventory) / Current Liabilities | Быстрая ликвидность |

### Проверки иерархии кредитных метрик

Убедись, что Upside демонстрирует самый сильный кредитный профиль:
- **Левередж**: Upside < Base < Downside (чем меньше, тем лучше)
- **Покрытие**: Upside > Base > Downside (чем больше, тем лучше)
- **Ликвидность**: Upside > Base > Downside (чем больше, тем лучше)

### Отслеживание соблюдения ковенант

Если известны долговые ковенанты, добавь явные проверки соблюдения, сравнивая фактические метрики с пороговыми значениями ковенант.
## Анализ сценариев (Base / Upside / Downside)

Используй переключатель сценариев (выпадающий список) во вкладке **Assumptions** с формулами `CHOOSE` или `INDEX/MATCH`.

| Scenario   | Description                                   |
|------------|-----------------------------------------------|
| Base Case  | Руководство компании или консенсусные оценки |
| Upside Case| Рост выше рекомендаций, расширение маржи    |
| Downside Case| Рост ниже тренда, сжатие маржи               |

**Ключевые драйверы для чувствительности**: рост выручки, валовая маржа, % SG&A, DSO/DIO/DPO, % CapEx, процентная ставка, налоговая ставка.

**Проверки аудита сценариев**: переключатели охватывают все строки, балансы **BS** во всех сценариях, проверка согласованности наличных, соблюдение иерархии (Upside > Base > Downside для NI, EBITDA, FCF, марж).

## Завершение шаблонов модели

В этом разделе даны общие рекомендации по заполнению любого шаблона финансовой модели из трёх отчётов, при сохранении существующих формул и обеспечении целостности данных.

### Шаг 1: Анализ структуры шаблона

Прежде чем вводить какие‑либо данные, тщательно изучи шаблон, чтобы понять его архитектуру:

**Определение ячеек ввода и формульных ячеек**
- Ищи визуальные подсказки (цвет шрифта, заливка ячеек), отличающие ячейки ввода от формульных.
- Распространённые соглашения: синий шрифт = ввод, чёрный шрифт = формулы, зелёный шрифт = ссылки на другие листы.
- Используй **Trace Precedents/Dependents** в Excel (Formulas → Trace Precedents) / (Formulas → Trace Dependents), чтобы понять взаимосвязи ячеек.
- Проверь именованные диапазоны, которые могут управлять ключевыми вводами (Formulas → Name Manager).

**Картирование потока шаблона**
- Определи, какие вкладки питают другие (например, Assumptions → IS → BS → CF).
- Обрати внимание на вспомогательные графики и их связи с основными отчётами.
- Задокументируй конкретные строки и структуру шаблона перед заполнением.

### Шаг 2: Заполнение данных без нарушения формул

**Золотые правила ввода данных**

| Правило | Описание |
|--------|----------|
| Редактировать только ячейки ввода | Никогда не перезаписывай ячейки, содержащие формулы, если только не заменяешь формулу намеренно. |
| Сохранять ссылки на ячейки | При копировании данных используй **Paste Values** (`Ctrl+Shift+V`), чтобы не перезаписать формулы форматированием источника. |
| Соответствовать единицам шаблона | Убедись, использует ли шаблон тысячи, миллионы или фактические значения, прежде чем вводить данные. |
| Соблюдать конвенцию знаков | Следуй уже установленной в шаблоне конвенции знаков (например, расходы как положительные или отрицательные). |
| Проверять на круговые ссылки | Если шаблон использует итеративные расчёты, убедись, что **Enable Iterative Calculation** включён. |

**Безопасный процесс ввода данных**
1. Определи точные ячейки, предназначенные для ввода (обычно подсвечены или помечены).
2. Сначала введи исторические данные, затем проверь, правильно ли формулы рассчитывают эти периоды.
3. Введи драйверы‑предположения, которые питают прогнозные расчёты.
4. Просмотри рассчитанные результаты, чтобы убедиться, что формулы работают как задумано.
5. Если необходимо изменить формульную ячейку, зафиксируй оригинальную формулу перед изменением.

**Работа с предустановленными формулами**
- Если формулы ссылаются на ячейки, которые ещё не заполнены, ожидай временных ошибок (`#REF!`, `#DIV/0!`) до завершения ввода всех данных.
- Когда формулы дают неожиданные результаты, используй **Trace Precedents**, чтобы найти недостающие или неверные вводы.
- Никогда не удаляй строки/столбцы без проверки зависимостей формул во всех листах.

### Шаг 3: Проверка формул

**Проверки целостности формул**

Перед тем как полагаться на выводы шаблона, убедись, что формулы работают корректно:

| Тип проверки | Метод |
|--------------|-------|
| Trace precedents | Выбери ячейку с формулой → Formulas → Trace Precedents, чтобы убедиться, что она ссылается на правильные вводы. |
| Trace dependents | Проверь, что ключевые вводы идут к ожидаемым ячейкам‑результатам. |
| Evaluate formula | Используй Formulas → Evaluate Formula, чтобы пошагово пройти сложные расчёты. |
| Проверка на хардкоды | Прогнозные формулы должны ссылаться на лист Assumptions, а не содержать фиксированные значения. |
| Тест с известными значениями | Введи простые тестовые значения, чтобы убедиться, что формулы дают ожидаемый результат. |
| Согласованность между листами | Убедись, что одинаковая логика формул применяется во всех периодах прогноза. |

**Распространённые проблемы с формулами**
- Смешанные абсолютные/относительные ссылки, вызывающие неверные результаты при копировании по периодам.
- Разорванные ссылки на внешние файлы или удалённые диапазоны (`#REF!` ошибки).
- Деление на ноль в ранних периодах до роста выручки (`#DIV/0!` ошибки).
- Предупреждения о круговых ссылках (могут быть намеренными для расчётов процентов).
- Несоответствия формул между столбцами прогноза (используй `Ctrl+\`, чтобы найти различия).

**Проверка связей между листами**
- Подтверди, что значения, появляющиеся на нескольких листах, связаны (а не дублируются).
- Убедись, что итоги графиков соответствуют соответствующим строкам в основных отчётах.
- Проверь, что метки периодов согласованы во всех листах.

### Шаг 4: Проверки качества по листам

Выполни эти проверки качества на каждом листе после заполнения шаблона:

**Проверки качества отчёта о прибылях и убытках (IS)**
- Показатели выручки соответствуют исходным данным за исторические периоды.
- Все строки расходов суммируются до заявленных итогов.
- Промежуточные итоги (Gross Profit, EBIT, EBT, Net Income) рассчитываются корректно.
- Логика расчёта налога адекватна (правильно обрабатывает убытки).
- Прогнозные драйверы ссылаются на лист Assumptions (без хардкодов).
- Изменения период‑за‑период выглядят разумно.

**Проверки качества баланса (BS)**
- Активы = Обязательства + Капитал для каждого периода (основная проверка).
- Остаток наличных совпадает с конечным остатком в отчёте о движении денежных средств.
- Счета оборотного капитала согласованы с поддерживающими графиками (если есть).
- Нераспределённая прибыль переносится правильно: Prior RE + Net Income – Dividends ± Adjustments = Ending RE.
- Остатки долга согласованы с графиком долга (если есть).
- Все позиции баланса имеют корректные знаки (активы – положительные, большинство обязательств – положительные).

**Проверки качества отчёта о движении денежных средств (CF)**
- Net Income в начале CFO совпадает с Net Income в IS.
- Неденежные корректировки (D&A, SBC и др.) согласованы с их исходными графиками/отчётами.
- Изменения оборотного капитала имеют правильные знаки (увеличение актива = использование наличных = отрицательно).
- CapEx согласован с графиком PP&E или roll‑forward фиксированных активов.
- Финансовая деятельность согласована с изменениями в долге и капитале на BS.
- Конечный остаток наличных совпадает с Cash на BS.
- Начальный остаток наличных равен конечному остатку предыдущего периода.

**Проверки качества вспомогательных графиков**
- Открывающие остатки равны закрывающим остаткам предыдущего периода.
- Логика roll‑forward полна (Beginning + Additions – Deductions = Ending).
- Итоги графиков согласованы с основными строками отчётов.
- Предположения, использованные в расчётах, соответствуют листу Assumptions.

### Шаг 5: Проверки целостности между отчётами

После проверки отдельных листов удостоверься, что три отчёта правильно интегрированы:

| Проверка | Формула | Ожидаемый результат |
|----------|---------|----------------------|
| Баланс листа | Assets – Liabilities – Equity | = 0 |
| Согласование наличных | CF Ending Cash – BS Cash | = 0 |
| Связь Net Income | IS Net Income – CF Starting Net Income | = 0 |
| Нераспределённая прибыль | Prior RE + NI – Dividends – BS Ending RE | = 0 (при необходимости скорректировать на SBC/другие статьи) |

### Шаг 6: Финальный обзор

Прежде чем считать модель завершённой:
- Переключайся между всеми сценариями (если применимо), чтобы убедиться, что проверки проходят в каждом случае.
- Просмотри все ошибки `#REF!`, `#DIV/0!`, `#VALUE!`, `#NAME?` и исправь их или задокументируй.
- Убедись, что все ячейки ввода заполнены (поиск placeholder‑значений).
- Проверь согласованность единиц во всех листах.
- Сохрани чистую версию перед внесением любых дополнительных изменений.

|---|---|---|
| Баланс листа | Assets - Liabilities - Equity | = 0 |

### Обработка циклических ссылок

Процентные расходы создают цикличность: Interest → Net Income → Cash → Debt Balance → Interest.

Включите итеративные расчёты в Excel: **File → Options → Formulas → Enable iterative calculation**. Установите максимум 100 итераций и максимальное изменение 0,001. Добавьте переключатель «circuit breaker» на вкладке **Assumptions**.

### Категории проверок

**Раздел 1: Последовательность валют**
- Валюта определена и задокументирована в **Assumptions**.
- Все листы используют одинаковый символ валюты и масштаб.
- Строка **Units** соответствует валюте модели.

**Раздел 2: Целостность баланса**
- Assets = Liabilities + Equity (для каждого периода).
- Формула: Assets - Liabilities - Equity (должно = 0).

**Раздел 3: Целостность отчёта о движении денежных средств**
- Наличные привязаны к BS (CF Ending Cash = BS Cash).
- Наличные (мес.) vs год: Closing Cash (Monthly) = Closing Cash (Annual).
- Чистая прибыль привязана к IS (CF Net Income = IS Net Income).
- D&A привязана к графику.
- SBC привязана к IS.
- ΔAR, ΔInventory, ΔAP привязаны к графику WC.
- CapEx привязан к графику DA.

**Раздел 4: Нераспределённая прибыль**
- Проверка переноса RE: Prior RE + NI + SBC - Dividends = Ending RE.
- Вывести разложение компонентов для отладки.

**Раздел 5: Оборотный капитал**
- AR, Inventory, AP привязаны к BS.
- Проверки разумности DSO, DIO, DPO (флаг, если выходит за обычные диапазоны).

**Раздел 6: График долга**
- Общий долг привязан к BS (Current + LT Debt).
- Расчёт процентов привязан к IS.

**Раздел 6b: Финансирование акций**
- Выручка от выпуска акций привязана к увеличению **Common Stock/APIC** в BS.
- Увеличение наличных от акций = увеличение счёта Equity (должно сбалансировать).
- Сверка выпуска акций: ΔCommon Stock/APIC (BS) = Equity Issuance (CFF) (должно = 0).
- Сверка акционерного капитала в год 0: Equity Raised (Year 0) = Beginning Equity Capital (Year 1).

**Раздел 6c: График NOL**
- Beginning NOL (Year 1 / Formation) = 0 (новый бизнес начинается с нулевым NOL).
- NOL увеличивается только когда EBT < 0 (убытки должны быть реализованы).
- DTA привязан к BS (NOL Schedule DTA = BS Deferred Tax Asset).
- Использование NOL ≤ 80 % от EBT (федеральное ограничение после 2017 г.).
- Баланс NOL неотрицателен (нельзя использовать больше, чем доступно).
- NOL генерируется только когда EBT < 0.
- Налоговый расход = 0, когда налогооблагаемый доход ≤ 0.

**Раздел 7: Иерархия сценариев**
- Абсолютные метрики: Upside > Base > Downside (NI, EBITDA, FCF).
- Маржи: Upside > Base > Downside (GM %, EBITDA %, NI %).
- Кредитные метрики: Upside < Base < Downside для левереджа (инвертировано).

**Раздел 8: Целостность формул**
- COGS, S&M, G&A, R&D, SBC рассчитываются как % от Revenue (без хардкодов).
- Последовательные формулы во всех прогнозных годах.
- Нет ошибок #REF!, #DIV/0!, #VALUE!.

**Раздел 9: Пороговые значения кредитных метрик**
- Флагировать метрики как Green/Yellow/Red в зависимости от порогов ковенант.
- Сводка любых красных флагов.

### Главная формула проверки

Соберите статусы всех разделов в одну главную проверку:
- Если все разделы прошли → `"✓ ALL CHECKS PASS"`
- Если хотя бы один раздел не прошёл → `"✗ ERRORS DETECTED - REVIEW BELOW"`

### Быстрый рабочий процесс отладки

Когда статус **Master** показывает ошибки:
1. Прокрути вниз, чтобы найти секции, выделенные красным.
2. Определи, в какой категории проверок есть сбои.
3. Перейди на соответствующий лист‑источник для расследования.
4. Исправь первопричину проблемы.
5. Вернись на лист **Checks** и проверь, устранена ли ошибка.
