## Процесс DCF

### Шаг 1: Получение и проверка данных

Получить данные с серверов MCP, пользовательские данные и из интернета.

**Приоритет источников данных:**
1. **MCP Servers** (если настроено) – структурированные финансовые данные от провайдеров, таких как Daloopa
2. **User‑Provided Data** – исторические финансовые показатели из их исследований
3. **Web Search/Fetch** – текущие цены, бета, долг и наличные при необходимости

**Контрольный список проверки:**
- Проверить чистый долг vs чистые наличные (критично для оценки)
- Подтвердить количество разводнённых акций в обращении (проверить недавние выкупы/эмиссии)
- Убедиться, что исторические маржи согласуются с бизнес‑моделью
- Сопоставить темпы роста выручки с отраслевыми бенчмарками
- Убедиться, что налоговая ставка разумна (обычно 21‑28 %)

### Шаг 2: Исторический анализ (3‑5 лет)

Проанализировать и задокументировать:
- **Тенденции роста выручки**: рассчитать CAGR, определить драйверы
- **Эволюцию маржи**: отслеживать валовую маржу, маржу EBIT, маржу FCF
- **Капитальную интенсивность**: D&A и CapEx в % от выручки
- **Эффективность оборотного капитала**: изменения NWC в % от роста выручки
- **Метрики доходности**: тенденции ROIC, ROE

Создать сводные таблицы, показывающие:
```
Historical Metrics (LTM):
Revenue: $X million
Revenue growth: X% CAGR
Gross margin: X%
EBIT margin: X%
D&A % of revenue: X%
CapEx % of revenue: X%
FCF margin: X%
```

### Шаг 3: Построение прогноза выручки

**Методология:**
1. Начать с последней фактической выручки (LTM или последнего финансового года)
2. Применить темпы роста для каждого года прогноза
3. Показать как денежные суммы, так и рассчитанные % роста

**Структура темпов роста:**
- Годы 1‑2: более высокий рост, отражающий близкую видимость
- Годы 3‑4: постепенное замедление к отраслевому среднему
- Годы 5+: приближение к терминальному темпу роста

**Структура формулы:**
- `Revenue(Year N) = Revenue(Year N‑1) × (1 + Growth Rate)`
- `Growth %(Year N) = Revenue(Year N) / Revenue(Year N‑1) - 1`

**Подход с тремя сценариями:**
```
Bear Case: Conservative growth (e.g., 8-12%)
Base Case: Most likely scenario (e.g., 12-16%)
Bull Case: Optimistic growth (e.g., 16-20%)
```

### Шаг 4: Моделирование операционных расходов

**Анализ фиксированных/переменных расходов:**

Операционные расходы должны моделировать реалистичный операционный рычаг:
- **Sales & Marketing**: обычно 15‑40 % от выручки в зависимости от бизнес‑модели
- **Research & Development**: обычно 10‑30 % для технологических компаний
- **General & Administrative**: обычно 8‑15 % от выручки, показатель рычага при масштабировании

**Ключевые принципы:**
- ВСЕ проценты рассчитываются от **REVENUE**, а не от валовой прибыли
- Моделировать операционный рычаг: % должно снижаться по мере роста выручки
- Сохранять отдельные строки для S&M, R&D, G&A
- Рассчитать `EBIT = Gross Profit - Total OpEx`

**Рамка расширения маржи:**
```
Current State → Target State (Year 5)
Gross Margin: X% → Y% (justify based on scale, efficiency)
EBIT Margin: X% → Y% (result of revenue growth + opex leverage)
```

### Шаг 5: Расчёт свободного денежного потока (FCF)

**Построить FCF в правильной последовательности:**
```
EBIT
(-) Taxes (EBIT × Tax Rate)
= NOPAT (Net Operating Profit After Tax)
(+) D&A (non-cash expense, % of revenue)
(-) CapEx (% of revenue, typically 4-8%)
(-) Δ NWC (change in working capital)
= Unlevered Free Cash Flow
```

**Моделирование оборотного капитала:**
- Рассчитывать как % от изменения выручки (Δ revenue)
- Типичный диапазон: –2 % … +2 % от изменения выручки
- Отрицательное число = источник наличности (освобождение оборотного капитала)
- Положительное число = использование наличности (наращивание оборотного капитала)

**Поддержка vs рост CapEx:**
- **Maintenance CapEx**: поддерживает текущие операции (~2‑3 % выручки)
- **Growth CapEx**: поддерживает расширение (дополнительные 2‑5 % выручки)
- Общий CapEx должен соответствовать стратегии роста компании

### Шаг 6: Исследование стоимости капитала (WACC)

**Методология CAPM для стоимости собственного капитала:**
```
Cost of Equity = Risk-Free Rate + Beta × Equity Risk Premium

Where:
- Risk-Free Rate = Current 10-Year Treasury Yield
- Beta = 5-year monthly stock beta vs market index
- Equity Risk Premium = 5.0-6.0% (market standard)
```

**Расчёт стоимости долга:**
```
After-Tax Cost of Debt = Pre-Tax Cost of Debt × (1 - Tax Rate)

Determine Pre-Tax Cost of Debt from:
- Credit rating (if available)
- Current yield on company bonds
- Interest expense / Total Debt from financials
```

**Веса структуры капитала:**
```
Market Value Equity = Current Stock Price × Shares Outstanding
Net Debt = Total Debt - Cash & Equivalents
Enterprise Value = Market Cap + Net Debt

Equity Weight = Market Cap / Enterprise Value
Debt Weight = Net Debt / Enterprise Value

WACC = (Cost of Equity × Equity Weight) + (After-Tax Cost of Debt × Debt Weight)
```

**Особые случаи:**
- **Net Cash Position**: если Cash > Debt, чистый долг **NEGATIVE**
  - Вес долга может быть отрицательным
  - Расчёт WACC корректируется соответствующим образом
- **No Debt**: WACC = Cost of Equity

**Типичные диапазоны WACC:**
- Крупные стабильные компании: 7‑9 %
- Ростовые компании: 9‑12 %
- Высокий рост/риск: 12‑15 %

### Шаг 7: Применение дисконтной ставки (прогноз 5‑10 лет)

**Конвенция середины года:**
- Денежные потоки считаются происходящими в середине года
- Период дисконтирования: 0.5, 1.5, 2.5, 3.5, 4.5 и т.д.
- Дисконтный фактор = `1 / (1 + WACC)^Period`

**Расчёт приведённой стоимости:**
```
For each projection year:
PV of FCF = Unlevered FCF × Discount Factor

Example (Year 1):
FCF = $1,000
WACC = 10%
Period = 0.5
Discount Factor = 1 / (1.10)^0.5 = 0.9535
PV = $1,000 × 0.9535 = $954
```

**Выбор периода прогноза:**
- **5 лет**: стандарт для большинства анализов
- **7‑10 лет**: ростовые компании с более длительным «взлётом»
- **3 года**: зрелый, стабильный бизнес

### Шаг 8: Расчёт терминальной стоимости

**Метод постоянного роста (предпочтительно):**
```
Terminal FCF = Final Year FCF × (1 + Terminal Growth Rate)
Terminal Value = Terminal FCF / (WACC - Terminal Growth Rate)

Critical Constraint: Terminal Growth < WACC (otherwise infinite value)
```

**Выбор терминального темпа роста:**
- Консервативный: 2.0‑2.5 % (рост ВВП)
- Умеренный: 2.5‑3.5 %
- Агрессивный: 3.5‑5.0 % (только для рыночных лидеров)

**Не превышать** безрисковую ставку или долгосрочный рост ВВП.

**Метод мультипликатора выхода (альтернатива):**
```
Terminal Value = Final Year EBITDA × Exit Multiple

Where Exit Multiple comes from:
- Industry comparable trading multiples
- Precedent transaction multiples
- Typical range: 8-15x EBITDA
```

**Приведённая стоимость терминальной стоимости:**
```
PV of Terminal Value = Terminal Value / (1 + WACC)^Final Period

Where Final Period accounts for timing:
5-year model with mid-year convention: Period = 4.5
```

**Проверка разумности терминальной стоимости:**
- Должна составлять 50‑70 % от **Enterprise Value**
- Если > 75 %, модель может слишком сильно полагаться на терминальные допущения
- Если < 40 %, проверь, не слишком консервативны ли допущения

### Шаг 9: Переход от Enterprise Value к Equity Value

**Структура сводки оценки:**
```
(+) Sum of PV of Projected FCFs = $X million
(+) PV of Terminal Value = $Y million
= Enterprise Value = $Z million

(-) Net Debt [or + Net Cash if negative] = $A million
= Equity Value = $B million

÷ Diluted Shares Outstanding = C million shares
= Implied Price per Share = $XX.XX

Current Stock Price = $YY.YY
Implied Return = (Implied Price / Current Price) - 1 = XX%
```

**Критические корректировки:**
- **Net Debt = Total Debt – Cash & Equivalents**
  - Если положительно: вычесть из EV (уменьшает equity value)
  - Если отрицательно (Net Cash): добавить к EV (увеличивает equity value)
- **Использовать Diluted Shares**: включает опционы, RSU, конвертируемые ценные бумаги
- **Другие корректировки** (при необходимости):
  - Миноритарные интересы
  - Пенсионные обязательства
  - Обязательства по операционной аренде

**Формат вывода оценки:**
```csv
Valuation Component,Amount ($M)
PV Explicit FCFs,X.X
PV Terminal Value,Y.Y
Enterprise Value,Z.Z
(-) Net Debt,A.A
Equity Value,B.B
,,
Shares Outstanding (M),C.C
Implied Price per Share,$XX.XX
Current Share Price,$YY.YY
Implied Upside/(Downside),+XX%
```

```
=INDEX(B10:D10, 1, $B$6)
```

**НЕ так – разбросанные `IF`‑выражения:**
```
=IF($B$6=1,[Bear block cell],IF($B$6=2,[Base block cell],[Bull block cell]))
```

Подход с консолидационным столбцом централизует логику и упрощает аудит модели.

### Правильный шаблон прогноза выручки

**Создать столбец консолидации с `INDEX`, затем использовать его в проекциях:**

**Шаг 1 – столбец консолидации для роста FY1:**
```
=INDEX([Bear FY1 growth]:[Bull FY1 growth], 1, $B$6)
```

**Шаг 2 – ссылка на столбец консолидации в расчёте выручки:**
```
Revenue Year 1: =D29*(1+$E$10)
```

Где:
- `D29` = выручка предыдущего года
- `$E$10` = ячейка консолидации для роста FY1 (содержит `INDEX`)
- `$B$6` = селектор случая (1=Bear, 2=Base, 3=Bull)

Такой подход чище, чем встраивание `IF` в каждую формулу, и облегчает аудит.

### Правильный шаблон формулы FCF

**Использовать столбцы консолидации с `INDEX`, затем ссылаться на них в расчётах FCF:**

**Подход с консолидационным столбцом:**
```csv

D&A,=E29*$E$21,$E$21 = consolidation column for D&A %
CapEx,=E29*$E$22,$E$22 = consolidation column for CapEx %
```

**Каждая ячейка консолидационного столбца содержит формулу `INDEX`,** вытягивая данные из соответствующего блока сценария по селектору случая. Это сохраняет формулы проекций чистыми и проверяемыми.

### Правильная реализация таблиц чувствительности

**ВАЖНО**: это НЕ функция Excel «Data Table». Это простые сетки, где пишутся обычные формулы через `openpyxl`. Да, около 75 формул (3 таблицы × 25 ячеек), но задача проста и обязательна.

**Программное заполнение формулами:**

Каждая таблица чувствительности должна быть полностью заполнена формулами, пересчитывающими подразумеваемую цену акции для каждой комбинации допущений. **Не использовать** функцию Data Table (требует ручного вмешательства и не автоматизируется `openpyxl`).

**Подход – КОНКРЕТНЫЙ ПРИМЕР:**

**Структура таблицы – сетка 5×5 (нечётное количество, базовый случай в центре):**

Если базовый `WACC` = 9.0 % и базовый терминальный рост = 3.0 %, построить оси симметрично вокруг этих значений:

```csv
WACC vs Terminal Growth,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,       [fml], [fml], [fml], [fml], [fml]
              8.5%,       [fml], [fml], [fml], [fml], [fml]
              9.0%,       [fml], [fml], [★  ], [fml], [fml]   ← middle row = base WACC
              9.5%,       [fml], [fml], [fml], [fml], [fml]
             10.0%,       [fml], [fml], [fml], [fml], [fml]
                                   ↑
                          middle col = base terminal g
```

**★ = центральная ячейка.** Её формула ДОЛЖНА возвращать фактическую цену акции из сводки оценки. Применить заливку `#BDD7EE` и полужирный шрифт, чтобы визуально зафиксировать базовый случай.

**Правило для значений осей:**
`axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]` – симметрично вокруг базового, нечётное количество гарантирует центр.

**Шаблон формулы – ячейка B88 (`WACC` = 8.0 %, `Terminal Growth` = 2.0 %):**

Формула в B88 должна пересчитывать цену, используя:
- Ставка дисконтирования из заголовка строки: `$A88` (8.0 %)
- Темп роста из заголовка столбца: `B$87` (2.0 %)

**Рекомендуемый подход:** сослаться на основной расчёт DCF, заменив эти параметры.

**Пример структуры формулы:**
```
=([SUM of PV FCFs using $A88 as discount rate] + [Terminal Value using B$87 as growth rate and $A88 as WACC] - [Net Debt]) / [Shares]
```

**КРИТИЧНО – написать формулу для КАЖДОЙ ячейки в сетке 5×5** (25 ячеек на таблицу, 75 ячеек всего). Использовать `openpyxl` для записи формул в цикле. Не пропускать ни одной ячейки и не оставлять текст‑заполнитель.

**Шаблон реализации на Python:**
```python
# Pseudocode for populating sensitivity table
for row_idx, wacc_value in enumerate(wacc_range):
```

### Стандарты форматирования

**ВАЖНО**: Следуй правилам построения формул и конвенциям числового форматирования из навыка `xlsx`. Навык DCF добавляет специфические стандарты визуального оформления.

#### Цветовая схема – два уровня

**Уровень 1: Цвета шрифта (ОБЯЗАТЕЛЬНО из навыка `xlsx`)**
- **Синий текст (RGB 0,0,255)** – ВСЕ жёстко заданные входные данные (цена акции, количество акций, исторические данные, предположения)
- **Чёрный текст (RGB 0,0,0)** – ВСЕ формулы и расчёты
- **Зелёный текст (RGB 0,128,0)** – Ссылки на другие листы (ссылка на лист WACC)

**Уровень 2: Цвета заливки – профессиональная палитра синих/серых (по умолчанию, если пользователь не указал иначе)**
- **Минимум** – используй только синие и серые цвета заливки. Не вводи зелёные, жёлтые, оранжевые или несколько акцентных цветов. Модель с избыточным количеством цветов выглядит непрофессионально.
- **Палитра заливки по умолчанию:**
  - **Заголовки разделов** – тёмно‑синий (RGB 31,78,121 / `#1F4E79`) фон с белым жирным текстом
  - **Подзаголовки/заголовки столбцов** – светло‑синий (RGB 217,225,242 / `#D9E1F2`) фон с чёрным жирным текстом
  - **Ячейки ввода** – светло‑серый (RGB 242,242,242 / `#F2F2F2`) фон с синим шрифтом — или просто белый фон с синим шрифтом, если нужен максимальный минимум
  - **Ячейки расчётов** – белый фон с чёрным шрифтом
  - **Строки вывода/итогов** (стоимость на акцию, EV и т.п.) – средний синий (RGB 189,215,238 / `#BDD7EE`) фон с чёрным жирным шрифтом

**Итого — 3 синих + 1 серый + белый.** Сопротивляйся желанию добавить больше.
Шаблоны, предоставленные пользователем, или явные предпочтения по цвету **всегда** переопределяют эти значения по умолчанию.

**Как работают уровни вместе:**
- Ячейка ввода: синий шрифт + светло‑серый фон = «жёстко заданный ввод»
- Ячейка формулы: чёрный шрифт + белый фон = «вычисленное значение»
- Ссылка на лист: зелёный шрифт + белый фон = «ссылка с другого листа»
- Ключевой вывод: чёрный жирный шрифт + средний синий фон = «это ответ»

**Цвет шрифта** указывает, **ЧТО** это (ввод/формула/ссылка). **Цвет заливки** указывает, **ГДЕ** ты находишься (заголовок/данные/итог).

## Реализация выбора сценария

**Трёхсценарный фреймворк:**

### Медвежий сценарий
- Консервативный рост выручки (нижний предел исторического диапазона)
- Сжатие маржи или отсутствие её расширения
- Более высокий WACC (рост риск‑премии)
- Более низкий темп роста терминала
- Более высокие предположения по CapEx

### Базовый сценарий
- Рост выручки согласно консенсусу или рекомендациям руководства
- Умеренное расширение маржи за счёт операционного рычага
- Текущий рыночный WACC
- Темп роста терминала, согласованный с ВВП (2,5‑3,0 %)
- Стандартные предположения по CapEx

### Бычий сценарий
- Оптимистичный рост выручки (верхний предел прогнозов)
- Значительное расширение маржи
- Более низкий WACC (сниженная риск‑премия)
- Более высокий темп роста терминала (3,5‑5,0 %)
- Сниженная интенсивность CapEx

**Реализация формул**

**НЕ используйте вложенные IF‑формулы, разбросанные по листу.** Вместо этого создайте столбец консолидации, который использует формулы `INDEX` или `OFFSET` для получения значений из соответствующего блока сценария.

**Рекомендуемый шаблон (с использованием `INDEX`):**
```
=INDEX(B10:D10, 1, $B$6)
```
где `B10:D10` — значения для медвежьего, базового и бычьего сценариев, `1` — смещение строки, `$B$6` — ячейка‑селектор сценария (1, 2 или 3).

**Затем используйте столбец консолидации** во всех прогнозах:
```
Revenue Year 1: =D29*(1+$E$10)
```
где `$E$10` — значение столбца консолидации для роста в первый год.

Такой подход централизует логику сценариев, делая модель проще для аудита и поддержки.
