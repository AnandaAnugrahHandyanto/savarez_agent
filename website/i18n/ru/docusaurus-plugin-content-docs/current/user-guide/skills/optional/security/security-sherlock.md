# Sherlock OSINT Username Search

Отслеживай аккаунты в социальных сетях по имени пользователя в более чем 400 соцсетях с помощью [Sherlock Project](https://github.com/sherlock-project/sherlock).

## Когда использовать

- Пользователь просит найти аккаунты, связанные с именем пользователя
- Пользователь хочет проверить доступность имени пользователя на платформах
- Пользователь проводит исследование OSINT или разведку
- Пользователь спрашивает «где зарегистрировано это имя пользователя?» или аналогично

## Требования

- Установлен Sherlock CLI: `pipx install sherlock-project` или `pip install sherlock-project`
- При желании: доступен Docker (`docker run -it --rm sherlock/sherlock`)
- Сетевой доступ для запросов к социальным платформам

## Процедура

### 1. Проверить, установлен ли Sherlock

**Перед выполнением любых действий** убедись, что `sherlock` доступен:

```bash
sherlock --version
```

Если команда завершилась ошибкой:
- Предложи установить: `pipx install sherlock-project` (рекомендовано) или `pip install sherlock-project`
- **Не** пытайся использовать несколько методов установки — выбери один и продолжай
- Если установка не удалась, сообщи пользователю и остановись

### 2. Извлечь имя пользователя

**Извлеки имя пользователя напрямую из сообщения, если оно явно указано.**

Примеры, когда **НЕ** следует уточнять:
- `Find accounts for nasa` → имя пользователя `nasa`
- `Search for johndoe123` → имя пользователя `johndoe123`
- `Check if alice exists on social media` → имя пользователя `alice`
- `Look up user bob on social networks` → имя пользователя `bob`

**Уточнять следует только если:**
- Упомянуто несколько потенциальных имён («search for alice or bob»)
- Формулировка неоднозначна («search for my username» без уточнения)
- Имя пользователя не указано вовсе («do an OSINT search»)

При извлечении берите **точное** имя пользователя, как оно указано — сохраняйте регистр, цифры, подчёркивания и т.д.

### 3. Сформировать команду

**Команда по умолчанию** (используй её, если пользователь явно не запросил иначе):

```bash
sherlock --print-found --no-color "<username>" --timeout 90
```

**Опциональные флаги** (добавляй только при явном запросе пользователя):
- `--nsfw` — включить NSFW‑сайты (только если пользователь просит)
- `--tor` — маршрутизировать через Tor (только если пользователь просит анонимность)

**Не задавай вопросов о параметрах через уточнение** — просто запусти поиск по умолчанию. Пользователь может запросить конкретные опции при необходимости.

### 4. Выполнить поиск

Запусти через инструмент `terminal`. Команда обычно занимает 30‑120 секунд в зависимости от сетевых условий и количества сайтов.

**Пример вызова терминала:**

```json
{
  "command": "sherlock --print-found --no-color \"target_username\"",
  "timeout": 180
}
```

### 5. Разобрать и представить результаты

Sherlock выводит найденные аккаунты в простом формате. Разбери вывод и представь:

1. **Строка‑резюме:** «Found X accounts for username 'Y'»
2. **Сгруппированные ссылки:** При необходимости группируй по типу платформ (социальные, профессиональные, форумы и т.д.)
3. **Местоположение файла вывода:** По умолчанию Sherlock сохраняет результаты в `<username>.txt`

**Пример разбора вывода:**

```
[+] Instagram: https://instagram.com/username
[+] Twitter: https://twitter.com/username
[+] GitHub: https://github.com/username
```

Представляй результаты как кликабельные ссылки, когда это возможно.

## Подводные камни

### Нет найденных результатов
Если Sherlock не нашёл аккаунтов, это часто правильно — имя пользователя может не быть зарегистрировано на проверяемых платформах. Предложи:
- Проверить орфографию/вариацию
- Попробовать похожие имена с подстановкой `?`: `sherlock "user?name"`
- Учесть, что у пользователя могут быть настройки приватности или удалённые аккаунты

### Проблемы с таймаутом
Некоторые сайты работают медленно или блокируют автоматические запросы. Используй `--timeout 120`, чтобы увеличить время ожидания, или `--site`, чтобы сузить область поиска.

### Настройка Tor
`--tor` требует запущенного демона Tor. Если пользователь хочет анонимность, но Tor недоступен, предложи:
- Установить сервис Tor
- Использовать `--proxy` с альтернативным прокси

### Ложные срабатывания
Некоторые сайты всегда возвращают «found» из‑за своей структуры ответа. Сравни неожиданные результаты с ручной проверкой.

### Ограничения скорости
Агрессивные запросы могут вызвать rate‑limiting. Для массового поиска имён пользователей добавляй задержки между вызовами или используй `--local` с кэшированными данными.
