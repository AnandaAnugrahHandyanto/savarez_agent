# Конвейер производства видео Manim
## Когда использовать

Используй, когда пользователи запрашивают: анимированные объяснения, математические анимации, визуализацию концепций, пошаговый разбор алгоритмов, технические разъяснения, видео в стиле 3Blue1Brown или любую программную анимацию с геометрическим / математическим содержанием. Создаёт объясняющие видео в стиле 3Blue1Brown, визуализации алгоритмов, вывод уравнений, схемы архитектуры и истории данных с помощью Manim Community Edition.
## Творческий стандарт

Это образовательное кино. Каждый кадр учит. Каждая анимация раскрывает структуру.

**Прежде чем написать единую строку кода**, сформулируй сюжетную арку. Какое заблуждение это исправляет? Каков «момент «ага»? Какую визуальную историю рассказать, чтобы зритель прошёл от путаницы к пониманию? Запрос пользователя — отправная точка — интерпретируй его с педагогической амбициозностью.

**Геометрия перед алгеброй.** Сначала показывай форму, потом уравнение. Визуальная память запоминается быстрее, чем символическая. Когда зритель видит геометрический паттерн до формулы, уравнение кажется заслуженным.

**Первый рендер — непереговорно.** Вывод должен быть визуально чётким и эстетически согласованным без раундов правок. Если что‑то выглядит захламлённым, плохо синхронным или как «AI‑сгенерированные слайды», это ошибка.

**Непрозрачность слоёв направляет внимание.** Никогда не показывай всё на полной яркости. Основные элементы — 1.0, контекстные — 0.4, структурные (оси, сетки) — 0.15. Мозг обрабатывает визуальную значимость слоями.

**Пространство для дыхания.** Каждая анимация требует `self.wait()` после неё. Зрителю нужно время, чтобы усвоить то, что только что появилось. Не спеши от одной анимации к другой. Пауза в 2 секунды после ключевого раскрытия никогда не тратится зря.

**Единый визуальный язык.** Все сцены используют одну цветовую палитру, согласованные размеры типографики, одинаковые скорости анимации. Технически корректное видео, где каждая сцена использует случайные разные цвета, — это эстетический провал.

## Конвейер

```
PLAN --> CODE --> RENDER --> STITCH --> AUDIO (optional) --> REVIEW
```

1. **PLAN** — создай `plan.md` с повествовательной аркой, списком сцен, визуальными элементами, цветовой палитрой и сценарием озвучки
2. **CODE** — создай `script.py`, где каждая сцена реализована отдельным классом, который можно рендерить независимо
3. **RENDER** — `manim -ql script.py Scene1 Scene2 …` — черновой рендер; `-qh` — рендер для продакшна
4. **STITCH** — склейка видеоклипов сцен с помощью `ffmpeg concat` в файл `final.mp4`
5. **AUDIO** (optional) — добавь озвучку и/или фоновую музыку через `ffmpeg`. См. `references/rendering.md`
6. **REVIEW** — отрендери превью‑кадры, сравни их с планом, при необходимости внеси правки

**Используй моноширинные шрифты для всего текста.** Рендерер Pango в Manim даёт сломанный кернинг при пропорциональных шрифтах любого размера. См. `references/visual-design.md` для полных рекомендаций.

### Вариации по сценам

Никогда не используй одинаковую конфигурацию для всех сцен. Для каждой сцены:
- **Разный доминирующий цвет** из палитры
- **Разный макет** — не всегда центрируй всё
- **Разный способ входа анимации** — чередуй Write, FadeIn, GrowFromCenter, Create
- **Разный визуальный вес** — в одних сценах плотность, в других — разреженность
## Рабочий процесс

### Шаг 1: План (plan.md)

Перед любым кодом создай `plan.md`. Смотри `references/scene-planning.md` для полного шаблона.

### Шаг 2: Код (script.py)

По одному классу на сцену. Каждая сцена рендерится независимо.

```python
from manim import *

BG = "#1C1C1C"
PRIMARY = "#58C4DD"
SECONDARY = "#83C167"
ACCENT = "#FFFF00"
MONO = "Menlo"

class Scene1_Introduction(Scene):
    def construct(self):
        self.camera.background_color = BG
        title = Text("Why Does This Work?", font_size=48, color=PRIMARY, weight=BOLD, font=MONO)
        self.add_subcaption("Why does this work?", duration=2)
        self.play(Write(title), run_time=1.5)
        self.wait(1.0)
        self.play(FadeOut(title), run_time=0.5)
```

Ключевые паттерны:
- **Субтитры** на каждой анимации: `self.add_subcaption("text", duration=N)` или `subcaption="text"` в `self.play()`
- **Общие константы цветов** в начале файла для согласованности между сценами
- **`self.camera.background_color`** задаётся в каждой сцене
- **Корректный выход** — FadeOut всех объектов в конце сцены: `self.play(FadeOut(Group(*self.mobjects)))`

### Шаг 3: Рендер

```bash
manim -ql script.py Scene1_Introduction Scene2_CoreConcept  # draft
manim -qh script.py Scene1_Introduction Scene2_CoreConcept  # production
```

## Критические замечания по реализации

### Сырые строки для LaTeX
```python
# WRONG: MathTex("\frac{1}{2}")
# RIGHT:
MathTex(r"\frac{1}{2}")
```

### buff >= 0.5 для текста краев
```python
label.to_edge(DOWN, buff=0.5)  # never < 0.5
```

### FadeOut перед заменой текста
```python
self.play(ReplacementTransform(note1, note2))  # not Write(note2) on top
```

### Никогда не анимировать не добавленные Mobjects
```python
self.play(Create(circle))  # must add first
self.play(circle.animate.set_color(RED))  # then animate
```

## Creative Divergence (использовать только по запросу пользователя на экспериментальный/креативный/уникальный вывод)

Если пользователь просит креативный, экспериментальный или нетрадиционный способ объяснения, выбери стратегию и обоснуй её **до** разработки анимации.

- **SCAMPER** — когда нужен свежий взгляд на стандартное объяснение
- **Assumption Reversal** — когда нужно поставить под вопрос привычный способ подачи материала

### Трансформация SCAMPER
Возьми обычную математическую/техническую визуализацию и преобразуй её:
- **Substitute**: замените привычную визуальную метафору (числовая прямая → извилистый путь, матрица → городская сетка)
- **Combine**: объедините два подхода к объяснению (алгебраический + геометрический одновременно)
- **Reverse**: выводите «обратно» — начните с результата и деконструируйте его до аксиом
- **Modify**: преувеличьте параметр, чтобы показать его значимость (10× скорость обучения, 1000× размер выборки)
- **Eliminate**: уберите всю нотацию — объясняйте только с помощью анимации и пространственных отношений

### Переворот предположений (Assumption Reversal)
1. Перечисли, что «стандартно» в визуализации этой темы (слева направо, 2‑D, дискретные шаги, формальная нотация).
2. Выбери самое фундаментальное предположение.
3. Переверни его (вывод справа налево, 3‑D‑встраивание 2‑D‑концепции, непрерывное морфирование вместо шагов, отсутствие нотации).
4. Исследуй, что раскрывает такой переворот, чего скрывает стандартный подход.
