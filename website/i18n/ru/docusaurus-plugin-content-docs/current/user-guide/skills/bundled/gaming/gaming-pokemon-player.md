# Pokemon Player

Играй в игры Pokemon через безголовую эмуляцию, используя пакет `pokemon-agent`.
## Когда использовать
- Пользователь говорит «play pokemon», «start pokemon», «pokemon game»
- Пользователь спрашивает о Pokemon Red, Blue, Yellow, FireRed и т.п.
- Пользователь хочет посмотреть, как ИИ играет в Pokemon
- Пользователь ссылается на ROM‑файл (.gb, .gbc, .gba)
## Процедура запуска

### 1. Первичная настройка (клонирование, venv, установка)
Репозиторий — NousResearch/pokemon-agent на GitHub. Клонируй его, затем создай виртуальное окружение Python 3.10+. Используй `uv` (предпочтительно для скорости) для создания venv и установки пакета в режиме редактирования с дополнением `pyboy`. Если `uv` недоступен, перейди к `python3 -m venv` + `pip`.

На этой машине он уже настроен в `/home/teknium/pokemon-agent` с готовым venv — просто перейди в каталог и активируй его командой `source .venv/bin/activate`.

Тебе также нужен файл ROM. Спроси пользователя о его файле. На этой машине он находится в `roms/pokemon_red.gb` внутри этого каталога.
**НИКОГДА** не скачивай и не предоставляй файлы ROM — всегда спрашивай пользователя.

### 2. Запуск игрового сервера
Находясь в каталоге `pokemon-agent` с активированным venv, запусти `pokemon-agent serve` с параметром `--rom`, указывающим путь к ROM, и `--port 9876`. Запусти его в фоновом режиме с помощью `&`. Чтобы продолжить с сохранённой игрой, добавь `--load-state` с именем сохранения. Подожди 4 секунды для запуска, затем проверь работоспособность запросом `GET /health`.

### 3. Настройка живой панели мониторинга для пользователя
Используй обратный SSH‑туннель через `localhost.run`, чтобы пользователь мог просматривать панель в своём браузере. Подключись через `ssh`, пробрасывая локальный порт `9876` к удалённому порту `80` на `nokey@localhost.run`. Перенаправь вывод в файл журнала, подожди 10 секунд, затем выполни `grep` в журнале для поиска URL‑а `.lhr.life`. Передай пользователю найденный URL с добавленным `/dashboard/`. URL туннеля меняется каждый раз — при перезапуске дай пользователю новый URL.
## Сохранение и загрузка

## Справочник действий
- press_a — подтвердить, поговорить, выбрать
- press_b — отменить, закрыть меню
- press_start — открыть игровое меню
- walk_up/down/left/right — переместиться на одну клетку
- hold_b_N — удерживать B в течение N кадров (используется для ускорения текста)
- wait_60 — ждать около 1 секунды (60 кадров)
- a_until_dialog_end — нажимать A повторно, пока диалог не завершится
## Критические советы из опыта

### ИСПОЛЬЗУЙ ВИДЕНИЕ ПОСТОЯННО
- Делай скриншот каждые 2‑4 шага перемещения.
- Состояние RAM показывает позицию и HP, но **не** показывает, что находится вокруг.
- Края, заборы, знаки, двери зданий, NPC — видны только на скриншоте.
- Задавай модели зрения конкретные вопросы: «что находится один тайл к северу от меня?».
- Когда застрял, всегда делай скриншот перед тем, как пробовать случайные направления.

### Переходы через варпы требуют дополнительного ожидания
При проходе двери или лестницы экран затемняется во время перехода карты. Ты **должен** дождаться завершения. Добавь 2‑3 действия `wait_60` после любого варпа через дверь/лестницу. Без ожидания позиция будет устаревшей, и ты подумаешь, что всё ещё находишься на старой карте.

### Ловушка выхода из здания
Когда выходишь из здания, ты оказываешься прямо **перед** дверью. Если пойдёшь на север, сразу вернёшься внутрь. **Всегда** сначала отойди в сторону, пройдя 2 тайла влево или вправо, а затем двигайся в нужном направлении.

### Обработка диалогов
Текст первого поколения прокручивается медленно буква за буквой. Чтобы ускорить диалог, удерживай **B** на 120 кадров, затем нажми **A**. Повтори при необходимости. Удерживание **B** выводит текст на максимальной скорости, после чего нажимай **A**, чтобы перейти к следующей строке. Действие `a_until_dialog_end` проверяет флаг диалога в RAM, но этот флаг не охватывает **все** состояния текста. Если диалог кажется застрявшим, используй шаблон `hold_b + press_a` вручную и проверь результат скриншотом.

### Края — односторонние
Края (маленькие обрывы) можно прыгать **только вниз** (на юг), но нельзя подниматься **вверх** (на север). Если путь заблокирован краем, ведущим на север, нужно идти влево или вправо, чтобы найти проход вокруг него. Используй зрение, чтобы определить, в какую сторону находится проход, и спроси модель зрения явно.

### Стратегия навигации
- Двигайся 2‑4 шага за раз, затем делай скриншот для проверки позиции.
- При входе в новую зону сразу делай скриншот, чтобы сориентироваться.
- Спрашивай модель зрения: «в каком направлении к [цель]?».
- Если застрял более 3 попыток, сделай скриншот и полностью переоцените ситуацию.
- Не спамь 10‑15 перемещений — ты перегнешь цель или застрянешь.

### Побег от диких боёв
В меню боя **RUN** находится в правом нижнем углу. Чтобы добраться до него из позиции курсора по умолчанию (**FIGHT**, верхний левый), нажми **down**, затем **right**, чтобы переместить курсор на **RUN**, и нажми **A**. Оберни это удержанием **B**, чтобы ускорить текст и анимацию.

### Бой (FIGHT)
В меню боя **FIGHT** находится в верхнем левом углу (позиция курсора по умолчанию). Нажми **A**, чтобы открыть выбор приёма, ещё раз **A**, чтобы использовать первый приём. Затем удерживай **B**, чтобы ускорить анимацию атаки и текст.
## Стратегия боя

### Дерево решений
1. Хочешь поймать? → Ослабь, а затем брось **Poké Ball**
2. Дикая, она тебе не нужна? → **RUN**
3. Преимущество по типу? → Используй суперэффективный приём
4. Нет преимущества? → Используй самый сильный **STAB‑приём**
5. Низкое HP? → Сменить покемона или использовать **Potion**

### Таблица типов Gen 1 (ключевые противостояния)
- **Water** побеждает **Fire**, **Ground**, **Rock**
- **Fire** побеждает **Grass**, **Bug**, **Ice**
- **Grass** побеждает **Water**, **Ground**, **Rock**
- **Electric** побеждает **Water**, **Flying**
- **Ground** побеждает **Fire**, **Electric**, **Rock**, **Poison**
- **Psychic** побеждает **Fighting**, **Poison** (доминирует в Gen 1!)

### Особенности Gen 1
- Статистика **Special** = и атака, и защита для специальных приёмов
- Тип **Psychic** переусилен (приёмы **Ghost** багнут)
- Критические удары зависят от **Speed**
- **Wrap/Bind** не дают противнику действовать
- Баг **Focus Energy**: **уменьшает** шанс критов вместо повышения
