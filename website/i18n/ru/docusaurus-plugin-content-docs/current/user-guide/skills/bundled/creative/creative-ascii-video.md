## Архитектура конвейера

Каждый режим использует один и тот же 6‑ступенчатый конвейер:

```
INPUT → ANALYZE → SCENE_FN → TONEMAP → SHADE → ENCODE
```

1. **INPUT** — Загрузка/декодирование исходного материала (видеокадры, аудиосэмплы, изображения или ничего)
2. **ANALYZE** — Выделение признаков для каждого кадра (аудиобанды, яркость и контуры видео, векторы движения)
3. **SCENE_FN** — Функция сцены рендерит в пиксельный канвас (`uint8 H,W,3`). Компонует несколько сеток персонажей через `_render_vf()` + режимы смешивания пикселей. См. `references/composition.md`
4. **TONEMAP** — Адаптивная нормализация яркости на основе процентилей. См. `references/composition.md` § Adaptive Tonemap
5. **SHADE** — Постобработка через `ShaderChain` + `FeedbackBuffer`. См. `references/shaders.md`
6. **ENCODE** — Передача необработанных RGB‑кадров в ffmpeg для кодирования в H.264/GIF

## Критические замечания по реализации

### Яркость — используйте `tonemap()`, а не линейные множители

Это самая критичная визуальная проблема. ASCII на чёрном фоне по умолчанию выглядит тёмным. **Никогда не используйте множители `canvas * N`** — они обрезают светлые участки. Используйте адаптивный тонмап:

```python
def tonemap(canvas, gamma=0.75):
    f = canvas.astype(np.float32)
    lo, hi = np.percentile(f[::4, ::4], [1, 99.5])
    if hi - lo < 10: hi = lo + 10
    f = np.clip((f - lo) / (hi - lo), 0, 1) ** gamma
    return (f * 255).astype(np.uint8)
```

Конвейер: `scene_fn() → tonemap() → FeedbackBuffer → ShaderChain → ffmpeg`

Гамма для каждой сцены: по умолчанию 0.75, соляризация 0.55, постеризация 0.50, яркие сцены 0.85. Используйте режим смешения `screen` (а не `overlay`) для тёмных слоёв.

### Высота ячейки шрифта

macOS Pillow: `textbbox()` возвращает неверную высоту. Используйте `font.getmetrics()`: `cell_height = ascent + descent`. См. `references/troubleshooting.md`.

### Deadlock при работе ffmpeg через pipe

Никогда не используйте `stderr=subprocess.PIPE` с длительно работающим ffmpeg — буфер заполняется до 64 KB и приводит к deadlock‑у. Перенаправляйте вывод в файл. См. `references/troubleshooting.md`.

### Совместимость шрифтов

Не все символы Unicode отображаются во всех шрифтах. Проверяйте палитры при инициализации — отрисовывайте каждый символ и проверяйте, что вывод не пустой. См. `references/troubleshooting.md`.

### Архитектура per‑clip

Для сегментированных видео (цитаты, сцены, главы) рендерите каждый фрагмент как отдельный клип‑файл, чтобы обеспечить параллельный рендеринг и выборочный повторный рендер. См. `references/scenes.md`.

## Creative Divergence (использовать только по запросу пользователя на экспериментальный/креативный/уникальный вывод)

Если пользователь просит креативный, экспериментальный, удивительный или нетрадиционный результат, выбери наиболее подходящую стратегию и продумай её шаги **до** генерации кода.

- **Forced Connections** — когда пользователь хочет вдохновения из другой области («сделай органичным», «индустриальная эстетика»)
- **Conceptual Blending** — когда пользователь назвал два объекта для объединения («океан встречает музыку», «космос + каллиграфия»)
- **Oblique Strategies** — когда пользователь полностью открыт («удиви меня», «что‑то, чего я ещё не видел»)

### Forced Connections
1. Выбери область, не связанную с визуальной задачей (метеорологические системы, микробиология, архитектура, гидродинамика, текстильное плетение).
2. Перечисли её ключевые визуальные/структурные элементы (эрозия → постепенное раскрытие; митоз → разветвление; плетение → взаимосвязанные узоры).
3. Отобрази эти элементы с помощью ASCII‑символов и анимационных паттернов.
4. Синтезируй — как будет выглядеть «эрозия» или «кристаллизация» в сетке символов?

### Conceptual Blending
1. Назови два разных визуальных/концептуальных пространства (например, волны океана + нотный лист).
2. Найди соответствия (пики = высокие ноты, впадины = паузы, пена = стаккато).
3. Смешай выборочно — оставь самые интересные соответствия, откажись от принудительных.
4. Разработай возникающие свойства, которые существуют только в результате смешения.

### Oblique Strategies
1. Выбери одну из подсказок: «Почитай свою ошибку как скрытое намерение», «Используй старую идею», «Что бы сделал твой лучший друг?», «Подчеркни недостатки», «Переверни всё», «Только часть, а не целое», «Обратный ход».
2. Интерпретируй директиву в контексте текущей задачи по ASCII‑анимации.
3. Применяй полученное латеральное решение к визуальному дизайну перед написанием кода.
