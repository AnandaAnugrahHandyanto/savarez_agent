## Идеи дизайна

**Не создавай скучные слайды.** Обычные маркеры на белом фоне никого не впечатлят. Рассмотри идеи из этого списка для каждого слайда.

### Перед началом

- **Выбери смелую, ориентированную на содержание цветовую палитру**: палитра должна выглядеть разработанной специально для ЭТОЙ темы. Если ты можешь просто поменять цвета на совершенно другую презентацию, и она всё равно «будет работать», значит, ты не сделал достаточно специфичных выборов.
- **Доминирование над равенством**: один цвет должен доминировать (60‑70 % визуального веса), с 1‑2 поддерживающими тонами и одним ярким акцентом. Никогда не давай всем цветам одинаковый вес.
- **Контраст тёмного/светлого**: тёмные фоны для титульных и заключительных слайдов, светлые — для контента («структура «сэндвича»). Либо используй тёмный фон на протяжении всей презентации для премиального ощущения.
- **Приверженность визуальному мотиву**: выбери ОДИН отличительный элемент и повторяй его — закруглённые рамки изображений, иконки в цветных кругах, толстые односторонние границы. Применяй его на каждом слайде.

### Цветовые палитры

Выбирай цвета, соответствующие теме — не используй стандартный синий. Используй эти палитры как вдохновение:

| **Coral Energy** | `F96167` (coral) | `F9E795` (gold) | `2F3C7E` (navy) |

| **Ocean Gradient** | `065A82` (deep blue) | `1C7293` (teal) | `21295C` (midnight) |

### Для каждого слайда

**Каждому слайду нужен визуальный элемент** — изображение, диаграмма, иконка или форма. Слайды только с текстом быстро забываются.

**Варианты макета:**
- Две колонки (текст слева, иллюстрация справа)
- Строки «иконка + текст» (иконка в цветном круге, жирный заголовок, описание ниже)
- Сетка 2×2 или 2×3 (изображение с одной стороны, блоки контента в виде сетки с другой)
- Полноширинное изображение (полностью слева или справа) с наложенным текстом

**Отображение данных:**
- Большие цифры‑выделения (числа 60‑72 pt с небольшими подписями снизу)
- Колонки сравнения (до/после, плюсы/минусы, варианты рядом)
- Хронология или поток процесса (нумерованные шаги, стрелки)

**Визуальная отделка:**
- Иконки в небольших цветных кругах рядом с заголовками разделов
- Курсивный акцентный текст для ключевых цифр или слоганов

### Типографика

**Выбери интересное сочетание шрифтов** — не используй по умолчанию Arial. Подбери шрифт заголовка с характером и сочетай его с чистым шрифтом основного текста.

| Шрифт заголовка | Шрифт основного текста |
|-----------------|------------------------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| Элемент | Размер |
|---------|--------|
| Заголовок слайда | 36‑44 pt bold |
| Заголовок раздела | 20‑24 pt bold |
| Основной текст | 14‑16 pt |
| Подписи | 10‑12 pt muted |

### Расстояния

- Минимальные отступы 0.5 "
- 0.3‑0.5 " между блоками контента
- Оставляй «дыхательное» пространство — не заполняй каждый дюйм

### Ошибки, которых следует избегать

- **Не повторяй один и тот же макет** — меняй колонки, карточки и выделения на разных слайдах.
- **Не центрируй основной текст** — выравнивай абзацы и списки по левому краю; центрируй только заголовки.
- **Не экономь на контрасте размеров** — заголовки должны быть 36 pt + и выделяться над 14‑16 pt основного текста.
- **Не используй синий по умолчанию** — выбирай цвета, отражающие конкретную тему.
- **Не перемешивай отступы случайным образом** — выбирай 0.3 " или 0.5 " промежутки и используй их последовательно.
- **Не оформляй один слайд, а остальные оставляй простыми** — либо полностью привержен стилю, либо держи всё простым на протяжении всей презентации.
- **Не создавай слайды только с текстом** — добавляй изображения, иконки, диаграммы или другие визуальные элементы; избегай простого заголовка + маркированных пунктов.
- **Не забывай про отступы в текстовых блоках** — при выравнивании линий или фигур с краями текста ставь `margin: 0` у текстового блока или смещай форму, учитывая отступы.
- **Не используй элементы с низким контрастом** — иконки **и** текст должны иметь сильный контраст с фоном; избегай светлого текста на светлом фоне или тёмного текста на тёмном фоне.
- **НИКОГДА не используй акцентные линии под заголовками** — это типичный признак слайдов, сгенерированных ИИ; вместо этого используй пустое пространство или цвет фона.
## QA (Required)

**Предположим, что есть проблемы. Твоя задача — найти их.**

Твой первый рендер почти никогда не бывает правильным. Подходи к QA как к охоте за багами, а не как к подтверждению. Если ты не нашёл ни одной проблемы при первом осмотре, значит, ты недостаточно внимательно искал.

### Контент QA

```bash
python -m markitdown output.pptx
```

Проверь отсутствие контента, опечатки, неправильный порядок.

**При использовании шаблонов проверяй наличие оставшегося текста‑заполнителя:**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

Если `grep` возвращает результаты, исправь их до того, как объявишь успех.

### Визуальный QA

**⚠️ ИСПОЛЬЗУЙ СУБАГЕНТЫ** — даже для 2‑3 слайдов. Ты смотришь на код и видишь то, что ожидаешь, а не то, что действительно есть. Субагенты смотрят свежим взглядом.

Преобразуй слайды в изображения (см. [Converting to Images](#converting-to-images)), затем используй следующий запрос:

```
Visually inspect these slides. Assume there are issues — find them.

Look for:
- Overlapping elements (text through shapes, lines through words, stacked elements)
- Text overflow or cut off at edges/box boundaries
- Decorative lines positioned for single-line text but title wrapped to two lines
- Source citations or footers colliding with content above
- Elements too close (< 0.3" gaps) or cards/sections nearly touching
- Uneven gaps (large empty area in one place, cramped in another)
- Insufficient margin from slide edges (< 0.5")
- Columns or similar elements not aligned consistently
- Low-contrast text (e.g., light gray text on cream-colored background)
- Low-contrast icons (e.g., dark icons on dark backgrounds without a contrasting circle)
- Text boxes too narrow causing excessive wrapping
- Leftover placeholder content

For each slide, list issues or areas of concern, even if minor.

Read and analyze these images:
1. /path/to/slide-01.jpg (Expected: [brief description])
2. /path/to/slide-02.jpg (Expected: [brief description])

Report ALL issues found, including minor ones.
```

### Цикл верификации

1. Сгенерировать слайды → Преобразовать в изображения → Проверить
2. **Список найденных проблем** (если проблем не найдено, посмотри ещё более критично)
3. Исправить проблемы
4. **Повторно проверить затронутые слайды** — одно исправление часто создаёт новую проблему
5. Повторять, пока полный проход не покажет отсутствие новых проблем

**Не объявляй успех, пока не завершишь хотя бы один цикл «исправить‑и‑проверить».**
## Преобразование в изображения

Преобразуй презентации в отдельные изображения слайдов для визуального просмотра:

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

Это создаст `slide-01.jpg`, `slide-02.jpg` и т.д.

Чтобы перерендерить конкретные слайды после исправлений:

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
