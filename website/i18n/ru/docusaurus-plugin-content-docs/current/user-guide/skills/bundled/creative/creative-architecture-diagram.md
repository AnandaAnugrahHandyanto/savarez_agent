# Навык «Диаграмма архитектуры»

Генерировать профессиональные, темные технические архитектурные диаграммы в виде автономных HTML‑файлов с встроенной SVG‑графикой. Без внешних инструментов, без API‑ключей, без библиотек рендеринга — просто создать HTML‑файл и открыть его в браузере.

## Область применения

**Наиболее подходит для:**
- Архитектуры программных систем (frontend / backend / уровни баз данных)
- Облачной инфраструктуры (VPC, регионы, подсети, управляемые сервисы)
- Топологии микросервисов / сервис‑меша
- Карты базы данных + API, схемы развертывания
- Любого предмета, связанного с технической инфраструктурой, который вписывается в темную, сетчатую эстетику

**Смотрите в другом месте сначала:**
- Физика, химия, математика, биология и другие научные темы
- Физические объекты (транспорт, оборудование, анатомия, разрезы)
- Планировки помещений, повествовательные путешествия, визуалы в стиле учебников
- Рисованные от руки наброски на доске (рассмотри `excalidraw`)
- Анимированные объяснения (рассмотри навык анимации)

Если существует более специализированный навык для данной темы, предпочтительно использовать его. Если подходящего нет, этот навык также может служить общим запасным вариантом SVG‑диаграммы — вывод будет просто иметь описанную ниже темную техно‑эстетику.

## Технические детали реализации

### Рендеринг компонентов
Компоненты — скруглённые прямоугольники (`rx="6"` ) с обводкой 1.5px. Чтобы стрелки не просвечивали сквозь полупрозрачные заполнения, использовать **двойную маску‑прямоугольник**:
1. Нарисовать непрозрачный фон‑прямоугольник (`#0f172a`)
2. Нарисовать полупрозрачный стилизованный прямоугольник сверху

### Правила соединений
- **Z‑порядок:** Рисовать стрелки *рано* в SVG (после сетки), чтобы они отрисовывались позади блоков компонентов
- **Кончики стрелок:** Определяются через SVG‑маркеры
- **Потоки безопасности:** Пунктирные линии розового цвета (`#fb7185`)
- **Границы:**
  - *Security Groups:* Пунктир (`4,4`), розовый цвет
  - *Regions:* Большой пунктир (`8,4`), янтарный цвет, `rx="12"`

### Логика отступов и размещения
- **Стандартная высота:** 60px (службы); 80‑120px (крупные компоненты)
- **Вертикальный зазор:** Минимум 40px между компонентами
- **Message Buses:** Должны располагаться *в зазоре* между службами, не перекрывая их
- **Размещение легенды:** **КРИТИЧЕСКО**. Должна находиться за пределами всех границ. Вычислить наибольшую Y‑координату всех границ и разместить легенду минимум на 20px ниже её.

## Структура документа

Сгенерированный HTML‑файл состоит из четырёх частей:
1. **Header:** Заголовок с индикатором‑пульсирующей точкой и подзаголовком
2. **Main SVG:** Диаграмма внутри карточки с закруглённой рамкой
3. **Summary Cards:** Сетка из трёх карточек под диаграммой с основными деталями
4. **Footer:** Минимальные метаданные
