# Obsidian Vault

Используй этот навык для работы с Obsidian vault, ориентированной на файловую систему: чтение заметок, перечисление заметок, поиск файлов заметок, создание заметок, добавление содержимого и добавление wikilink‑ов.

## Путь к хранилищу

Укажи известный или разрешённый путь к хранилищу перед вызовом файловых инструментов.

Документированная конвенция `vault-path` — переменная окружения `OBSIDIAN_VAULT_PATH`, например из `~/.hermes/.env`. Если она не задана, используй `~/Documents/Obsidian Vault`.

Файловые инструменты не расширяют переменные оболочки. Не передавай пути, содержащие `$OBSIDIAN_VAULT_PATH`, в `read_file`, `write_file`, `patch` или `search_files`; сначала разреши путь к хранилищу и передай конкретный абсолютный путь. Путь к хранилищу может содержать пробелы, что является ещё одной причиной предпочитать файловые инструменты вместо команд оболочки.

Если путь к хранилищу неизвестен, `terminal` допустим для разрешения `OBSIDIAN_VAULT_PATH` или проверки существования запасного пути. Как только путь известен, переключайся обратно на файловые инструменты.

## Чтение заметки

Используй `read_file` с разрешённым абсолютным путём к заметке. Предпочитай его `cat`, потому что он возвращает номера строк и поддерживает постраничный вывод.

## Перечисление заметок

Используй `search_files` с `target: "files"` и разрешённым путём к хранилищу. Предпочитай его `find` или `ls`.

- Чтобы перечислить все markdown‑заметки, используй `pattern: "*.md"` под путём к хранилищу.
- Чтобы перечислить подпапку, ищи в абсолютном пути этой подпапки.

## Поиск

Используй `search_files` как для поиска по имени файла, так и по содержимому. Предпочитай его `grep`, `find` или `ls`.

- Для поиска по именам файлов используй `search_files` с `target: "files"` и шаблоном `pattern`.
- Для поиска по содержимому заметок используй `search_files` с `target: "content"`, регулярным выражением в `pattern` и `file_glob: "*.md"`, если нужно ограничить совпадения только markdown‑заметками.

## Создание заметки

Используй `write_file` с разрешённым абсолютным путём и полным markdown‑содержимым. Предпочитай его heredoc‑команды оболочки или `echo`, так как это избегает проблем с экранированием в оболочке и возвращает структурированные результаты.

## Добавление к заметке

Предпочитай нативный рабочий процесс с файловыми инструментами, если это не неудобно:

- Прочитай целевую заметку с помощью `read_file`.
- Используй `patch` для привязанного добавления, когда есть стабильный контекст, например добавление раздела после существующего заголовка или добавление перед известным завершающим блоком.
- Используй `write_file`, когда перезапись всей заметки яснее, чем построение хрупкого патча.

Для привязанного добавления с `patch` замени якорь на якорь + новое содержимое.

Для простого добавления без стабильного контекста допустим `terminal`, если это самый понятный и безопасный вариант.

## Целевые правки

Используй `patch` для точечных изменений заметки, когда текущий контент даёт стабильный контекст. Предпочитай его переписыванию текста через оболочку.

## Wikilinks

Obsidian связывает заметки с помощью синтаксиса `[[Note Name]]`. При создании заметок используй их для ссылки на связанное содержание.
