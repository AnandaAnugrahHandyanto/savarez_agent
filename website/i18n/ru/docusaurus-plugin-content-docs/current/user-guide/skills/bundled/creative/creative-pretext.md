# Pretext Creative Demos
## Обзор

[`@chenglou/pretext`](https://github.com/chenglou/pretext) — 15 KB библиотека TypeScript без зависимостей от Cheng Lou (React core, ReasonML, Midjourney) для **измерения и разметки многострочного текста без DOM**. Она делает одну вещь: получив `(text, font, width)`, возвращает разрывы строк, ширины каждой строки, позиции каждой графемы и общую высоту — всё через измерения canvas, без перерасчётов разметки.

Это звучит как «трубопровод». Не так. Благодаря скорости и геометрическому подходу, это **творческий примитив**: можно переразмещать абзацы вокруг движущегося спрайта со скоростью 60 fps, создавать игры, где геометрия уровня состоит из реальных слов, управлять ASCII‑логотипами через прозу, разбивать текст на частицы с точными начальными позициями каждой графемы или упаковывать многострочный UI без какого‑либо `getBoundingClientRect`‑шторма.

Этот **skill** существует, чтобы Hermes мог делать **крутые демо** с его помощью — те, что люди публикуют в X. Смотри `pretext.cool` и `chenglou.me/pretext` для демонстрационного корпуса сообщества.
## Когда использовать

Используй, когда пользователь просит:
- «pretext‑demo» / «крутая pretext‑вещь» / «текст‑как‑X»
- Текст, обтекающий движущуюся форму (hero‑sections, редакционные макеты, анимированные длинные страницы)
- Эффекты ASCII‑арт с **реальными словами или прозой**, а не моноширинными растрами
- Игры, где игровое поле / препятствия / кирпичи сделаны из текста (Tetris‑from‑letters, Breakout‑of‑prose)
- Кинетическая типография с физикой по глифам (shatter, scatter, flock, flow)
- Генеративное типографическое искусство, особенно с нелатинскими или смешанными скриптами
- Многострочный UI «shrink‑wrap» (наименьшая ширина контейнера, всё ещё вмещающая текст)
- Всё, что требует знать переносы строк *до* рендеринга

Не используй для:
- Статических SVG/HTML‑страниц, где CSS уже решает макет — просто используй CSS
- Редакторов форматированного текста, общих движков инлайн‑форматирования (pretext намеренно узок)
- Преобразования изображение → текст (используй `ascii-art` / `ascii-video` skills)
- Чистого генеративного искусства на canvas без роли текста — используй `p5js`
## Creative Standard

Это визуальное искусство, отрисованное в браузере. Pretext возвращает числа; **ты** рисуешь объект.

- **Не поставляй демо «hello world».** Шаблон `hello-orb-flow.html` — это *отправная* точка. Каждый готовый демо‑пример должен добавить осознанный цвет, движение, композицию и одну визуальную деталь, которую пользователь не запрашивал, но оценит.
- **Тёмные фоны, тёплые ядра, продуманные палитры.** Классический янтарный на чёрном (CRT / терминал) работает, но также подходят холодный белый на угольном (редакционный) и десатурированные пастельные (рисограф). Выбери один вариант и придерживайся его.
- **Пропорциональные шрифты — суть.** Вся атмосфера Pretext — «не моноширинный» — используй это. Применяй Iowan Old Style, Inter, JetBrains Mono, Helvetica Neue или переменный шрифт. Никогда не используй шрифт по умолчанию.
- **Настоящий источник/текст, а не lorem ipsum.** Корпус должен иметь смысл. Краткие манифесты, поэзия, реальный исходный код, найденный текст, README библиотеки — никогда `lorem ipsum`.
- **Первый кадр безупречен.** Без состояний загрузки, без пустых кадров. Демо должно выглядеть готовым к выпуску сразу после открытия.
## Стек

Один автономный HTML‑файл на каждый демонстрационный пример. Сборка не требуется.

| Слой | Инструмент | Назначение |
|------|------------|------------|
| Core | `@chenglou/pretext` via `esm.sh` CDN | Измерение текста + раскладка строк |
| Render | HTML5 Canvas 2D | Отрисовка глифов, композиция кадра |
| Segmentation | `Intl.Segmenter` (built-in) | Разделение графем для эмодзи / CJK / комбинирующих знаков |
| Interaction | Raw DOM events | Мышь / касание / колесо — без фреймворка |

```html
<script type="module">
import {
  prepare, layout,                   // use-case 1: simple height
  prepareWithSegments, layoutWithLines,  // use-case 2a: fixed-width lines
  layoutNextLineRange, materializeLineRange, // use-case 2b: streaming / variable width
  measureLineStats, walkLineRanges,  // stats without string allocation
} from "https://esm.sh/@chenglou/pretext@0.0.6";
</script>
```

Зафиксируй версию. `@0.0.6` на момент написания — проверь [npm](https://www.npmjs.com/package/@chenglou/pretext) для актуальной, если поведение демо отличается.
## Два сценария использования

Почти всё сводится к одной из этих двух форм. Выучи оба.

### Сценарий 1 — измерить, затем отрисовать с помощью CSS/DOM

```js
const prepared = prepare(text, "16px Inter");
const { height, lineCount } = layout(prepared, 320, 20);
```

Ты всё равно позволяешь браузеру рисовать текст. `pretext` просто сообщает, насколько высоким будет блок при заданной ширине, **без** чтения из DOM. Используй для:
- Виртуализированных списков, где строки содержат переносимый текст
- Masonry с точными высотами карточек
- Проверок «Помещается ли эта метка?» во время разработки
- Предотвращения сдвига макета, когда загружается удалённый текст

**Держи `font` и `letterSpacing` точно синхронными с твоим CSS.** Формат `canvas ctx.font` (например, `"16px Inter"`, `"500 17px 'JetBrains Mono'"`) должен соответствовать отрисованному CSS, иначе измерения будут отклоняться.

### Сценарий 2 — измерить *и* отрисовать сам

```js
const prepared = prepareWithSegments(text, FONT);
const { lines } = layoutWithLines(prepared, 320, 26);
for (let i = 0; i < lines.length; i++) {
  ctx.fillText(lines[i].text, 0, i * 26);
}
```

Здесь происходит творческая работа. Ты контролируешь отрисовку, поэтому можешь:
- Рисовать в `canvas`, `SVG`, `WebGL` или любой системе координат
- Подменять трансформации отдельного глифа (поворот, дрожание, масштаб, непрозрачность)
- Использовать метаданные строк (ширина, позиции графем) как геометрию

Для **переменной ширины строки** (текст вокруг формы, текст в кольцевой ленте, текст в непрямоугольной колонке):

```js
let cursor = { segmentIndex: 0, graphemeIndex: 0 };
let y = 0;
while (true) {
  const lineWidth = widthAtY(y);  // your function: how wide is the corridor at this y?
  const range = layoutNextLineRange(prepared, cursor, lineWidth);
  if (!range) break;
  const line = materializeLineRange(prepared, range);
  ctx.fillText(line.text, leftEdgeAtY(y), y);
  cursor = range.end;
  y += lineHeight;
}
```

Это самый важный шаблон во всей библиотеке. Он открывает возможность «текст, текущий вокруг перетаскиваемого спрайта» — демо, которое стало вирусным в X.

### Полезные вспомогательные функции

- `measureLineStats(prepared, maxWidth)` → `{ lineCount, maxLineWidth }` — самая широкая строка, то есть ширина, получающаяся при обтекании несколькими строками.
- `walkLineRanges(prepared, maxWidth, callback)` — перебирает строки без создания строковых объектов. Используй для статистики/физики над графемами, когда символы не нужны.
- `@chenglou/pretext/rich-inline` — та же система, но для абзацев, смешивающих шрифты / чипы / упоминания. Импортируй из подкаталога.
## Примеры паттернов рецептов

Корпус сообщества (см. `references/patterns.md`) группируется в несколько сильных паттернов. Выбери один и импровизируй — не придумывай новую категорию, если тебя не просят.

| Паттерн | Ключевой API | Пример идеи |
|---|---|---|
| **Обтекание препятствия** | `layoutNextLineRange` + функция ширины строки | Редакционный абзац, который обтекает перетаскиваемый спрайт курсора |
| **Игра «текст как геометрия»** | `layoutWithLines` + прямоугольники столкновений для каждой строки | Breakout, где каждый кирпич — измеренное слово |
| **Разрушение / частицы** | `walkLineRanges` → per-grapheme (x,y) → physics | Предложение, которое взрывается в буквы по клику |
| **ASCII‑типография с препятствиями** | `layoutNextLineRange` + измеренные препятствующие отрезки в каждой строке | Битмап‑логотип ASCII, морфинг форм и перетаскиваемые проволочные объекты, заставляющие текст открываться вокруг их реальной геометрии |
| **Редакционный многоколоночный** | `layoutNextLineRange` per column + shared cursor | Анимированный разворот журнала с выносками |
| **Кинетический тип** | `layoutWithLines` + трансформация каждой строки во времени | Краул Star Wars, волна, подпрыгивание, глитч |
| **Многострочный shrink‑wrap** | `measureLineStats` | Карточка с цитатой, автоматически подгоняющаяся к самому тесному контейнеру |

   - `write_file` в новый файл `.html` в каталоге `/tmp/` или в рабочем пространстве пользователя.
3. **Замени корпус** на материал, соответствующий задаче. Реальный текст, 10–100 предложений, без «lorem».
4. **Настрой эстетику** — шрифт, палитру, композицию, взаимодействие. Это и есть работа; не пропускай этот шаг.
5. **Проверь локально**:
      ```sh
   cd <dir-with-html> && python3 -m http.server 8765
   # then open http://localhost:8765/<file>.html
   ```
6. **Посмотри консоль** — `pretext` выбросит ошибку, если `prepareWithSegments` вызвать с некорректной строкой шрифта; `Intl.Segmenter` доступен во всех современных браузерах.
7. **Покажи пользователю путь к файлу**, а не только код — ему нужно открыть файл.
## Примечания по производительности

- `prepare()` / `prepareWithSegments()` — дорогой вызов. Выполняй его **один раз** на каждую пару текст + шрифт. Кешируй дескриптор.
- При изменении размеров перезапускай только `layout()` / `layoutWithLines()` — никогда не вызывай `prepare` повторно.
- Для покадровых анимаций, когда текст не меняется, а меняется геометрия, `layoutNextLineRange` в плотном цикле достаточно лёгок, чтобы выполнять его каждый кадр при 60 fps для обычных абзацев.
- При рендеринге ASCII‑масок каждый кадр держи буфер ячеек (`Uint8Array`/typed arrays), получай измеренные по‑строчным отрезки препятствий из ячеек или проецируемой геометрии, объединяй отрезки и передавай их в `layoutNextLineRange` перед отрисовкой текста.
- Синхронно анимируй визуальную часть и разметку. Если сфера превращается в куб, интерполируй одновременно буфер ячеек и отрезки препятствий одним и тем же значением; иначе демо выглядит «нарисованным», а не физически перераспределённым.
- Для затуханий предпочтительнее менять непрозрачность слоя, а не интенсивность глифов или масштаб препятствий. Размести временные ASCII‑спрайты на отдельном канвасе и меняй его непрозрачность через CSS/GSAP, чтобы геометрия не казалась сжимающейся.
- Установка `ctx.font` у Canvas оказывается неожиданно медленной; задавай её **один раз** за кадр, если шрифт не меняется, а не при каждом вызове `fillText`.
## Распространённые подводные камни

1. **Дрейф строк шрифтов CSS/Canvas.** `ctx.font = "16px Inter"` измеряется, но в CSS указано `font-family: Inter, sans-serif; font-size: 16px`. Всё в порядке, *если* шрифт Inter загружается. Если Inter возвращает 404, CSS переходит на `sans-serif`, и измерения отклоняются на 5–20 %. Всегда `preload` шрифт или используй веб‑безопасное семейство.

2. **Повторная подготовка внутри цикла анимации.** Дешево только `layout*`. Повторный вызов `prepare` каждый кадр сильно тормозит производительность. Храни подготовленный хэндл в области модуля.

3. **Забываешь `Intl.Segmenter` для разбиения на графемы.** Emoji, комбинирующие знаки, CJK — `"é".split("")` даёт два символа. Используй `new Intl.Segmenter(undefined, { granularity: "grapheme" })`, когда нужно выбирать отдельные видимые глифы.

4. **Чипы `break: 'never'` без `extraWidth`.** В `rich-inline`, если ты используешь `break: 'never'` для атомарного чипа/упоминания, также необходимо задать `extraWidth` для отступов pill — иначе хром чипа выходит за пределы контейнера.

5. **Использование `@chenglou/pretext` из `unpkg` с экспортом только для TypeScript.** Пользуйся `esm.sh` — он автоматически компилирует TS‑экспорты в готовый к браузеру ESM. `unpkg` вернёт 404 или сырой TS.

6. **Моноширинные fallback‑ы тихо уничтожают смысл.** Пользователи, видящие вывод в моноширинном виде, часто имеют в CSS `font-family`, который упал до `monospace`. Проверь фактический отрисованный шрифт через DevTools.

7. **Пропуск строк вместо изменения ширины** при обходе формы. Если коридор в этой строке слишком узок, чтобы вместить строку, *пропусти строку* (`y += lineHeight; continue;`), а не передавай крошечный `maxWidth` в `layoutNextLineRange` — pretext вернёт строки из одной графемы, которые выглядят сломанными.

8. **Отправка «холодного» демо.** По умолчанию первый кадр выглядит как учебный материал. Добавь: виньетку, лёгкую скан‑линию, плавное автодвижение в простое состояние, один тщательно подобранный интерактивный отклик (drag, hover, scroll, click). Без этого «крутое демо pretext» превращается в «внутреннюю репродукцию README».
