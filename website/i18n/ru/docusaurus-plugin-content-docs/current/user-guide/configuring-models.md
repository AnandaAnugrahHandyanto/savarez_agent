# Настройка моделей

Hermes использует два типа слотов моделей:

- **Основная модель** — та, с помощью которой агент мыслит. Каждый пользовательский запрос, каждый цикл вызова инструмента, каждый потоковый ответ проходят через эту модель.
- **Вспомогательные модели** — небольшие побочные задачи, которые агент делегирует: сжатие контекста, зрение (анализ изображений), суммирование веб‑страниц, оценка одобрения, маршрутизация инструментов MCP, генерация названия сессии и поиск навыков. У каждой есть свой слот, и их можно переопределять независимо.

Эта страница описывает настройку обеих моделей через панель управления. Если ты предпочитаешь файлы конфигурации или CLI, переходи к разделу [Альтернативные методы](#alternative-methods) внизу.

:::tip Самый быстрый путь: Nous Portal
[Nous Portal](/user-guide/features/tool-gateway) предоставляет более 300 моделей в рамках одной подписки. При свежей установке запусти `hermes setup --portal`, чтобы войти в систему и установить Nous в качестве провайдера одной командой. Посмотри, что подключено, с помощью `hermes portal status`.

- Подписчики Portal также получают **10 % скидку** на провайдеров, оплачиваемых токенами.
:::

:::note `model:` schema — пустая строка vs. словарь
В новой установке включённый конфиг по умолчанию содержит `model: ""` (пустая строка‑сентинел, означающая «ещё не настроено»). При первом запуске `hermes setup` или `hermes model` этот ключ преобразуется на месте в словарь с под‑ключами `provider`, `default`, `base_url` и `api_mode` — форма, показанная на этой странице и в [`profiles.md`](./profiles.md) / [`configuration.md`](./configuration.md). Если ты увидишь пустую строку в `config.yaml`, запусти `hermes model` (или нажми **Change** в панели управления), и Hermes запишет для тебя словарную форму.
:::
## Страница моделей

Открой панель управления и нажми **Models** в боковой панели. Ты увидишь два раздела:

1. **Model Settings** — верхняя панель, где ты назначаешь модели слотам.
2. **Usage analytics** — ранжированные карточки, показывающие каждую модель, запустившую сессию за выбранный период, с подсчётом токенов, стоимостью и значками возможностей.

![Models page overview](/img/docs/dashboard-models/overview.png)

Верхняя карточка — панель **Model Settings**. Основная строка всегда показывает, какую модель агент запустит для новых сессий. Нажми **Change**, чтобы открыть выбор модели.
## Настройка основной модели

Нажми **Change** в строке **Main model**:

![Model picker dialog](/img/docs/dashboard-models/picker-dialog.png)

В диалоговом окне два столбца:

- **Left** — аутентифицированные провайдеры. Здесь отображаются только те провайдеры, которые ты настроил (указан API‑ключ, выполнен OAuth или задан пользовательский endpoint). Если провайдер отсутствует, перейди в **Keys** и добавь его учётные данные.
- **Right** — курируемый список моделей для выбранного провайдера. Это агентные модели, которые Hermes рекомендует для данного провайдера, а не «сырой» дамп `/models` (в OpenRouter он содержит более 400 моделей, включая TTS, генераторы изображений и ранжировщики).

Введи запрос в поле фильтра, чтобы сузить список по названию провайдера, slug или ID модели.

Выбери модель, нажми **Switch**, и Hermes запишет её в `~/.hermes/config.yaml` в секцию `model`. **Это применяется только к новым сессиям** — любые уже открытые вкладки чата продолжают работать с той моделью, с которой они были запущены. Чтобы мгновенно переключить текущий чат, используй слеш‑команду `/model` внутри него.
## Настройка вспомогательных моделей

Нажми **Show auxiliary**, чтобы открыть восемь слотов задач:

![Auxiliary panel expanded](/img/docs/dashboard-models/auxiliary-expanded.png)

Каждая вспомогательная задача по умолчанию имеет значение `auto` — то есть Hermes использует твою основную модель и для этой работы. Переопредели конкретную задачу, если нужен более дешёвый или быстрый вариант модели для побочной задачи.

### Распространённые шаблоны переопределения

| Задача | Когда переопределять |
|---|---|
| **Title Gen** | Почти всегда. Модель `$0.10/M flash` пишет названия сессий так же, как Opus. В конфигурации по умолчанию установлено `google/gemini-3-flash-preview` на OpenRouter. |
| **Vision** | Когда у основной модели нет поддержки зрения. Укажи `google/gemini-2.5-flash` или `gpt-4o-mini`. |
| **Compression** | Когда ты тратишь токены рассуждений на Opus/M2.7 только для суммирования контекста. Быстрая чат‑модель справится с задачей за 1/50 стоимости. |
| **Approval** | Для `approval_mode: smart` — быстрая/дешёвая модель (haiku, flash, gpt-5-mini) решает, одобрять ли автоматически команды с низким риском. Дорогие модели здесь — пустая трата. |
| **Web Extract** | Когда ты часто используешь `web_extract`. Та же логика, что и у сжатия — суммирование не требует рассуждений. |
| **Skills Hub** | `hermes skills search` использует её. Обычно достаточно `auto`. |
| **MCP** | Маршрутизация инструмента MCP. Обычно достаточно `auto`. |

### Переопределение для отдельной задачи

Нажми **Change** в любой строке вспомогательной задачи. Откроется тот же селектор, будет то же поведение — выбери провайдера + модель, нажми **Switch**. Строка обновится и покажет `provider · model` вместо `auto (use main model)`.

### Сбросить всё в режим `auto`

Если ты слишком «подтюнил» и хочешь начать заново, нажми **Reset all to auto** вверху раздела вспомогательных задач. Все слоты вернутся к использованию основной модели.
## Выпадающий список «Use as»

Каждая карточка модели на странице имеет выпадающий список **Use as**. Это быстрый способ — выбираешь модель, которую видишь в аналитике, нажимаешь **Use as** и назначаешь её в основной слот или в любой конкретный вспомогательный **task** одним кликом:

![Use as dropdown](/img/docs/dashboard-models/use-as-dropdown.png)

В выпадающем списке есть:

- **Main model** — то же, что нажать **Change** в основной строке.
- **All auxiliary tasks** — назначает эту модель сразу во все 8 aux‑слотов. Полезно, когда хочется, чтобы каждый побочный процесс использовал дешёвую flash‑модель.
- **Individual task options** — Vision, Web Extract, Compression и т.д. Текущая назначенная модель для каждой задачи помечена `current`.

Карточки помечаются бейджами `main` или `aux · <task>`, когда они сейчас назначены — так ты сразу видишь, какие из твоих исторических моделей где подключены.
## Что записывается в `config.yaml`

При сохранении через дашборд Hermes записывает в `~/.hermes/config.yaml`:

**Основная модель:**
```yaml
model:
  provider: openrouter
  default: anthropic/claude-opus-4.7
  base_url: ''        # cleared on provider switch
  api_mode: chat_completions
```

**Вспомогательное переопределение (пример — vision на gemini-flash):**
```yaml
auxiliary:
  vision:
    provider: openrouter
    model: google/gemini-2.5-flash
    base_url: ''
    api_key: ''
    timeout: 120
    extra_body: {}
    download_timeout: 30
```

**Вспомогательная модель в режиме `auto` (по умолчанию):**
```yaml
auxiliary:
  compression:
    provider: auto
    model: ''
    base_url: ''
    # ... other fields unchanged
```

`provider: auto` с `model: ''` сообщает Hermes использовать основную модель для этой задачи.
## Когда это вступает в силу?

- **CLI** (`hermes chat`): при следующем вызове `hermes chat`.
- **Gateway** (Telegram, Discord, Slack и т.д.): при следующей *новой* сессии. Существующие сессии сохраняют свою модель. Перезапусти шлюз (`hermes gateway restart`), если нужно принудительно применить изменение ко всем сессиям.
- **Вкладка чата в Dashboard** (`/chat`): при следующем новом PTY. Текущий открытый чат сохраняет свою модель — используй `/model` внутри него для горячей замены.

Изменения никогда не делают недействительными кеши подсказок в запущенных сессиях. Это сделано намеренно: замена основной модели внутри сессии требует сброса кеша (системная подсказка содержит содержимое, зависящее от модели), и мы оставляем это только для явной slash‑команды `/model` в чате.
## Устранение неполадок

### «No authenticated providers» в выборе провайдера

Hermes отображает провайдера только если у него есть действительные учётные данные. Проверь **Keys** в боковой панели — ты должен увидеть один из вариантов: API‑ключ, успешный OAuth или пользовательский URL конечного пункта. Если нужного провайдера нет, запусти `hermes setup`, чтобы настроить его, или перейди в **Keys** и добавь переменную окружения.

### Основная модель не изменилась в запущенном чате

Ожидаемое поведение. Панель управления записывает `config.yaml`, который читают новые сессии. Текущий открытый чат — это живой процесс агента, он сохраняет ту модель, с которой был запущен. Используй `/model <name>` внутри чата, чтобы мгновенно переключить именно эту сессию.

### Переопределение вспомогательного инструмента «не вступило в силу»

Проверь три вещи:

1. **Запустил ли ты новую сессию?** Существующие чаты не перечитывают конфиг.
2. **Установлен ли `provider` в значение, отличное от `auto`?** Если поле показывает `auto`, задача всё ещё использует твою основную модель. Нажми **Change** и выбери реального провайдера.
3. **Аутентифицирован ли провайдер?** Если ты назначил `minimax` задаче, но у тебя нет API‑ключа MiniMax, задача переключится на запасной вариант OpenRouter и запишет предупреждение в `agent.log`.

### Я выбрал модель, а Hermes переключил провайдера

На OpenRouter (или любом агрегаторе) «голые» названия моделей сначала разрешаются *внутри* агрегатора. Поэтому `claude-sonnet-4` на OpenRouter превращается в `anthropic/claude-sonnet-4.6`, оставаясь под твоей аутентификацией OpenRouter. Но если ты ввёл `claude-sonnet-4` при прямой аутентификации Anthropic, он останется как `claude-sonnet-4-6`. Если ты видишь неожиданное переключение провайдера, проверь, что текущий провайдер соответствует твоим ожиданиям — выборщик всегда отображает текущий основной провайдер в верхней части диалога.
## Альтернативные методы

### CLI‑слеш‑команда

В любой сессии `hermes chat`:

```
/model gpt-5.4 --provider openrouter             # session-only
/model gpt-5.4 --provider openrouter --global    # also persists to config.yaml
```

`--global` делает то же, что кнопка **Change** в дашборде, плюс переключает текущую сессию на месте.

### Пользовательские алиасы

Определи свои короткие имена для часто используемых моделей, а затем используй `/model <alias>` в CLI или любой платформе обмена сообщениями. Существует два эквивалентных формата — выбирай тот, который лучше вписывается в твой рабочий процесс.

**Канонический (верхний уровень `model_aliases:`)** — полный контроль над провайдером + `base_url`:

```yaml
# ~/.hermes/config.yaml
model_aliases:
  fav:
    model: claude-sonnet-4.6
    provider: anthropic
  grok:
    model: grok-4
    provider: x-ai
```

**Краткая строковая форма (`model.aliases.<name>: provider/model`)** — удобно в оболочке, потому что `hermes config set` записывает только скалярные значения, но нельзя задать пользовательский `base_url`:

```bash
hermes config set model.aliases.fav anthropic/claude-opus-4.6
hermes config set model.aliases.grok x-ai/grok-4
```

Оба пути используют один и тот же загрузчик (`hermes_cli/model_switch.py`). Записи, объявленные в `model_aliases:`, имеют приоритет над записями в `model.aliases:` с тем же именем.

Затем `/model fav` или `/model grok` в чате. Пользовательские алиасы перекрывают встроенные короткие имена (`sonnet`, `kimi`, `opus` и т.д.). См. [Custom model aliases](/reference/slash-commands#custom-model-aliases) для полной справки.

### Подкоманда `hermes model`

```bash
hermes model            # Interactive provider + model picker (the canonical way to switch defaults)
```

`hermes model` проводит тебя через выбор провайдера, аутентификацию (OAuth‑потоки открывают браузер; провайдеры с API‑ключом запрашивают ключ), а затем выбор конкретной модели из курированного каталога провайдера. Выбор записывается в `model.provider` и `model.model` в `~/.hermes/config.yaml`.

Чтобы перечислить провайдеров/моделей без запуска мастера, используй дашборд или REST‑эндпоинты ниже. Чтобы увидеть, что CLI действительно использует сейчас: `hermes config get model` и `hermes status`.

### Прямое редактирование конфигурации

Отредактируй `~/.hermes/config.yaml` и перезапусти приложение, которое её читает. См. [Configuration reference](./configuration.md) для полной схемы.

### REST API

Дашборд использует три эндпоинта. Полезно для скриптов:

```bash
# List authenticated providers + curated model lists
curl -H "X-Hermes-Session-Token: $TOKEN" http://localhost:PORT/api/model/options

# Read current main + auxiliary assignments
curl -H "X-Hermes-Session-Token: $TOKEN" http://localhost:PORT/api/model/auxiliary

# Set the main model
curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
  -d '{"scope":"main","provider":"openrouter","model":"anthropic/claude-opus-4.7"}' \
  http://localhost:PORT/api/model/set

# Override a single auxiliary task
curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
  -d '{"scope":"auxiliary","task":"vision","provider":"openrouter","model":"google/gemini-2.5-flash"}' \
  http://localhost:PORT/api/model/set

# Assign one model to every auxiliary task
curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
  -d '{"scope":"auxiliary","task":"","provider":"openrouter","model":"google/gemini-2.5-flash"}' \
  http://localhost:PORT/api/model/set

# Reset all auxiliary tasks to auto
curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
  -d '{"scope":"auxiliary","task":"__reset__","provider":"","model":""}' \
  http://localhost:PORT/api/model/set
```

Токен сессии внедряется в HTML дашборда при запуске и обновляется при каждой перезагрузке сервера. Получи его из инструментов разработчика браузера (`window.__HERMES_SESSION_TOKEN__`), если пишешь скрипт, работающий с запущенным дашбордом.
