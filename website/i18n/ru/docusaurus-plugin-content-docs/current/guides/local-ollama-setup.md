# Запусти Hermes локально с Ollama — без затрат на API
## Проблема

Облачные API LLM берут оплату за токен. Интенсивная сессия программирования может стоить $5–20. Для личных проектов, обучения или работы с конфиденциальными данными это быстро накапливается — и при этом ты отправляешь каждый диалог третьей стороне.
## Что решает это руководство

Ты настроишь Hermes Agent, работающего полностью на твоём собственном оборудовании, используя [Ollama](https://ollama.com) в качестве бэкенда модели. Без API‑ключей, без подписок, без передачи данных с твоего компьютера. После настройки Hermes будет работать точно так же, как с OpenRouter или Anthropic — терминальные команды, редактирование файлов, веб‑поиск, делегирование — но модель будет запускаться локально.

К концу ты получишь:

- Ollama, обслуживающую одну или несколько моделей с открытыми весами
- Hermes, подключённый к Ollama как пользовательскую конечную точку
- Рабочий локальный агент, способный редактировать файлы, выполнять команды и просматривать веб
- По желанию: бот для Telegram/Discord, полностью работающий на твоём оборудовании
## Что тебе нужно

| Компонент | Минимум | Рекомендовано |
|-----------|---------|--------------|
| **RAM** | 8 ГБ (для моделей 3B) | 32+ ГБ (для моделей 27B+) |
| **Storage** | 5 ГБ свободно | 30+ ГБ (для нескольких моделей) |
| **CPU** | 4 ядра | 8+ ядер (AMD EPYC, Ryzen, Intel Xeon) |
| **GPU** | Не требуется | NVIDIA GPU с 8+ ГБ VRAM значительно ускоряет работу |

:::tip CPU‑only работает, но ожидай более медленных ответов
Ollama работает на серверах только с CPU. Модель 9B на современном 8‑ядерном CPU дает ~10 токенов/сек. Модель 31B на CPU работает медленнее (~2–5 токенов/сек) — каждый ответ занимает 30–120 секунд, но работает. GPU значительно улучшает производительность. Для конфигураций только с CPU увеличь таймаут API через переменную окружения (это не ключ `config.yaml`):

```bash
# ~/.hermes/.env
HERMES_API_TIMEOUT=1800   # 30 minutes — generous for slow local models
```
:::
## Шаг 1: Установи Ollama

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Убедись, что он запущен:

```bash
ollama --version
curl http://localhost:11434/api/tags   # Should return {"models":[]}
```
## Шаг 2: Загрузка модели

Выбирай в зависимости от твоего оборудования:

| Модель | Размер на диске | Требуемая ОЗУ | Вызов инструментов | Оптимально для |
|-------|----------------|--------------|:-----------------:|-----------------|
| `gemma4:31b` | ~20 ГБ | 24+ ГБ | Да | Лучшее качество — активное использование инструментов и рассуждения |
| `gemma2:27b` | ~16 ГБ | 20+ ГБ | Нет | Диалоговые задачи, без использования инструментов |
| `gemma2:9b` | ~5 ГБ | 8+ ГБ | Нет | Быстрый чат, вопросы‑ответы — без вызова инструментов |
| `llama3.2:3b` | ~2 ГБ | 4+ ГБ | Нет | Лёгкие быстрые ответы только |

:::warning Важность вызова инструментов
Hermes — это **агентный** помощник: он редактирует файлы, выполняет команды и просматривает веб через вызовы инструментов. Модели без поддержки вызова инструментов могут только вести диалог; они не способны выполнять действия. Чтобы получить полный опыт Hermes, используй модель, поддерживающую инструменты (например, `gemma4:31b`).
:::

Загрузи выбранную модель:

```bash
ollama pull gemma4:31b
```

:::info Несколько моделей
Ты можешь загрузить несколько моделей и переключаться между ними внутри Hermes с помощью `/model`. Ollama загружает активную модель в память по запросу и автоматически выгружает неиспользуемые.
:::

Проверь, что модель работает:

```bash
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemma4:31b",
    "messages": [{"role": "user", "content": "Say hello"}],
    "max_tokens": 50
  }'
```

Ты должен увидеть JSON‑ответ с репликой модели.
## Шаг 3: Настройка Hermes

Запусти мастер настройки Hermes:

```bash
hermes setup
```

Когда будет запрошен provider, выбери **Custom Endpoint** и введи:

- **Base URL:** `http://localhost:11434/v1`
- **API Key:** Оставь пустым или введи `no-key` (Ollama не требует ключа)
- **Model:** `gemma4:31b` (или любую другую модель, которую ты скачал)

Либо отредактируй `~/.hermes/config.yaml` напрямую:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"
```
## Шаг 4: Начни использовать Hermes

```bash
hermes
```

Вот и всё. Теперь у тебя запущен полностью локальный агент. Попробуй его:

```
You: List all Python files in this directory and count the lines of code in each

You: Read the README.md and summarize what this project does

You: Create a Python script that fetches the weather for Ho Chi Minh City
```

Hermes будет использовать инструмент терминала, операции с файлами и твою локальную модель — без облачных вызовов.
## Шаг 5: Выбери правильную модель для своей задачи

Не каждая задача требует самой большой модели. Вот практическое руководство:

| Задача | Рекомендуемая модель | Почему |
|------|-------------------|-----|
| Правка файлов, код, команды терминала | `gemma4:31b` | Единственная модель с надёжным вызовом инструментов |
| Быстрый Q&A (без использования инструментов) | `gemma2:9b` | Быстрые ответы для разговорных задач |
| Лёгкий чат | `llama3.2:3b` | Самая быстрая, но с очень ограниченными возможностями |

:::note
Для полноценной агентной работы (правка файлов, выполнение команд, браузинг) `gemma4:31b` в настоящее время лучший локальный вариант с поддержкой вызова инструментов. Смотри [библиотеку моделей Ollama](https://ollama.com/library) для новых моделей — поддержка вызова инструментов быстро расширяется.
:::

Переключай модели «на лету» внутри сессии:

```
/model gemma2:9b
```
## Шаг 6: Оптимизация скорости

### Увеличение окна контекста Ollama

По умолчанию Ollama использует контекст в 2048 токенов. Hermes требует минимум 64 000 токенов для агентной работы с инструментами:

```bash
# Create a Modelfile that extends context
cat > /tmp/Modelfile << 'EOF'
FROM gemma4:31b
PARAMETER num_ctx 64000
EOF

ollama create gemma4-64k -f /tmp/Modelfile
```

Затем обнови конфигурацию Hermes, указав `gemma4-64k` в качестве имени модели.

### Держать модель загруженной

По умолчанию Ollama выгружает модели после 5 минут бездействия. Для постоянного шлюз‑бота держи её загруженной:

```bash
# Set keep-alive to 24 hours
curl http://localhost:11434/api/generate \
  -d '{"model": "gemma4:31b", "keep_alive": "24h"}'
```

Или установи это глобально в окружении Ollama:

```bash
# /etc/systemd/system/ollama.service.d/override.conf
[Service]
Environment="OLLAMA_KEEP_ALIVE=24h"
```

### Использование GPU‑offloading (если доступно)

Если у тебя есть GPU NVIDIA, Ollama автоматически offload‑ит слои на него. Проверь с помощью:

```bash
ollama ps   # Shows which model is loaded and how many GPU layers
```

Для модели 31 B на GPU с 12 ГБ ты получишь частичную выгрузку (~40 слоёв на GPU, остальные — CPU), что всё равно даёт значительное ускорение.
## Шаг 7: Запуск в качестве шлюз‑бота (необязательно)

После того как Hermes работает локально в CLI, ты можешь развернуть его как бота в Telegram или Discord — при этом он полностью работает на твоём оборудовании.

### Telegram

1. Создай бота через [@BotFather](https://t.me/BotFather) и получи токен.
2. Добавь в свой `~/.hermes/config.yaml`:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"

platforms:
  telegram:
    enabled: true
    token: "YOUR_TELEGRAM_BOT_TOKEN"
```

3. Запусти шлюз:

```bash
hermes gateway
```

Теперь отправляй сообщения своему боту в Telegram — он отвечает, используя твою локальную модель.

### Discord

1. Создай приложение Discord на [discord.com/developers](https://discord.com/developers/applications).
2. Добавь в конфигурацию:

```yaml
platforms:
  discord:
    enabled: true
    token: "YOUR_DISCORD_BOT_TOKEN"
```

3. Запусти: `hermes gateway`
## Шаг 8: Настройка запасного (варианта) (необязательно)

Локальные модели могут сталкиваться со сложными задачами. Настрой облачный запасной (вариант), который активируется только когда локальная модель не справляется:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"

fallback_providers:
  - provider: openrouter
    model: anthropic/claude-sonnet-4
```

Таким образом, 90 % твоего использования будет бесплатным (локально), и только сложные задачи будут обращаться к платному API.
## Устранение неполадок

### «Connection refused» при запуске

Ollama не запущена. Запусти её:

```bash
sudo systemctl start ollama
# or
ollama serve
```

### Медленные ответы

- **Проверь размер модели относительно памяти:** Если модели требуется больше ОЗУ, чем доступно, она будет использовать своп на диск. Используй меньшую модель или добавь ОЗУ.
- **Проверь `ollama ps`:** Если нет выгруженных слоёв GPU, ответы будут ограничены CPU. Это нормально для серверов без GPU.
- **Сократи контекст:** Большие диалоги замедляют инференс. Регулярно используй `/compress` или задай более низкий порог сжатия в конфигурации.

### Модель не следует вызовам инструментов

Меньшие модели (3B, 7B) иногда игнорируют инструкции по вызову инструментов и выводят обычный текст вместо структурированных вызовов функций. Решения:

- **Используй более большую модель** — `gemma4:31b` или `gemma2:27b` обрабатывают вызовы инструментов гораздо лучше, чем модели 3B/7B.
- **Hermes имеет авто‑исправление** — она обнаруживает некорректные вызовы инструментов и пытается исправить их автоматически.
- **Настрой запасной вариант** — если локальная модель не справится три раза, Hermes переключится на облачного провайдера.

### Ошибки окна контекста

Контекст Ollama по умолчанию (2048 токенов) слишком мал для агентной работы. См. [Step 6](#step-6-optimize-for-speed) для увеличения его размера.
## Сравнение стоимости

Вот что экономит запуск локально по сравнению с облачными API, исходя из типичной сессии программирования (~100 К токенов ввода, ~20 К токенов вывода):

| Провайдер | Стоимость за сессию | В месяц (при ежедневном использовании) |
|----------|---------------------|----------------------------------------|
| Anthropic Claude Sonnet | ~$0.80 | ~$24 |
| OpenRouter (GPT-4o) | ~$0.60 | ~$18 |
| **Ollama (local)** | **$0.00** | **$0.00** |

Твои единственные расходы — электроэнергия, примерно $0.01–0.05 за сессию в зависимости от оборудования.
## Что хорошо работает локально

- **Редактирование файлов и генерация кода** — модели 9B+ справляются с этим отлично
- **Команды терминала** — Hermes оборачивает команду, выполняет её, читает вывод независимо от модели
- **Веб‑поиск** — инструмент браузера осуществляет загрузку; модель лишь интерпретирует результаты
- **Cron‑задачи и запланированные задачи** — работают так же, как в облачных настройках
- **Мультиплатформенный шлюз инструментов** — Telegram, Discord, Slack работают с локальными моделями
## Что лучше с облачными моделями

- **Очень сложные многошаговые рассуждения** — модели 70 B+ или облачные модели типа Claude Opus заметно лучше справляются
- **Большие окна контекста** — облачные модели предлагают 100 K–1 M токенов; локальные рантаймы часто по умолчанию ниже минимального порога Hermes — 64 K, если их не настроить
- **Скорость при больших ответах** — облачный вывод быстрее, чем только CPU‑локальный, при генерации длинных ответов

Оптимальный вариант: использовать локально для повседневных задач, настроить облачный запасной вариант для сложных задач.
