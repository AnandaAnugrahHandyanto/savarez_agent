# Использовать голосовой режим с Hermes

Это практическое руководство — компаньон к [Справка по функции Voice Mode](/user-guide/features/voice-mode).

Если на странице функции объясняется, что может делать голосовой режим, то в этом руководстве показано, как действительно эффективно его использовать.

:::tip
[Nous Portal](/integrations/nous-portal) объединяет как LLM, так и TTS через один OAuth — голосовой режим работает сквозным образом без дополнительных учётных данных.
:::
## Для чего полезен голосовой режим

Голосовой режим особенно полезен, когда:
- ты хочешь работать в CLI без рук
- тебе нужны голосовые ответы в Telegram или Discord
- ты хочешь, чтобы Hermes находился в голосовом канале Discord для живого общения
- нужно быстро фиксировать идеи, отлаживать или вести диалог, пока ты ходишь, а не печатаешь
## Выбери настройку голосового режима

Существует действительно три разных голосовых режима в Hermes.

| Режим | Лучшее применение | Платформа |
|---|---|---|
| Интерактивный микрофонный цикл | Персональное использование без рук во время кодирования или исследования | CLI |
| Голосовые ответы в чате | Произносимые ответы наряду с обычными сообщениями | Telegram, Discord |
| Бот в живом голосовом канале | Групповой или персональный живой разговор в голосовом канале | Discord voice channels |

Хороший путь:
1. сначала настрой работу с текстом
2. затем включи голосовые ответы
3. в конце перейди к голосовым каналам Discord, если хочешь полный набор функций
## Шаг 1: убедись, что обычный Hermes работает

Прежде чем переходить к голосовому режиму, проверь, что:
- Hermes запускается
- твой провайдер настроен
- агент может нормально отвечать на текстовые запросы

```bash
hermes
```

Спроси что‑нибудь простое:

```text
What tools do you have available?
```

Если это ещё не стабильно, сначала исправь режим текста.
## Шаг 2: установить необходимые дополнения

### CLI‑микрофон + воспроизведение

```bash
pip install "hermes-agent[voice]"
```

### Платформы обмена сообщениями

```bash
pip install "hermes-agent[messaging]"
```

### Премиум ElevenLabs TTS

```bash
pip install "hermes-agent[tts-premium]"
```

### Локальный NeuTTS (необязательно)

```bash
python -m pip install -U neutts[all]
```

### Все

```bash
pip install "hermes-agent[all]"
```
## Шаг 3: установка системных зависимостей

### macOS

```bash
brew install portaudio ffmpeg opus
brew install espeak-ng
```

### Ubuntu / Debian

```bash
sudo apt install portaudio19-dev ffmpeg libopus0
sudo apt install espeak-ng
```

Почему это важно:
- `portaudio` → ввод и воспроизведение микрофона для режима голосового CLI
- `ffmpeg` → конвертация аудио для TTS и доставки сообщений
- `opus` → поддержка голосового кодека Discord
- `espeak-ng` → бэкенд фонемизатора для NeuTTS
## Шаг 4: выбор провайдеров STT и TTS

Hermes поддерживает как локальные, так и облачные стеки распознавания и синтеза речи.

### Самая простая / дешёвая конфигурация

Используй локальный STT и бесплатный Edge TTS:
- Провайдер STT: `local`
- Провайдер TTS: `edge`

Обычно это лучшее место для начала.

### Пример файла окружения

Добавь в `~/.hermes/.env`:

```bash
# Cloud STT options (local needs no key)
GROQ_API_KEY=***
VOICE_TOOLS_OPENAI_KEY=***

# Premium TTS (optional)
ELEVENLABS_API_KEY=***
```

### Рекомендации по провайдерам

#### Speech-to-text

- `local` → лучший вариант по умолчанию для конфиденциальности и нулевой стоимости
- `groq` → очень быстрая облачная транскрипция
- `openai` → хороший платный запасной вариант

#### Text-to-speech

- `edge` → бесплатно и достаточно хорошо для большинства пользователей
- `neutts` → бесплатный локальный/на‑устройстве TTS
- `elevenlabs` → наилучшее качество
- `openai` → хороший средний вариант
- `mistral` → многоязычный, нативный Opus

### Если ты используешь `hermes setup`

Если в мастере настройки ты выбираешь NeuTTS, Hermes проверяет, установлен ли `neutts`. Если его нет, мастер сообщает, что NeuTTS требует Python‑пакет `neutts` и системный пакет `espeak-ng`, предлагает установить их за тебя, устанавливает `espeak-ng` через менеджер пакетов твоей платформы, а затем выполняет:

```bash
python -m pip install -U neutts[all]
```

Если ты пропустишь эту установку или она завершится ошибкой, мастер переключается на Edge TTS.
## Шаг 5: рекомендуемая конфигурация

```yaml
voice:
  record_key: "ctrl+b"
  max_recording_seconds: 120
  auto_tts: false
  beep_enabled: true
  silence_threshold: 200
  silence_duration: 3.0

stt:
  provider: "local"
  local:
    model: "base"

tts:
  provider: "edge"
  edge:
    voice: "en-US-AriaNeural"
```

Это хорошая консервативная настройка по умолчанию для большинства пользователей.

Если ты хочешь использовать локальный TTS, замени блок `tts` на:

```yaml
tts:
  provider: "neutts"
  neutts:
    ref_audio: ''
    ref_text: ''
    model: neuphonic/neutts-air-q4-gguf
    device: cpu
```
## Сценарий применения 1: голосовой режим CLI
## Включить

Запусти Hermes:

```bash
hermes
```

Внутри CLI:

```text
/voice on
```

### Процесс записи

Клавиша по умолчанию:
- `Ctrl+B`

Рабочий процесс:
1. нажми `Ctrl+B`
2. говори
3. дождись, пока детектор тишины автоматически остановит запись
4. Hermes транскрибирует и отвечает
5. если включён TTS, он озвучивает ответ
6. цикл может автоматически перезапуститься для непрерывного использования

### Полезные команды

```text
/voice
/voice on
/voice off
/voice tts
/voice status
```

### Хорошие рабочие процессы в CLI

#### Отладка на ходу

Скажи:

```text
I keep getting a docker permission error. Help me debug it.
```

Затем продолжай без рук:
- «Прочитай последнюю ошибку ещё раз»
- «Объясни причину проще»
- «Сейчас дай точное исправление»

#### Исследования / мозговой штурм

Отлично подходит для:
- прогулок во время размышлений
- диктовки недописанных идей
- просьбы к Hermes структурировать мысли в реальном времени

#### Доступность / сессии с минимальным набором текста

Если печатать неудобно, голосовой режим — один из самых быстрых способов оставаться в полном цикле Hermes.
## Настройка поведения CLI

### Порог тишины

Если Hermes слишком агрессивно начинает/останавливает запись, отрегулируй:

```yaml
voice:
  silence_threshold: 250
```

Более высокий порог = менее чувствительно.

### Продолжительность тишины

Если между фразами часто делаются паузы, увеличь:

```yaml
voice:
  silence_duration: 4.0
```

### Клавиша записи

Если `Ctrl+B` конфликтует с твоими настройками терминала или tmux:

```yaml
voice:
  record_key: "ctrl+space"
```
## Use case 2: голосовые ответы в Telegram или Discord

Этот режим проще, чем полные голосовые каналы.

Hermes остаётся обычным чат‑ботом, но может озвучивать ответы.

### Запуск gateway

```bash
hermes gateway
```

### Включение голосовых ответов

Внутри Telegram или Discord:

```text
/voice on
```

или

```text
/voice tts
```

### Режимы

| Mode | Meaning |
|---|---|
| `off` | только текст |
| `voice_only` | озвучивать только когда пользователь отправил голосовое сообщение |
| `all` | озвучивать каждый ответ |

### Когда использовать какой режим

- `/voice on` — если нужны голосовые ответы только для сообщений, пришедших в виде голоса
- `/voice tts` — если нужен полностью озвученный помощник всё время

### Хорошие сценарии обмена сообщениями

#### Помощник в Telegram на твоём телефоне

Используй, когда:
- ты не у своего компьютера;
- хочешь отправлять голосовые заметки и получать быстрые голосовые ответы;
- хочешь, чтобы Hermes функционировал как портативный исследовательский или операционный помощник.

#### Прямые сообщения в Discord с голосовым выводом

Полезно, когда нужен приватный диалог без поведения упоминаний в серверных каналах.
## Сценарий использования 3: голосовые каналы Discord

Это самый продвинутый режим.

Hermes присоединяется к голосовому каналу Discord, прослушивает речь пользователя, транскрибирует её, запускает обычный конвейер агента и озвучивает ответы обратно в канал.
## Требуемые разрешения Discord

Помимо обычной настройки текстового бота, убедись, что у бота есть:
- Connect
- Speak
- желательно **Use Voice Activity**

Также включи привилегированные интенты в **Developer Portal**:
- Presence Intent
- Server Members Intent
- Message Content Intent
## Присоединение и выход

В текстовом канале Discord, где присутствует бот:

```text
/voice join
/voice leave
/voice status
```

### Что происходит при присоединении

- пользователи говорят в голосовом канале
- Hermes определяет границы речи
- транскрипты публикуются в связанном текстовом канале
- Hermes отвечает текстом и аудио
- текстовым каналом считается тот, в котором была выполнена команда `/voice join`

### Лучшие практики использования голосовых каналов Discord

- ограничивай `DISCORD_ALLOWED_USERS`
- сначала используй отдельный канал для бота/тестирования
- убедись, что STT и TTS работают в обычном режиме голосового чата перед попыткой использовать режим голосового канала
## Рекомендации по качеству озвучки

### Лучшее качество

- STT: local `large-v3` или Groq `whisper-large-v3`
- TTS: ElevenLabs

### Оптимальное сочетание скорости и удобства

- STT: local `base` или Groq
- TTS: Edge

### Самый экономичный вариант

- STT: local
- TTS: Edge
## Общие причины сбоев

### «Не найдено аудио‑устройство»

Установи `portaudio`.

### «Бот присоединился, но ничего не слышит»

Проверь:
- твой Discord‑ID находится в `DISCORD_ALLOWED_USERS`;
- ты не отключил микрофон;
- включены привилегированные intents;
- у бота есть права Connect/Speak.

### «Бот транскрибирует, но не озвучивает»

Проверь:
- конфигурацию провайдера TTS;
- API‑ключ / квоту для ElevenLabs или OpenAI;
- установку `ffmpeg` для путей конвертации Edge.

### «Whisper выдаёт мусор»

Попробуй:
- более тихую обстановку;
- увеличить `silence_threshold`;
- другой провайдер/модель STT;
- более короткие, чёткие реплики.

### «Работает в личных сообщениях, но не в каналах сервера»

Часто связано с политикой упоминаний.

По умолчанию бот требует `@mention` в текстовых каналах Discord‑сервера, если не настроено иначе.
## Предлагаемая настройка первой недели

Если ты хочешь самый короткий путь к успеху:

1. запусти текстовый Hermes
2. установи `hermes-agent[voice]`
3. используй голосовой режим CLI с локальным STT + Edge TTS
4. затем включи `/voice on` в Telegram или Discord
5. только после этого попробуй режим Discord VC

Такой порядок сохраняет небольшую поверхность отладки.
