# Учебник: Создание бота ежедневных брифингов

В этом учебнике ты создашь персонального бота‑брифинга, который каждое утро просыпается, исследует интересующие тебя темы, подводит итоги и доставляет лаконичный бриф прямо в твой Telegram или Discord.

К концу ты получишь полностью автоматизированный рабочий процесс, объединяющий **веб‑поиск**, **планирование cron**, **делегирование** и **обмен сообщениями** — без написания кода.

:::tip
Этот рецепт охватывает веб‑поиск, суммирование и опциональный TTS — всё в рамках подписки Portal. Самый быстрый способ установки — `hermes setup --portal`. Смотри [Nous Portal](/integrations/nous-portal).
:::

## Что мы будем создавать

Вот схема процесса:

1. **08:00** — Планировщик cron запускает твою задачу
2. **Hermes** поднимает новую сессию агента с твоим запросом
3. **Веб‑поиск** собирает последние новости по твоим темам
4. **Суммирование** превращает их в чистый формат брифинга
5. **Доставка** отправляет бриф в твой Telegram или Discord

Всё работает без твоего участия. Ты просто читаешь бриф за утренним кофе.

## Предварительные требования

Перед началом убедись, что у тебя есть:

- **Hermes Agent установлен** — смотри [Руководство по установке](/getting-started/installation)
- **Gateway запущен** — демон шлюза обрабатывает выполнение cron:
  ```bash
  hermes gateway install   # Install as a user service
  sudo hermes gateway install --system   # Linux servers: boot-time system service
  # or
  hermes gateway           # Run in foreground
  ```
- **API‑ключ Firecrawl** — задай `FIRECRAWL_API_KEY` в окружении для веб‑поиска
- **Настроенный мессенджер** (опционально, но рекомендуется) — [Telegram](/user-guide/messaging/telegram) или Discord с домашним каналом

:::tip No messaging? No problem
Ты всё равно можешь пройти этот учебник, используя `deliver: "local"`. Брифы будут сохраняться в `~/.hermes/cron/output/`, и ты сможешь читать их в любое время.
:::

## Шаг 1: Протестировать рабочий процесс вручную

Прежде чем автоматизировать, убедимся, что бриф работает. Запусти чат‑сессию:

```bash
hermes
```

Затем введи следующий запрос:

```
Search for the latest news about AI agents and open source LLMs.
Summarize the top 3 stories in a concise briefing format with links.
```

Hermes выполнит веб‑поиск, просмотрит результаты и выдаст нечто вроде:

```
☀️ Your AI Briefing — March 8, 2026

1. Qwen 3 Released with 235B Parameters
   Alibaba's latest open-weight model matches GPT-4.5 on several
   benchmarks while remaining fully open source.
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain Launches Agent Protocol Standard
   A new open standard for agent-to-agent communication gains
   adoption from 15 major frameworks in its first week.
   → https://blog.langchain.dev/agent-protocol/

3. EU AI Act Enforcement Begins for General-Purpose Models
   The first compliance deadlines hit, with open source models
   receiving exemptions under the 10M parameter threshold.
   → https://artificialintelligenceact.eu/updates/

---
3 stories • Sources searched: 8 • Generated by Hermes Agent
```

Если всё работает, можно переходить к автоматизации.

:::tip Iterate on the format
Пробуй разные запросы, пока не получишь желаемый вывод. Добавляй инструкции вроде «использовать эмодзи‑заголовки» или «каждое резюме не более 2 предложений». Что бы ты ни выбрал, это попадёт в задачу cron.
:::

## Шаг 2: Создать задачу cron

Теперь запланируем автоматический запуск каждое утро. Есть два способа.

Перед созданием задач cron убедись, что у Hermes настроена модель и провайдер по умолчанию глобально. Если нужно, чтобы конкретная задача использовала другие значения, укажи явные переопределения модели/провайдера при её создании.

### Вариант A: Естественный язык (в чате)

Просто скажи Hermes, что тебе нужно:

```
Every morning at 8am, search the web for the latest news about AI agents
and open source LLMs. Summarize the top 3 stories in a concise briefing
with links. Use a friendly, professional tone. Deliver to telegram.
```

Hermes создаст задачу cron, используя единый инструмент `cronjob`.

### Вариант B: CLI‑команда

Используй команду `/cron` для большего контроля:

```
/cron add "0 8 * * *" "Search the web for the latest news about AI agents and open source LLMs. Find at least 5 recent articles from the past 24 hours. Summarize the top 3 most important stories in a concise daily briefing format. For each story include: a clear headline, a 2-sentence summary, and the source URL. Use a friendly, professional tone. Format with emoji bullet points and end with a total story count."
```

### Золотое правило: Самодостаточные запросы

:::warning Critical concept
Задачи cron запускаются в **полностью новой сессии** — без памяти о предыдущих разговорах, без контекста о том, что ты «настраивал ранее». Твой запрос должен содержать **всё**, что агенту нужно для выполнения задачи.
:::

**Плохой запрос:**
```
Do my usual morning briefing.
```

**Хороший запрос:**
```
Search the web for the latest news about AI agents and open source LLMs.
Find at least 5 recent articles from the past 24 hours. Summarize the
top 3 most important stories in a concise daily briefing format. For each
story include: a clear headline, a 2-sentence summary, and the source URL.
Use a friendly, professional tone. Format with emoji bullet points.
```

Хороший запрос конкретизирует **что искать**, **сколько статей**, **в каком формате** и **в каком тоне**. Всё, что нужно агенту, находится в одном запросе.

## Шаг 3: Настроить брифинг

Когда базовый бриф работает, можно проявлять креатив.

### Мульти‑тематические брифинги

Объединяй несколько областей в одном брифе:

```
/cron add "0 8 * * *" "Create a morning briefing covering three topics. For each topic, search the web for recent news from the past 24 hours and summarize the top 2 stories with links.

Topics:
1. AI and machine learning — focus on open source models and agent frameworks
2. Cryptocurrency — focus on Bitcoin, Ethereum, and regulatory news
3. Space exploration — focus on SpaceX, NASA, and commercial space

Format as a clean briefing with section headers and emoji. End with today's date and a motivational quote."
```

### Делегирование для параллельных исследований

Для ускорения брифов попроси Hermes делегировать каждую тему суб‑агенту:

```
/cron add "0 8 * * *" "Create a morning briefing by delegating research to sub-agents. Delegate three parallel tasks:

1. Delegate: Search for the top 2 AI/ML news stories from the past 24 hours with links
2. Delegate: Search for the top 2 cryptocurrency news stories from the past 24 hours with links
3. Delegate: Search for the top 2 space exploration news stories from the past 24 hours with links

Collect all results and combine them into a single clean briefing with section headers, emoji formatting, and source links. Add today's date as a header."
```

Каждый суб‑агент ищет независимо и параллельно, после чего главный агент собирает всё в один отполированный бриф. Подробнее смотри [Документацию по делегированию](/user-guide/features/delegation).

### Расписание только по будням

Не нужны брифы в выходные? Используй cron‑выражение, охватывающее понедельник‑пятницу:

```
/cron add "0 8 * * 1-5" "Search for the latest AI and tech news..."
```

### Два брифа в день

Утренний обзор и вечернее резюме:

```
/cron add "0 8 * * *" "Morning briefing: search for AI news from the past 12 hours..."
/cron add "0 18 * * *" "Evening recap: search for AI news from the past 12 hours..."
```

### Добавление личного контекста через память

Если у тебя включена [память](/user-guide/features/memory), можно хранить предпочтения между сессиями. Но помни — задачи cron работают в новых сессиях без разговорной памяти. Чтобы добавить личный контекст, включи его прямо в запрос:

```
/cron add "0 8 * * *" "You are creating a briefing for a senior ML engineer who cares about: PyTorch ecosystem, transformer architectures, open-weight models, and AI regulation in the EU. Skip stories about product launches or funding rounds unless they involve open source.

Search for the latest news on these topics. Summarize the top 3 stories with links. Be concise and technical — this reader doesn't need basic explanations."
```

:::tip Tailor the persona
Указание, для кого предназначен бриф, значительно повышает релевантность. Расскажи агенту о своей роли, интересах и что следует пропустить.
:::

## Шаг 4: Управление задачами

### Список всех запланированных задач

В чате:
```
/cron list
```

Или из терминала:
```bash
hermes cron list
```

Ты увидишь вывод вроде:

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### Удалить задачу

В чате:
```
/cron remove a1b2c3d4
```

Или попроси в разговоре:
```
Remove my morning briefing cron job.
```

Hermes использует `cronjob(action="list")` для поиска и `cronjob(action="remove")` для удаления.

### Проверить статус Gateway

Убедись, что планировщик действительно работает:

```bash
hermes cron status
```

Если шлюз не запущен, задачи не будут выполняться. Установи его как фоновый сервис для надёжности:

```bash
hermes gateway install
# or on Linux servers
sudo hermes gateway install --system
```
