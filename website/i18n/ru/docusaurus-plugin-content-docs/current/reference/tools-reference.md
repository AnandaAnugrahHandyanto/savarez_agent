# Справочник встроенных инструментов

Эта страница описывает встроенные инструменты Hermes, сгруппированные по набору инструментов. Доступность зависит от платформы, учётных данных и включённых наборов инструментов.

**Быстрый подсчёт (текущий реестр):** ~64 инструмента — 10 браузерных инструментов (ядро) + 2 инструмента браузера с ограничением CDP, 4 файловых инструмента, 4 инструмента Home Assistant, 2 терминальных инструмента, 2 веб‑инструмента, 5 инструментов Feishu, 7 инструментов Spotify (зарегистрированы встроенным плагином `spotify`), 5 инструментов Yuanbao, 9 kanban‑инструментов (зарегистрированы, когда диспетчер kanban порождает агента), 2 инструмента Discord и несколько автономных инструментов (`memory`, `clarify`, `delegate_task`, `execute_code`, `cronjob`, `session_search`, `skill_view`/`skill_manage`/`skills_list`, `text_to_speech`, `image_generate`, `video_generate`, `vision_analyze`, `video_analyze`, `mixture_of_agents`, `send_message`, `todo`, `computer_use`, `process`).

:::tip MCP Tools
Помимо встроенных инструментов, Hermes может загружать инструменты динамически с серверов MCP. Инструменты MCP отображаются с префиксом `mcp_<server>_` (например, `mcp_github_create_issue` для сервера MCP `github`). См. [MCP Integration](/user-guide/features/mcp) для настройки.
:::
## `browser` toolset

| Инструмент | Описание | Требует окружения |
|------|-------------|----------------------|
| `browser_back` | Перейти назад к предыдущей странице в истории браузера. Требует предварительного вызова `browser_navigate`. | — |
| `browser_click` | Кликнуть по элементу, идентифицированному по его ref‑ID из снимка (например, `@e5`). Ref‑ID отображаются в квадратных скобках в выводе снимка. Требует предварительных вызовов `browser_navigate` и `browser_snapshot`. | — |
| `browser_console` | Получить вывод консоли браузера и ошибки JavaScript с текущей страницы. Возвращает сообщения `console.log/warn/error/info` и непойманные исключения JS. Используй для обнаружения тихих ошибок JavaScript, неудачных API‑вызовов и предупреждений приложения. | — |
| `browser_get_images` | Получить список всех изображений на текущей странице с их URL и альтернативным текстом. Полезно для поиска изображений для анализа инструментом vision. Требует предварительного вызова `browser_navigate`. | — |
| `browser_navigate` | Перейти по URL в браузере. Инициализирует сессию и загружает страницу. Должен вызываться перед другими инструментами браузера. Для простого получения информации предпочтительнее `web_search` или `web_extract` (быстрее, дешевле). Используй инструменты браузера, когда требуется интерактивное взаимодействие. | — |
| `browser_press` | Нажать клавишу клавиатуры. Полезно для отправки форм (`Enter`), навигации (`Tab`) или горячих клавиш. Требует предварительного вызова `browser_navigate`. | — |
| `browser_scroll` | Прокрутить страницу в заданном направлении. Используй для раскрытия содержимого, которое может находиться ниже или выше текущего окна просмотра. Требует предварительного вызова `browser_navigate`. | — |
| `browser_snapshot` | Получить текстовый снимок дерева доступности текущей страницы. Возвращает интерактивные элементы с ref‑ID (например, `@e1`, `@e2`) для `browser_click` и `browser_type`. `full=false` (по умолчанию): компактный вид с интерактивными элементами. `full=true`: полный вид. | — |
| `browser_type` | Ввести текст в поле ввода, идентифицированное по его ref‑ID. Сначала очищает поле, затем вводит новый текст. Требует предварительных вызовов `browser_navigate` и `browser_snapshot`. | — |
| `browser_vision` | Сделать скриншот текущей страницы и проанализировать его с помощью vision‑AI. Используй, когда нужно визуально понять, что находится на странице — особенно полезно для CAPTCHA, визуальных проверок, сложных макетов или когда текст сложно извлечь. | — |

## `kanban` toolset

Регистрируется, когда агент либо (a) запускается диспетчером kanban (`HERMES_KANBAN_TASK` установлен в переменной окружения), либо (b) работает в профиле, который явно включает набор инструментов `kanban`. Workers, ограниченные задачей, используют инструменты жизненного цикла для своей назначенной задачи; профили‑оркестраторы дополнительно получают инструменты маршрутизации доски, такие как `kanban_list` и `kanban_unblock`. См. [Kanban Multi-Agent](/user-guide/features/kanban) для полного описания рабочего процесса.

| Инструмент | Описание | Требует окружения |
|------|-------------|----------------------|
| `kanban_show` | Показать активную задачу kanban, назначенную этому worker’у (заголовок, описание, комментарии, зависимости). | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_list` | Вывести список задач доски с фильтрами. Только для оркестратора; скрыто от workers, запущенных диспетчером. | профиль с набором инструментов `kanban` |
| `kanban_complete` | Отметить текущую задачу выполненной, передав структурированный payload (результаты, артефакты, последующие действия). | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_block` | Заблокировать текущую задачу вопросом для пользователя — диспетчер приостанавливается, выводит вопрос и возобновляется после ответа человека. | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_heartbeat` | Отправлять heartbeat прогресса во время длительной операции, чтобы диспетчер знал, что worker всё ещё жив. | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_comment` | Добавить комментарий в ветку задачи без изменения её состояния — полезно для отображения промежуточных выводов. | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_create` | Расширить текущую задачу дочерними задачами. Используется оркестраторами и workers, создающими последующие задачи. | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_link` | Связать задачи ребром зависимости «родитель → дочерняя». | `HERMES_KANBAN_TASK` или набор инструментов `kanban` |
| `kanban_unblock` | Вернуть заблокированную задачу в состояние `ready`. Только для оркестратора; скрыто от workers, запущенных диспетчером. | профиль с набором инструментов `kanban` |

## `video_gen` toolset

Опциональный набор инструментов (не загружается в наборе по умолчанию `hermes-cli`). Добавляется через `--toolsets video_gen` или включается в `hermes tools` → Video Generation, где также предлагается выбрать бэкенд.

Бэкенды поставляются как плагины в каталоге `plugins/video_gen/<name>/`:

- **xAI Grok-Imagine** — text‑to‑video и image‑to‑video (SuperGrok OAuth или `XAI_API_KEY`).
- **FAL.ai** — Veo 3.1, Pixverse v6, Kling O3 (требуется `FAL_KEY`).

Единый инструмент `video_generate` покрывает обе модальности — передай `image_url`, чтобы анимировать статичное изображение, либо не указывай его, чтобы сгенерировать только из текста. Активный бэкенд автоматически маршрутизирует запрос к нужному эндпоинту. Описание инструмента пересобирается при старте сессии, отражая реальные возможности активного бэкенда (модальности, соотношения сторон, разрешения, диапазон длительности, максимальное количество референс‑изображений, поддержка аудио). См. [Video Generation Provider Plugins](/developer-guide/video-gen-provider-plugin) для создания бэкенда.

| Tool | Description | Requires environment |
|------|-------------|----------------------|
| `video_generate` | Сгенерировать видео по текстовому запросу (text‑to‑video) или анимировать статичное изображение (image‑to‑video) с использованием настроенного пользователем бэкенда генерации видео. Передай `image_url`, чтобы анимировать это изображение; не указывай его, чтобы сгенерировать только из текста. Бэкенд автоматически маршрутизирует запрос к нужному эндпоинту. Возвращает либо HTTP‑URL, либо абсолютный путь к файлу в поле `video`. | Активный плагин `video_gen` + его учётные данные (например, `XAI_API_KEY`, `FAL_KEY`) |
