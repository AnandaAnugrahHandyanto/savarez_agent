# CLI Interface

CLI Hermes Agent — це повноцінний інтерфейс користувача в терміналі (TUI), а не веб‑інтерфейс. Він підтримує багаторядкове редагування, автодоповнення slash‑команд, історію розмов, переривання та перенаправлення, а також стрімінговий вивід інструментів. Створений для людей, які живуть у терміналі.

:::tip Перший запуск
Одна команда — `hermes setup --portal` — і ти готовий до `hermes chat`. Дивись [Nous Portal](/integrations/nous-portal).
:::

:::tip
Hermes також постачається з сучасним TUI, що має модальні накладки, вибір мишею та неблокуючий ввід. Запусти його за допомогою `hermes --tui` — дивись посібник [TUI](tui.md).
:::
## Запуск CLI

```bash
# Start an interactive session (default)
hermes

# Single query mode (non-interactive)
hermes chat -q "Hello"

# With a specific model
hermes chat --model "anthropic/claude-sonnet-4"

# With a specific provider
hermes chat --provider nous        # Use Nous Portal
hermes chat --provider openrouter  # Force OpenRouter

# With specific toolsets
hermes chat --toolsets "web,terminal,skills"

# Start with one or more skills preloaded
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -q "open a draft PR"

# Resume previous sessions
hermes --continue             # Resume the most recent CLI session (-c)
hermes --resume <session_id>  # Resume a specific session by ID (-r)

# Verbose mode (debug output)
hermes chat --verbose

# Isolated git worktree (for running multiple agents in parallel)
hermes -w                         # Interactive mode in worktree
hermes -w -q "Fix issue #123"     # Single query in worktree
```

### Рядок стану

Постійний рядок стану розташований над областю вводу та оновлюється в реальному часі:

```
 ⚕ claude-sonnet-4-20250514 │ 12.4K/200K │ [██████░░░░] 6% │ $0.06 │ 15m
```

| Елемент | Опис |
|---------|------|
| Model name | Поточна модель (скорочена, якщо довша за 26 символів) |
| Token count | Токени контексту, що використані / максимальне вікно контексту |
| Context bar | Візуальний індикатор заповнення з кольоровими порогами |
| Cost | Оцінка вартості сесії (або `n/a` для моделей без ціни/з нульовою вартістю) |
| 🗜️ N | **Кількість стискань контексту** — скільки разів поточна сесія була автоматично стиснута. Показується після першого стискання. |
| ▶ N | **Активні фонові завдання** — скільки запитів `/background` ще працюють у поточній сесії. Показується, коли хоча б одне завдання виконується. |
| Duration | Час, що минув з початку сесії |
| ⚠ YOLO | **Попередження режиму YOLO** — показується, коли `HERMES_YOLO_MODE` увімкнено (або `hermes --yolo` при запуску, або `/yolo` під час сесії). Відображає попередження в рядку банера, щоб ти не забув, що працюєш у режимі автоматичного схвалення. |

Рядок адаптується до ширини терміналу — повний макет при ≥ 76 колонках, компактний при 52–75, мінімальний (модель + тривалість, плюс значок YOLO, коли активний) при < 52.

**Кольорова індикація контексту:**

| Колір | Поріг | Значення |
|-------|------|----------|
| Green | < 50 % | Достатньо місця |
| Yellow | 50–80 % | Заповнюється |
| Orange | 80–95 % | Підходить до межі |
| Red | ≥ 95 % | Майже переповнення — розглянь `/compress` |

Використовуй `/usage` для докладного розподілу, включаючи витрати за категоріями (вхідні vs вихідні токени).

### Відображення відновлення сесії

При відновленні попередньої сесії (`hermes -c` або `hermes --resume <id>`), між банером і підказковим рядком з’являється панель «Previous Conversation», що показує стислий підсумок історії розмови. Дивись [Sessions — Conversation Recap on Resume](sessions.md#conversation-recap-on-resume) для деталей та налаштувань.
## Клавіші

| Клавіша | Дія |
|-----|--------|
| `Enter` | Надіслати повідомлення |
| `Alt+Enter`, `Ctrl+J` або `Shift+Enter` | Новий рядок (багаторядковий ввід). `Shift+Enter` потребує терміналу, який розрізняє його від `Enter` — дивись нижче. У Windows Terminal `Alt+Enter` перехоплюється терміналом (перемикач повноекранного режиму); використай `Ctrl+Enter` або `Ctrl+J` замість цього. |
| `Alt+V` | Вставити зображення з буфера обміну, якщо термінал це підтримує |
| `Ctrl+V` | Вставити текст і, за можливості, прикріпити зображення з буфера |
| `Ctrl+B` | Почати/зупинити запис голосу, коли режим голосу ввімкнено (`voice.record_key`, за замовчуванням: `ctrl+b`) |
| `Ctrl+G` | Відкрити поточний буфер вводу у `$EDITOR` (vim/nvim/nano/VS Code тощо). Зберегти і вийти, щоб надіслати відредагований текст як наступний запит — ідеально для довгих, багатопараграфних запитів. |
| `Ctrl+X Ctrl+E` | Альтернативна прив’язка в стилі Emacs для зовнішнього редактора (те ж саме, що і `Ctrl+G`). |
| `Ctrl+C` | Перервати агента (подвійне натискання протягом 2 с для примусового виходу) |
| `Ctrl+D` | Вихід |
| `Ctrl+Z` | Призупинити Hermes у фоні (лише Unix). Запусти `fg` у оболонці, щоб відновити. |
| `Tab` | Прийняти автопідказку (ghost text) або автодоповнити slash‑команди |

**Попередній перегляд багаторядкової вставки.** Коли ти вставляєш багаторядковий блок, CLI виводить компактний однорядковий попередній перегляд (`[pasted: 47 lines, 1,842 chars — press Enter to send]`) замість того, щоб виводити весь вміст у скролбек. Повний вміст все одно надсилається; це лише візуальне оформлення.

**Видалення Markdown у фінальних відповідях.** CLI видаляє найоб’ємніші markdown‑огородження та обгортки `**bold**` / `*italic*` з *фінальних* відповідей агента, щоб вони відображалися як читабельний текст у терміналі, а не як сирий код. Блоки коду та списки зберігаються. Це не впливає на платформи‑шлюзи або результати інструментів — вони залишають свій markdown для рідного рендерингу.
## Slash Commands

Натисни `/`, щоб побачити випадаюче автодоповнення. Hermes підтримує великий набір CLI slash‑команд, динамічних команд навичок та користувацьких швидких команд.

Типові приклади:

| Command | Description |
|---------|-------------|
| `/help` | Показати довідку по команді |
| `/model` | Показати або змінити поточну модель |
| `/tools` | Перелік доступних інструментів |
| `/skills browse` | Переглянути центр навичок та офіційні додаткові навички |
| `/background <prompt>` | Запустити запит у окремій фоновій сесії |
| `/skin` | Показати або переключити активну CLI тему |
| `/voice on` | Увімкнути голосовий режим CLI (натисни `Ctrl+B` для запису) |
| `/voice tts` | Перемкнути озвучення відповідей Hermes |
| `/reasoning high` | Збільшити інтенсивність роздумів |
| `/title My Session` | Назвати поточну сесію |
| `/status` | Показати інформацію про сесію — модель/профіль/токени/тривалість — та блок **Session recap** з останніми підрахунками ходів, топ‑інструментами, зміненими файлами, останнім запитом користувача + відповіддю асистента. Чисто локальне обчислення; без виклику LLM. |
| `/sessions` | Відкрити інтерактивний вибір сесій прямо в класичному CLI (те ж саме, що й у TUI). Введи текст для фільтрації, використай стрілки для навігації, Enter — для відновлення. |

Для повного списку вбудованих CLI та команд обміну повідомленнями дивись [Slash Commands Reference](../reference/slash-commands.md).

Для налаштувань, провайдерів, регулювання тиші та використання голосу в обміні повідомленнями/Discord, дивись [Voice Mode](features/voice-mode.md).

:::tip
Команди не чутливі до регістру — `/HELP` працює так само, як `/help`. Встановлені навички також автоматично стають slash‑командами.
:::
## Швидкі команди

Ти можеш визначати власні команди, які миттєво виконують shell‑команди без залучення LLM. Вони працюють і в CLI, і в платформах обміну повідомленнями (Telegram, Discord тощо).

```yaml
# ~/.hermes/config.yaml
quick_commands:
  status:
    type: exec
    command: systemctl status hermes-agent
  gpu:
    type: exec
    command: nvidia-smi --query-gpu=utilization.gpu,memory.used --format=csv,noheader
  restart:
    type: alias
    target: /gateway restart
```

Потім введи `/status`, `/gpu` або `/restart` у будь‑якому чаті. Дивись [Посібник з налаштування](/user-guide/configuration#quick-commands) для додаткових прикладів.
## Попереднє завантаження skills при запуску

Якщо ти вже знаєш, які skills хочеш активувати для сесії, передай їх під час запуску:

```bash
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -s github-auth
```

Hermes завантажує кожен названий skill у підказку сесії перед першим ходом. Така ж опція працює в інтерактивному режимі та в режимі одиночного запиту.
## Slash‑команди навичок

Кожна встановлена навичка в `~/.hermes/skills/` автоматично реєструється як slash‑команда. Назва навички стає командою:

```
/gif-search funny cats
/axolotl help me fine-tune Llama 3 on my dataset
/github-pr-workflow create a PR for the auth refactor

# Just the skill name loads it and lets the agent ask what you need:
/excalidraw
```
## Персональності

Встанови попередньо визначену персональність, щоб змінити тон агента:

```
/personality pirate
/personality kawaii
/personality concise
```

Вбудовані персональності включають: `helpful`, `concise`, `technical`, `creative`, `teacher`, `kawaii`, `catgirl`, `pirate`, `shakespeare`, `surfer`, `noir`, `uwu`, `philosopher`, `hype`.

Ти також можеш визначити власні персональності у `~/.hermes/config.yaml`:

```yaml
personalities:
  helpful: "You are a helpful, friendly AI assistant."
  kawaii: "You are a kawaii assistant! Use cute expressions..."
  pirate: "Arrr! Ye be talkin' to Captain Hermes..."
  # Add your own!
```
## Багаторядковий ввід

Існує два способи вводити багаторядкові повідомлення:

1. **`Alt+Enter`, `Ctrl+J`, або `Shift+Enter`** — вставляє новий рядок
2. **Продовження за допомогою зворотного слешу** — закінчи рядок `\`, щоб продовжити:

```
❯ Write a function that:\
  1. Takes a list of numbers\
  2. Returns the sum
```

:::info
Вставка багаторядкового тексту підтримується — використай будь‑яку з клавіш нових рядків вище або просто встав текст безпосередньо.
:::

### Сумісність Shift+Enter

Більшість терміналів за замовчуванням надсилає однакову послідовність байтів для `Enter` і `Shift+Enter`, тому програми не можуть їх розрізнити. Hermes розпізнає `Shift+Enter` лише коли термінал надсилає окрему послідовність через [протокол клавіатури Kitty](https://sw.kovidgoyal.net/kitty/keyboard-protocol/) або режим `modifyOtherKeys` у xterm.

| Термінал | Статус |
|---|---|
| Kitty, foot, WezTerm, Ghostty | Окремий `Shift+Enter` увімкнено за замовчуванням |
| iTerm2 (недавні), Alacritty, VS Code terminal, Warp | Підтримується після ввімкнення протоколу Kitty у налаштуваннях |
| Windows Terminal Preview 1.25+ | Підтримується після ввімкнення протоколу Kitty у налаштуваннях |
| macOS Terminal.app, стандартний Windows Terminal (stable) | Не підтримується — `Shift+Enter` не відрізняється від `Enter` |

Там, де термінал не може їх розрізнити, `Alt+Enter` і `Ctrl+J` працюють скрізь. **У Windows Terminal саме `Alt+Enter` перехоплюється терміналом (перемикає повноекранний режим) і ніколи не доходить до Hermes — використай `Ctrl+Enter` (надсилається як `Ctrl+J`) або безпосередньо `Ctrl+J` для нового рядка.**
## Переривання агента

Ти можеш переривати агента в будь‑який момент:

- **Введи нове повідомлення + Enter**, доки агент працює — воно перериває і обробляє твої нові інструкції
- **`Ctrl+C`** — перервати поточну операцію (натисни двічі протягом 2 с, щоб примусово вийти)
- Команди, що виконуються в терміналі, миттєво завершаються (спочатку `SIGTERM`, потім `SIGKILL` через 1 с)
- Кілька повідомлень, введених під час переривання, об’єднуються в один запит

### Режим зайвого вводу

Ключ конфігурації `display.busy_input_mode` визначає, що відбувається, коли ти натискаєш **Enter**, доки агент працює:

| Mode | Behavior |
|------|----------|
| `"interrupt"` (за замовчуванням) | Твоє повідомлення перериває поточну операцію і обробляється негайно |
| `"queue"` | Твоє повідомлення тихо ставиться в чергу і надсилається як наступний хід після завершення роботи агента |
| `"steer"` | Твоє повідомлення впроваджується у поточний запуск через `/steer`, доходячи до агента після наступного виклику інструменту — без переривання, без нового ходу |

```yaml
# ~/.hermes/config.yaml
display:
  busy_input_mode: "steer"   # or "queue" or "interrupt" (default)
```

Режим `"queue"` корисний, коли потрібно підготувати наступні повідомлення, не скасовуючи випадково роботу, що вже виконується. Режим `"steer"` корисний, коли треба перенаправити агента під час виконання завдання без переривання — напр., «насправді, ще перевір тести», коли код ще редагується. Невідомі значення повертаються до `"interrupt"`.

У `"steer"` є два автоматичні запасні варіанти: якщо агент ще не розпочав роботу, або якщо до повідомлення прикріплені зображення, повідомлення переходить у поведінку `"queue"`, щоб нічого не втратити.

Ти також можеш змінити це всередині CLI:

```text
/busy queue
/busy steer
/busy interrupt
/busy status
```

:::tip First-touch hint
The very first time you press Enter while Hermes is working, Hermes prints a one-line reminder explaining the `/busy` knob (`"(tip) Your message interrupted the current run…"`). It only fires once per install — a flag in `config.yaml` under `onboarding.seen.busy_input_prompt` latches it. Delete that key to see the tip again.
:::

### Призупинення у фон

У системах Unix натисни **`Ctrl+Z`**, щоб призупинити Hermes у фон — так само, як будь‑який процес у терміналі. Оболонка виводить підтвердження:

```
Hermes Agent has been suspended. Run `fg` to bring Hermes Agent back.
```

Введи `fg` у своїй оболонці, щоб відновити сесію саме там, де ти зупинився. Це не підтримується у Windows.
## Відображення прогресу інструменту

CLI показує анімований зворотний зв’язок під час роботи агента:

**Анімація мислення** (під час API‑викликів):
```
  ◜ (｡•́︿•̀｡) pondering... (1.2s)
  ◠ (⊙_⊙) contemplating... (2.4s)
  ✧٩(ˊᗜˋ*)و✧ got it! (3.1s)
```

**Вивід виконання інструменту:**
```
  ┊ 💻 terminal `ls -la` (0.3s)
  ┊ 🔍 web_search (1.2s)
  ┊ 📄 web_extract (2.1s)
```

Перемикай режими відображення за допомогою `/verbose`: `off → new → all → verbose`. Цю команду також можна ввімкнути для платформ обміну повідомленнями — дивись [configuration](/user-guide/configuration#display-settings).

### Довжина попереднього перегляду інструменту

Ключ конфігурації `display.tool_preview_length` керує максимальною кількістю символів, що показуються у рядках попереднього перегляду викликів інструменту (наприклад, шляхи до файлів, команди терміналу). За замовчуванням `0`, що означає відсутність обмеження — показуються повні шляхи та команди.

```yaml
# ~/.hermes/config.yaml
display:
  tool_preview_length: 80   # Truncate tool previews to 80 chars (0 = no limit)
```

Це корисно в вузьких терміналах або коли аргументи інструменту містять дуже довгі шляхи до файлів.
## Управління сесіями

### Відновлення сесій

Коли ти виходиш із CLI‑сесії, виводиться команда відновлення:

```
Resume this session with:
  hermes --resume 20260225_143052_a1b2c3

Session:        20260225_143052_a1b2c3
Duration:       12m 34s
Messages:       28 (5 user, 18 tool calls)
```

Опції відновлення:

```bash
hermes --continue                          # Resume the most recent CLI session
hermes -c                                  # Short form
hermes -c "my project"                     # Resume a named session (latest in lineage)
hermes --resume 20260225_143052_a1b2c3     # Resume a specific session by ID
hermes --resume "refactoring auth"         # Resume by title
hermes -r 20260225_143052_a1b2c3           # Short form
```

Відновлення відновлює повну історію розмови з SQLite. Агент бачить усі попередні повідомлення, виклики інструментів і відповіді — так, ніби ти ніколи не залишав сесію.

Використовуй `/title My Session Name` у чаті, щоб назвати поточну сесію, або `hermes sessions rename <id> <title>` у командному рядку. Команда `hermes sessions list` дозволяє переглянути минулі сесії.

### Зберігання сесій

CLI‑сесії зберігаються у SQLite‑базі стану Hermes за шляхом `~/.hermes/state.db`. База містить:

- метадані сесії (ID, назва, мітки часу, лічильники токенів)
- історію повідомлень
- лінійку походження між стиснутими/відновленими сесіями
- індекси повнотекстового пошуку, які використовує `session_search`

Деякі адаптери обміну повідомленнями також зберігають файли транскриптів для кожної платформи поруч із базою даних, проте сам CLI відновлює сесію лише з SQLite‑сховища.

### Стиснення контексту

Довгі розмови автоматично підсумовуються при наближенні до меж контексту:

```yaml
# In ~/.hermes/config.yaml
compression:
  enabled: true
  threshold: 0.50    # Compress at 50% of context limit by default

# Summarization model configured under auxiliary:
auxiliary:
  compression:
    model: ""  # Leave empty to use the main chat model (default). Or pin a cheap fast model, e.g. "google/gemini-3-flash-preview".
```

Коли спрацьовує стиснення, середні ходи підсумовуються, тоді як перші 3 та останні 20 ходів завжди зберігаються.
## Фонові сесії

Запусти підказку в окремій **фоновой** сесії, продовжуючи використовувати CLI для іншої роботи:

⟦HOLD_17⟩

Hermes одразу підтверджує завдання і повертає тобі підказку:

⟦HOLD_18⟩

### Як це працює

Кожна підказка `/background` створює **повністю окрему сесію агента** у демон‑потоку:

- **Ізольована розмова** — фоновий агент не має знань про історію твоєї поточної сесії. Він отримує лише підказку, яку ти йому передаєш.
- **Ті ж налаштування** — фоновий агент успадковує твою модель, провайдера, набори інструментів, налаштування міркувань та запасну модель з поточної сесії.
- **Не блокує** — твоя **основна** сесія залишається повністю інтерактивною. Ти можеш спілкуватися, виконувати команди або навіть запускати інші фоновые завдання.
- **Кілька завдань** — можна запускати кілька фоновых завдань одночасно. Кожному присвоюється номерний ID.

### Результати

Коли фонове завдання завершується, результат з’являється у вигляді панелі у твоєму терміналі:

⟦HOLD_19⟩

Якщо завдання не вдається, замість цього буде показано сповіщення про помилку. Якщо `display.bell_on_complete` увімкнено у твоїй конфігурації, при завершенні завдання дзвонить сигнал терміналу.

### Приклади використання

- **Тривале дослідження** — `"/background research the latest developments in quantum error correction"` під час роботи над кодом
- **Обробка файлів** — `"/background analyze all Python files in this repo and list any security issues"` під час продовження розмови
- **Паралельні розслідування** — запуск кількох фоновых завдань для одночасного дослідження різних аспектів

:::info
Фонові сесії не з’являються в історії твоєї основної розмови. Це окремі сесії зі своїм ID завдання (наприклад, `bg_143022_a1b2c3`).
:::
## Тихий режим

За замовчуванням CLI працює в тихому режимі, який:
- Прибирає докладне журналювання інструментів
- Вмикає анімований зворотний зв’язок у стилі «кавайї»
- Забезпечує чистий та зручний для користувача вивід

Для виведення налагоджувальної інформації:
```bash
hermes chat --verbose
```
