# TUI

TUI — це сучасний фронтенд для Hermes, який працює в терміналі та використовує той самий Python‑runtime, що й [Classic CLI](cli.md). Той самий агент, ті самі сесії, ті самі slash‑команди; чистіший, більш чуйний інтерфейс для взаємодії з ними.

Це рекомендований спосіб запуску Hermes у інтерактивному режимі.
## Запуск

```bash
# Launch the TUI
hermes --tui

# Resume the latest TUI session (falls back to the latest classic session)
hermes --tui -c
hermes --tui --continue

# Resume a specific session by ID or title
hermes --tui -r 20260409_000000_aa11bb
hermes --tui --resume "my t0p session"

# Run source directly — skips the prebuild step (for TUI contributors)
hermes --tui --dev
```

Ти також можеш увімкнути його через змінну середовища:

```bash
export HERMES_TUI=1
hermes          # now uses the TUI
hermes chat     # same
```

Класичний CLI залишається доступним за замовчуванням. Все, що задокументовано в [CLI Interface](cli.md) — slash‑команди, швидкі команди, попереднє завантаження skill, персональності, багаторядковий ввід, переривання — працює в TUI ідентично.
## Чому TUI

- **Instant first frame** — банер малюється ще до завершення завантаження програми, тому термінал ніколи не здається «завислим», коли Hermes запускається.
- **Non-blocking input** — можна вводити та ставити в чергу повідомлення ще до того, як сесія готова. Твій перший запит надсилається в момент, коли агент підключається.
- **Rich overlays** — вибір моделі, вибір сесії, запити на підтвердження та уточнення відображаються у вигляді модальних панелей, а не вбудованих потоків.
- **Live session panel** — інструменти та навички заповнюються поступово під час ініціалізації.
- **Mouse-friendly selection** — перетягування для виділення з уніфікованим фоном замість SGR‑інверсії. Копіюй за допомогою звичного жесту копіювання у твоєму терміналі.
- **Alternate-screen rendering** — диференційовані оновлення запобігають мерехтінню під час трансляції, не залишають сміття в історії після виходу.
- **Composer affordances** — вбудоване згортання вставок для довгих фрагментів, `Cmd+V` / `Ctrl+V` вставка тексту з резервом для зображень у буфері, безпечне вставлення у дужках, а також нормалізація прикріплення зображень/шляхів до файлів.

Ті ж [шкірси](features/skins.md) і [особистості](features/personality.md) застосовуються. Перемикай їх під час сесії за допомогою `/skin ares`, `/personality pirate`, і інтерфейс перефарбовується в реальному часі. Дивись [Skins & Themes](features/skins.md) для повного списку налаштовуваних клавіш і того, які з них діють у класичному режимі, а які у TUI — TUI дотримується палітри банера, кольорів UI, гліфа/кольору запиту, відображення сесії, меню автодоповнення, фону виділення, `tool_prefix` та `help_header`.

### Розкривні розділи банера

Банер запуску TUI групує інформацію про середовище у чотири розкривні розділи, кожен з яких відображається зі стрілкою `▸` / `▾` поруч із назвою розділу:

| Розділ | Типовий стан |
|--------|--------------|
| Tools | Відкрито |
| Skills | Згорнуто |
| System Prompt | Згорнуто |
| MCP Servers | Згорнуто |

Клацни будь-де на заголовку розділу (або на його стрілці), щоб перемкнути його. Список **Tools** відкритий за замовчуванням, бо це найчастіше перевірений розділ на старті сесії; **Skills**, **System Prompt** і **MCP Servers** згортаються за замовчуванням, щоб банер залишався компактним, навіть якщо ти встановив десятки навичок або підключив багато серверів MCP. Стан зберігається локально для конкретного екземпляра банера, тому наступний запуск скидає його до типових значень.
## Вимоги

- **Node.js** ≥ 20 — TUI працює як підпроцес, запущений з Python CLI. `hermes doctor` перевіряє це.
- **TTY** — як і у класичному CLI, передача stdin або запуск у неінтерактивному середовищі переходить у режим одного запиту.

При першому запуску Hermes встановлює Node‑залежності TUI у `ui-tui/node_modules` (одноразово, кілька секунд). Подальші запуски проходять швидко. Якщо ти оновиш Hermes до нової версії, bundle TUI буде перебудовано автоматично, коли джерела новіші за `dist`.

### Зовнішній попередньо зібраний bundle

Дистрибутиви, які постачають попередньо зібраний bundle (Nix, системні пакети), можуть вказати Hermes на нього:

```bash
export HERMES_TUI_DIR=/path/to/prebuilt/ui-tui
hermes --tui
```

Каталог має містити `dist/entry.js`.
## Комбінації клавіш

Комбінації клавіш відповідають [Classic CLI](cli.md#keybindings) точно. Єдині відмінності в поведінці:

- **Mouse drag** підсвічує текст рівномірним фоном виділення.
- **`Cmd+V` / `Ctrl+V`** спочатку намагається вставити звичайний текст, потім переходить до читання буфера обміну через OSC52/нативний буфер, і врешті‑решт додає зображення, якщо буфер або вставлений вміст є зображенням.
- **`/terminal-setup`** встановлює локальні прив’язки терміналу VS Code / Cursor / Windsurf для кращого `Cmd+Enter` та паритету скасування/повернення на macOS.
- **Slash autocompletion** відкривається у вигляді плаваючої панелі з описами, а не як вбудований випадаючий список.
- **`Ctrl+X`** відкриває перемикач живих сесій. Коли виділено повідомлення в черзі (надіслане, поки агент ще працював), воно все одно видаляє це повідомлення в черзі. **`Esc`** скасовує редагування та знімає виділення без видалення.
- **`Ctrl+G` / `Ctrl+X Ctrl+E`** — відкрити поточний буфер вводу у `$EDITOR` для багаторядкового / довгого складання запиту; збереження і вихід надсилає вміст назад як запит.
## Slash commands

Усі slash‑команди працюють без змін. Декілька з них належать TUI — вони виводять більш насичений результат або відображаються як накладки замість вбудованих панелей:

| Command | TUI behavior |
|---------|--------------|
| `/help` | Накладка з категоризованими командами, навігація клавішами‑стрілками |
| `/sessions` (alias `/switch`) | Живий перемикач сесій — список відкритих TUI‑сесій, перемикання між ними, закриття або запуск нової |
| `/model` | Модальне вікно вибору моделі, згруповане за провайдером, з підказками вартості |
| `/skin` | Живий попередній перегляд — зміна теми застосовується під час перегляду |
| `/details` | Перемикач детального виводу викликів інструментів (глобально або для окремого розділу) |
| `/usage` | Насичена панель токенів / вартості / контексту |
| `/agents` (alias `/tasks`) | Накладка спостереження — живе дерево підагентів з керуванням kill/pause, підсумковою вартістю / токенами / файлами для кожної гілки, історією крок за кроком |
| `/reload` | Перечитує `~/.hermes/.env` у запущений процес TUI, щоб нові API‑ключі набули сили без перезапуску |
| `/mouse [on\|off\|toggle\|wheel\|buttons\|all]` | Вибір пресету відстеження миші під час роботи (також зберігається в `display.mouse_tracking` у `config.yaml`). `wheel` (1000+1006) зберігає прокручування колеса без подій hover, які змушують tmux спамити «No image in clipboard» над рядком підказки; `buttons` додає перетягування для вибору; `all` — типове налаштування з UI, керованим hover. |

Усі інші slash‑команди (включаючи встановлені skills, швидкі команди та перемикачі персональностей) працюють ідентично класичному CLI. Дивись [Slash Commands Reference](../reference/slash-commands.md).
## Перемикач живих сесій

Використовуй перемикач живих сесій, коли потрібно, щоб один термінал діяв як диспетчер для кількох TUI‑сесій. Він показує лише сесії, які зараз активні в цьому процесі TUI; закриті сесії залишаються збереженими транскриптами і їх все ще можна відкрити за допомогою `/resume` або `hermes --tui --resume <id-or-title>`.

Відкрий його будь‑яким із цих способів:

- `Ctrl+X` у TUI.
- `/sessions` або `/switch`.
- `/sessions new` — створити нову живу сесію одразу.
- Клацни на кількість `N live sessions` у рядку стану.

<video controls muted loop playsInline src="/img/docs/tui-session-orchestrator/session-orchestrator-demo.mp4" title="Hermes TUI Session Orchestrator demo" />

Всередині перемикача:

- `↑` / `↓` переміщують вибір; кліки миші також вибирають рядки.
- `Enter` перемикає на вибрану живу сесію.
- `Ctrl+D` закриває вибрану живу сесію.
- `Ctrl+N` запускає порожню живу сесію.
- `Ctrl+R` оновлює список живих сесій.
- `Esc` закриває перемикач.
- Вибери `+new`, введи підказку та натисни `Enter`, щоб диспетчеризувати нову живу сесію. Спочатку натисни `Tab`, якщо хочеш вибрати модель лише для цієї нової сесії.
## Відображення LaTeX‑математики

Текстовий інтерфейс (TUI) у своєму markdown‑конвеєрі рендерить LaTeX‑математику inline: `$E = mc^2$` і `$$\frac{a}{b}$$` відображаються як формули у форматі Unicode замість сирого TeX‑коду. Працює як для inline, так і для block‑математики; неподтримуваний синтаксис повертається у вигляді буквального TeX, обгорнутого в кодовий спан, щоб його можна було скопіювати.

Це завжди ввімкнено — нічого не потрібно налаштовувати. Класичний CLI залишає сирий TeX.
## Виявлення світлих терміналів

TUI автоматично виявляє світлі термінали та перемикає тему на світлу відповідно. Виявлення працює у трьох шарах:

1. `HERMES_TUI_THEME` env var — найвищий пріоритет. Значення: `light`, `dark` або сирий 6‑символьний hex кольору фону (наприклад `ffffff`, `1a1a2e`).
2. `COLORFGBG` env var — класична підказка «який у мене колір фону?», що використовується терміналами, похідними від xterm.
3. Запит фону терміналу через OSC 11 — працює у сучасних терміналах (Ghostty, Warp, iTerm2, WezTerm, Kitty), які не встановлюють `COLORFGBG`.

Якщо ти хочеш постійно використовувати світлу тему незалежно від терміналу:

```bash
export HERMES_TUI_THEME=light
```
## Стилі індикатора зайнятості

Індикатор зайнятості в рядку стану підключається як плагін — за замовчуванням він обертає kawaii‑обличчя Hermes кожні 2,5 секунди під час роботи агента. Вибери інший стиль за допомогою конфігурації або slash‑команди `/indicator`:

```yaml
display:
  tui_status_indicator: kaomoji   # kaomoji | emoji | unicode | ascii
```

Або під час сесії: `/indicator emoji` (тощо). Стилі постачаються з відповідною шириною гліфів, тому решта рядка стану не «дрижить» під час обертання.
## Автоматичне відновлення

За замовчуванням `hermes --tui` запускає нову сесію при кожному запуску. Щоб автоматично повторно підключитися до останньої TUI‑сесії (корисно, коли твій термінал або SSH‑з’єднання несподівано розривається), увімкни цю опцію:

```bash
export HERMES_TUI_RESUME=1          # most-recent TUI session
# or:
export HERMES_TUI_RESUME=<session-id>   # specific session
```

Скинь змінну або явно передай `--resume <id>`, щоб перевизначити її для окремого запуску.
## Рядок стану

Рядок стану TUI відстежує стан агента в режимі реального часу:

| Статус | Значення |
|--------|----------|
| `starting agent…` | Ідентифікатор сесії активний; інструменти та навички ще підключаються. Ти можеш вводити — повідомлення ставляться в чергу і надсилаються, коли готові. |
| `ready` | Агент бездіяльний, приймає ввід. |
| `thinking…` / `running…` | Агент міркує або виконує інструмент. |
| `interrupted` | Поточний хід скасовано; натисни Enter, щоб надіслати знову. |
| `forging session…` / `resuming…` | Початкове підключення або рукостискання `--resume`. |

Кольори та пороги рядка стану для кожної теми спільні з класичним CLI — дивись [Skins](features/skins.md) для налаштувань.

Рядок стану також показує:

- **Робоча директорія з гілкою git** — `~/projects/hermes-agent (docs/two-week-gap-sweep)`. Суфікс гілки оновлюється, коли ти виконуєш `git checkout` у боковому терміналі (mtime‑кешовано), тож TUI відображає твою фактичну активну гілку, а не ту, що була під час запуску.
- **Час, що минув з моменту prompt** — `⏱ 12s/3m 45s` під час виконання ходу (у реальному часі), заморожується до `⏲ 32s / 3m 45s` після завершення ходу. Перше число — час з останнього повідомлення користувача; друге — загальна тривалість сесії. Скидається при кожному новому prompt.
- **`🗜️ N`** — кількість автоматичних стискань поточної сесії. Показується, коли спрацьовує перше стискання.
- **`▶ N`** — кількість задач `/background`, що зараз виконуються в цій сесії. Показується, коли хоча б одна задача запущена.
- **`⚠ YOLO`** — видимий попереджувальний сигнал, коли режим YOLO увімкнено (`hermes --yolo`, `/yolo` або `HERMES_YOLO_MODE=1`). Така ж іконка також з’являється у стартовому банері, тому ти не можеш запустити сесію з автоматичним схваленням, не помітивши цього.
## Конфігурація

TUI дотримується всіх стандартних налаштувань Hermes: `~/.hermes/config.yaml`, профілі, персональності, скіни, швидкі команди, пул облікових даних, провайдери пам'яті, ввімкнення інструментів/навичок. Файлу конфігурації, специфічного для TUI, не існує.

Невелика кількість ключів налаштовує саме інтерфейс TUI:

```yaml
display:
  skin: default              # any built-in or custom skin
  personality: helpful
  details_mode: collapsed    # hidden | collapsed | expanded — global accordion default
  sections:                  # optional: per-section overrides (any subset)
    thinking: expanded       # always open
    tools: expanded          # always open
    activity: collapsed      # opt back IN to the activity panel (hidden by default)
  mouse_tracking: all        # off | wheel | buttons | all (or true/false for back-compat).
                             #   wheel   — 1000+1006 (scroll + click; no drag, no hover —
                             #             recommended inside tmux to silence the prompt-row
                             #             "No image in clipboard" spam from hover events)
                             #   buttons — adds 1002 for terminal-side drag selection
                             #   all     — adds 1003 for hover (scrollbar paginate-on-hover,
                             #             link mouseenter, etc.)
```

Перемикачі під час виконання:

- `/details [hidden|collapsed|expanded|cycle]` — встановити глобальний режим
- `/details <section> [hidden|collapsed|expanded|reset]` — перевизначити один розділ
  (розділи: `thinking`, `tools`, `subagents`, `activity`)

**Видимість за замовчуванням**

TUI постачається з продуманими значеннями за замовчуванням для кожного розділу, які транслюють хід у вигляді живої транскрипції, а не стіни шевронів:

- `thinking` — **expanded**. Потік міркувань виводиться вбудовано, коли модель його генерує.
- `tools` — **expanded**. Виклики інструментів та їх результати відображаються відкритими.
- `subagents` — успадковує глобальний `details_mode` (за замовчуванням **collapsed** під шевроном — залишається тихим, доки не відбудеться делегування).
- `activity` — **hidden**. Додаткова мета‑інформація (підказки шлюзу, підказки терміналу, фонові сповіщення) зайва для більшості щоденного використання. Помилки інструментів все ще виводяться вбудовано у рядку з помилковим інструментом; ambient‑помилки/попередження з’являються у вигляді плаваючого сповіщення, коли всі панелі приховані.

Перевизначення на рівні розділу мають пріоритет над значенням за замовчуванням розділу та глобальним `details_mode`. Щоб змінити розташування:

- `display.sections.thinking: collapsed` — повернути мислення під шеврон
- `display.sections.tools: collapsed` — повернути виклики інструментів під шеврон
- `display.sections.activity: collapsed` — включити панель активності
- `/details <section> <mode>` під час виконання

Будь‑яке явне встановлення в `display.sections` переважає над значеннями за замовчуванням, тому існуючі конфігурації продовжують працювати без змін.
## Сесії

Сесії спільні між TUI та класичним CLI — обидва записують у той самий `~/.hermes/state.db`. Ти можеш розпочати сесію в одному інтерфейсі, а продовжити в іншому. Вибірник сесій показує сесії з обох джерел, з міткою джерела.

Дивись [Sessions](sessions.md) для інформації про життєвий цикл, пошук, стискання та експорт.
## Приєднання до запущеного шлюзу

За замовчуванням TUI створює власний шлюз у процесі, тому кожен екземпляр TUI є самодостатнім. Якщо у тебе вже працює довготривалий шлюз (наприклад `hermes gateway run` у tmux або сервіс systemd / launchd), ти можеш вказати TUI підключитися до цього шлюзу — тоді TUI стає тонким клієнтом і ділиться станом з усіма іншими поверхнями (платформи обміну повідомленнями, веб‑дашборд, інші сесії TUI), які підключені до того самого шлюзу.

Встанови URL веб‑сокету через змінну середовища перед запуском:

```bash
export HERMES_TUI_GATEWAY_URL="ws://localhost:8765/api/ws?token=<auth-token>"
hermes --tui
```

Токен береться з конфігурації автентифікації API шлюзу (див. [API Server](features/api-server.md)). Коли змінна середовища встановлена, TUI:

- Повністю пропускає створення локального шлюзу — без дублювання адаптерів платформи, без конфліктів портів.
- Маршрутизує кожну дію (slash‑команди, прикріплення зображень, прогрес браузера, голосові події, …) через веб‑сокет до спільного шлюзу.
- Автоматично перепідключається, якщо URL шлюзу змінюється (новий токен) між запитами.

Це той самий канал, який використовує вбудований у веб‑дашборд TUI (див. [Web Dashboard](features/web-dashboard.md#chat)) — один шлюз, багато клієнтів.
## Повернення до класичного CLI

Запуск `hermes` (без `--tui`) залишає класичний CLI. Щоб змусити машину використовувати TUI, встанови `HERMES_TUI=1` у профілі оболонки. Щоб повернутися назад, скинь цю змінну середовища.

Якщо TUI не вдається запустити (відсутній Node, бракує пакету, проблема з TTY), Hermes виводить діагностичне повідомлення і переходить у запасний (фолбек) режим — замість того, щоб залишити тебе в стані зависання.
