# Налаштування Email

Hermes може отримувати та відповідати на листи, використовуючи стандартні протоколи IMAP та SMTP. Надішли лист на адресу агента, і він відповість у темі — без спеціального клієнта чи API бота. Працює з Gmail, Outlook, Yahoo, Fastmail або будь‑яким провайдером, що підтримує IMAP/SMTP.

:::info Gateway adapter only: no external dependencies
Ця сторінка охоплює адаптер шлюзу Email, який використовує вбудовані Python‑модулі `imaplib`, `smtplib` та `email`. Додаткові пакети чи зовнішні сервіси не потрібні для цього шляху шлюзу.
:::

Це окремо від вбудованого [Himalaya email skill](/docs/user-guide/skills/bundled/email/email-himalaya), який дозволяє агенту керувати поштою через термінальні команди та вимагає зовнішнього CLI `himalaya` і конфігураційного файлу `~/.config/himalaya/config.toml`.

| Випадок використання | Що налаштовувати | Зовнішня залежність |
|---|---|---|
| Дозволити людям надсилати листи агенту Hermes і отримувати відповіді | Адаптер шлюзу Email на цій сторінці | Ні, окрім облікового запису IMAP/SMTP |
| Дозволити агенту переглядати, створювати, переміщати та керувати повідомленнями поштової скриньки за допомогою термінальних інструментів | Himalaya email skill | CLI `himalaya` та `~/.config/himalaya/config.toml` |

## Передумови

- **Окремий обліковий запис електронної пошти** для твого агента Hermes (не використовуйте особисту пошту)
- **IMAP увімкнено** в обліковому записі
- **Пароль додатку**, якщо використовуєш Gmail або інший провайдер з 2FA

### Налаштування Gmail

1. Увімкни двофакторну автентифікацію в Google Account
2. Перейди до [App Passwords](https://myaccount.google.com/apppasswords)
3. Створи новий пароль додатку (вибери «Mail» або «Other»)
4. Скопіюй 16‑символьний пароль — він буде використовуватись замість звичайного пароля

### Outlook / Microsoft 365

1. Перейди до [Security Settings](https://account.microsoft.com/security)
2. Увімкни 2FA, якщо ще не активовано
3. Створи пароль додатку у розділі «Additional security options»
4. IMAP‑хост: `outlook.office365.com`, SMTP‑хост: `smtp.office365.com`

### Інші провайдери

Більшість провайдерів підтримують IMAP/SMTP. Перевір документацію провайдера щодо:
- IMAP‑хосту та порту (зазвичай порт 993 з SSL)
- SMTP‑хосту та порту (зазвичай порт 587 з STARTTLS)
- Чи потрібні паролі додатків

## Крок 1: Налаштування Hermes

Найпростіший спосіб:

```bash
hermes gateway setup
```

Вибери **Email** у меню платформи. Майстер запросить твою електронну адресу, пароль, хости IMAP/SMTP та дозволені відправники.

### Ручна конфігурація

Додай до `~/.hermes/.env`:

```bash
# Required
EMAIL_ADDRESS=hermes@gmail.com
EMAIL_PASSWORD=abcd efgh ijkl mnop    # App password (not your regular password)
EMAIL_IMAP_HOST=imap.gmail.com
EMAIL_SMTP_HOST=smtp.gmail.com

# Security (recommended)
EMAIL_ALLOWED_USERS=your@email.com,colleague@work.com

# Optional
EMAIL_IMAP_PORT=993                    # Default: 993 (IMAP SSL)
EMAIL_SMTP_PORT=587                    # Default: 587 (SMTP STARTTLS)
EMAIL_POLL_INTERVAL=15                 # Seconds between inbox checks (default: 15)
EMAIL_HOME_ADDRESS=your@email.com      # Default delivery target for cron jobs
```

---

## Крок 2: Запуск шлюзу

```bash
hermes gateway              # Run in foreground
hermes gateway install      # Install as a user service
sudo hermes gateway install --system   # Linux only: boot-time system service
```

При старті адаптер:
1. Перевіряє з’єднання IMAP і SMTP
2. Позначає всі існуючі повідомлення вхідної скриньки як «прочитані» (обробляє лише нові листи)
3. Починає опитування нових повідомлень

## Як це працює

### Отримання повідомлень

Адаптер опитує IMAP‑скриньку на предмет **UNSEEN**‑повідомлень з налаштовуваним інтервалом (за замовчуванням — 15 секунд). Для кожного нового листа:

- **Тема** включається в контекст (наприклад, `[Subject: Deploy to production]`)
- **Відповіді** (тема починається з `Re:`) пропускають префікс теми — контекст вже встановлений
- **Вкладення** кешуються локально:
  - Зображення (JPEG, PNG, GIF, WebP) → доступні інструменту зору
  - Документи (PDF, ZIP тощо) → доступні для доступу до файлів
- **Листи лише в HTML** очищуються від тегів для отримання чистого тексту
- **Самостійні повідомлення** фільтруються, щоб уникнути циклів відповіді
- **Автоматичні/недоступні відправники** ігноруються без повідомлення — `noreply@`, `mailer-daemon@`, `bounce@`, `no-reply@` та листи з заголовками `Auto-Submitted`, `Precedence: bulk` або `List-Unsubscribe`

### Надсилання відповідей

Відповіді надсилаються через SMTP з правильним формуванням теми:

- Заголовки **In-Reply-To** і **References** зберігають структуру теми
- **Тема** зберігається з префіксом `Re:` (без подвійного `Re: Re:`)
- **Message-ID** генерується з доменом агента
- Відповіді надсилаються у вигляді простого тексту (UTF‑8)

### Вкладення файлів

Агент може надсилати файли у відповідях. Додай `MEDIA:/path/to/file` у відповідь, і файл буде прикріплений до вихідного листа.

### Пропуск вкладень

Щоб ігнорувати всі вхідні вкладення (з міркувань безпеки або економії трафіку), додай у `config.yaml`:

```yaml
platforms:
  email:
    skip_attachments: true
```

Коли увімкнено, вкладення та вбудовані частини пропускаються до декодування корисного навантаження. Текст листа все одно обробляється звичайно.

## Контроль доступу

Доступ до Email працює за тим же принципом, що й інші платформи Hermes:

1. **`EMAIL_ALLOWED_USERS` встановлено** → обробляються лише листи з цих адрес
2. **Не встановлено allowlist** → невідомі відправники отримують код парування
3. **`EMAIL_ALLOW_ALL_USERS=true`** → приймаються листи від будь‑кого (використовуй обережно)

:::warning
**Завжди налаштовуй `EMAIL_ALLOWED_USERS`.** Без цього будь‑хто, хто знає адресу агента, може надсилати команди. Агент має доступ до терміналу за замовчуванням.
:::

## Устранення проблем

| Проблема | Рішення |
|---------|--------|
| **"IMAP connection failed"** під час старту | Перевір `EMAIL_IMAP_HOST` і `EMAIL_IMAP_PORT`. Переконайся, що IMAP увімкнено в обліковому записі. Для Gmail увімкни його в Settings → Forwarding and POP/IMAP. |
| **"SMTP connection failed"** під час старту | Перевір `EMAIL_SMTP_HOST` і `EMAIL_SMTP_PORT`. Переконайся, що пароль правильний (використовуй App Password для Gmail). |
| **Листи не надходять** | Переконайся, що `EMAIL_ALLOWED_USERS` містить адресу відправника. Перевір папку спаму — деякі провайдери позначають автоматичні відповіді як спам. |
| **"Authentication failed"** | Для Gmail треба використовувати App Password, а не звичайний пароль. Спершу увімкни 2FA. |
| **Дублюються відповіді** | Переконайся, що запущений лише один екземпляр шлюзу. Перевір `hermes gateway status`. |
| **Повільна реакція** | Інтервал опитування за замовчуванням — 15 секунд. Зменш його, встановивши `EMAIL_POLL_INTERVAL=5` для швидшої відповіді (але більше з’єднань IMAP). |
| **Відповіді не групуються в темі** | Адаптер використовує заголовки In-Reply-To. Деякі клієнти (особливо веб‑версії) можуть не групувати автоматичні листи правильно. |

## Безпека

:::warning
**Використовуй окремий обліковий запис електронної пошти.** Не використовуйте особисту пошту — агент зберігає пароль у `.env` і має повний доступ до скриньки через IMAP.
:::

- Використовуй **App Passwords** замість основного пароля (обов’язково для Gmail з 2FA)
- Встанови `EMAIL_ALLOWED_USERS`, щоб обмежити, хто може взаємодіяти з агентом
- Пароль зберігається у `~/.hermes/.env` — захисти цей файл (`chmod 600`)
- IMAP використовує SSL (порт 993), а SMTP — STARTTLS (порт 587) за замовчуванням — з’єднання зашифровані

| Змінна | Обов’язкова | За замовчуванням | Опис |
|----------|--------------|-------------------|------|
| `EMAIL_ADDRESS` | Так | — | Адреса електронної пошти агента |
| `EMAIL_PASSWORD` | Так | — | Пароль або пароль додатку |
| `EMAIL_IMAP_HOST` | Так | — | Хост IMAP‑сервера (наприклад, `imap.gmail.com`) |
| `EMAIL_SMTP_HOST` | Так | — | Хост SMTP‑сервера (наприклад, `smtp.gmail.com`) |
| `EMAIL_IMAP_PORT` | Ні | `993` | Порт IMAP‑сервера |
| `EMAIL_SMTP_PORT` | Ні | `587` | Порт SMTP‑сервера |
| `EMAIL_POLL_INTERVAL` | Ні | `15` | Кількість секунд між перевірками скриньки |
| `EMAIL_ALLOWED_USERS` | Ні | — | Список дозволених адрес відправників, розділених комами |
| `EMAIL_HOME_ADDRESS` | Ні | — | Типова адреса доставки для cron‑завдань |
| `EMAIL_ALLOW_ALL_USERS` | Ні | `false` | Дозволити всіх відправників (не рекомендовано) |
