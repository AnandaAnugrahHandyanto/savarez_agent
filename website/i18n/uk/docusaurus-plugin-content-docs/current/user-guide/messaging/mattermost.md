# Налаштування Matterprompt

Hermes Agent інтегрується з Mattermost як бот, дозволяючи спілкуватися зі своїм AI‑асистентом через прямі повідомлення або канали команди. Mattermost — це самохостингова, open‑source альтернатива Slack, яку ти розгортаєш на власній інфраструктурі, зберігаючи повний контроль над даними. Бот підключається через REST API Mattermost (v4) та WebSocket для подій у реальному часі, обробляє повідомлення через конвеєр Hermes Agent (включаючи використання інструментів, пам'ять та міркування) і відповідає миттєво. Підтримуються текст, вкладення файлів, зображення та slash‑команди.

Зовнішня бібліотека Mattermost не потрібна — адаптер використовує `aiohttp`, який вже є залежністю Hermes.

Перед налаштуванням, ось те, що більшість людей хоче знати: як Hermes поводиться, коли він вже запущений у твоєму екземплярі Mattermost.
## Як поводиться Hermes

| Context | Behavior |
|---------|----------|
| **DMs** | Hermes відповідає на кожне повідомлення. Не потрібне `@mention`. Кожен DM має свою сесію. |
| **Public/private channels** | Hermes відповідає, коли ти `@mention` його. Без згадки Hermes ігнорує повідомлення. |
| **Threads** | Якщо `MATTERMOST_REPLY_MODE=thread`, Hermes відповідає в темі під твоїм повідомленням. Контекст теми залишається ізольованим від батьківського каналу. |
| **Shared channels with multiple users** | За замовчуванням Hermes ізолює історію сесії для кожного користувача в каналі. Двоє людей, які спілкуються в одному каналі, не діляться одним транскриптом, якщо ти явно не вимкнеш це. |

:::tip
Якщо ти хочеш, щоб Hermes відповідав у вигляді вкладених розмов (під твоїм оригінальним повідомленням), встанови `MATTERMOST_REPLY_MODE=thread`. За замовчуванням — `off`, що надсилає плоскі повідомлення в канал.
:::

### Модель сесії в Mattermost

За замовчуванням:

- кожен DM отримує свою сесію
- кожна тема отримує власний простір імен сесії
- кожен користувач у спільному каналі отримує свою сесію всередині цього каналу

Це контролюється у `config.yaml`:

```yaml
group_sessions_per_user: true
```

Встанови `false` лише якщо ти явно хочеш одну спільну розмову для всього каналу:

```yaml
group_sessions_per_user: false
```

Спільні сесії можуть бути корисними для колаборативного каналу, але вони також означають:

- користувачі діляться ростом контексту та витратами токенів
- довге інструментальне завдання однієї людини може збільшити контекст усіх інших
- запущений процес однієї людини може перервати подальший запит іншої людини в тому ж каналі

Цей посібник проведе тебе через повний процес налаштування — від створення бота в Mattermost до надсилання першого повідомлення.
## Крок 1: Увімкнути облікові записи ботів

Облікові записи ботів мають бути увімкнені на вашому сервері Mattermost, перш ніж ви зможете їх створити.

1. Увійди в Mattermost як **System Admin**.
2. Перейди до **System Console** → **Integrations** → **Bot Accounts**.
3. Встанови **Enable Bot Account Creation** у **true**.
4. Натисни **Save**.

:::info
Якщо у тебе немає доступу **System Admin**, попроси адміністратора Mattermost увімкнути облікові записи ботів і створити їх для тебе.
:::
## Крок 2: Створити обліковий запис бота

1. У Mattermost натисни меню **☰** (зверху‑зліва) → **Integrations** → **Bot Accounts**.
2. Натисни **Add Bot Account**.
3. Заповни дані:
   - **Username**: напр., `hermes`
   - **Display Name**: напр., `Hermes Agent`
   - **Description**: за бажанням
   - **Role**: `Member` достатньо
4. Натисни **Create Bot Account**.
5. Mattermost покаже **bot token**. **Скопіюй його одразу**.

:::warning[Token shown only once]
Токен бота відображається лише один раз під час створення облікового запису бота. Якщо ти його втратиш, доведеться регенерувати його в налаштуваннях облікового запису бота. Ніколи не публікуй токен публічно і не додавай його до Git — будь‑хто, хто має цей токен, отримує повний контроль над ботом.
:::

Збережи токен у безпечному місці (наприклад, у менеджері паролів). Тобі знадобиться він у Кроці 5.

:::tip
Ти також можеш використати **personal access token** замість облікового запису бота. Перейди до **Profile** → **Security** → **Personal Access Tokens** → **Create Token**. Це корисно, якщо хочеш, щоб Hermes публікував від імені твого власного користувача, а не окремого бота.
:::
## Крок 3: Додати бота до каналів

Бот має бути учасником будь‑якого каналу, у якому ти хочеш, щоб він відповідав:

1. Відкрий канал, у якому хочеш додати бота.
2. Клацни назву каналу → **Add Members**.
3. Знайди ім’я користувача свого бота (наприклад, `hermes`) і додай його.

Для приватних повідомлень просто відкрий пряме повідомлення з ботом — він зможе відповісти одразу.
## Крок 4: Знайди свій Mattermost User ID

Hermes Agent використовує твій Mattermost User ID, щоб контролювати, хто може взаємодіяти з ботом. Щоб знайти його:

1. Клацни на свою **аватар** (у верхньому лівому куті) → **Профіль**.
2. Твій User ID відображається у діалоговому вікні профілю — клацни його, щоб скопіювати.

Твій User ID — це 26‑символьний алфавітно‑цифровий рядок, наприклад `3uo8dkh1p7g1mfk49ear5fzs5c`.

:::warning
Твій User ID **не** є твоїм ім’ям користувача. Ім’я користувача — це те, що показується після `@` (наприклад, `@alice`). User ID — це довгий алфавітно‑цифровий ідентифікатор, який Mattermost використовує внутрішньо.
:::

**Альтернатива**: Ти також можеш отримати свій User ID через API:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-mattermost-server/api/v4/users/me | jq .id
```

:::tip
Щоб отримати **Channel ID**: клацни назву каналу → **Переглянути інформацію**. Channel ID показується у панелі інформації. Тобі це знадобиться, якщо хочеш вручну встановити домашній канал.
:::
## Крок 5: Налаштування Hermes Agent

### Варіант A: Інтерактивне налаштування (Рекомендовано)

Запусти команду інтерактивного налаштування:

```bash
hermes gateway setup
```

Вибери **Mattermost**, коли буде запит, потім встав URL сервера, токен бота та ID користувача, коли буде запитано.

### Варіант B: Ручна конфігурація

Додай наступне у файл `~/.hermes/.env`:

```bash
# Required
MATTERMOST_URL=https://mm.example.com
MATTERMOST_TOKEN=***
MATTERMOST_ALLOWED_USERS=3uo8dkh1p7g1mfk49ear5fzs5c

# Multiple allowed users (comma-separated)
# MATTERMOST_ALLOWED_USERS=3uo8dkh1p7g1mfk49ear5fzs5c,8fk2jd9s0a7bncm1xqw4tp6r3e

# Optional: reply mode (thread or off, default: off)
# MATTERMOST_REPLY_MODE=thread

# Optional: respond without @mention (default: true = require mention)
# MATTERMOST_REQUIRE_MENTION=false

# Optional: channels where bot responds without @mention (comma-separated channel IDs)
# MATTERMOST_FREE_RESPONSE_CHANNELS=channel_id_1,channel_id_2
```

Опціональні налаштування поведінки у `~/.hermes/config.yaml`:

```yaml
group_sessions_per_user: true
```

- `group_sessions_per_user: true` зберігає контекст кожного учасника ізольовано в межах спільних каналів і тем

### Запуск шлюзу

Після налаштування запусти шлюз Mattermost:

```bash
hermes gateway
```

Бот має підключитися до твого сервера Mattermost протягом кількох секунд. Надішли йому повідомлення — будь то DM або в канал, куди його додали — щоб перевірити.

:::tip
Ти можеш запустити `hermes gateway` у фоні або як службу systemd для постійної роботи. Дивись документацію з розгортання для деталей.
:::
## Канал «домашній»

Ти можеш призначити «домашній канал», куди бот надсилатиме проактивні повідомлення (наприклад, виведення cron‑завдань, нагадування та сповіщення). Є два способи його встановити:

### За допомогою слеш‑команди

Введи `/sethome` у будь‑якому каналі Mattermost, де присутній бот. Цей канал стане домашнім каналом.

### Ручна конфігурація

Додай це у свій `~/.hermes/.env`:

```bash
MATTERMOST_HOME_CHANNEL=abc123def456ghi789jkl012mn
```

Замініть ID на фактичний ідентифікатор каналу (клацни назву каналу → View Info → скопіюй ID).
## Режим відповіді

Налаштування `MATTERMOST_REPLY_MODE` керує тим, як Hermes публікує відповіді:

| Режим | Поведінка |
|------|----------|
| `off` (за замовчуванням) | Hermes надсилає прості повідомлення в канал, як звичайний користувач. |
| `thread` | Hermes відповідає у **треді** під твоїм оригінальним повідомленням. Зберігає канали чистими, коли багато обміну. |

Встанови його у своєму `~/.hermes/.env`:

```bash
MATTERMOST_REPLY_MODE=thread
```
## Поведінка згадок

За замовчуванням бот відповідає у каналах лише коли його `@mentioned`. Це можна змінити:

| Змінна | За замовчуванням | Опис |
|----------|-------------------|------|
| `MATTERMOST_REQUIRE_MENTION` | `true` | Встановити `false`, щоб відповідати на всі повідомлення в каналах (DM завжди працюють). |
| `MATTERMOST_FREE_RESPONSE_CHANNELS` | _(none)_ | Список ID каналів, розділених комами, у яких бот відповідає без `@mention`, навіть коли `require_mention` встановлено в `true`. |

Щоб знайти ID каналу в Mattermost: відкрий канал, натисни заголовок назви каналу і шукай ID у URL або в деталях каналу.

Коли бот `@mentioned`, згадка автоматично видаляється з повідомлення перед обробкою.
## Дозволений список каналів (`allowed_channels`)

Обмежити бота фіксованим набором каналів Mattermost. Коли встановлено, бот **лише** відповідає в каналах, ідентифікатор яких присутній у списку — повідомлення з будь‑якого іншого каналу ігноруються без повідомлення, навіть якщо бот `@mentioned`.

**Прямі повідомлення** не підпадають під цей фільтр, тому уповноважені користувачі завжди можуть звертатися до бота в особистих повідомленнях.

```yaml
mattermost:
  allowed_channels:
    - "abc123def456ghi789jkl012mno"   # #ops
    - "xyz987uvw654rst321opq098nml"   # #incident-response
```

Або через змінну середовища (через кому):

```bash
MATTERMOST_ALLOWED_CHANNELS="abc123def456ghi789jkl012mno,xyz987uvw654rst321opq098nml"
```

**Поведінка**:

- Порожньо / не встановлено → жодних обмежень (повна зворотна сумісність).
- Не порожньо → ідентифікатор каналу має бути у списку, інакше повідомлення відкидається ще до будь‑якого іншого обмеження (вимога згадки, `MATTERMOST_FREE_RESPONSE_CHANNELS` тощо).
- Знайти ідентифікатор каналу можна в інтерфейсі Mattermost → заголовок каналу → «View Info», або прочитати його з URL каналу.

Дивись також: [розподіл slash‑команд для admin/user](../../reference/slash-commands.md#permissions-and-adminuser-split).
## Усунення неполадок

### Бот не відповідає на повідомлення

**Причина**: Бот не є учасником каналу, або `MATTERMOST_ALLOWED_USERS` не містить твій User ID.

**Виправлення**: Додай бота до каналу (назва каналу → Add Members → пошук бота). Перевір, чи твій User ID присутній у `MATTERMOST_ALLOWED_USERS`. Перезапусти шлюз.

### Помилки 403 Forbidden

**Причина**: Токен бота недійсний, або у бота немає прав публікувати в каналі.

**Виправлення**: Переконайся, що `MATTERMOST_TOKEN` у твоєму файлі `.env` правильний. Перевір, чи обліковий запис бота не був деактивований. Переконайся, що бот доданий до каналу. Якщо використовується персональний токен доступу, впевнись, що твій обліковий запис має необхідні дозволи.

### Відключення WebSocket / цикли перепідключення

**Причина**: Нестабільність мережі, перезапуск сервера Mattermost або проблеми з брандмауером/проксі щодо WebSocket‑з’єднань.

**Виправлення**: Адаптер автоматично перепідключається з експоненціальним збільшенням інтервалу (2 s → 60 s). Перевір конфігурацію WebSocket на своєму сервері — зворотні проксі (nginx, Apache) мають мати налаштовані заголовки оновлення WebSocket. Переконайся, що брандмауер не блокує WebSocket‑з’єднання на твоєму сервері Mattermost.

Для nginx переконайся, що у твоїй конфігурації є:

```nginx
location /api/v4/websocket {
    proxy_pass http://mattermost-backend;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_read_timeout 600s;
}
```

### «Failed to authenticate» під час запуску

**Причина**: Токен або URL сервера неправильні.

**Виправлення**: Переконайся, що `MATTERMOST_URL` вказує на твій сервер Mattermost (включай `https://`, без кінцевого слешу). Перевір, що `MATTERMOST_TOKEN` дійсний — спробуй його за допомогою curl:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-server/api/v4/users/me
```

Якщо це повертає інформацію про користувача твого бота, токен дійсний. Якщо повертає помилку, згенеруй новий токен.

### Бот офлайн

**Причина**: Шлюз Hermes не запущений, або не вдалося підключитися.

**Виправлення**: Перевір, чи запущений `hermes gateway`. Подивись вивід терміналу на предмет повідомлень про помилки. Поширені проблеми: неправильний URL, прострочений токен, недоступний сервер Mattermost.

### «User not allowed» / Бот ігнорує тебе

**Причина**: Твій User ID не включений у `MATTERMOST_ALLOWED_USERS`.

**Виправлення**: Додай свій User ID до `MATTERMOST_ALLOWED_USERS` у `~/.hermes/.env` і перезапусти шлюз. Пам’ятай: User ID — це 26‑символьний алфавітно‑цифровий рядок, а не твій `@username`.
## Підказки для кожного каналу

Призначай тимчасові системні підказки конкретним каналам Mattermost. Підказка інжектується під час виконання на кожному кроці — ніколи не зберігається в історії транскрипту — тому зміни набувають чинності одразу.

```yaml
mattermost:
  channel_prompts:
    "channel_id_abc123": |
      You are a research assistant. Focus on academic sources,
      citations, and concise synthesis.
    "channel_id_def456": |
      Code review mode. Be precise about edge cases and
      performance implications.
```

Ключі — це ідентифікатори каналів Mattermost (знайди їх у URL каналу або через API). Усі повідомлення в відповідному каналі отримують підказку як тимчасову системну інструкцію.
## Безпека

:::warning
Завжди встановлюй `MATTERMOST_ALLOWED_USERS`, щоб обмежити, хто може взаємодіяти з ботом. Без цього **gateway** за замовчуванням відхиляє всіх користувачів як захисний захід. Додавай лише ID користувачів людей, яким ти довіряєш — уповноважені користувачі мають повний доступ до можливостей агента, включаючи використання інструментів та доступ до системи.
:::

Для отримання додаткової інформації про захист розгортання Hermes Agent, переглянь [Посібник з безпеки](../security.md).
## Примітки

- **Self-hosted friendly**: Працює з будь‑яким самостійно розгорнутим екземпляром Mattermost. Не потрібен обліковий запис або підписка Mattermost Cloud.
- **No extra dependencies**: Адаптер використовує `aiohttp` для HTTP та WebSocket, який вже включений у Hermes Agent.
- **Team Edition compatible**: Працює як з безкоштовною Mattermost Team Edition, так і з Enterprise Edition.
