# Налаштування WhatsApp

Hermes підключається до WhatsApp через вбудований шлюз на основі **Baileys**. Це працює шляхом емуляції сесії WhatsApp Web — **не** через офіційний WhatsApp Business API. Обліковий запис розробника Meta або верифікація Business не потрібні.

> Запусти `hermes gateway setup` і обери **WhatsApp** для покрокового налаштування.

:::warning Неофіційний API — ризик блокування
WhatsApp **не** офіційно підтримує сторонні боти поза Business API. Використання стороннього шлюзу несе невеликий ризик обмеження облікового запису. Щоб мінімізувати ризик:
- **Використовуй окремий номер телефону** для бота (не свій особистий номер)
- **Не надсилай масові/спам‑повідомлення** — тримай використання у розмовному форматі
- **Не автоматизуй вихідні повідомлення** людям, які не писали спочатку
:::

:::warning Оновлення протоколу WhatsApp Web
WhatsApp періодично оновлює свій Web‑протокол, що може тимчасово порушити сумісність
з сторонніми шлюзами. Коли це трапляється, Hermes оновить залежність шлюзу. Якщо
бот перестане працювати після оновлення WhatsApp, завантаж останню версію Hermes і повторно підключи.
:::
## Два режими

| Режим | Як працює | Найкраще підходить |
|------|-----------|--------------------|
| **Окремий номер бота** (рекомендовано) | Признач окремий номер телефону боту. Люди надсилають повідомлення безпосередньо на цей номер. | Чистий UX, кілька користувачів, нижчий ризик бану |
| **Особистий самочат** | Використовуй свій WhatsApp. Надсилай повідомлення самому собі, щоб спілкуватися з агентом. | Швидке налаштування, один користувач, тестування |

---
## Prerequisites

- **Node.js v18+** та **npm** — місток WhatsApp працює як процес Node.js
- **Телефон з встановленим WhatsApp** (для сканування QR‑коду)

На відміну від старих браузерних містків, поточний місток на базі Baileys **не** вимагає локального стеку залежностей Chromium або Puppeteer.

---
## Крок 1: Запусти майстра налаштування

```bash
hermes whatsapp
```

Майстер виконає:

1. Запитає, який режим ти хочеш (**bot** або **self-chat**)
2. Встановить залежності мосту, якщо потрібно
3. Показуватиме **QR‑код** у твоєму терміналі
4. Дочекається, поки ти його відскануєш

**Щоб відсканувати QR‑код:**

1. Відкрий WhatsApp на телефоні
2. Перейди до **Settings → Linked Devices**
3. Натисни **Link a Device**
4. Наведи камеру на QR‑код у терміналі

Після успішного з’єднання майстер підтверджує підключення і завершує роботу. Твоя сесія зберігається автоматично.

:::tip
Якщо QR‑код виглядає спотвореним, переконайся, що твій термінал має щонайменше 60 колонок у ширину та підтримує
Unicode. Можеш також спробувати інший емулятор терміналу.
:::

## Крок 2: Отримання другого номера телефону (режим Bot)

Для режиму Bot потрібен номер телефону, який ще не зареєстрований у WhatsApp. Три варіанти:

| Option | Cost | Notes |
|--------|------|-------|
| **Google Voice** | Free | Тільки США. Отримай номер на [voice.google.com](https://voice.google.com). Підтверджуй WhatsApp через SMS у додатку Google Voice. |
| **Prepaid SIM** | $5–15 одноразово | Будь‑який оператор. Активуй, підтвердь WhatsApp, після чого SIM‑картка може залишитися у ящику. Номер має залишатися активним (здійснюй дзвінок кожні 90 днів). |
| **VoIP services** | Free–$5/month | TextNow, TextFree або подібні. Деякі VoIP‑номери блокуються WhatsApp — спробуй кілька, якщо перший не працює. |

Після отримання номера:

1. Встанови WhatsApp на телефон (або використай WhatsApp Business з dual‑SIM)
2. Зареєструй новий номер у WhatsApp
3. Запусти `hermes whatsapp` і відскануй QR‑код з цього облікового запису WhatsApp

## Крок 3: Налаштуй Hermes

Додай наступне у файл `~/.hermes/.env`:

```bash
# Required
WHATSAPP_ENABLED=true
WHATSAPP_MODE=bot                          # "bot" or "self-chat"

# Access control — pick ONE of these options:
WHATSAPP_ALLOWED_USERS=15551234567         # Comma-separated phone numbers (with country code, no +)
# WHATSAPP_ALLOWED_USERS=*                 # OR use * to allow everyone
# WHATSAPP_ALLOW_ALL_USERS=true            # OR set this flag instead (same effect as *)
```

:::tip Allow‑all shorthand
Встановлення `WHATSAPP_ALLOWED_USERS=*` дозволяє **всіх** відправників (еквівалентно `WHATSAPP_ALLOW_ALL_USERS=true`).
Це узгоджується з [Signal group allowlists](/reference/environment-variables).
Щоб використати процес спарювання, видали обидві змінні та покладися на
[DM pairing system](/user-guide/security#dm-pairing-system).
:::

Опціональні налаштування поведінки у `~/.hermes/config.yaml`:

```yaml
unauthorized_dm_behavior: pair

whatsapp:
  unauthorized_dm_behavior: ignore
```

- `unauthorized_dm_behavior: pair` — глобальне значення за замовчуванням. Невідомі DM‑відправники отримують код спарювання.
- `whatsapp.unauthorized_dm_behavior: ignore` змушує WhatsApp залишатися мовчазним для неавторизованих DM, що зазвичай кращий вибір для приватного номера.

Потім запусти **gateway**:

```bash
hermes gateway              # Foreground
hermes gateway install      # Install as a user service
sudo hermes gateway install --system   # Linux only: boot-time system service
```

**gateway** автоматично запускає bridge WhatsApp, використовуючи збережену сесію.

## Збереження сесії

Місток Baileys зберігає свою сесію у `~/.hermes/platforms/whatsapp/session`. Це означає:

- **Сесії зберігаються після перезапуску** — тобі не потрібно щоразу сканувати QR‑код
- Дані сесії включають ключі шифрування та облікові дані пристрою
- **Не ділися цим каталогом сесії і не додавай його до репозиторію** — він надає повний доступ до облікового запису WhatsApp

---
## Перепарування

Якщо сесія розривається (перезавантаження телефону, оновлення WhatsApp, ручне від’єднання), у логах шлюзу з’являються помилки підключення. Щоб виправити це:

```bash
hermes whatsapp
```

Це генерує новий QR‑код. Скануй його ще раз, і сесія буде відновлена. Шлюз автоматично обробляє **тимчасові** роз’єднання (збої мережі, короткочасне відключення телефону) за допомогою логіки повторного підключення.

## Голосові повідомлення

Hermes підтримує голос у WhatsApp:

- **Вхідні:** Голосові повідомлення (`.ogg` opus) автоматично транскрибуються за допомогою налаштованого STT‑провайдера: локальний `faster-whisper`, Groq Whisper (`GROQ_API_KEY`) або OpenAI Whisper (`VOICE_TOOLS_OPENAI_KEY`)
- **Вихідні:** Відповіді TTS надсилаються як вкладення у форматі MP3‑аудіо
- Відповіді агента за замовчуванням мають префікс «⚕ **Hermes Agent**». Ти можеш налаштувати або вимкнути це у `config.yaml`:

```yaml
# ~/.hermes/config.yaml
whatsapp:
  reply_prefix: ""                          # Empty string disables the header
  # reply_prefix: "🤖 *My Bot*\n──────\n"  # Custom prefix (supports \n for newlines)
```

## Форматування повідомлень та доставка

WhatsApp підтримує **потокові (прогресивні) відповіді** — бот редагує своє повідомлення в реальному часі під час генерації тексту ШІ, так само як Discord і Telegram. Внутрішньо WhatsApp класифікується як платформа TIER_MEDIUM за можливостями доставки.

### Чанкування

Довгі відповіді автоматично розбиваються на кілька повідомлень розміром **4 096 символів** на чанку (практичне обмеження відображення в WhatsApp). Нічого налаштовувати не потрібно — шлюз інструментів обробляє розбиття і надсилає чанки послідовно.

### Сумісний з WhatsApp Markdown

Стандартний Markdown у відповідях ШІ автоматично конвертується у власне форматування WhatsApp:

| Markdown | WhatsApp | Відображається як |
|----------|----------|-------------------|
| `**bold**` | `*bold*` | **bold** |
| `~~strikethrough~~` | `~strikethrough~` | ~~strikethrough~~ |
| `# Heading` | `*Heading*` | Жирний текст (без власних заголовків) |
| `[link text](url)` | `link text (url)` | Вбудоване посилання |

Блоки коду та інлайн‑код зберігаються без змін, оскільки WhatsApp підтримує форматування трьома зворотними апострофами.

### Прогрес інструментів

Коли агент викликає інструменти (веб‑пошук, операції з файлами тощо), WhatsApp показує індикатори реального часу, що вказують, який інструмент виконується. Це ввімкнено за замовчуванням — налаштування не потрібні.
## Устранення проблем

| Проблема | Рішення |
|---------|----------|
| **QR‑код не сканується** | Переконайся, що термінал достатньо широкий (60+ колонок). Спробуй інший термінал. Переконайся, що скануєш з правильного облікового запису WhatsApp (номер бота, а не особистий). |
| **QR‑код закінчує термін дії** | QR‑коди оновлюються приблизно кожні 20 секунд. Якщо час вичерпано, перезапусти `hermes whatsapp`. |
| **Сесія не зберігається** | Перевір, що `~/.hermes/platforms/whatsapp/session` існує і доступний для запису. Якщо працюєш у контейнері, змонтуй його як постійний том. |
| **Неочікуваний вихід з системи** | WhatsApp від’єднує пристрої після тривалої неактивності. Тримай телефон увімкненим і підключеним до мережі, потім при потребі повторно підключи за допомогою `hermes whatsapp`. |
| **Bridge падає або виникають цикли перепідключення** | Перезапусти **gateway**, онови Hermes і повторно підключи, якщо сесію було анульовано через зміну протоколу WhatsApp. |
| **Бот перестає працювати після оновлення WhatsApp** | Онови Hermes, щоб отримати останню версію bridge, потім повторно підключи. |
| **macOS: “Node.js not installed”, хоча node працює в терміналі** | Служби launchd не успадковують PATH твого шелу. Запусти `hermes gateway install`, щоб зберегти поточний PATH у plist, потім `hermes gateway start`. Дивись [документацію сервісу gateway](./index.md#macos-launchd) для деталей. |
| **Повідомлення не надходять** | Переконайся, що `WHATSAPP_ALLOWED_USERS` містить номер відправника (з кодом країни, без `+` та пробілів), або встанови `*`, щоб дозволити всіх. Встанови `WHATSAPP_DEBUG=true` у `.env` і перезапусти gateway, щоб бачити сирі події повідомлень у `bridge.log`. |
| **Бот відповідає незнайомцям кодом парування** | Додай `whatsapp.unauthorized_dm_behavior: ignore` у `~/.hermes/config.yaml`, якщо хочеш, щоб неавторизовані DM ігнорувалися. |
## Безпека

:::warning
**Налаштуй контроль доступу** перед запуском у продакшн. Встанови `WHATSAPP_ALLOWED_USERS` зі специфічними
номерами телефонів (включаючи код країни, без `+`), використай `*`, щоб дозволити всіх, або встанови
`WHATSAPP_ALLOW_ALL_USERS=true`. Якщо нічого з цього не вказано, gateway **відхиляє всі вхідні
повідомлення** як запобіжний захід.
:::

За замовчуванням неавторизовані DMs все ще отримують відповідь з кодом парування. Якщо ти хочеш, щоб приватний номер WhatsApp залишався повністю мовчазним для сторонніх, встанови:

```yaml
whatsapp:
  unauthorized_dm_behavior: ignore
```

- Каталог `~/.hermes/platforms/whatsapp/session` містить повні облікові дані сесії — захищай його як пароль
- Встанови права доступу до файлу: `chmod 700 ~/.hermes/platforms/whatsapp/session`
- Використовуй **виділений номер телефону** для бота, щоб ізолювати ризик від свого особистого облікового запису
- Якщо підозрюєш компрометацію, відв'яжи пристрій у WhatsApp → Settings → Linked Devices
- Номери телефонів у логах частково приховані, проте переглянь свою політику зберігання логів
