# Polymarket — дані ринків прогнозів

Запитуй дані ринків прогнозів з Polymarket, використовуючи їх публічні REST API.
Усі кінцеві точки лише для читання і не вимагають автентифікації.

Дивись `references/api-endpoints.md` для повного переліку кінцевих точок з прикладами curl.

## Коли використовувати

- Користувач запитує про ринки прогнозів, коефіцієнти ставок або ймовірності подій
- Користувач хоче дізнатися «яка ймовірність того, що X станеться?»
- Користувач запитує саме про Polymarket
- Користувач хоче отримати ціни ринку, дані ордербуку або історію цін
- Користувач просить моніторити або відстежувати рухи ринку прогнозів

## Ключові концепції

- **Події** містять один або кілька **ринків** (зв’язок 1:багато)
- **Ринки** мають бінарні результати з цінами Yes/No у діапазоні від 0.00 до 1.00
- Ціни — це ймовірності: ціна 0.65 означає, що ринок вважає ймовірність 65 %
- Поле `outcomePrices`: JSON‑закодований масив типу `["0.80", "0.20"]`
- Поле `clobTokenIds`: JSON‑закодований масив двох ідентифікаторів токенів [Yes, No] для запитів price/book
- Поле `conditionId`: шістнадцятковий рядок, що використовується для запитів історії цін
- Обсяг вказується в USDC (долари США)

## Три публічні API

1. **Gamma API** на `gamma-api.polymarket.com` — відкриття, пошук, перегляд
2. **CLOB API** на `clob.polymarket.com` — ціни в реальному часі, ордербуки, історія
3. **Data API** на `data-api.polymarket.com` — торги, відкритий інтерес

## Типовий робочий процес

Коли користувач запитує про коефіцієнти ринку прогнозів:

1. **Пошук** за допомогою публічного endpoint Gamma API `public-search` з його запитом
2. **Парсинг** відповіді — витягти події та їхні вкладені ринки
3. **Презентація** питання ринку, поточних цін у відсотках та обсягу
4. **Глибокий аналіз** за запитом — використати `clobTokenIds` для ордербуку, `conditionId` для історії

## Представлення результатів

Форматуй ціни у відсотки для зручності читання:
- `outcomePrices` `["0.652", "0.348"]` → «Yes: 65.2 %, No: 34.8 %»
- Завжди показуй питання ринку та ймовірність
- Додавай обсяг, якщо він доступний

Приклад: `"Will X happen?" — 65.2 % Yes ($1.2M volume)`

## Парсинг подвійно закодованих полів

Gamma API повертає `outcomePrices`, `outcomes` та `clobTokenIds` як JSON‑рядки всередині JSON‑відповідей (подвійне кодування). При обробці в Python розпарсь їх за допомогою `json.loads(market['outcomePrices'])`, щоб отримати реальний масив.

## Ліміти запитів

Щедрі — малоймовірно перевищити при нормальному використанні:
- Gamma: 4 000 запитів за 10 секунд (загальне)
- CLOB: 9 000 запитів за 10 секунд (загальне)
- Data: 1 000 запитів за 10 секунд (загальне)

## Обмеження

- Ця навичка лише для читання — вона не підтримує розміщення торгів
- Торгівля вимагає автентифікації гаманцем (підписи EIP‑712)
- Деякі нові ринки можуть мати порожню історію цін
- Географічні обмеження застосовуються до торгів, але дані лише для читання доступні глобально
