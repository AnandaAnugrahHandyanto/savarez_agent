# Навичка «Діаграма архітектури»

Генерує професійні, темні технічні діаграми архітектури у вигляді окремих HTML‑файлів з вбудованою SVG‑графікою. Без зовнішніх інструментів, без API‑ключів, без бібліотек рендерингу — просто створює файл HTML і відкриває його в браузері.

## Обсяг

**Найкраще підходить для:**
- Архітектури програмного забезпечення (шари фронтенду / бекенду / бази даних)
- Хмарної інфраструктури (VPC, регіони, підмережі, керовані сервіси)
- Топології мікросервісів / service‑mesh
- Карт баз даних + API, діаграми розгортання
- Будь‑що з технічно‑інфраструктурною тематикою, що підходить до темної, сіткової естетики

**Шукайте інше спочатку:**
- Фізика, хімія, математика, біологія чи інші наукові теми
- Фізичні об’єкти (транспорт, апаратура, анатомія, перерізи)
- Планування приміщень, наративні подорожі, візуали у стилі підручників
- Накреслені від руки схеми на білих дошках (розглянь `excalidraw`)
- Анімовані пояснювальні ролики (розглянь навичку анімації)

Якщо існує більш спеціалізована навичка для даної теми, віддавай перевагу їй. Якщо жодної не підходить, ця навичка може слугувати загальним запасним варіантом SVG‑діаграми — результат просто матиме темну технічну естетику, описану нижче.

## Технічні деталі реалізації

### Відображення компонентів
Компоненти — це закруглені прямокутники (`rx="6"`), контуром 1.5 px. Щоб стрілки не просвічували через напівпрозорі заповнення, використовуйте **подвійне маскування прямокутника**:
1. Намалюйте непрозорий фон‑прямокутник (`#0f172a`)
2. Над ним намалюйте напівпрозорий стильовий прямокутник

### Правила з’єднань
- **Z‑порядок:** Малюйте стрілки *рано* у SVG (після сітки), щоб вони розташовувалися позаду блоків компонентів
- **Стрілкові головки:** Визначені через SVG‑маркері
- **Потоки безпеки:** Пунктирні лінії рожевого кольору (`#fb7185`)
- **Кордони:**
  - *Security Groups:* Пунктир (`4,4`), рожевий колір
  - *Regions:* Великий пунктир (`8,4`), амбер‑колір, `rx="12"`

### Логіка відступів та розташування
- **Стандартна висота:** 60 px (Сервіси); 80‑120 px (Великі компоненти)
- **Вертикальний проміжок:** Мінімум 40 px між компонентами
- **Message Bus:** Має розташовуватись *у проміжку* між сервісами, не перекриваючи їх
- **Розташування легенди:** **КРИТИЧНО.** Легенда повинна бути поза межами всіх коробок‑кордонів. Обчисліть найнижчу Y‑координату всіх кордонів і розмістіть легенду принаймні на 20 px нижче.

## Структура документа

Згенерований HTML‑файл має чотиристоронню структуру:
1. **Header:** Заголовок з індикатором‑точкою, що пульсує, та підзаголовком
2. **Main SVG:** Діаграма у межах картки з округленою рамкою
3. **Summary Cards:** Сітка з трьох карток під діаграмою для високорівневих деталей
4. **Footer:** Мінімальні метадані

### Шаблон інформаційної картки
```html
<div class="card">
  <div class="card-header">
    <div class="card-dot cyan"></div>
    <h3>Title</h3>
  </div>
  <ul>
    <li>• Item one</li>
    <li>• Item two</li>
  </ul>
</div>
```

## Вимоги до виводу
- **Один файл:** Єдиний самодостатній `.html` файл
- **Без зовнішніх залежностей:** Увесь CSS та SVG вбудовані (крім Google Fonts)
- **Без JavaScript:** Для анімацій (наприклад, пульсуючі точки) використовуйте чистий CSS
- **Сумісність:** Має коректно відображатися в будь‑якому сучасному веб‑браузері
