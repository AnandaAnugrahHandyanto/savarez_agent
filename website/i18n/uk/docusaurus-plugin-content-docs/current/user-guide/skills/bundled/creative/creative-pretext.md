# Претекстові креативні демо
## Огляд

[`@chenglou/pretext`](https://github.com/chenglou/pretext) — це 15 KB бібліотека TypeScript без залежностей від Cheng Lou (React core, ReasonML, Midjourney) для **вимірювання та розташування багаторядкового тексту без DOM**. Вона робить одне: отримавши `(text, font, width)`, повертає розриви рядків, ширини кожного рядка, позиції кожної графеми та загальну висоту — все за допомогою вимірювань canvas, без повторного рендеру.

Звучить як «трубопровід». Це не так. Оскільки вона швидка та геометрична, це **креативний примітив**: ти можеш переформатовувати абзаци навколо рухомого спрайту зі швидкістю 60 fps, створювати ігри, у яких геометрія рівня складається з реальних слів, керувати ASCII‑логотипами за допомогою прози, розбивати текст на частинки з точними початковими позиціями кожної графеми або упакувати багаторядковий UI без жодного шуму `getBoundingClientRect`.

Цей інструмент існує, щоб Hermes міг створювати **класні демо** за його допомогою — ті, що люди публікують у X. Дивись `pretext.cool` та `chenglou.me/pretext` для демо‑корпусу спільноти.
## Коли використовувати

Використовуй, коли користувач просить:
- “pretext demo” / “cool pretext thing” / “text-as-X”
- Текст, що обтікає рухому форму (hero‑секції, редакційні макети, анімовані довгі сторінки)
- ASCII‑арт‑ефекти за допомогою **справжніх слів або прози**, а не моноширинних растерів
- Ігри, у яких ігрове поле / перешкоди / цеглини складаються з тексту (Tetris‑з‑літер, Breakout‑з‑прози)
- Кінетична типографіка з фізикою per‑glyph (розбиття, розкидання, зграя, потік)
- Генеративне типографічне мистецтво, особливо з нелатинськими або змішаними скриптами
- Багаторядковий UI “shrink‑wrap” (найменша ширина контейнера, що ще вміщає текст)
- Будь‑що, що потребує знання розривів рядків *заздалегідь* перед рендерингом

Не використовуйте для:
- Статичних SVG/HTML‑сторінок, де CSS вже вирішує розташування — просто використай CSS
- Редакторів багатого тексту, загальних движків інлайн‑форматування (pretext навмисно вузькоспеціалізований)
- Image → text (використовуй `ascii-art` / `ascii-video` інструменти)
- Чистого canvas‑генеративного мистецтва без ролі тексту — використай `p5js`
## Creative Standard

Це візуальне мистецтво, що відображається в браузері. Pretext повертає числа; **ти** малюєш це.

- **Не доставляй демо «hello world».** Шаблон `hello-orb-flow.html` є *початковою* точкою. Кожне готове демо має додати навмисний колір, рух, композицію та одну візуальну деталь, яку користувач не просив, але оцінить.
- **Темне тло, теплі ядра, продумана палітра.** Класичний амбер на чорному (CRT / термінал) підходить, так само як холодний білий на вугіллі (редакційний) і десатуровані пастелі (рисограф). Обери один варіант і дотримуйся його.
- **Пропорційні шрифти — це суть.** Уся атмосфера Pretext — «не моноширинний» — використай це. Використовуй Iowan Old Style, Inter, JetBrains Mono, Helvetica Neue або змінний шрифт. Ніколи не використовуйте стандартний sans.
- **Реальне джерело/текст, а не lorem ipsum.** Корпус має мати сенс. Короткі маніфести, поезія, реальний вихідний код, знайдений текст, README бібліотеки — ніколи `lorem ipsum`.
- **Відмінність першого кадру.** Без станів завантаження, без порожніх кадрів. Демо має виглядати готовим до випуску одразу після відкриття.
## Стек

Один самодостатній HTML‑файл на кожну демонстрацію. Без кроку збірки.

| Layer | Tool | Purpose |
|-------|------|---------|
| Core | `@chenglou/pretext` via `esm.sh` CDN | Вимірювання тексту + розташування рядків |
| Render | HTML5 Canvas 2D | Відображення гліфів, композиція кадру за кадром |
| Segmentation | `Intl.Segmenter` (built-in) | Розбиття на графеми для емодзі / CJK / комбінованих знаків |
| Interaction | Raw DOM events | Миша / дотик / колесо — без фреймворку |

```html
<script type="module">
import {
  prepare, layout,                   // use-case 1: simple height
  prepareWithSegments, layoutWithLines,  // use-case 2a: fixed-width lines
  layoutNextLineRange, materializeLineRange, // use-case 2b: streaming / variable width
  measureLineStats, walkLineRanges,  // stats without string allocation
} from "https://esm.sh/@chenglou/pretext@0.0.6";
</script>
```

Зафіксуй версію. `@0.0.6` на момент написання — перевір [npm](https://www.npmjs.com/package/@chenglou/pretext) для останньої, якщо поведінка демо відрізняється.
## Два випадки використання

Практично все зводиться до одного з цих двох підходів. Вивчи обидва.

### Випадок використання 1 — виміряти, а потім відобразити за допомогою CSS/DOM

```js
const prepared = prepare(text, "16px Inter");
const { height, lineCount } = layout(prepared, 320, 20);
```

Ти все ще дозволяєш браузеру малювати текст. Pretext просто підказує, якої висоти буде блок при заданій ширині, **без** читання DOM. Використовуй для:
- Віртуалізованих списків, де рядки містять переносний текст
- Masonry з точними висотами карток
- Перевірок під час розробки «Чи вміщується ця мітка?»
- Запобігання зсуву розмітки, коли завантажується віддалений текст

**Тримай `font` і `letterSpacing` точно синхронізованими з твоїм CSS.** Формат `ctx.font` канви (наприклад `"16px Inter"`, `"500 17px 'JetBrains Mono'"`) має відповідати відрендереному CSS, інакше вимірювання будуть неточними.

### Випадок використання 2 — виміряти *і* відобразити самостійно

```js
const prepared = prepareWithSegments(text, FONT);
const { lines } = layoutWithLines(prepared, 320, 26);
for (let i = 0; i < lines.length; i++) {
  ctx.fillText(lines[i].text, 0, i * 26);
}
```

Тут і живе творча робота. Ти керуєш малюванням, тому можеш:
- Відображати на canvas, SVG, WebGL або будь‑якій системі координат
- Підміняти трансформації окремих гліфів (обертання, трясіння, масштаб, прозорість)
- Використовувати метадані рядка (ширина, позиції графем) як геометрію

Для **змінної ширини рядка** (текст навколо форми, текст у кільцевій смузі, текст у неректанґульній колонці):

```js
let cursor = { segmentIndex: 0, graphemeIndex: 0 };
let y = 0;
while (true) {
  const lineWidth = widthAtY(y);  // your function: how wide is the corridor at this y?
  const range = layoutNextLineRange(prepared, cursor, lineWidth);
  if (!range) break;
  const line = materializeLineRange(prepared, range);
  ctx.fillText(line.text, leftEdgeAtY(y), y);
  cursor = range.end;
  y += lineHeight;
}
```

Це найважливіший шаблон у всій бібліотеці. Він відкриває можливість «тексту, що обтікає перетягнутий спрайт» — демо, яке стало вірусним у X.

### Корисні допоміжні функції

- `measureLineStats(prepared, maxWidth)` → `{ lineCount, maxLineWidth }` — найширший рядок, тобто ширина, яку займає багаторядковий shrink‑wrap.
- `walkLineRanges(prepared, maxWidth, callback)` — ітерує рядки без створення рядкових об’єктів. Використовуй для статистики/фізики над графемами, коли тобі не потрібні символи.
- `@chenglou/pretext/rich-inline` — та сама система, але для абзаців, що змішують шрифти / чіпи / згадки. Імпортуй з підшляху.

| **Кінетичний тип** | `layoutWithLines` + трансформація рядка з часом | Прокрутка у стилі «Star Wars», хвиля, підстрибування, гліч |

## Примітки щодо продуктивності

- `prepare()` / `prepareWithSegments()` – це дорогий виклик. Виконуй його **один раз** для кожної пари текст + шрифт. Кешуй дескриптор.
- При зміні розміру лише повторно викликай `layout()` / `layoutWithLines()` — ніколи не пере‑підготовлюй.
- Для анімацій у кожному кадрі, коли текст не змінюється, а змінюється геометрія, `layoutNextLineRange` у щільному циклі досить швидкий, щоб виконувати його кожен кадр при 60 fps для звичайних абзаців.
- При рендерингу ASCII‑масок у кожному кадрі тримай буфер клітинок (`Uint8Array`/typed arrays), отримуй виміряні ділянки перешкод по рядках з клітинок або проєктованої геометрії, об’єднуй ділянки, а потім передавай їх у `layoutNextLineRange` перед малюванням тексту.
- Тримай візуальну анімацію та анімацію розташування синхронізованими. Якщо сфера перетворюється на куб, інтерполюй і буфер клітинок, і ділянки перешкод одним і тим же значенням; інакше демо виглядатиме намальованим, а не фізично переформованим.
- Для згасань краще змінювати прозорість шару, ніж інтенсивність гліфу або масштаб перешкоди. Розмісти тимчасові ASCII‑спрайти на окремому канвасі і згасай канвас за допомогою CSS/GSAP opacity, щоб геометрія не виглядала стискаючоюся.
- Налаштування `ctx.font` у Canvas виявляється надзвичайно повільним; встановлюй його **один раз** за кадр, якщо шрифт не змінюється, а не при кожному виклику `fillText`.
## Типові підводні камені

1. **Дрейф рядків шрифтів у CSS/Canvas.** `ctx.font = "16px Inter"` вимірюється, а в CSS вказано `font-family: Inter, sans-serif; font-size: 16px`. Працює, *якщо* Inter завантажується. Якщо Inter повертає 404, CSS переходить на `sans-serif`, і вимірювання відхиляються на 5‑20 %. Завжди `preload` шрифт або використай веб‑безпечну сімейство.

2. **Повторна підготовка всередині циклу анімації.** Дешевими є лише функції `layout*`. Виклик `prepare` кожного кадру різко знижує продуктивність. Тримай підготовлений дескриптор у модульній області.

3. **Забування про `Intl.Segmenter` для розбиття на графеми.** Emoji, комбіновані діакритики, CJK — `"é".split("")` повертає два символи. Використовуй `new Intl.Segmenter(undefined, { granularity: "grapheme" })`, коли треба отримати окремі видимі гліфи.

4. **Чіпи `break: 'never'` без `extraWidth`.** У `rich-inline`, якщо ти задаєш `break: 'never'` для атомарного чіпа/згадки, треба також вказати `extraWidth` для відступу «pill» — інакше обводка чіпа виходить за межі контейнера.

5. **Використання `@chenglou/pretext` з `unpkg` з лише TypeScript‑входом.** Користуйся `esm.sh` — він автоматично компілює TS‑експорти у готовий для браузера ESM. `unpkg` поверне 404 або сирий TS‑файл.

6. **Monospace‑fallback, що тихо руйнує задум.** Користувачі, які бачать вивід у моноширинному шрифті, часто мають у CSS `font-family`, який впав до `monospace`. Перевір фактичний рендерений шрифт у DevTools.

7. **Пропуск рядків vs корекція ширини** при обтікання форми. Якщо коридор у цьому рядку занадто вузький для рядка, *пропусти рядок* (`y += lineHeight; continue;`) замість передачі крихітного `maxWidth` у `layoutNextLineRange` — `pretext` поверне однографемні рядки, які виглядають пошкодженими.

8. **Випуск «холодної» демо‑версії.** За замовчуванням перший кадр виглядає як навчальний приклад. Додай: віньєтку, легку скан‑лінію, пасивний автодвижок, одну ретельно обрану інтерактивну реакцію (drag, hover, scroll, click). Без цього «крута демо‑версія pretext» сприйматиметься як «репродукція інтерна README».
