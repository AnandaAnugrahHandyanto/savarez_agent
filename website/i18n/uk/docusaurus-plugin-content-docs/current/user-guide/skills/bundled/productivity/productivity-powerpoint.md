## Ідеї дизайну

**Не створюй нудні слайди.** Прості маркери на білому фоні нікого не вразять. Розглянь ідеї з цього списку для кожного слайду.

### Перед початком

- **Вибери сміливу, орієнтовану на зміст кольорову палітру**: Палітра має виглядати створеною спеціально для ЦЬОГО топіка. Якщо ти можеш просто підмінити свої кольори в іншу презентацію і вона все ще «працюватиме», ти не зробив достатньо специфічних виборів.
- **Домінування над рівністю**: Один колір має домінувати (60‑70 % візуальної ваги), з 1‑2 підтримуючими тонами і одним яскравим акцентом. Ніколи не давай усім кольорам однакову вагу.
- **Контраст темного/світлого**: Темний фон для титульних і заключних слайдів, світлий — для контенту («структура «сендвіч»). Або ж використай темний фон протягом усієї презентації для преміального вигляду.
- **Прийми візуальний мотив**: Вибери ОДИН відмінний елемент і повторюй його — округлені рамки зображень, іконки в кольорових колах, товсті односторонні рамки. Носи його через усі слайди.

### Для кожного слайду

**Кожен слайд потребує візуального елементу** — зображення, діаграми, іконки або форми. Слайди лише з текстом легко забути.

**Варіанти розташування:**
- Два стовпці (текст зліва, ілюстрація справа)
- Рядки іконка + текст (іконка в кольоровому колі, жирний заголовок, опис під ним)
- Сітка 2×2 або 2×3 (зображення з однієї сторони, блоки контенту у сітці з іншої)
- Зображення, що простягається на половину (повна ліва або права сторона) з накладеним контентом

**Відображення даних:**
- Великі блоки статистики (числа 60‑72 pt з маленькими підписами)
- Колонки порівняння (до/після, плюси/мінуси, варіанти поруч)
- Хронологія або процес (нумеровані кроки, стрілки)

**Візуальна довершеність:**
- Іконки в маленьких кольорових колах поруч із заголовками розділів
- Курсивний акцентний текст для ключових статистик або слоганів

### Типографіка

**Обери цікаву пару шрифтів** — не залишайся на Arial. Підбери шрифт заголовка з характером і поєднай його з чистим шрифтом для основного тексту.

| Шрифт заголовка | Шрифт основного |
|-----------------|-----------------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| Елемент | Розмір |
|---------|--------|
| Заголовок слайду | 36‑44 pt **жирний** |
| Заголовок розділу | 20‑24 pt **жирний** |
| Основний текст | 14‑16 pt |
| Підписи | 10‑12 pt **приглушений** |

### Відступи

- Мінімальні поля 0.5 "
- 0.3‑0.5 " між блоками контенту
- Забезпечуй «дихаючий простір» — не заповнюй кожен дюйм

### Уникати (поширені помилки)

- **Не повторюй один і той же макет** — варіюй колонки, картки, блоки викликів на різних слайдах
- **Не центруй основний текст** — вирівнюй абзаци та списки по лівому краю; центруй лише заголовки
- **Не економ на контрасті розмірів** — заголовки мають бути 36 pt+ і виділятись над 14‑16 pt основним текстом
- **Не використовуйте синій за замовчуванням** — підбирай кольори, що відображають конкретну тему
- **Не змішуй відступи випадково** — обирай 0.3 " або 0.5 " і використай послідовно
- **Не стилізуй один слайд, а інші залишай простими** — або стилізуй усі, або залишай їх простими, підтримуй консистентність
- **Не створюй слайди лише з текстом** — додавай зображення, іконки, діаграми або інші візуальні елементи; уникай простих «заголовок + маркери»
- **Не забувай про відступи текстових блоків** — коли вирівнюєш лінії або форми з краями тексту, встанови `margin: 0` у текстовому боксі або зсунь форму, щоб урахувати відступ
- **Не використовуйте елементи з низьким контрастом** — іконки **і** текст повинні мати сильний контраст до фону; уникай світлого тексту на світлому фоні чи темного тексту на темному фоні
- **НІКОЛИ не використовуйте акцентні лінії під заголовками** — це ознака слайдів, створених ШІ; замість цього використай простір або колір фону
## QA (Required)

**Припустимо, що є проблеми. Твоє завдання — знайти їх.**

Твій перший рендер майже ніколи не є правильним. Підходь до QA як до полювання на баги, а не як до кроку підтвердження. Якщо ти не виявив жодної проблеми під час першої інспекції, ти не досить ретельно шукав.

### Content QA

```bash
python -m markitdown output.pptx
```

Перевір на відсутність контенту, типографічні помилки, неправильний порядок.

**При використанні шаблонів перевір на залишковий текст‑заповнювач:**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

Якщо `grep` повертає результати, виправ їх перед оголошенням успіху.

### Visual QA

**⚠️ USE SUBAGENTS** — навіть для 2‑3 слайдів. Ти дивишся на код і бачиш те, що очікуєш, а не те, що є. Субагенти мають свіжий погляд.

Перетвори слайди у зображення (див. [Converting to Images](#converting-to-images)), потім використай цей промпт:

```
Visually inspect these slides. Assume there are issues — find them.

Look for:
- Overlapping elements (text through shapes, lines through words, stacked elements)
- Text overflow or cut off at edges/box boundaries
- Decorative lines positioned for single-line text but title wrapped to two lines
- Source citations or footers colliding with content above
- Elements too close (< 0.3" gaps) or cards/sections nearly touching
- Uneven gaps (large empty area in one place, cramped in another)
- Insufficient margin from slide edges (< 0.5")
- Columns or similar elements not aligned consistently
- Low-contrast text (e.g., light gray text on cream-colored background)
- Low-contrast icons (e.g., dark icons on dark backgrounds without a contrasting circle)
- Text boxes too narrow causing excessive wrapping
- Leftover placeholder content

For each slide, list issues or areas of concern, even if minor.

Read and analyze these images:
1. /path/to/slide-01.jpg (Expected: [brief description])
2. /path/to/slide-02.jpg (Expected: [brief description])

Report ALL issues found, including minor ones.
```

### Verification Loop

1. Генеруй слайди → Перетвори у зображення → Перевір
2. **Список виявлених проблем** (якщо їх немає, подивись ще критичніше)
3. Виправ проблеми
4. **Повторно перевір змінені слайди** — одне виправлення часто створює іншу проблему
5. Повторюй, доки повний прохід не виявить нових проблем

**Не оголошуй успіх, доки не завершиш хоча б один цикл виправлення‑перевірки.**
## Перетворення у зображення

Перетворюй презентації у окремі зображення слайдів для візуальної перевірки:

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

Це створює `slide-01.jpg`, `slide-02.jpg` тощо.

Щоб повторно перерендерити конкретні слайди після виправлень:

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
