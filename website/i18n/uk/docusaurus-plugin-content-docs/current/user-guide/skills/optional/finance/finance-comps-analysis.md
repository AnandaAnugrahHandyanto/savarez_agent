## Розділ 2: Операційна статистика та фінансові метрики

### Основні стовпці (почни з цих)
1. **Company** – назви з уніфікованим форматуванням
2. **Revenue** – метрика розміру (може бути LTM, квартальна або річна, залежно від контексту)
3. **Revenue Growth** – річна зміна у відсотках
4. **Gross Profit** – дохід мінус собівартість проданих товарів
5. **Gross Margin** – GP/Revenue (фундаментальна прибутковість)
6. **EBITDA** – прибуток до вирахування відсотків, податків, амортизації та зносу
7. **EBITDA Margin** – EBITDA/Revenue (операційна ефективність)

### Додаткові стовпці (вибирай за галуззю/ціллю)
- **Quarterly vs LTM** – включи обидва, якщо важлива сезонність
- **Free Cash Flow** – для капіталомістких або SaaS‑бізнесів
- **FCF Margin** – FCF/Revenue (ефективність генерування готівки)
- **Net Income** – для зрілих, прибуткових компаній
- **Operating Income** – для компаній з різною амортизацією та зносом
- **CapEx metrics** – для галузей з важкими активами
- **Rule of 40** – спеціально для SaaS (Growth % + Margin %)
- **FCF Conversion** – для аналізу якості прибутку (просунутий)

### Приклади формул (на прикладі рядка 7)
```excel
// Core ratios - these are always calculated
Gross Margin (F7): =E7/C7
EBITDA Margin (H7): =G7/C7

// Optional ratios - include if relevant
FCF Margin: =[FCF]/[Revenue]
Net Margin: =[Net Income]/[Revenue]
Rule of 40: =[Growth %]+[FCF Margin %]
```

**Золоте правило:** Кожне співвідношення має бути [Щось] / [Revenue] або [Щось] / [Щось з цього листа]. Тримай все просто.

### Блок статистики (після даних компаній)

**CRITICAL: Додай формули статистики для всіх порівнянних метрик (коефіцієнти, маржі, темпи росту, мультиплікатори).**
```
[Leave one blank row for visual separation]
- Maximum: =MAX(B7:B9)
- 75th Percentile: =QUARTILE(B7:B9,3)

```

**Стовпці, які ПОВИННІ мати статистику (порівнянні метрики):**
- Revenue Growth %, Gross Margin %, EBITDA Margin %, EPS
- EV/Revenue, EV/EBITDA, P/E, Dividend Yield %, Beta

**Стовпці, які НЕ ПОВИННІ мати статистику (метрики розміру):**
- Revenue, EBITDA, Net Income (абсолютний розмір змінюється залежно від масштабу компанії)
- Market Cap, Enterprise Value (не порівнюються між компаніями різного розміру)

**Примітка:** Додай один порожній рядок між даними компаній та рядками статистики для візуального розділення. Не додавай заголовок рядка «SECTOR STATISTICS» або «VALUATION STATISTICS».

## Розділ 3: Оцінкові множники та інвестиційні метрики

### Основні колонки оцінки (почни з цих)
1. **Company** – Та ж послідовність, що й у розділі операцій
2. **Market Cap** – Поточна ринкова оцінка
3. **Enterprise Value** – Market Cap ± Net Debt/Cash
4. **EV/Revenue** – Скільки ринок платить за долар продажу
5. **EV/EBITDA** – Скільки ринок платить за долар прибутку
6. **P/E Ratio** – Ціна щодо чистого прибутку

### Додаткові метрики оцінки (вибирай за контекстом)
- **FCF Yield** – FCF/Market Cap (для аналізу, орієнтованого на готівку)
- **PEG Ratio** – P/E/Growth Rate (для компаній росту)
- **Price/Book** – Ринкова вартість проти балансової вартості (для активних бізнесів)
- **ROE/ROA** – Метрики прибутковості (для порівняння прибутковості)
- **Revenue/EBITDA CAGR** – Історичні темпи зростання (для аналізу тенденцій)
- **Asset Turnover** – Revenue/Assets (для операційної ефективності)
- **Debt/Equity** – Кредитне навантаження (для аналізу структури капіталу)

**Ключовий принцип:** Включи 3‑5 основних множників, які важливі для твоєї індустрії. Не додавай усі можливі метрики лише тому, що можеш.

### Приклади формул
```excel
// Core multiples - always include these
EV/Revenue: =[Enterprise Value]/[LTM Revenue]
EV/EBITDA: =[Enterprise Value]/[LTM EBITDA]
P/E Ratio: =[Market Cap]/[Net Income]

// Optional multiples - include if data available
FCF Yield: =[LTM FCF]/[Market Cap]
PEG Ratio: =[P/E]/[Growth Rate %]
```

### Правило крос‑посилань
**CRITICAL:** Оцінкові множники ПОВИННІ посилатися на розділ операційних метрик. Не вводь ті ж самі сирі дані двічі. Якщо revenue знаходиться в C7, то формула EV/Revenue має посилатися на C7.

### Блок статистики
Така ж структура, як у розділі операцій: Max, 75‑й, Median, 25‑й, Min для кожної метрики. Додай один порожній рядок для візуального розділення між даними компаній та статистикою. Не додавай рядок заголовка «VALUATION STATISTICS».

- Time period definitions (LTM, CAGR calculation periods)

**Valuation Methodology:**
- How was Enterprise Value calculated? (Market Cap + Net Debt)
- What growth rates were used? (Historical CAGR, forward estimates)
- Any adjustments made? (One‑time items excluded, normalized margins)

**Analysis Framework:**
- What's the investment thesis? (Cloud/SaaS efficiency)
- What metrics matter most? (Cash generation, capital efficiency)
- How should readers interpret the statistics? (Quartiles provide context)

## Розділ 5: Вибір правильних метрик (рамки прийняття рішень)

### Почни з «На яке питання я відповідаю?»

**«Яка компанія недооцінена?»**
→ Зосередитися на: EV/Revenue, EV/EBITDA, P/E, Market Cap
→ Пропустити: Operational details, growth metrics

**«Яка компанія найефективніша?»**
→ Зосередитися на: Gross Margin, EBITDA Margin, FCF Margin, Asset Turnover
→ Пропустити: Size metrics, absolute dollar amounts

**«Яка компанія росте найшвидше?»**
→ Зосередитися на: Revenue Growth %, EBITDA CAGR, User/Customer Growth
→ Пропустити: Margin metrics, leverage ratios

**«Яка компанія генерує найбільше готівки?»**
→ Зосередитися на: FCF, FCF Margin, FCF Conversion, CapEx intensity
→ Пропустити: EBITDA, P/E ratios

### Вибір метрик за галуззю

**Software/SaaS:**
Обов’язково: Revenue Growth, Gross Margin, Rule of 40
Опційно: ARR, Net Dollar Retention, CAC Payback
Пропустити: Asset Turnover, Inventory metrics

**Manufacturing/Industrials:**
Обов’язково: EBITDA Margin, Asset Turnover, CapEx/Revenue
Опційно: ROA, Inventory Turns, Backlog
Пропустити: Rule of 40, SaaS metrics

**Financial Services:**
Обов’язково: ROE, ROA, Efficiency Ratio, P/E
Опційно: Net Interest Margin, Loan Loss Reserves
Пропустити: Gross Margin, EBITDA (не має сенсу для банків)

**Retail/E-commerce:**
Обов’язково: Revenue Growth, Gross Margin, Inventory Turnover
Опційно: Same-Store Sales, Customer Acquisition Cost
Пропустити: Heavy R&D або CapEx метрики

### Правило «5‑10»

**5 операційних метрик** – Revenue, Growth, 2‑3 margins/efficiency metrics
**5 метрик оцінки** – Market Cap, EV, 3 multiples
**= 10 загальних колонок** – Достатньо, щоб розповісти історію, не так багато, щоб втратити нить

Якщо у тебе більше 15 метрик, ймовірно, ти включаєш шум. Редагуй безжально.
## Розділ 6: Кращі практики та перевірки якості

### Перед початком
1. **Визначити групу порівняння** – компанії мають бути дійсно порівнянними (подібна бізнес‑модель, масштаб, географія)
2. **Обрати правильний період** – LTM згладжує сезонність; квартальний показник відображає тренди
3. **Стандартизувати одиниці заздалегідь** – вибір між мільйонами та мільярдами впливає на все
4. **Скласти карту джерел даних** – знати, звідки береться кожне число

### Під час створення
1. **Ввести всі необроблені дані спочатку** – заповнити синій текст перед написанням формул
2. **Додати коментарі до всіх жорстко закодованих вводів** – правий клік по клітинці → *Insert Comment* → документувати джерело **або** припущення

   **Для даних з джерел вкажи точно, звідки вони взяті:**
   - Приклад: “Bloomberg Terminal – MSFT Equity DES, accessed 2024-10-02”
   - Приклад: “Q4 2024 10‑K filing, page 42, line item ‘Total Revenue’”
   - Приклад: “FactSet consensus estimate as of 2024-10-02”
   - **Додай гіперпосилання, коли можливо:** правий клік по клітинці → *Link* → вставити URL до SEC filing, джерела даних або звіту

   **Для припущень поясни обґрунтування:**
   - Приклад: “Assumed 15 % EBITDA margin based on peer median, company does not disclose”
   - Приклад: “Estimated Enterprise Value as Market Cap + $50M net debt (from Q3 balance sheet, Q4 not yet available)”

   **Чому це важливо:** забезпечує аудиторські сліди, верифікацію даних, прозорість припущень та можливість майбутніх оновлень

3. **Будувати формули рядок за рядком** – тестувати кожний розрахунок перед переходом далі
4. **Використовувати абсолютні посилання для заголовків** – `$C$6` фіксує рядок заголовка
5. **Форматувати послідовно** – відсотки як відсотки, а не як десяткові дроби
6. **Додати умовне форматування** – автоматично підсвічувати аномалії

### Перевірки розумності
- **Тест маржі:** Gross margin > EBITDA margin > Net margin (завжди вірно за визначенням)
- **Обґрунтованість множників:**
  - EV/Revenue: зазвичай 0.5‑20× (значно варіюється за галуззю)
  - EV/EBITDA: зазвичай 8‑25× (досить стабільно між галузями)
  - P/E: зазвичай 10‑50× (залежить від темпів зростання)
- **Кореляція зростання‑множник:** вища швидкість зростання зазвичай означає вищі множники
- **Компроміс розміру‑ефективності:** більші компанії часто мають кращі маржі (ефекти масштабу)

### Поширені помилки, яких слід уникати
❌ Змішування market cap та enterprise value у формулах
❌ Використання різних часових періодів для чисельника та знаменника (LTM vs quarterly)
❌ Жорстке кодування чисел у формулах замість посилань на клітинки
❌ **Жорстко закодовані входи без коментарів клітинок, що вказують джерело **або** пояснюють припущення**
❌ Відсутність гіперпосилань на SEC filing або інші джерела даних, коли вони доступні
❌ Надмірна кількість метрик без чіткого призначення
❌ Включення непорівнянних компаній (різні бізнес‑моделі)
❌ Використання застарілих даних без розкриття
❌ Неправильне обчислення середніх значень відсотків (повинно бути median)

## Розділ 11: Довідник формул

### Основні формули Excel
```excel
// Statistical Functions
=AVERAGE(range)          // Simple mean
=MEDIAN(range)           // Middle value
=QUARTILE(range, 1)      // 25th percentile
=QUARTILE(range, 3)      // 75th percentile
=MAX(range)              // Maximum value
=MIN(range)              // Minimum value
=STDEV.P(range)          // Standard deviation

// Financial Calculations
=B7/C7                   // Simple ratio (Margin)
=SUM(B7:B9)/3            // Average of multiple companies
=IF(B7>0, C7/B7, "N/A")  // Conditional calculation
=IFERROR(C7/D7, 0)       // Handle divide by zero

// Cross-Sheet References
='Sheet1'!B7             // Reference another sheet
=VLOOKUP(A7, Table1, 2)  // Lookup from data table
=INDEX(MATCH())          // Advanced lookup

// Formatting
=TEXT(B7, "0.0%")        // Format as percentage
=TEXT(C7, "#,##0")       // Thousands separator
```

### Формули спільного відношення
```excel
Gross Margin = Gross Profit / Revenue
EBITDA Margin = EBITDA / Revenue
FCF Margin = Free Cash Flow / Revenue
FCF Conversion = FCF / Operating Cash Flow
ROE = Net Income / Shareholders' Equity
ROA = Net Income / Total Assets
Asset Turnover = Revenue / Total Assets
Debt/Equity = Total Debt / Shareholders' Equity
```

## Ключові принципи підсумок

1. **Структура забезпечує інсайт** – правильні заголовки змушують мислити правильно
2. **Менше — більше** – 5‑10 важливих метрик краще, ніж 20 непотрібних
3. **Обирай метрики під своє питання** – аналіз оцінки вартості ≠ аналіз ефективності
4. **Статистика показує патерни** – медіана/квартилі розкривають більше, ніж середнє
5. **Прозорість краща за складність** – прості формули, які розуміє кожен
6. **Порівнянність — це головне** – краще виключити, ніж змушувати погане порівняння
7. **Документуй свої вибори** – поясни, які метрики і чому у розділі нотаток

## Постійне вдосконалення

Після завершення аналізу компенсації запитай:
1. Чи виявила статистика неочікувані інсайти?
2. Чи були прогалини в даних, які обмежували аналіз?
3. Чи запитували зацікавлені сторони метрики, яких ти не включив?
4. Скільки часу це зайняло порівняно з тим, скільки мало зайняти?
5. Що зробить цей аналіз кориснішим наступного разу?

Найкращі аналізи компенсації розвиваються з кожною ітерацією. Зберігай шаблони, вчись на зворотному зв’язку та уточнюй структуру на основі того, що дійсно використовують приймаючі рішення.
## Джерела даних — спочатку MCP, резервний варіант — веб

Багато розділів нижче говорять «використовуй S&P Kensho MCP / Daloopa MCP / FactSet MCP». Це комерційні фінансові MCP з оригінального контексту плагіна Cowork. У Hermes:

- **Якщо у тебе налаштовано будь‑який структурований фінансовий MCP** (Hermes підтримує MCP — дивись навичку `native-mcp`), надавай перевагу йому для порівняння у конкретний момент часу, прецедентних транзакцій та звітів.
- **Інакше** використай резервний варіант:
  - `web_search` / `web_extract` проти SEC EDGAR (`https://www.sec.gov/cgi-bin/browse-edgar`) для американських звітів
  - Сторінки IR компаній для прес‑релізів, презентацій прибутковості
  - `browser_navigate` для інтерактивних порталів даних
  - Дані, надані користувачем (явно запитуй, коли контекст їх не містить)
- **Ніколи не вигадуй**. Якщо число множини, прецеденту чи звіту не може бути отримане, познач клітинку як `[UNSOURCED]` і відобрази це користувачеві.
