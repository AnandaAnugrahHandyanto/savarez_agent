# Діаграми концепцій

Генеруй діаграми у форматі SVG високої якості з уніфікованою плоскою, мінімалістичною системою дизайну. Вихідний файл — це один самодостатній HTML‑файл, який відображається ідентично в будь‑якому сучасному браузері, з автоматичним перемиканням між світлим та темним режимами.
## Scope

**Найкраще підходить для:**
- Фізичні налаштування, хімічні механізми, математичні криві, біологія
- Фізичні об’єкти (літаки, турбіни, смартфони, механічні годинники, клітини)
- Анатомія, перерізи, розгорнуті вигляди шарів
- Плани поверхів, архітектурні перетворення
- Наративні подорожі (життєвий цикл X, процес Y)
- Інтеграції систем «центр‑спиця» (розумне місто, мережі IoT, електричні мережі)
- Освітні/підручникові візуалізації у будь‑якій галузі
- Кількісні діаграми (груповані стовпці, енергетичні профілі)

**Шукайте інше спочатку:**
- Спеціалізоване програмне забезпечення / архітектура хмарної інфраструктури з темною технічною естетикою (розглянь `architecture-diagram`, якщо доступно)
- Ручні ескізи на дошці (розглянь `excalidraw`, якщо доступно)
- Анімовані пояснювальні ролики або відео‑вихід (розглянь навичку анімації)

Якщо існує більш спеціалізована навичка для теми, віддай перевагу їй. Якщо жодна не підходить, ця навичка може слугувати універсальним резервом для SVG‑діаграм — результат матиме чисту освітню естетику, описану нижче, що є розумним варіантом за замовчуванням для майже будь‑якої теми.
## Робочий процес

1. Визнач тип діаграми (див. типи діаграм нижче).
2. Розмісти компоненти, дотримуючись правил Design System.
3. Напиши повну HTML‑сторінку, використовуючи `templates/template.html` як обгортку — встав свій SVG у місце, де шаблон містить `<!-- PASTE SVG HERE -->`.
4. Збережи як окремий файл `.html` (наприклад `~/my-diagram.html` або `./my-diagram.html`).
5. Користувач відкриває його безпосередньо в браузері — без сервера, без залежностей.

Опційно: якщо користувач хоче мати галерею кількох діаграм, дивись розділ «Local Preview Server» внизу.

Завантаження HTML‑шаблону:
```
skill_view(name="concept-diagrams", file_path="templates/template.html")
```

Шаблон вбудовує повну CSS‑систему дизайну (`c-*` колірні класи, текстові класи, змінні light/dark, стилі маркерів стрілок). SVG, який ти генеруєш, покладається на наявність цих класів на хостинговій сторінці.
## Design System

### Філософія

- **Flat**: без градієнтів, тіней, розмиття, сяйва чи неонових ефектів.
- **Minimal**: показуй лише суттєве. Без декоративних іконок всередині блоків.
- **Consistent**: однакові кольори, відступи, типографіка та товщина штрихів у всіх діаграмах.
- **Dark-mode ready**: всі кольори автоматично адаптуються через CSS‑класи — без окремих SVG для кожного режиму.

#### Правила призначення кольорів

Колір кодує **значення**, а не послідовність. Не використовуйте кольори як веселку.

- Групуйте вузли за **категорією** — усі вузли одного типу мають один колір.
- Для нейтральних/структурних вузлів (початок, кінець, загальні кроки, користувачі) використовуйте `c-gray`.
- Використовуйте **2‑3 кольори на діаграму**, а не 6+.
- Для загальних категорій перевагу надавайте `c-purple`, `c-teal`, `c-coral`, `c-pink`.
- `c-blue`, `c-green`, `c-amber`, `c-red` залишайте для семантичного значення (info, success, warning, error).

Відповідність світлих/темних відтінків (обробляється шаблонним CSS — просто використовуйте клас):
- Світлий режим: 50 fill + 600 stroke + 800 title / 600 subtitle
- Темний режим: 800 fill + 200 stroke + 100 title / 200 subtitle

### Типографіка

Лише два розміри шрифтів. Без винятків.

| Клас | Розмір | Товщина | Використання |
|------|--------|---------|--------------|
| `th` | 14px | 500 | Заголовки вузлів, мітки регіонів |
| `ts` | 12px | 400 | Підзаголовки, описи, підписи стрілок |
| `t`  | 14px | 400 | Загальний текст |

- **Sentence case** завжди. Ніколи Title Case, ніколи ALL CAPS.
- Кожен `<text>` ПОВИНЕН мати клас (`t`, `ts` або `th`). Текст без класу заборонений.
- `dominant-baseline="central"` у всьому тексті всередині блоків.
- `text-anchor="middle"` для центрованого тексту в блоках.

**Оцінка ширини (приблизно):**
- 14 px, товщина 500: ~8 px на символ
- 12 px, товщина 400: ~6.5 px на символ
- Завжди перевіряй: `box_width >= (char_count × px_per_char) + 48` (по 24 px відступу з кожного боку)

### Відступи та розташування

- **ViewBox**: `viewBox="0 0 680 H"`, де H = висота контенту + 40 px буфер.
- **Безпечна зона**: x=40 до x=640, y=40 до y=(H‑40).
- **Між блоками**: мінімум 60 px.
- **Всередині блоків**: 24 px горизонтальний відступ, 12 px вертикальний.
- **Проміжок між кінчиком стрілки та краєм блоку**: 10 px.
- **Блок в один рядок**: висота 44 px.
- **Блок у два рядки**: висота 56 px, 18 px між базовими лініями заголовка та підзаголовка.
- **Внутрішній відступ контейнера**: мінімум 20 px.
- **Максимальна глибина вкладеності**: 2‑3 рівні. Глибше стає нечитаємим при ширині 680 px.

### Штрих та форма

- **Товщина штриху**: 0.5 px у всіх межах вузлів. Не 1 px, не 2 px.
- **Закруглення прямокутників**: `rx="8"` для вузлів, `rx="12"` для внутрішніх контейнерів, `rx="16"`‑`rx="20"` для зовнішніх контейнерів.
- **Шляхи з’єднань**: МАЮТЬ мати `fill="none"`. Інакше SVG за замовчуванням застосовує `fill: black`.

### Маркер стрілки

Додайте цей блок `<defs>` на початок **кожного** SVG:

```xml
<defs>
  <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
          markerWidth="6" markerHeight="6" orient="auto-start-reverse">
    <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
          stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
  </marker>
</defs>
```

Використовуйте `marker-end="url(#arrow)"` на лініях. Кінець стрілки успадковує колір лінії через `context-stroke`.

### CSS‑класи (надає шаблон)

Шаблонна сторінка надає:

- Текст: `.t`, `.ts`, `.th`
- Нейтральні: `.box`, `.arr`, `.leader`, `.node`
- Кольорові градієнти: `.c-purple`, `.c-teal`, `.c-coral`, `.c-pink`, `.c-gray`, `.c-blue`, `.c-green`, `.c-amber`, `.c-red` (усі з автоматичним перемиканням світлого/темного режиму)

Ви **не** повинні переозначати їх — просто застосовуйте у вашому SVG. Файл шаблону містить повні визначення CSS.
## SVG Boilerplate

Кожен SVG у шаблонній сторінці починається з точно такої структури:

```xml
<svg width="100%" viewBox="0 0 680 {HEIGHT}" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
            markerWidth="6" markerHeight="6" orient="auto-start-reverse">
      <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
            stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
    </marker>
  </defs>

  <!-- Diagram content here -->

</svg>
```

Заміні `{HEIGHT}` на фактично обчислену висоту (нижня межа останнього елементу + 40 px).

### Node Patterns

**Однорядковий вузол (44 px):**
```xml
<g class="node c-blue">
  <rect x="100" y="20" width="180" height="44" rx="8" stroke-width="0.5"/>
  <text class="th" x="190" y="42" text-anchor="middle" dominant-baseline="central">Service name</text>
</g>
```

**Дворядковий вузол (56 px):**
```xml
<g class="node c-teal">
  <rect x="100" y="20" width="200" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="200" y="38" text-anchor="middle" dominant-baseline="central">Service name</text>
  <text class="ts" x="200" y="56" text-anchor="middle" dominant-baseline="central">Short description</text>
</g>
```

**З’єднувач (без мітки):**
```xml
<line x1="200" y1="76" x2="200" y2="120" class="arr" marker-end="url(#arrow)"/>
```

**Контейнер (пунктирний або суцільний):**
```xml
<g class="c-purple">
  <rect x="40" y="92" width="600" height="300" rx="16" stroke-width="0.5"/>
  <text class="th" x="66" y="116">Container label</text>
  <text class="ts" x="66" y="134">Subtitle info</text>

```

## Типи діаграм

Вибери макет, який підходить темі:

1. **Flowchart** — CI/CD конвеєри, життєвий цикл запитів, процеси схвалення, обробка даних. Потік в одному напрямку (зверху вниз або зліва направо). Максимум 4‑5 вузлів у рядку.
2. **Structural / Containment** — Вкладеність хмарної інфраструктури, архітектура системи з шарами. Великі зовнішні контейнери з внутрішніми регіонами. Пунктирні прямокутники для логічних групувань.
3. **API / Endpoint Map** — REST‑маршрути, схеми GraphQL. Дерево від кореня, розгалуження до груп ресурсів, кожна з яких містить вузли кінцевих точок.
4. **Microservice Topology** — Сервісна сітка, подієво‑орієнтовані системи. Сервіси як вузли, стрілки для шаблонів комунікації, черги повідомлень між ними.
5. **Data Flow** — ETL конвеєри, потокові архітектури. Потік зліва направо від джерел через обробку до приймачів.
6. **Physical / Structural** — Транспортні засоби, будівлі, апаратне забезпечення, анатомія. Використовуй форми, що відповідають фізичній формі — `<path>` для вигнутих корпусів, `<polygon>` для звужених форм, `<ellipse>`/`<circle>` для циліндричних частин, вкладені `<rect>` для відсіків. Дивись `references/physical-shape-cookbook.md`.
7. **Infrastructure / Systems Integration** — Розумні міста, IoT‑мережі, багатодоменні системи. Макет «центр‑спиця» з центральною платформою, що з’єднує підсистеми. Семантичні стилі ліній (`.data-line`, `.power-line`, `.water-pipe`, `.road`). Дивись `references/infrastructure-patterns.md`.
8. **UI / Dashboard Mockups** — Адміністративні панелі, моніторингові дашборди. Рамка екрану з вкладеними елементами діаграм/приборів/індикаторів. Дивись `references/dashboard-patterns.md`.

Для фізичних, інфраструктурних та дашборд‑діаграм завантаж відповідний файл‑посилання перед генерацією — кожен з них містить готові CSS‑класи та примітиви форм.
## Перевірочний список валідації

Перш ніж завершувати будь‑який SVG, переконайся, що **всі** наступні пункти виконані:

1. Кожен `<text>` має клас `t`, `ts` або `th`.
2. Кожен `<text>` всередині боксу має `dominant-baseline="central"`.
3. Кожен з’єднувальний `<path>` або `<line>`, що використовується як стрілка, має `fill="none"`.
4. Жодна стрілка не перетинає жодного іншого боксу.
5. `box_width >= (longest_label_chars × 8) + 48` для тексту розміром 14 px.
6. `box_width >= (longest_label_chars × 6.5) + 48` для тексту розміром 12 px.
7. Висота `ViewBox` = найнижчий елемент + 40 px.
8. Увесь вміст знаходиться в межах `x=40` до `x=640`.
9. Класи кольорів (`c-*`) розташовуються на `<g>` або елементах‑фігурах, ніколи не на `<path>`‑з’єднувачах.
10. Присутній блок `<defs>` для стрілок.
11. Не використовується градієнти, тіні, розмиття або ефекти світіння.
12. Ширина штриху становить 0,5 px на всіх межах вузлів.

### За бажанням: локальний сервер попереднього перегляду (галерея з кількома діаграмами)

**Правила:**

- Сервер є необов’язковим і включається лише за запитом — спочатку надавай окремий HTML‑файл.

Рекомендований шаблон (дозволяє ОС вибрати вільний епхемерний порт):

```bash
# Put each diagram in its own folder under .diagrams/
mkdir -p .diagrams/sn2-mechanism
# ...write .diagrams/sn2-mechanism/index.html...

# Serve on loopback only, free port
cd .diagrams && python3 -c "
import http.server, socketserver
with socketserver.TCPServer(('127.0.0.1', 0), http.server.SimpleHTTPRequestHandler) as s:
    print(f'Serving at http://127.0.0.1:{s.server_address[1]}/')
    s.serve_forever()
" &
```

Якщо користувач наполягає на фіксованому порту, використай `127.0.0.1:<port>` — все одно ніколи `0.0.0.0`. Документуй, як зупинити сервер (`kill %1` або `pkill -f "http.server"`).
