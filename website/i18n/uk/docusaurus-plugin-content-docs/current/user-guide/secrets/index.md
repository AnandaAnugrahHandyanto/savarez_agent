# Secrets

Hermes може отримувати API‑ключі з зовнішніх менеджерів секретів під час запуску процесу замість зберігання їх у `~/.hermes/.env`. Bootstrap‑токен для менеджера секретів зберігається в `.env`; усі інші ключі провайдерів (OpenAI, Anthropic, OpenRouter тощо) можуть залишатися в менеджері та оновлюватися централізовано.

Підтримувані:

- [Bitwarden Secrets Manager](./bitwarden) — `bws` CLI, lazy‑installed, безкоштовний тариф працює.

Інші бекенди (Vault, AWS Secrets Manager, 1Password CLI) легко додати за тим же інтерфейсом — достатньо створити один модуль у `agent/secret_sources/` і один CLI‑обробник. Подай запит, якщо маєш на увазі конкретний бекенд.
