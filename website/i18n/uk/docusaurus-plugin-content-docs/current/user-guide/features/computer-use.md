# Використання комп’ютера (macOS)

Hermes Agent може керувати робочим столом твого Mac — клацати, вводити текст, прокручувати,
перетягувати — у **фоновому** режимі. Курсор не переміщується, фокус клавіатури
не змінюється, і macOS не перемикає Spaces. Ти і агент
спільно працюєте на одній машині.

На відміну від більшості інтеграцій для використання комп’ютера, це працює з **будь‑яким інструментально‑здатним
моделем** — Claude, GPT, Gemini або відкритою моделлю на локальній vLLM‑точці.
Не потрібно турбуватись про схеми, специфічні для Anthropic.

## Як це працює

Набір інструментів `computer_use` спілкується з MCP через stdio до [`cua-driver`](https://github.com/trycua/cua),
macOS‑драйвера, який використовує приватні SPI SkyLight (`SLEventPostToPid`,
`SLPSPostEventRecordTo`) та SPI `_AXObserverAddNotificationAndCheckRemote`
для доступності, щоб:

- Надсилати синтезовані події безпосередньо до цільових процесів — без HID‑tap,
  без переміщення курсора.
- Перемикати активний стан AppKit без підняття вікон — без перемикання Space.
- Підтримувати дерева доступності Chromium/Electron живими, коли вікна
  приховані.

Це поєднання — те, що OpenAI Codex називає «фонова робота з комп’ютером».
cua-driver є його відкритим аналогом.

## Увімкнення

Обери будь‑який зручний шлях — обидва запускають один і той же upstream‑установник:

**Варіант 1: окрема команда CLI (найпряміший).**

```
hermes computer-use install
```

Це завантажує та запускає upstream‑установник cua-driver:
`curl -fsSL https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.sh`.
Використай `hermes computer-use status`, щоб перевірити встановлення.

**Варіант 2: увімкнути набір інструментів інтерактивно.**

1. Запусти `hermes tools`, вибери `🖱️ Computer Use (macOS)` → `cua-driver (background)`.
2. Налаштування запускає upstream‑установник (те саме, що у Варіанті 1).

Після встановлення, незалежно від обраного шляху:

3. Надай macOS доступ, коли буде запит:
   - **System Settings → Privacy & Security → Accessibility** → дозволь
     терміналу (або Hermes‑додатку).
   - **System Settings → Privacy & Security → Screen Recording** → дозволь
     те саме.
4. Запусти сесію з увімкненим набором інструментів:
   ```
   hermes -t computer_use chat
   ```
   або додай `computer_use` до `enabled toolsets` у `~/.hermes/config.yaml`.

## Оновлення cua-driver

Проєкт cua-driver регулярно випускає виправлення (наприклад, v0.1.6 виправив баг
з фокусом вікна Safari для робочих процесів UTM). Hermes оновлює бінарник у двох
місцях, щоб ти не залишився на застарілій версії:

- **`hermes update`** — коли оновлюєш сам Hermes, якщо `cua-driver` є в PATH,
  upstream‑установник запускається знову в кінці оновлення.
  Не робить нічого для користувачів не‑macOS та для тих, у кого cua-driver не встановлено.
- **`hermes computer-use install --upgrade`** — ручне примусове оновлення.
  Запускає upstream‑установник незалежно від того, чи встановлено вже cua-driver.
  Використовуй, коли потрібне останнє виправлення без очікування наступного оновлення агента.

`hermes computer-use status` показує встановлену версію поруч із шляхом до бінарника.

## Швидкий приклад

Запит користувача: *«Знайди моє останнє листування від Stripe і підсумуй, що вони хочуть, щоб я зробив.»*

План агента:

1. `computer_use(action="capture", mode="som", app="Mail")` — зробити
   скріншот Mail з нумерацією всіх елементів бокової панелі, кнопок
   інструментальної панелі та рядків листів.
2. `computer_use(action="click", element=14)` — клацнути поле пошуку
   (елемент #14 зі скріншоту).
3. `computer_use(action="type", text="from:stripe")`
4. `computer_use(action="key", keys="return", capture_after=True)` — відправити
   і отримати новий скріншот.
5. Клацнути верхній результат, прочитати тіло, підсумувати.

Протягом усього часу курсор залишається там, де ти його залишив, а Mail ніколи
не піднімається на передній план.

## Сумісність провайдерів

| Provider | Vision? | Works? | Notes |
|---|---|---|---|
| Anthropic (Claude Sonnet/Opus 3+) | ✅ | ✅ | Найкраще загалом; SOM + чисті координати. |
| OpenRouter (any vision model) | ✅ | ✅ | Підтримуються багаточастинні повідомлення інструментів. |
| OpenAI (GPT-4+, GPT-5) | ✅ | ✅ | Те саме, що вище. |
| Local vLLM / LM Studio (vision model) | ✅ | ✅ | Якщо модель підтримує багаточастинний інструментний контент. |
| Text‑only models | ❌ | ✅ (degraded) | Використовуй `mode="ax"` для роботи лише з деревом доступності. |

Скріншоти надсилаються інлайн разом з результатами інструментів у вигляді частин `image_url` у стилі OpenAI. Для Anthropic адаптер перетворює їх у нативні блоки `tool_result`‑зображень.

## Безпека

Hermes застосовує багаторівневі захисні механізми:

- Деструктивні дії (click, type, drag, scroll, key, focus_app) вимагають
  схвалення — або інтерактивно через діалог CLI, або через кнопки
  схвалення у платформі обміну повідомленнями.
- Жорстко заблоковані комбінації клавіш на рівні інструменту: очистити смітник, примусове видалення,
  блокування екрану, вихід з системи, примусовий вихід.
- Жорстко заблоковані шаблони вводу: `curl | bash`, `sudo rm -rf /`, fork‑бомби тощо.
- Системний промпт агента явно забороняє: кліки по діалогах дозволу, ввід паролів,
  виконання інструкцій, вбудованих у скріншоти.

Додай `approvals.mode: manual` у `~/.hermes/config.yaml`, якщо хочеш, щоб кожна дія підтверджувалась.

## Ефективність токенів

Скріншоти дорогі. Hermes застосовує чотири рівні оптимізації:

- **Видалення скріншотів** — адаптер Anthropic зберігає лише 3 останні скріншоти в контексті; старі замінюються плейсхолдерами `[screenshot removed to save context]`.
- **Обрізка стиснення на клієнті** — компресор контексту виявляє багатомодальні результати інструментів і видаляє частини зображень зі старих.
- **Оцінка токенів з урахуванням зображень** — кожне зображення рахується як ~1500 токенів (фіксована ставка Anthropic) замість довжини base64‑рядка.
- **Редагування контексту на сервері (лише Anthropic)** — коли активовано, адаптер вмикає `clear_tool_uses_20250919` через `context_management`, і API Anthropic очищає старі результати інструментів на боці сервера.

Сесія з 20 діями на дисплеї 1568×900 зазвичай коштує ~30 К токенів скріншотного контексту, а не ~600 К.

## Обмеження

- **Лише macOS.** cua-driver використовує приватні Apple SPI, яких немає в Linux чи Windows. Для крос‑платформенної автоматизації GUI використай набір інструментів `browser`.
- **Ризик приватних SPI.** Apple може змінити символічну поверхню SkyLight у будь‑якому оновленні OS. Зафіксуй версію драйвера за допомогою змінної середовища `HERMES_CUA_DRIVER_VERSION`, якщо потрібна відтворюваність після оновлення macOS.
- **Продуктивність.** Фоновий режим повільніший за передній — події, маршрутизовані через SkyLight, займають ~5‑20 мс проти прямого HID‑постингу. Не помітно при звичайних кліках агента; помітно, якщо намагаєшся записати спід‑ран.
- **Немає вводу паролів з клавіатури.** `type` має жорсткі блоки для командних payload‑ів; для паролів використай автозаповнення системи.

## Конфігурація

Перевизначити шлях до бінарника драйвера (тести / CI):

```
HERMES_CUA_DRIVER_CMD=/opt/homebrew/bin/cua-driver
HERMES_CUA_DRIVER_VERSION=0.5.0    # optional pin
```

Повністю замінити бекенд (для тестування):

```
HERMES_COMPUTER_USE_BACKEND=noop   # records calls, no side effects
```

## Устранення проблем

**`computer_use backend unavailable: cua-driver is not installed`** — Запусти
`hermes computer-use install`, щоб завантажити бінарник cua-driver, або
використай `hermes tools` і увімкни набір інструментів Computer Use.

**Кліки не мають ефекту** — зроби capture і перевір. Можливо, модальне вікно,
яке ти не помітив, блокує ввід. Закрий його за допомогою `escape` або кнопки закриття.

**Індекси елементів застаріли** — SOM‑індекси дійсні лише до наступного `capture`. Зроби новий capture після будь‑якої дії, що змінює стан.

**"blocked pattern in type text"** — Текст, який ти намагався `type`,
відповідає списку небезпечних шаблонів. Розбий команду на частини або переглянь її.
