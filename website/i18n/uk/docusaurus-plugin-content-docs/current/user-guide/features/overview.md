# Огляд можливостей

Hermes Agent включає багатий набір функцій, які виходять далеко за межі базового чату. Від постійної пам’яті та контексту, що враховує файли, до автоматизації браузера та голосових розмов — ці можливості працюють разом, роблячи Hermes потужним автономним помічником.

:::tip Не знаєш, з чого почати?
`hermes setup --portal` охоплює постачальника моделей та всі чотири інструменти Tool Gateway (веб‑пошук, генерація зображень, TTS, браузер) в одній команді. Дивись [Nous Portal](/integrations/nous-portal).
:::

## Core

- **[Tools & Toolsets](tools.md)** — Інструменти — це функції, які розширюють можливості агента. Вони організовані в логічні набори інструментів, які можна вмикати або вимикати per platform, охоплюючи веб‑пошук, виконання в терміналі, редагування файлів, пам’ять, делегування та інше.
- **[Skills System](skills.md)** — Документи знань за запитом, які агент може завантажити за потреби. Skills слідують патерну прогресивного розкриття, щоб мінімізувати використання токенів, і сумісні з відкритим стандартом [agentskills.io](https://agentskills.io/specification).
- **[Persistent Memory](memory.md)** — Обмежена, курована пам’ять, що зберігається між сесіями. Hermes пам’ятає твої уподобання, проєкти, середовище та те, чого навчився через `MEMORY.md` та `USER.md`.
- **[Context Files](context-files.md)** — Hermes автоматично виявляє та завантажує файли контексту проєкту (`.hermes.md`, `AGENTS.md`, `CLAUDE.md`, `SOUL.md`, `.cursorrules`), які формують його поведінку у твоєму проєкті.
- **[Context References](context-references.md)** — Введи `@` і посилання, щоб вставити файли, папки, git‑diff’и та URL‑адреси безпосередньо у свої повідомлення. Hermes розгортає посилання inline і автоматично додає вміст.
- **[Checkpoints](../checkpoints-and-rollback.md)** — Hermes автоматично створює знімки твоєї робочої теки перед внесенням змін у файли, забезпечуючи захисну сітку для відкату за допомогою `/rollback`, якщо щось піде не так.

## Automation

- **[Scheduled Tasks (Cron)](cron.md)** — Плануй завдання для автоматичного виконання за допомогою природної мови або cron‑виразів. Завдання можуть підключати skills, доставляти результати на будь‑яку платформу та підтримують операції паузи/продовження/редагування.
- **[Subagent Delegation](delegation.md)** — Інструмент `delegate_task` створює дочірні інстанси агента з ізольованим контекстом, обмеженими наборами інструментів та власними термінальними сесіями. За замовчуванням запускає 3 одночасних субагенти (конфігуровано) для паралельних робочих потоків.
- **[Code Execution](code-execution.md)** — Інструмент `execute_code` дозволяє агенту писати Python‑скрипти, які програмно викликають інструменти Hermes, стискаючи багатокрокові робочі процеси в один хід LLM через пісочничне RPC‑виконання.
- **[Event Hooks](hooks.md)** — Запускай власний код у ключових точках життєвого циклу. Hook‑и шлюзу обробляють логування, сповіщення та вебхуки; hook‑и плагінів — перехоплення інструментів, метрики та захисні бар’єри.
- **[Batch Processing](batch-processing.md)** — Запускай агента Hermes на сотнях чи тисячах запитів паралельно, генеруючи структуровані дані траєкторії у форматі ShareGPT для створення навчальних наборів або оцінки.

## Media & Web

- **[Voice Mode](voice-mode.md)** — Повна голосова взаємодія в CLI та платформах обміну повідомленнями. Говори з агентом через мікрофон, слухай його відповіді та веди живі голосові розмови у голосових каналах Discord.
- **[Browser Automation](browser.md)** — Повна автоматизація браузера з кількома бекендами: хмара Browserbase, хмара Browser Use, локальний Chrome/Brave/Chromium/Edge через CDP або локальний Chromium. Навігай сайти, заповнюй форми та витягай інформацію.
- **[Vision & Image Paste](vision.md)** — Підтримка мультимодальної візії. Вставляй зображення з буфера обміну у CLI та проси агента аналізувати, описувати або працювати з ними за допомогою будь‑якої моделі, що підтримує візію.
- **[Image Generation](image-generation.md)** — Генеруй зображення за текстовими підказками через FAL.ai. Підтримується дев’ять моделей (FLUX 2 Klein/Pro, GPT‑Image 1.5/2, Nano Banana Pro, Ideogram V3, Recraft V4 Pro, Qwen, Z‑Image Turbo); обери одну за допомогою `hermes tools`.
- **[Voice & TTS](tts.md)** — Вивід тексту в мову та транскрипція голосових повідомлень на всіх платформах обміну, з десятьма вбудованими провайдерами: Edge TTS (безкоштовно), ElevenLabs, OpenAI TTS, MiniMax, Mistral Voxtral, Google Gemini, xAI, NeuTTS, KittenTTS та Piper — плюс кастомні провайдери команд для будь‑якого локального TTS CLI.

## Integrations

- **[MCP Integration](mcp.md)** — Підключення до будь‑якого сервера MCP через stdio або HTTP‑транспорт. Доступ до зовнішніх інструментів з GitHub, баз даних, файлових систем та внутрішніх API без написання власних інструментів Hermes. Включає фільтрацію інструментів per server та підтримку семплінгу.
- **[Provider Routing](provider-routing.md)** — Тонке керування тим, які AI‑провайдери обробляють твої запити. Оптимізуй за вартістю, швидкістю або якістю за допомогою сортування, біллістів, чорних листів та пріоритетного порядку.
- **[Fallback Providers](fallback-providers.md)** — Автоматичний запасний варіант до резервних LLM‑провайдерів, коли основна модель повертає помилки, включаючи окремий запасний варіант для допоміжних завдань, таких як візія та компресія.
- **[Credential Pools](credential-pools.md)** — Розподіляй API‑виклики між кількома ключами одного провайдера. Автоматичне чергування при обмеженнях швидкості або збоях.
- **[Prompt caching](../configuration#prompt-caching)** — Вбудований крос‑сесійний кеш префіксів на 1 годину для Claude на нативному Anthropic, OpenRouter та Nous Portal. Завжди ввімкнено; налаштування не потрібні.
- **[Memory Providers](memory-providers.md)** — Підключай зовнішні бекенди пам’яті (Honcho, OpenViking, Mem0, Hindsight, Holographic, RetainDB, ByteRover, Supermemory) для крос‑сесійного моделювання користувачів та персоналізації поза вбудованою системою пам’яті.
- **[API Server](api-server.md)** — Експонуй Hermes як сумісну з OpenAI HTTP‑точку. Підключай будь‑який фронтенд, що розуміє формат OpenAI — Open WebUI, LobeChat, LibreChat та інші.
- **[IDE Integration (ACP)](acp.md)** — Використовуй Hermes у редакторах, сумісних з ACP, таких як VS Code, Zed та JetBrains. Чат, активність інструментів, diff‑файлів та термінальні команди відображаються всередині редактора.
- **[Batch Processing](batch-processing.md)** — Запускай агента над багатьма запитами або завданнями паралельно з CLI, отримуючи структуровані виходи та траєкторії, придатні для оцінок або подальших навчальних конвеєрів.

## Customization

- **[Personality & SOUL.md](personality.md)** — Повністю налаштовувана особистість агента. `SOUL.md` — основний файл ідентичності, перше, що потрапляє в system prompt, і ти можеш замінювати його вбудованими або кастомними пресетами `/personality` per session.
- **[Skins & Themes](skins.md)** — Налаштуй візуальне оформлення CLI: кольори банера, обличчя спінерів та дієслова, мітки у вікні відповіді, брендований текст та префікс активності інструментів.
- **[Plugins](plugins.md)** — Додавай власні інструменти, hook‑и та інтеграції без зміни ядра. Три типи плагінів: загальні плагіни (інструменти/hook‑и), провайдери пам’яті (знання між сесіями) та контекстні движки (альтернативне управління контекстом). Керується через уніфікований інтерактивний UI `hermes plugins`.
