# Режим результату

Коли Hermes Agent працює всередині шлюзу обміну повідомленнями (Slack, Discord, Telegram,
WhatsApp, Signal тощо), він може доставляти згенеровані файли безпосередньо в
чат — не як шляхи, які користувачеві доведеться копіювати, а як рідні вкладення.

Діаграма відображається як вбудоване зображення. PDF‑звіт — як файл для
завантаження. Таблицю можна завантажити у форматі `.xlsx`. Агенту не потрібно
писати тег `MEDIA:` чи робити щось особливе — він просто генерує файл і згадує його
абсолютний шлях у відповіді. Шлюз витягує шлях із тексту, видаляє його з видимого
повідомлення та завантажує файл як рідне вкладення.

## Як це працює

Три частини працюють разом:

1. **У агента є інструменти, які створюють файли.** `execute_code` для діаграм через
   matplotlib, навичка `latex-pdf-report` для PDF, навичка `powerpoint` для презентацій,
   `image_generate` для зображень, `text_to_speech` для аудіо тощо.

2. **Шлюз сканує відповіді агента на предмет шляхів до файлів.** Будь‑який абсолютний шлях
   (`/tmp/...`) або шлях, відносний до домашньої теки (`~/...`), що закінчується підтримуваним
   розширенням, витягується. Шляхи всередині блоків коду та інлайн‑коду ігноруються, щоб
   зразки коду не були пошкоджені.

3. **Шлюз розподіляє за типом файлу.** Зображення вбудовуються в рядок, якщо платформа це підтримує;
   відео вбудовуються в рядок; аудіо надсилається як голосове/аудіо‑вкладення; все інше — як файл‑вкладення.

## Підтримувані розширення файлів

| Категорія | Розширення | Доставка |
|---|---|---|
| Зображення | `.png .jpg .jpeg .gif .webp .bmp .tiff .svg` | Вбудоване в рядок |
| Відео | `.mp4 .mov .avi .mkv .webm` | Вбудоване в рядок (де підтримується) |
| Аудіо | `.mp3 .wav .ogg .m4a .flac` | Голосове / аудіо‑вкладення |
| Документи | `.pdf .docx .doc .odt .rtf .txt .md` | Завантаження файлу |
| Дані | `.xlsx .xls .csv .tsv .json .xml .yaml .yml` | Завантаження файлу |
| Презентації | `.pptx .ppt .odp` | Завантаження файлу |
| Архіви | `.zip .tar .gz .tgz .bz2 .7z` | Завантаження файлу |
| Веб | `.html .htm` | Завантаження файлу |

`.py`, `.log` та інші розширення вихідних файлів навмисно виключені, щоб агент не надсилав
випадкові вихідні файли; якщо потрібно відправити код користувачу, використай блок коду.

## Заохочення агента створювати артефакти

За замовчуванням агент не генерує артефакти — йому треба це явно вказати. Два способи підштовхнути його:

**Per‑session:** запитай явно («надішли мені порівняння у вигляді діаграми»,
«поверни дані у CSV») або створити власні інструкції/особистість, які схиляють до
відповідей у вигляді артефактів у месенджерах.

**Project‑level:** додай упереджувальний запис у `AGENTS.md` / `CLAUDE.md` /
`.cursorrules` у проєкті, у глобальну персональність у `~/.hermes/SOUL.md`, або як
іменований пресет у `agent.personalities` у `~/.hermes/config.yaml` (перемикається
після сесії через `/personality`).

Механізм, який агент має використовувати, простий: згенерувати файл за
абсолютним шляхом (наприклад `/tmp/q3-revenue.png`) і згадати цей шлях у
простому тексті відповіді. Шлюз робить усе інше. Шляхи всередині блоків коду або
обратних лапок ігноруються, щоб зразки коду не були пошкоджені.

## Kanban: артефакти у сповіщеннях про завершення

Якщо ти користуєшся багатофункціональним робочим процесом kanban у Hermes,
робітники можуть прикріплювати файли результату до виклику `kanban_complete`:

```python
kanban_complete(
    summary="rendered Q3 revenue chart and report",
    artifacts=[
        "/tmp/q3-revenue.png",
        "/tmp/q3-report.pdf",
    ],
)
```

Коли шлюз‑повідомлювач надсилає повідомлення «завдання завершено» всім, хто підписався
на завдання у Slack/Telegram тощо, він також завантажує кожен артефакт як рідне
вкладення у чат. Людина отримує результат і підсумок в одному місці.

Файли, які не існують на диску під час роботи повідомлювача, пропускаються без повідомлення.

## Підключення інших сервісів через MCP

Окрім конвеєра доставки артефактів, агент може взаємодіяти з іншими сервісами через MCP (Model Context Protocol).
Екосистема MCP постачається з серверними реалізаціями для більшості популярних інструментів — встанови ті, які потрібні:

| Сервіс | Що відкриває |
|---|---|
| **Notion** | Читання/запис сторінок, баз даних, запити робочого простору |
| **GitHub** | Issues, PR, коментарі, пошук у репозиторіях поза CLI gh |
| **Linear** | Тікети, проєкти, цикли |
| **Slack** | Пошук по всьому робочому простору, читання інших каналів |
| **Gmail** | Тріаж вхідних, надсилання листів, управління мітками |
| **Salesforce** | Ліди, можливості, дані облікових записів |
| **Snowflake / BigQuery** | SQL‑запити до сховищ даних |
| **Google Drive** | Пошук файлів, вміст, управління спільним доступом |

Встанови сервери MCP у `~/.hermes/config.yaml` у розділі `mcp_servers`. Дивись
[MCP integration](./mcp.md) для повного посібника з налаштування.

## Порівняння з Perplexity Computer у Slack

Інтеграція Perplexity Computer у Slack базується на тому ж принципі: агент генерує артефакт (діаграму, PDF, презентацію) і публікує його у темі як рідне вкладення. Режим результату Hermes Agent надає той самий користувацький патерн локально:

- Генерація відбувається у власному venv/пісочниці користувача (без віддаленого тенанта).
- Файли потрапляють у чат через той самий Slack API `files.uploadV2`.
- Широта підключень забезпечується MCP, а не каталогом з 400 готових інтеграцій — встановлюй лише ті, що дійсно використовуєш.

OAuth‑токени залишаються на машині користувача у `auth.json` / `.env`. Ніякого хостингу токенів. Ніякого мульти‑тенантного microVM. Той самий кінцевий результат.
