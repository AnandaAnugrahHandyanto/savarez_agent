# Посилання на контекст

Введи `@` і посилання, щоб безпосередньо вставити вміст у своє повідомлення. Hermes розширює посилання в рядку та додає вміст у розділ `--- Attached Context ---`.

## Підтримувані посилання

| Синтаксис | Опис |
|-----------|------|
| `@file:path/to/file.py` | Вставити вміст файлу |
| `@file:path/to/file.py:10-25` | Вставити конкретний діапазон рядків (нумерація з 1, включно) |
| `@folder:path/to/dir` | Вставити дерево каталогу з метаданими файлів |
| `@diff` | Вставити `git diff` (незастейджені зміни) |
| `@staged` | Вставити `git diff --staged` (застейджені зміни) |
| `@git:5` | Вставити останні N комітів з патчами (максимум 10) |
| `@url:https://example.com` | Завантажити та вставити вміст веб‑сторінки |

## Приклади використання

```text
Review @file:src/main.py and suggest improvements

What changed? @diff

Compare @file:old_config.yaml and @file:new_config.yaml

What's in @folder:src/components?

Summarize this article @url:https://arxiv.org/abs/2301.00001
```

Кілька посилань можна використати в одному повідомленні:

```text
Check @file:main.py, and also @file:test.py.
```

Заключна пунктуація (`,`, `.`, `;`, `!`, `?`) автоматично видаляється зі значень посилань.

## Автодоповнення в CLI

У інтерактивному CLI введення `@` викликає автодоповнення:

- `@` показує всі типи посилань (`@diff`, `@staged`, `@file:`, `@folder:`, `@git:`, `@url:`)
- `@file:` і `@folder:` запускають автодоповнення шляхів файлової системи з метаданими розміру файлу
- Саме `@`, за яким слідує частковий текст, показує відповідні файли та папки з поточного каталогу

## Діапазони рядків

Посилання `@file:` підтримує діапазони рядків для точного вставлення вмісту:

```text
@file:src/main.py:42        # Single line 42
@file:src/main.py:10-25     # Lines 10 through 25 (inclusive)
```

Рядки нумеруються з 1. Неправильні діапазони ігноруються без повідомлення (повертається весь файл).

## Обмеження розміру

Посилання на контекст мають обмеження, щоб не перевантажити вікно контексту моделі:

| Поріг | Значення | Поведінка |
|-------|----------|----------|
| М’яке обмеження | 25 % довжини контексту | Додається попередження, розширення продовжується |
| Жорстке обмеження | 50 % довжини контексту | Розширення відхиляється, оригінальне повідомлення повертається без змін |
| Записи у папці | максимум 200 файлів | Надлишкові записи замінюються на `- ...` |
| Коміти Git | максимум 10 | `@git:N` обмежується діапазоном [1, 10] |

## Безпека

### Блокування чутливих шляхів

Ці шляхи завжди блокуються для посилань `@file:` задля запобігання витоку облікових даних:

- SSH‑ключі та конфігурація: `~/.ssh/id_rsa`, `~/.ssh/id_ed25519`, `~/.ssh/authorized_keys`, `~/.ssh/config`
- Профілі оболонки: `~/.bashrc`, `~/.zshrc`, `~/.profile`, `~/.bash_profile`, `~/.zprofile`
- Файли облікових даних: `~/.netrc`, `~/.pgpass`, `~/.npmrc`, `~/.pypirc`
- Hermes‑env: `$HERMES_HOME/.env`

Ці каталоги повністю блокуються (будь‑який файл всередині):

- `~/.ssh/`, `~/.aws/`, `~/.gnupg/`, `~/.kube/`, `$HERMES_HOME/skills/.hub/`

### Захист від обходу шляхів

Усі шляхи розв’язуються відносно робочого каталогу. Посилання, які виходять за межі дозволеного кореня робочої області, відхиляються.

### Виявлення бінарних файлів

Бінарні файли визначаються за MIME‑типом та скануванням на нуль‑байти. Відомі текстові розширення (`.py`, `.md`, `.json`, `.yaml`, `.toml`, `.js`, `.ts` тощо) обходять MIME‑детекцію. Бінарні файли відхиляються з попередженням.

## Доступність на платформах

Посилання на контекст — це в першу чергу **функція CLI**. Вони працюють в інтерактивному CLI, де `@` викликає автодоповнення, а посилання розширюються перед відправкою повідомлення агенту.

У **месенджерах** (Telegram, Discord тощо) синтаксис `@` не розширюється шлюзом — повідомлення передається «як є». Сам агент все одно може звертатися до файлів за допомогою інструментів `read_file`, `search_files` та `web_extract`.

## Взаємодія з компресією контексту

Коли контекст розмови стискається, розширений вміст посилання включається у підсумок стискання. Це означає:

- Великі файли, вставлені через `@file:`, споживають контекст
- При подальшому стисканні вміст файлу підсумовується (не зберігається дослівно)
- Для дуже великих файлів краще використовувати діапазони рядків (`@file:main.py:100-200`), щоб вставити лише потрібні частини

## Поширені шаблони

```text
# Code review workflow
Review @diff and check for security issues

# Debug with context
This test is failing. Here's the test @file:tests/test_auth.py
and the implementation @file:src/auth.py:50-80

# Project exploration
What does this project do? @folder:src @file:README.md

# Research
Compare the approaches in @url:https://arxiv.org/abs/2301.00001
and @url:https://arxiv.org/abs/2301.00002
```

## Обробка помилок

Неправильні посилання генерують попередження в рядку, а не помилки:

| Умова | Поведінка |
|-------|----------|
| Файл не знайдено | Попередження: «file not found» |
| Бінарний файл | Попередження: «binary files are not supported» |
| Папку не знайдено | Попередження: «folder not found» |
| Помилка Git‑команди | Попередження з `git stderr` |
| URL не повернув вміст | Попередження: «no content extracted» |
| Чутливий шлях | Попередження: «path is a sensitive credential file» |
| Шлях поза робочою областю | Попередження: «path is outside the allowed workspace» |
