# Генерація зображень

Hermes Agent генерує зображення за текстовими підказками через FAL.ai. Підтримуються одинадцять моделей «з коробки», кожна з різними компромісами швидкості, якості та вартості. Активну модель можна налаштувати за допомогою `hermes tools`, і вона зберігається у `config.yaml`.

## Налаштування

:::tip Nous Subscribers
Якщо у тебе є платна підписка на [Nous Portal](https://portal.nousresearch.com), ти можеш користуватись генерацією зображень через **[Tool Gateway](tool-gateway.md)** без ключа API FAL. Вибір моделі зберігається для обох шляхів. Нові інсталяції можуть виконати `hermes setup --portal` для входу та одночасного ввімкнення всіх інструментів шлюзу; існуючі інсталяції можуть обрати **Nous Subscription** як бекенд генерації зображень за допомогою `hermes tools`.

Якщо керований шлюз повертає `HTTP 4xx` для конкретної моделі, ця модель ще не проксірується на стороні порталу — агент повідомить про це та запропонує кроки виправлення (встановити `FAL_KEY` для прямого доступу або вибрати іншу модель).
:::

### Отримати ключ API FAL

1. Зареєструйся на [fal.ai](https://fal.ai/)
2. Згенеруй API‑ключ у своєму кабінеті

### Налаштувати та вибрати модель

Запусти команду tools:

```bash
hermes tools
```

Перейди до **🎨 Image Generation**, вибери бекенд (Nous Subscription або FAL.ai), після чого у таблиці‑вибірнику будуть показані всі підтримувані моделі у колонковому вигляді — клавішами‑стрілками навігація, Enter — вибір:

```
  Model                          Speed    Strengths                    Price
  fal-ai/flux-2/klein/9b         <1s      Fast, crisp text             $0.006/MP   ← currently in use
  fal-ai/flux-2-pro              ~6s      Studio photorealism          $0.03/MP
  fal-ai/z-image/turbo           ~2s      Bilingual EN/CN, 6B          $0.005/MP
  ...
```

Твій вибір зберігається у `config.yaml`:

```yaml
image_gen:
  model: fal-ai/flux-2/klein/9b
  use_gateway: false            # true if using Nous Subscription
```

### Якість GPT‑Image

Запит якості для `fal-ai/gpt-image-1.5` та `fal-ai/gpt-image-2` зафіксовано на рівні `medium` (~$0.034–$0.06/image при 1024×1024). Ми не надаємо користувачеві варіанти `low` / `high`, щоб білінг у Nous Portal залишався передбачуваним для всіх — різниця у вартості між рівнями 3–22×. Якщо потрібен дешевший варіант, обери Klein 9B або Z‑Image Turbo; якщо потрібна вища якість — Nano Banana Pro або Recraft V4 Pro.

## Використання

Схема, що бачить агент, навмисно мінімальна — модель бере те, що ти налаштував:

```
Generate an image of a serene mountain landscape with cherry blossoms
```

```
Create a square portrait of a wise old owl — use the typography model
```

```
Make me a futuristic cityscape, landscape orientation
```

## Співвідношення сторін

Кожна модель приймає ті ж три співвідношення сторін з точки зору агента. Внутрішньо нативний розмір моделі заповнюється автоматично:

| Agent input | image_size (flux/z-image/qwen/recraft/ideogram) | aspect_ratio (nano-banana-pro) | image_size (gpt-image-1.5) | image_size (gpt-image-2) |
|---|---|---|---|---|
| `landscape` | `landscape_16_9` | `16:9` | `1536x1024` | `landscape_4_3` (1024×768) |
| `square` | `square_hd` | `1:1` | `1024x1024` | `square_hd` (1024×1024) |
| `portrait` | `portrait_16_9` | `9:16` | `1024x1536` | `portrait_4_3` (768×1024) |

GPT Image 2 мапиться на пресети 4:3, а не 16:9, бо його мінімальна кількість пікселів — 655 360; пресет `landscape_16_9` (1024×576 = 589 824) був би відхилений.

Це перетворення виконується у `_build_fal_payload()` — код агента ніколи не повинен знати про різниці у схемах між моделями.

## Автоматичне підвищення роздільної здатності

Підвищення роздільної здатності через **Clarity Upscaler** від FAL контролюється per-model:

| Model | Upscale? | Why |
|---|---|---|
| `fal-ai/flux-2-pro` | ✓ | Сумісність назад (був попереднім вибором) |
| All others | ✗ | Швидкі моделі втратили б свою підсекундна цінність; hi‑res моделі його не потребують |

Коли підвищення роздільної здатності виконується, використовуються такі налаштування:

| Setting | Value |
|---|---|
| Upscale factor | 2× |
| Creativity | 0.35 |
| Resemblance | 0.6 |
| Guidance scale | 4 |
| Inference steps | 18 |

Якщо підвищення не вдається (проблема мережі, ліміт запитів), автоматично повертається оригінальне зображення.

## Як це працює всередині

1. **Визначення моделі** — `_resolve_fal_model()` читає `image_gen.model` з `config.yaml`, підставляє змінну середовища `FAL_IMAGE_MODEL`, а потім за замовчуванням `fal-ai/flux-2/klein/9b`.
2. **Формування payload** — `_build_fal_payload()` переводить твоє `aspect_ratio` у нативний формат моделі (enum пресету, enum співвідношення або літерал GPT), об’єднує параметри за замовчуванням моделі, застосовує будь‑які перевизначення виклику, потім фільтрує їх згідно whitelist `supports`, щоб непідтримувані ключі ніколи не надсилались.
3. **Відправка** — `_submit_fal_request()` маршрутизує запит через прямі облікові дані FAL або керований шлюз Nous.
4. **Підвищення роздільної здатності** — виконується лише якщо у метаданих моделі `upscale: True`.
5. **Доставка** — фінальний URL зображення повертається агенту, який генерує тег `MEDIA:<url>`, а адаптери платформ конвертують його у нативний медіа‑формат.

## Налагодження

Увімкни журнал налагодження:

## Обмеження

- **Потрібні облікові дані FAL** (прямий `FAL_KEY` або підписка Nous)
- **Лише text‑to‑image** — без inpainting, img2img чи редагування цим інструментом
- **Тимчасові URL** — FAL повертає хостовані URL, які спрацьовують лише кілька годин/днів; за потреби збережи їх локально
- **Обмеження per-model** — деякі моделі не підтримують `seed`, `num_inference_steps` тощо. Фільтр `supports` тих параметрів просто відкидає, що є очікуваною поведінкою.
