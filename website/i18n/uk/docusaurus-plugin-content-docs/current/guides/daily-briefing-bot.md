# Посібник: Створи бота щоденного брифінгу

У цьому посібнику ти створиш персонального бота‑брифінгу, який щоранку прокидається, досліджує теми, що тебе цікавлять, підсумовує результати та надсилає стислий бриф прямо в твій Telegram або Discord.

Після завершення у тебе буде повністю автоматизований робочий процес, що поєднує **веб‑пошук**, **cron‑планування**, **делегування** та **обмін повідомленнями** — без написання коду.

:::tip
Цей рецепт охоплює веб‑пошук, підсумовування та необов’язковий TTS — все в одному підписці Portal. Найшвидше налаштування — `hermes setup --portal`. Дивись [Nous Portal](/integrations/nous-portal).
:::

## Що ми створюємо

Ось схема процесу:

1. **8:00** — Планувальник cron запускає твою задачу
2. **Hermes** піднімає нову сесію агента з твоїм запитом
3. **Веб‑пошук** отримує останні новини за твоїми темами
4. **Підсумовування** перетворює їх у чистий формат брифінгу
5. **Доставка** надсилає бриф до твого Telegram або Discord

Все працює без твоєї участі. Ти лише читаєш бриф за чашкою ранкової кави.

## Передумови

Перш ніж почати, переконайся, що у тебе є:

- **Hermes Agent встановлений** — дивись [Посібник зі встановлення](/getting-started/installation)
- **Gateway запущений** — демон шлюзу обробляє виконання cron:
    ```bash
  hermes gateway install   # Install as a user service
  sudo hermes gateway install --system   # Linux servers: boot-time system service
  # or
  hermes gateway           # Run in foreground
  ```
- **API‑ключ Firecrawl** — встанови `FIRECRAWL_API_KEY` у середовищі для веб‑пошуку
- **Налаштований обмін повідомленнями** (необов’язково, але рекомендовано) — [Telegram](/user-guide/messaging/telegram) або Discord, налаштовані з домашнім каналом

:::tip No messaging? No problem
Ти все одно можеш пройти цей посібник, використовуючи `deliver: "local"`. Брифінги будуть збережені у `~/.hermes/cron/output/`, і ти зможеш читати їх у будь‑який час.
:::

## Крок 1: Перевірка робочого процесу вручну

Перш ніж автоматизовувати, переконайся, що бриф працює. Запусти чат‑сесію:

```bash
hermes
```

Потім введи цей запит:

```
Search for the latest news about AI agents and open source LLMs.
Summarize the top 3 stories in a concise briefing format with links.
```

Hermes здійснить веб‑пошук, перегляне результати та виведе щось на кшталт:

```
☀️ Your AI Briefing — March 8, 2026

1. Qwen 3 Released with 235B Parameters
   Alibaba's latest open-weight model matches GPT-4.5 on several
   benchmarks while remaining fully open source.
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain Launches Agent Protocol Standard
   A new open standard for agent-to-agent communication gains
   adoption from 15 major frameworks in its first week.
   → https://blog.langchain.dev/agent-protocol/

3. EU AI Act Enforcement Begins for General-Purpose Models
   The first compliance deadlines hit, with open source models
   receiving exemptions under the 10M parameter threshold.
   → https://artificialintelligenceact.eu/updates/

---
3 stories • Sources searched: 8 • Generated by Hermes Agent
```

Якщо все працює, ти готовий автоматизувати.

:::tip Iterate on the format
Спробуй різні запити, доки не отримаєш результат, який тобі подобається. Додай інструкції типу «використовуй емодзі‑заголовки» або «кожен підсумок не більше 2 речень». Що б ти не обрав, це потрапить у cron‑задачу.
:::

## Крок 2: Створення cron‑задачі

Тепер заплануємо автоматичний запуск щоранку. Є два способи.

Перед створенням cron‑задач переконайся, що Hermes має глобально налаштовану модель і провайдера за замовчуванням. Якщо потрібно, щоб окрема задача використовувала інші значення, вкажи їх явно під час створення.

### Варіант A: Натуральна мова (у чаті)

Просто скажи Hermes, що тобі потрібно:

```
Every morning at 8am, search the web for the latest news about AI agents
and open source LLMs. Summarize the top 3 stories in a concise briefing
with links. Use a friendly, professional tone. Deliver to telegram.
```

Hermes створить cron‑задачу за допомогою уніфікованого інструменту `cronjob`.

### Варіант B: CLI‑команда

Використай команду `/cron` для більшого контролю:

```
/cron add "0 8 * * *" "Search the web for the latest news about AI agents and open source LLMs. Find at least 5 recent articles from the past 24 hours. Summarize the top 3 most important stories in a concise daily briefing format. For each story include: a clear headline, a 2-sentence summary, and the source URL. Use a friendly, professional tone. Format with emoji bullet points and end with a total story count."
```

### Золоте правило: Самодостатні запити

:::warning Critical concept
Cron‑задачі виконуються в **повністю новій сесії** — без пам’яті про попередні розмови, без контексту того, що ти «налаштував раніше». Твій запит має містити **все**, що агенту потрібно для виконання задачі.
:::

**Поганий запит:**
```
Do my usual morning briefing.
```

**Добрий запит:**
```
Search the web for the latest news about AI agents and open source LLMs.
Find at least 5 recent articles from the past 24 hours. Summarize the
top 3 most important stories in a concise daily briefing format. For each
story include: a clear headline, a 2-sentence summary, and the source URL.
Use a friendly, professional tone. Format with emoji bullet points.
```

Добрий запит конкретизує **що шукати**, **скільки статей**, **у якому форматі**, **яким тоном**. Це все, що агенту потрібно одразу.

## Крок 3: Налаштування брифінгу

Коли базовий бриф працює, можна проявити креативність.

### Брифінги за кількома темами

Об’єднай кілька напрямків в одному брифі:

```
/cron add "0 8 * * *" "Create a morning briefing covering three topics. For each topic, search the web for recent news from the past 24 hours and summarize the top 2 stories with links.

Topics:
1. AI and machine learning — focus on open source models and agent frameworks
2. Cryptocurrency — focus on Bitcoin, Ethereum, and regulatory news
3. Space exploration — focus on SpaceX, NASA, and commercial space

Format as a clean briefing with section headers and emoji. End with today's date and a motivational quote."
```

### Делегування для паралельного дослідження

Для швидших брифінгів попроси Hermes делегувати кожну тему під‑агенту:

```
/cron add "0 8 * * *" "Create a morning briefing by delegating research to sub-agents. Delegate three parallel tasks:

1. Delegate: Search for the top 2 AI/ML news stories from the past 24 hours with links
2. Delegate: Search for the top 2 cryptocurrency news stories from the past 24 hours with links
3. Delegate: Search for the top 2 space exploration news stories from the past 24 hours with links

Collect all results and combine them into a single clean briefing with section headers, emoji formatting, and source links. Add today's date as a header."
```

Кожен під‑агент шукає незалежно та паралельно, а головний агент об’єднує все в один відполірований бриф. Дивись [Документацію з делегування](/user-guide/features/delegation) для деталей.

### Розклад лише у будні дні

Не потрібні брифінги у вихідні? Використай cron‑вираз, що охоплює понеділок‑п’ятницю:

```
/cron add "0 8 * * 1-5" "Search for the latest AI and tech news..."
```

### Двічі на день

Отримуй огляд вранці та підсумок ввечері:

```
/cron add "0 8 * * *" "Morning briefing: search for AI news from the past 12 hours..."
/cron add "0 18 * * *" "Evening recap: search for AI news from the past 12 hours..."
```

### Додавання особистого контексту за допомогою пам’яті

Якщо у тебе ввімкнено [memory](/user-guide/features/memory), ти можеш зберігати уподобання, які зберігаються між сесіями. Але пам’ятай — cron‑задачі запускаються у нових сесіях без розмовної пам’яті. Щоб додати особистий контекст, включи його безпосередньо у запит:

```
/cron add "0 8 * * *" "You are creating a briefing for a senior ML engineer who cares about: PyTorch ecosystem, transformer architectures, open-weight models, and AI regulation in the EU. Skip stories about product launches or funding rounds unless they involve open source.

Search for the latest news on these topics. Summarize the top 3 stories with links. Be concise and technical — this reader doesn't need basic explanations."
```

:::tip Tailor the persona
Включення деталей про те, *для кого* створюється бриф, значно підвищує релевантність. Розкажи агенту про свою роль, інтереси та що варто пропустити.
:::

## Крок 4: Керування задачами

### Перегляд усіх запланованих задач

У чаті:
```
/cron list
```

Або з терміналу:
```bash
hermes cron list
```

Ти побачиш щось подібне:

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### Видалення задачі

У чаті:
```
/cron remove a1b2c3d4
```

Або запитай у розмові:
```
Remove my morning briefing cron job.
```

Hermes використає `cronjob(action="list")`, щоб знайти її, і `cronjob(action="remove")`, щоб видалити.

### Перевірка стану шлюзу

Переконайся, що планувальник дійсно працює:

```bash
hermes cron status
```

Якщо шлюз не запущений, твої задачі не будуть виконуватись. Встанови його як фонову службу для надійності:

```bash
hermes gateway install
# or on Linux servers
sudo hermes gateway install --system
```
