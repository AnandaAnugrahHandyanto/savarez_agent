# Запусти Hermes локально за допомогою Ollama — нульова вартість API
## Проблема

API великих мовних моделей (LLM) стягують оплату за токен. Інтенсивна сесія кодування може коштувати $5–20. Для особистих проєктів, навчання чи роботи з чутливими даними це швидко накопичується — і ти надсилаєш кожну розмову третій стороні.
## Що вирішує цей посібник

Ти налаштуєш Hermes Agent, який працюватиме повністю на твоєму обладнанні, використовуючи [Ollama](https://ollama.com) як бекендом моделі. Без API‑ключів, без підписок, без передачі даних з твого комп’ютера. Після налаштування Hermes працює точно так само, як і з OpenRouter або Anthropic — термінальні команди, редагування файлів, веб‑перегляд, делегування — лише модель працює локально.

Після завершення ти матимеш:

- Ollama, що обслуговує одну або кілька моделей з відкритою вагою
- Hermes, підключений до Ollama як власну кінцеву точку
- Робочого локального агента, який може редагувати файли, виконувати команди та переглядати веб
- За бажанням: бот Telegram/Discord, який працює повністю на твоєму обладнанні
## Що тобі потрібно

| Компонент | Мінімум | Рекомендовано |
|-----------|---------|----------------|
| **RAM** | 8 ГБ (для моделей 3B) | 32+ ГБ (для моделей 27B+) |
| **Storage** | 5 ГБ вільного місця | 30+ ГБ (для декількох моделей) |
| **CPU** | 4 ядра | 8+ ядер (AMD EPIC, Ryzen, Intel Xeon) |
| **GPU** | Не потрібен | NVIDIA GPU з 8+ ГБ VRAM значно прискорює роботу |

:::tip CPU‑only працює, але очікуй повільніші відповіді
Ollama працює на серверах лише з CPU. Модель 9B на сучасному 8‑ядерному CPU дає ~10 токенів/сек. Модель 31B на CPU працює повільніше (~2–5 токенів/сек) — кожна відповідь займає 30–120 секунд, проте це працює. GPU значно покращує продуктивність. Для налаштувань лише з CPU збільште тайм‑аут API через змінну середовища (це не ключ `config.yaml`):

```bash
# ~/.hermes/.env
HERMES_API_TIMEOUT=1800   # 30 minutes — generous for slow local models
```
:::
## Крок 1: Встановити Ollama

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Перевір, чи запущено:

```bash
ollama --version
curl http://localhost:11434/api/tags   # Should return {"models":[]}
```
## Крок 2: Завантажити модель

Оберіть відповідно до вашого обладнання:

| Model | Size on Disk | RAM Needed | Tool Calling | Best For |
|-------|-------------|------------|:------------:|----------|
| `gemma4:31b` | ~20 GB | 24+ GB | Yes | Найвища якість — активне використання інструментів та розуміння |
| `gemma2:27b` | ~16 GB | 20+ GB | No | Розмовні завдання, без використання інструментів |
| `gemma2:9b` | ~5 GB | 8+ GB | No | Швидкий чат, Q&A — не може викликати інструменти |
| `llama3.2:3b` | ~2 GB | 4+ GB | No | Легковаговий, лише швидкі відповіді |

:::warning Tool calling matters
Hermes — це **агентний** помічник, який редагує файли, виконує команди та переглядає веб через виклики інструментів. Моделі без підтримки викликів інструментів можуть лише спілкуватися; вони не можуть виконувати дії. Для повного досвіду Hermes використовуйте модель, що підтримує інструменти (наприклад `gemma4:31b`).
:::

Завантажте обрану модель:

```bash
ollama pull gemma4:31b
```

:::info Multiple models
Ви можете завантажити кілька моделей і перемикатися між ними всередині Hermes за допомогою `/model`. Ollama завантажує активну модель у пам’ять за потребою та автоматично вивантажує неактивні.
:::

Перевірте, чи модель працює:

```bash
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemma4:31b",
    "messages": [{"role": "user", "content": "Say hello"}],
    "max_tokens": 50
  }'
```

Ви повинні побачити JSON‑відповідь з відповіддю моделі.
## Крок 3: Налаштуй Hermes

Запусти майстра налаштування Hermes:

```bash
hermes setup
```

Коли буде запитано про провайдера, вибери **Custom Endpoint** і введи:

- **Base URL:** `http://localhost:11434/v1`
- **API Key:** Залиши порожнім або введи `no-key` (Ollama не потребує ключа)
- **Model:** `gemma4:31b` (або будь‑яку іншу модель, яку ти завантажив)

Альтернативно, відредагуй `~/.hermes/config.yaml` безпосередньо:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"
```
## Крок 4: Почати використовувати Hermes

```bash
hermes
```

Ось і все. Ти тепер запускаєш повністю локальний агент. Спробуй його:

```
You: List all Python files in this directory and count the lines of code in each

You: Read the README.md and summarize what this project does

You: Create a Python script that fetches the weather for Ho Chi Minh City
```

Hermes буде використовувати інструмент терміналу, файлові операції та твою локальну модель — без викликів у хмару.
## Крок 5: Обери правильну модель для свого завдання

Не кожне завдання потребує найбільшої моделі. Ось практичний посібник:

| Завдання | Рекомендована модель | Чому |
|------|-------------------|-----|
| Редагування файлів, код, команди терміналу | `gemma4:31b` | Єдина модель із надійним викликом інструментів |
| Швидкі питання‑відповіді (без використання інструментів) | `gemma2:9b` | Швидкі відповіді для розмовних завдань |
| Легкий чат | `llama3.2:3b` | Найшвидша, але дуже обмежені можливості |

:::note
Для повноцінної агентної роботи (редагування файлів, виконання команд, перегляд), `gemma4:31b` наразі є найкращим локальним варіантом із підтримкою виклику інструментів. Перевір [бібліотеку моделей Ollama](https://ollama.com/library) на наявність нових моделей — підтримка виклику інструментів швидко розширюється.
:::

Перемикай моделі «на льоту» під час сесії:

```
/model gemma2:9b
```
## Крок 6: Оптимізація швидкості

### Збільшення вікна контексту Ollama

За замовчуванням Ollama використовує контекст у 2048 токенів. Hermes потребує щонайменше 64 000 токенів для агентської роботи з інструментами:

```bash
# Create a Modelfile that extends context
cat > /tmp/Modelfile << 'EOF'
FROM gemma4:31b
PARAMETER num_ctx 64000
EOF

ollama create gemma4-64k -f /tmp/Modelfile
```

Потім онови конфігурацію Hermes, вказавши `gemma4-64k` як назву моделі.

### Тримати модель завантаженою

За замовчуванням Ollama вивантажує моделі після 5 хвилин бездействія. Для постійного шлюз‑бота тримай її завантаженою:

```bash
# Set keep-alive to 24 hours
curl http://localhost:11434/api/generate \
  -d '{"model": "gemma4:31b", "keep_alive": "24h"}'
```

Або встанови це глобально в середовищі Ollama:

```bash
# /etc/systemd/system/ollama.service.d/override.conf
[Service]
Environment="OLLAMA_KEEP_ALIVE=24h"
```

### Використання GPU‑offloading (за наявності)

Якщо у тебе є GPU NVIDIA, Ollama автоматично offload‑ить шари на нього. Перевір це за допомогою:

```bash
ollama ps   # Shows which model is loaded and how many GPU layers
```

Для моделі 31 B на GPU 12 GB ти отримаєш частковий offload (~40 шарів на GPU, решта на CPU), що все одно дає значне прискорення.
## Крок 7: Запуск як шлюз‑бота (необов’язково)

Після того, як Hermes працює локально в CLI, ти можеш надати його у вигляді бота Telegram або Discord — при цьому він продовжує працювати повністю на твоєму обладнанні.

### Telegram

1. Створи бота через [@BotFather](https://t.me/BotFather) і отримай токен.
2. Додай до свого `~/.hermes/config.yaml`:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"

platforms:
  telegram:
    enabled: true
    token: "YOUR_TELEGRAM_BOT_TOKEN"
```

3. Запусти шлюз:

```bash
hermes gateway
```

Тепер напиши повідомлення своєму боту в Telegram — він відповість, використовуючи твою локальну модель.

### Discord

1. Створи Discord‑застосунок на [discord.com/developers](https://discord.com/developers/applications).
2. Додай до конфігурації:

```yaml
platforms:
  discord:
    enabled: true
    token: "YOUR_DISCORD_BOT_TOKEN"
```

3. Запусти: `hermes gateway`
## Крок 8: Налаштування запасного (варіанту) (необов’язково)

Локальні моделі можуть мати труднощі зі складними завданнями. Налаштуй хмарний запасний (варіант), який активується лише коли локальна модель не справляється:

```yaml
model:
  default: "gemma4:31b"
  provider: "custom"
  base_url: "http://localhost:11434/v1"

fallback_providers:
  - provider: openrouter
    model: anthropic/claude-sonnet-4
```

Таким чином, 90 % твоїх запитів будуть безкоштовними (локальними), а лише складні завдання будуть виконуватись через платний API.
## Troubleshooting

### «Connection refused» під час запуску

Ollama не працює. Запусти його:

```bash
sudo systemctl start ollama
# or
ollama serve
```

### Повільні відповіді

- **Перевір розмір моделі щодо RAM:** Якщо твоїй моделі потрібно більше оперативної пам’яті, ніж доступно, вона буде використовувати підкачку на диск. Використай меншу модель або додай RAM.
- **Перевір `ollama ps`:** Якщо жодні шари GPU не відвантажені, відповіді будуть обмежені CPU. Це нормально для серверів лише з CPU.
- **Зменши контекст:** Великі розмови уповільнюють інференс. Регулярно використовуйте `/compress` або встановіть нижчий поріг стиснення в конфігурації.

### Модель не виконує виклики інструментів

Менші моделі (3B, 7B) іноді ігнорують інструкції щодо викликів інструментів і повертають простий текст замість структурованих викликів функцій. Рішення:

- **Використай більшу модель** — `gemma4:31b` або `gemma2:27b` набагато краще обробляють виклики інструментів, ніж моделі 3B/7B.
- **Hermes має авто‑ремонт** — вона виявляє некоректні виклики інструментів і автоматично намагається їх виправити.
- **Налаштуй запасний (варіант)** — якщо локальна модель не вдається 3 рази, Hermes переходить до хмарного провайдера.

### Помилки вікна контексту

Типове вікно контексту Ollama (2048 токенів) занадто маленьке для агентної роботи. Дивись [Step 6](#step-6-optimize-for-speed), щоб збільшити його.
## Порівняння вартості

Ось скільки економиться, коли працюєш локально замість хмарних API, на прикладі типового сеансу кодування (~100 Т токенів на вхід, ~20 Т токенів на вихід):

| Провайдер | Вартість за сесію | Щомісячно (щоденне використання) |
|----------|-----------------|---------------------|
| Anthropic Claude Sonnet | ~$0.80 | ~$24 |
| OpenRouter (GPT-4o) | ~$0.60 | ~$18 |
| **Ollama (local)** | **$0.00** | **$0.00** |

Твоя єдина витрата — електрика, приблизно $0.01–0.05 за сесію залежно від обладнання.
## Що добре працює локально

- **File editing and code generation** — моделі 9B+ добре справляються з цим
- **Terminal commands** — Hermes обгортає команду, виконує її, читає вивід незалежно від моделі
- **Web browsing** — інструмент браузера виконує отримання даних; модель лише інтерпретує результати
- **Cron jobs and scheduled tasks** — працюють так само, як у хмарних налаштуваннях
- **Multi-platform gateway** — Telegram, Discord, Slack працюють з локальними моделями
## Що краще з хмарними моделями

- **Дуже складне багатокрокове мислення** — 70 B+ або хмарні моделі типу Claude Opus помітно кращі
- **Довгі контекстні вікна** — хмарні моделі пропонують 100 K–1 M токенів; локальні середовища часто мають значення нижче мінімуму Hermes у 64 K, якщо їх не налаштувати
- **Швидкість при великих відповідях** — хмарний інференс швидший, ніж лише CPU‑локальний, для довгих генерацій

Оптимальний варіант: використай локальний для щоденних завдань, налаштуй хмарний запасний (варіант) для складних випадків.
