# Каталог моделей

Hermes отримує підготовлені списки моделей для **OpenRouter** та **Nous Portal** з JSON‑манифесту, розміщеного поруч із сайтом документації. Це дозволяє підтримувачам оновлювати списки вибору без випуску нової версії `hermes-agent`.

Коли манифест недоступний (офлайн, блокування мережі, збій хостингу), Hermes тихо переходить до знімка, що зберігається в репозиторії і постачається разом із CLI. Манифест ніколи не ламає вибірник — у найгіршому випадку ти бачиш список, який був включений у встановлену версію.

## URL живого манифесту

```
https://hermes-agent.nousresearch.com/docs/api/model-catalog.json
```

Публікується при кожному злитті в `main` через існуючий конвеєр `deploy-site.yml` GitHub Pages. Джерело правди знаходиться в репозиторії за шляхом `website/static/api/model-catalog.json`.

## Схема

```json
{
  "version": 1,
  "updated_at": "2026-04-25T22:00:00Z",
  "metadata": {},
  "providers": {
    "openrouter": {
      "metadata": {},
      "models": [
        {"id": "moonshotai/kimi-k2.6", "description": "recommended", "metadata": {}},
        {"id": "openai/gpt-5.4",       "description": ""}
      ]
    },
    "nous": {
      "metadata": {},
      "models": [
        {"id": "anthropic/claude-opus-4.7"},
        {"id": "moonshotai/kimi-k2.6"}
      ]
    }
  }
}
```

Примітки до полів:

- **`version`** — цілочисельна версія схеми. Майбутні схеми підвищують це значення; Hermes відхиляє манифести з версіями, які він не розуміє, і переходить до жорстко закодованого знімка.
- **`metadata`** — довільний словник на рівні манифесту, провайдера та моделі. Будь‑які ключі. Hermes ігнорує невідомі поля, тому ти можеш додавати анотації (`"tier": "paid"`, `"tags": [...]` тощо) без необхідності змінювати схему.
- **`description`** — лише для OpenRouter. Керує текстом бейджика у вибірнику (`"recommended"`, `"free"` або порожньо`). Nous Portal не використовує це — доступність безкоштовного рівня визначається в режимі реального часу через кінцеву точку цінування Порталу.
- **Ціни та довжина контексту** НЕ містяться в манифесті. Вони надходять з живих API провайдерів (`/v1/models` endpoint, models.dev) під час отримання.

## Поведінка отримання

| Коли | Що відбувається |
|---|---|
| `/model` або `hermes model` | Отримує, якщо дисковий кеш застарілий, інакше використовує кеш |
| Дисковий кеш свіжий (< TTL) | Без мережевого запиту |
| Збій мережі з наявним кешем | Тихий запасний варіант — використання кешу, один рядок логу |
| Збій мережі, без кешу | Тихий запасний варіант — використання знімка в репозиторії |
| Манифест не проходить валідацію схеми | Вважається недоступним |

Розташування кешу: `~/.hermes/cache/model_catalog.json`.

## Конфігурація

```yaml
model_catalog:
  enabled: true
  url: https://hermes-agent.nousresearch.com/docs/api/model-catalog.json
  ttl_hours: 24
  providers: {}
```

Встанови `enabled: false`, щоб повністю вимкнути віддалене отримання і завжди використовувати знімок в репозиторії.

### Перевизначення URL‑ів для провайдерів

Треті сторони можуть самостійно розміщувати власний список підбору, використовуючи ту ж схему. Вкажи провайдеру кастомний URL:

```yaml
model_catalog:
  providers:
    openrouter:
      url: https://example.com/my-openrouter-curation.json
```

Перевизначений манифест має заповнити лише блок(и) провайдера, які його стосуються. Інші провайдери продовжують отримувати дані з головного URL.

## Оновлення манифесту

Підтримувачі:

```bash
# Re-generate from the in-repo hardcoded lists (keeps manifest in sync after
# editing OPENROUTER_MODELS or _PROVIDER_MODELS["nous"] in hermes_cli/models.py).
python scripts/build_model_catalog.py
```

Після цього створіть PR з змінами у `website/static/api/model-catalog.json` у гілку `main`. Сайт документації автоматично розгортається після злиття, і новий манифест стає активним протягом кількох хвилин.

Ти також можеш вручну редагувати JSON для тонких змін метаданих, які не повинні потрапляти у знімок репозиторію — скрипт‑генератор є лише зручністю, а не єдиним джерелом правди.
