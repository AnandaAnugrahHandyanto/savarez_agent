# Довідник вбудованих інструментів

Ця сторінка документує вбудовані інструменти Hermes, згруповані за набором інструментів. Доступність залежить від платформи, облікових даних та увімкнених наборів інструментів.

**Швидкі підрахунки (поточний реєстр):** ~64 інструменти — 10 інструментів браузера (ядро) + 2 інструменти браузера з CDP‑гейтом, 4 інструменти роботи з файлами, 4 інструменти Home Assistant, 2 інструменти терміналу, 2 веб‑інструменти, 5 інструментів Feishu, 7 інструментів Spotify (зареєстрованих плагіном `spotify`), 5 інструментів Yuanbao, 9 інструментів kanban (реєструються, коли диспетчер kanban створює агента), 2 інструменти Discord та кілька окремих інструментів (`memory`, `clarify`, `delegate_task`, `execute_code`, `cronjob`, `session_search`, `skill_view`/`skill_manage`/`skills_list`, `text_to_speech`, `image_generate`, `video_generate`, `vision_analyze`, `video_analyze`, `mixture_of_agents`, `send_message`, `todo`, `computer_use`, `process`).

:::tip MCP Tools
Окрім вбудованих інструментів, Hermes може завантажувати інструменти динамічно з серверів MCP. Інструменти MCP мають префікс `mcp_<server>_` (наприклад, `mcp_github_create_issue` для сервера `github`). Дивись [MCP Integration](/user-guide/features/mcp) для налаштування.
:::

| `browser_type` | Ввести текст у поле вводу, ідентифіковане за його ref‑ID. Спочатку очищає поле, потім вводить новий текст. Потрібно спочатку викликати `browser_navigate` і `browser_snapshot`. | — |
| `browser_vision` | Зробити скріншот поточної сторінки та проаналізувати його за допомогою AI‑vision. ## `kanban` toolset

Зареєстровано, коли агент або (a) створений диспетчером kanban (`HERMES_KANBAN_TASK` env set), або (b) працює у профілі, який явно вмикає `kanban` toolset. Робітники, прив’язані до завдання, використовують інструменти життєвого циклу для свого завдання; профілі‑оркестратори додатково отримують інструменти маршрутизації дошки, такі як `kanban_list` і `kanban_unblock`. Дивись [Kanban Multi-Agent](/user-guide/features/kanban) для повного робочого процесу.

| Tool | Description | Requires environment |
|------|-------------|----------------------|
| `kanban_show` | Показати активне kanban‑завдання, призначене цьому робітнику (заголовок, опис, коментарі, залежності). | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_list` | Перелік завдань дошки з фільтрами. Тільки оркестратор; приховано від робітників, створених диспетчером. | профіль з `kanban` toolset |
| `kanban_complete` | Позначити поточне завдання виконаним зі структурованим payload‑ом передачі (результати, артефакти, подальші дії). | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_block` | Заблокувати поточне завдання питанням для користувача — диспетчер паузить, показує питання і продовжує після відповіді людини. | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_heartbeat` | Надіслати heartbeat прогресу під час довготривалої операції, щоб диспетчер знав, що робітник ще живий. | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_comment` | Додати коментар до потоку завдання без зміни його стану — корисно для показу проміжних результатів. | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_create` | Розгалужувати дочірні завдання з поточного завдання. Використовується оркестраторами та робітниками, що створюють подальші завдання. | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_link` | Зв’язати завдання залежністю батько → дитина. | `HERMES_KANBAN_TASK` або `kanban` toolset |
| `kanban_unblock` | Повернути заблоковане завдання у стан `ready`. Тільки оркестратор; приховано від робітників, створених диспетчером. | профіль з `kanban` toolset |

## `video_gen` toolset

Опціональний набір інструментів (не завантажується у стандартному наборі `hermes-cli`). Додай його за допомогою `--toolsets video_gen` або увімкни у `hermes tools` → Video Generation, де також буде проведено процес вибору бекенду.

Бекенди постачаються як плагіни у `plugins/video_gen/<name>/`:

- **xAI Grok-Imagine** — text-to-video та image-to-video (SuperGrok OAuth або `XAI_API_KEY`).
- **FAL.ai** — Veo 3.1, Pixverse v6, Kling O3 (потрібен `FAL_KEY`).

Єдиний інструмент `video_generate` охоплює обидві модальності — передай `image_url`, щоб анімувати статичне зображення, або пропусти його, щоб згенерувати лише з тексту. Активний бекенд автоматично маршрутує запит до потрібної кінцевої точки. Опис інструмента перебудовується на початку сесії, щоб відображати реальні можливості активного бекенду (модальності, співвідношення сторін, роздільну здатність, діапазон тривалості, максимальну кількість референс‑зображень, підтримку аудіо). Дивись [Video Generation Provider Plugins](/developer-guide/video-gen-provider-plugin) для створення бекендів.

| Tool | Description | Requires environment |
|------|-------------|----------------------|
| `video_generate` | Генерує відео за текстовим запитом (text-to-video) або анімує статичне зображення (image-to-video), використовуючи налаштований користувачем бекенд генерації відео. Передай `image_url`, щоб анімувати це зображення; пропусти його, щоб згенерувати лише з тексту. Бекенд автоматично маршрутує запит до правильної кінцевої точки. Повертає або HTTP‑URL, або абсолютний шлях до файлу у полі `video`. | Active `video_gen` plugin + its credential (e.g. `XAI_API_KEY`, `FAL_KEY`) |
