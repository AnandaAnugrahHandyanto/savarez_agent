# OPERATOR-INBOX

Items operator MUST act on. Three admission criteria:

1. Blocking decision - ambiguity only operator can resolve
2. Ratification request - agent did something needing operator sign-off
3. External-blocker surfacing - Clay ruling, provider outage, upstream regression

NOT: status reports. NOT: session logs. NOT: summaries.

Filename: `<YYYY-MM-DD>-<short-slug>.md`. Body cites which admission criterion applies.

Clay-specific items go to `OPERATOR-INBOX/clay/` with same conventions.
