Prepared for: Wael Helmi  
Date: 2026-04-15

## Objective
Identify 20 Finnish organizations across different sectors, map practical AI bot implementations, and estimate token usage, costs, pricing, margins, competitiveness and sales approach.

## Assumptions used for token/cost model
- Input tokens: **$0.40 / 1M**
- Output tokens: **$1.60 / 1M**
- Embeddings: **$0.02 / 1M**
- FX assumption: **1 USD = 0.92 EUR**
- Monthly delivery cost = AI token cost + infra + monitoring/support baseline

## Universal implementation feature set
- Multilingual (FI/EN/SV), channel support (web + WhatsApp + Slack/Teams)
- RAG over SOPs, policies, manuals, ticket history and intranet docs
- Role + group access controls
- Policy engine (confidence threshold, forced escalation keywords, PII redaction)
- Human handoff workflow with SLA tracking
- CRM/helpdesk integration (HubSpot/Salesforce/Zendesk/Jira)
- Audit logs + analytics dashboard (answer rate, confidence, escalations)
- Continuous improvement loop (feedback, retraining prompts, content gaps)

## Why this is competitive in Finland
1. **Outcome-first offer**: you sell reduced operational load and faster response times, not a generic chatbot.
2. **Governance-first design**: stronger fit for healthcare/finance/energy where risk and auditability matter.
3. **Fast deployment**: 2–4 week pilot beats long SI projects.
4. **Productized pricing**: easier procurement and faster decision cycle.
5. **Local-language advantage**: FI/EN/SV support increases usability and adoption.

## How to improve win-rate (make sure it is sold)
1. Start with a **14-day paid pilot** and one KPI commitment (e.g., -25% repetitive tickets).
2. Use a **risk-reversal clause**: if KPI baseline not reached, month-2 optimization at no extra fee.
3. Pitch both audiences:
   - Executive: ROI + risk + timeline
   - Operational owner: daily workflow simplification
4. Include **security & compliance page** in every proposal.
5. Run **account-specific demo scripts** using their business language and likely workflows.
6. Use **land-and-expand**: one function first (support/ops), then upsell to sales/compliance/HR.
