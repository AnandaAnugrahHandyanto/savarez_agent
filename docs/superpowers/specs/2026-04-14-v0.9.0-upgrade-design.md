## Section 2: Dockerfile

### Approach

Adopt the upstream v0.9.0 Dockerfile structure exactly, layering mechovation custom packages on top. Retain the venv/non-root structure to minimize future merge friction with upstream.

### Key upstream v0.9.0 changes

- Multi-stage build: `uv` and `gosu` pulled from pinned official images
- Non-root `hermes` user (UID 10000) created at build time
- `uv venv` for Python dependency isolation (venv inside container)
- `PYTHONUNBUFFERED=1` and `PLAYWRIGHT_BROWSERS_PATH=/opt/hermes/.playwright` env vars
- `python3-pip` removed from apt (uv handles everything)
- `procps` added (provides `ps`, used by process monitoring)

### Custom mechovation additions

**Python packages** (installed as `hermes` user inside venv, after `uv pip install -e ".[all]"`):
- `crawl4ai==0.8.6`
- `uptime-kuma-api==1.2.1`
- `pirateweather==1.0.0` — `--no-deps` flag required
- `python-pptx` — `--no-deps` flag required
- `openpyxl xlsxwriter pandas python-docx docxtpl` — previously apt packages, moved to pip so they land in the venv

**Note:** Office packages (`openpyxl`, `pandas`, etc.) move from apt → pip. The venv is the Python environment hermes runs in, so pip is the correct target.

**Node:** whatsapp bridge install (`scripts/whatsapp-bridge`) preserved as root after main npm install.

**Not included:** `git`/`openssh-client` — were added to `fork/main` but not present in the working Dockerfile; omitted unless needed.

### Target Dockerfile structure

```dockerfile
FROM ghcr.io/astral-sh/uv:0.11.6-python3.13-trixie@sha256:<pin> AS uv_source
FROM tianon/gosu:1.19-trixie@sha256:<pin>                        AS gosu_source

FROM debian:13.4

ENV PYTHONUNBUFFERED=1
ENV PLAYWRIGHT_BROWSERS_PATH=/opt/hermes/.playwright

# System deps (root)
RUN apt-get update && apt-get install -y --no-install-recommends \
        build-essential nodejs npm python3 ripgrep ffmpeg \
        gcc python3-dev libffi-dev procps && \
    rm -rf /var/lib/apt/lists/*

# Non-root hermes user + gosu + uv
RUN useradd -u 10000 -m -d /opt/data hermes
COPY --chmod=0755 --from=gosu_source /gosu /usr/local/bin/
COPY --chmod=0755 --from=uv_source /usr/local/bin/uv /usr/local/bin/uvx /usr/local/bin/

COPY . /opt/hermes
WORKDIR /opt/hermes
RUN chown -R hermes:hermes /opt/hermes

# Python deps as hermes user (venv)
USER hermes
RUN uv venv && \
    uv pip install --no-cache-dir -e ".[all]" && \
    uv pip install --no-cache-dir crawl4ai==0.8.6 uptime-kuma-api==1.2.1 && \
    uv pip install --no-cache-dir --no-deps pirateweather==1.0.0 python-pptx && \
    uv pip install --no-cache-dir openpyxl xlsxwriter pandas python-docx docxtpl

# Node deps + Playwright (needs root/apt)
USER root
RUN npm install --prefer-offline --no-audit && \
    npx playwright install --with-deps chromium --only-shell && \
    cd /opt/hermes/scripts/whatsapp-bridge && \
    npm install --prefer-offline --no-audit && \
    npm cache clean --force

RUN chmod +x /opt/hermes/docker/entrypoint.sh

ENV HERMES_HOME=/opt/data
VOLUME ["/opt/data"]
ENTRYPOINT ["/opt/hermes/docker/entrypoint.sh"]
```
