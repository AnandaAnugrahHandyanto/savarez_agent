## What is still wrong
The user is not mainly complaining about:
- architecture
- presence of a separate balloon layer
- fake text generated by fal
- generic white-box masking

The user is specifically pointing to the **visual relationship between balloon and character**.

So the failure is likely in one or more of these:
1. tail anchor is too naive
2. balloon body sits too far from or too awkwardly above the speaker
3. tail angle / base position does not feel natural to the head or mouth
4. balloon body proportions do not match the speaker position and panel composition
5. the chosen safe zone may be technically safe but emotionally wrong
6. multi-character panels are using panel-level safe space instead of actor-local composition

## Strong hypothesis
The current renderer is still too box-first / zone-first.
It finds a safe rectangle, then adds a tail.
That preserves collision safety but often produces a balloon that feels only loosely attached to the speaker.

The next planning pass should explicitly rethink **attachment quality** as a first-class problem.

## What the new ralplan should focus on
Not a broad re-architecture from scratch.
Instead, a very targeted replanning around:

### Main objective
Make balloon-to-character attachment look natural.

### Likely planning topics
- stronger speaker anchor inference
- head / mouth / shoulder local attachment geometry
- tail routing rules
- actor-local preferred balloon zones instead of only global safe zones
- panel-specific override system for hard shots
- visual heuristics for "attachment plausibility"
- verification method for whether a balloon actually feels attached to the speaker

## Important direction constraints
- Keep `generated_fal_v3` as the source art baseline
- Keep art and balloons separate
- Do not revert to dense heuristic white overlays
- Keep the existing split pipeline if possible
- Prefer improving the analysis + rendering logic rather than restarting the whole system
- The user wants a **new plan**, not generic commentary

## Current likely weak spots in code
The replanning pass should inspect at least:
- `render_balloons.py`
  - `build_tail_points(...)`
  - placement choice logic in `choose_placement(...)`
  - how speech candidates are selected vs chat/caption
- `analyze_balloon_zones.py`
  - how `speaker_points` are inferred
  - whether speaker-local geometry is rich enough
- `balloon_layout_utils.py`
  - style defaults for speech bubble proportions
  - line fitting rules that affect balloon body proportions

## Planning questions Codex should answer
A good replanning pass should concretely answer:
1. Should attachment be modeled from **mouth/head anchor first**, then balloon body placement second?
2. Should safe zones be split into:
   - global safe zones
   - speaker-local preferred zones
3. Should tails be simple triangles, or need curved / biased routing logic?
4. Should hard panels support explicit per-panel override anchors or per-item attachment hints?
5. How should we test or verify "attachment naturalness" beyond collision checks?
