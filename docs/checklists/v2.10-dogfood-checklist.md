### B3. Ambiguity cards are disabled by default

**Step:** Check the environment variable default.

```bash
grep "AMBIGUITY_CARD_ENABLED" agent/managed_agents/feishu_session_resolver.py
```

**Expected:**
- Line shows `os.getenv("FEISHU_AMBIGUITY_CARD_ENABLED", "false")`
- Default is `"false"` — ambiguity detection is disabled until explicitly enabled
- Setting this to `"true"` requires a deliberate action

**Risk:** If someone accidentally sets this env var, ambiguity cards could fire. This is by design — the flag exists to control rollout timing.

| Check | Pass / Fail |
|---|---|
| Default is "false" | [ ] |
| Requires explicit env var to enable | [ ] |

### B4. Permission gate is advisory only

**Step:** Open `agent/managed_agents/feishu_permission_gate.py` and read the module docstring.

**Expected:**
- Docstring says: "This module is advisory input only. It does not modify routing or policy execution."
- Produces `PermissionCheckResult` objects with `allowed: bool` and `reason: str`
- Does not auto-block, auto-demote, or auto-switch agents
- Does not raise exceptions in deny path (returns `allowed=False` instead)

**Risk:** If someone wires `check()` into `_check_sender_allowed()` without understanding the advisory boundary, they might think it auto-blocks. The docstring should prevent this misunderstanding.

| Check | Pass / Fail |
|---|---|
| Advisory-only language in docstring | [ ] |
| Deny path returns PermissionCheckResult, not exception | [ ] |
| No agent blocking code in this module | [ ] |
