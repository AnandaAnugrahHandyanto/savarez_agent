# Hermes v2.10.1 Dogfood UX Cleanup

## What changed

- Added a Web Console status panel titled `Hermes v2.10 Session Binding Status` on the Sessions page.
- Clarified the difference between product release `v2.10-multi-entry-session-binding` and the App/UI runtime version.
- Made workspace, session, binding mode, Feishu, Discord, Permission Gate, and Lark CLI status visible at a glance.
- Updated session list wording so untitled legacy sessions show explicit `Source`, `Binding`, `Workspace`, and `Status` labels.

## What did not change

- No Feishu transport behavior changed.
- No Discord production entrypoint was added.
- No scheduler, router, policy, agent runtime, or Permission Gate enforcement behavior changed.
- No Lark CLI runtime, `LarkCliToolRuntime`, or `LarkSkillLoader` was implemented.
- No approval buttons, streaming cards, or production task enforcement were added.

## Why this is UX/status/copy only

The Web Console now surfaces existing local status from the current `/api/status` and `/api/v2.10/*` endpoints. It does not add new backend endpoints, write new session binding data, call external Feishu or Discord APIs, or change execution decisions.

## Manual verification checklist

- Open the Web Console Sessions page.
- Confirm the `Hermes v2.10 Session Binding Status` panel appears near the top.
- Confirm the panel shows product release `v2.10-multi-entry-session-binding`.
- Confirm App/UI version is shown separately from the product release.
- Confirm workspace/session values show `unknown`, `unavailable`, or `none selected` when the UI cannot prove a healthier state.
- Confirm Feishu copy says connected status does not imply full production execution.
- Confirm Discord is described as dormant/fallback and not part of the current Feishu-first workflow.
- Confirm Permission Gate is described as advisory only.
- Confirm Lark CLI is described as not implemented and deferred to the v2.11 Feishu Operation Layer.
- Confirm recent/session list rows show `Untitled legacy session`, `Source`, `Binding`, `Workspace`, and `Status` labels.

## Known limitations

- The Web Console does not have a current selected v2.10 session binding field, so the status panel must show `none selected` or `unknown` instead of inventing a current session.
- Legacy session records do not expose workspace binding metadata, so the session list shows `Workspace: unknown`.
- Feishu adapter visibility remains distinct from production task execution readiness.
- Permission Gate remains advisory only.
