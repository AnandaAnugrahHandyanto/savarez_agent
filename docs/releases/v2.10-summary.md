## Safety Notes

1. **Ambiguity cards are disabled by default.** `FEISHU_AMBIGUITY_CARD_ENABLED` defaults to `"false"`. No interactive card is ever sent unless explicitly enabled.

2. **Feishu transport integration is a sidecar.** The bridge (`feishu_session_binding_bridge.py`) runs as a try/except sidecar. Failures cannot block the main Feishu message flow.

3. **Permission gate is advisory only.** It produces structured `PermissionCheckResult` objects but does not auto-block, auto-demote, or auto-switch agents. Production enforcement is deferred to a future phase.

4. **No global fallback for ambiguous messages.** The resolver produces a deterministic default group session only after all explicit binding steps fail. There is no "I don't know" silent fallback that picks a random session.

5. **Router remains advisory only.** No automatic demotion, no automatic weight mutation, no agent hard-blocking. All v2.9 invariants preserved.

6. **Discord adapter is dormant.** It shows as disconnected/dormant in the dashboard. No false health status.
