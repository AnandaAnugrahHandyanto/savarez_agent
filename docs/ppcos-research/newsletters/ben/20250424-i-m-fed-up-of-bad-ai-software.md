Hey folks,
i’m going to kick off today with a mini-rant, if you allow me…thanks, here it is.
i’m so f*ckin’ sick of bad AI software. we’ve been promised the world will change, everyone will get rich, no-one has to work again bla bla bla. and then we get tools that claim they are great at xyz or ‘never use/do abc again’.
it’s all bullshit.
take perplexity for example. they’ve raised ~$900M, have ~100 employees and they just shipped their assistant to kill siri [ https://substack.com/redirect/4dfd75b1-52c9-4d88-90b1-e20eab53d3de?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ] (apparently).
“it can book a reservation for you”
no it cant. it just adds search params to opentable. thats fine(ish), but did no one check it in a different location than where they are based? didnt work for me
“it can set up reminders for you”
not really. it added a reminder on my reminders app (fine), set to recurring (didnt ask for that), and set the time for yesterday (🙄 - and got the time wrong)
this mini essay [ https://substack.com/redirect/d384cebb-e6c1-4c4f-87b5-296ecf267913?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ], with demos and examples, is fantastic and hits the nail on the head. Why are so many companies shipping just crappy software with AI tacked on? Every email writer I’ve tested is awful, many text-to-app builders that claim to be better than the current front-runners are also awful.
it feels like there’s so much hype about ai that everyone just markets to the hype. with little to no substance.
another post, this time reviewing Cluely - which went viral as it allows you to ‘cheat’ in realtime. but, the title of the post says it all “I used the ‘cheat on everything’ AI tool and it didn’t help me cheat on anything [ https://substack.com/redirect/48678f51-a250-4062-8a72-45edeea294ac?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ]”
i think hyped launches add new paying users which is contributing to these not quite true $000’s M ARR stats we’re seeing. it’ll all come out in the wash.
rant over.
on to the good stuff!

Stop over-building
We’ve all done it. Big idea → ask an llm → cusror/replit/lovable/etc → build → bug-fix … hours pass, dopamine gone.
maaybe you power through!
The trap
I initially set out to build a founder analysis tool that would take GitHub, LinkedIn, and Twitter URLs of startup founders and generate scores and insights about them. My instinct & LLMs input said to build a full-stack application with:
A front-end interface
Backend APIs to handle the data collection
Integration with various platforms
Storage for the analysis results
Complex logic to determine scoring
Sound familiar?
But as the project grew more complex, I found myself spending more time on scaffolding than on the actual analysis capabilities I wanted to build.
Here’s my full video walkthrough:
link to video [ https://substack.com/redirect/d9a497c4-5a78-4237-b616-7f9334419977?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ]
Breaking down to the essentials
When I stepped back and analysed what I really needed, it boiled down to:
A way to input URLs
A system to gather information from those URLs
AI to analyse that information
A clear output format for the results
That's when it hit me—I didn't need a full-stack application at all. I just needed an LLM with function calling capabilities.

Taking it further with o3
Then o3 came out a day later…
So I experimented with it’s agentic capabilities, uploading the same URLs and having it research and analyse the data. It managed to generate similar scores and insights without requiring any extra MCP integrations at all.
This further reinforced my realisation: we're often building complex systems when the foundation—the AI itself—is already capable of doing what we need with minimal scaffolding.
Simple is advanced
I think lots of us are overbuilding when actually the simplest version is the best.
There's something almost counterintuitive about this approach. In an era of complex frameworks and microservices, returning to something as "primitive" as a chat interface with AI and function calling feels like a step backward.
This realisation has changed how I approach new projects. Now, before diving into development, I ask myself: "Could this be built with just an LLM and some function calls?" Surprisingly often, the answer is yes.
Are you over-building?
Does your product mainly shuffle data between APIs?
Could an LLM decide the “how” (which API, which field) better than hard-coded logic?
Would a chat UI be enough for v1? (Honestly ask.)
Is the only reason you’re diving into [insert dev term] because Twitter said so?
If you nodded more than once, close the IDE and open a chat.
The next time you find yourself planning a tool, consider whether you're overbuilding. The most elegant solution might be the simplest one—a powerful AI model with just the right extensions to accomplish your task.
