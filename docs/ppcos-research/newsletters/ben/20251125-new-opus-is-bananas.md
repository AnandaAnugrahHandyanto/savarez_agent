Everyone’s rushing to launch their thing before American Turkey day.
I built a social tracker over the weekend, design inspired by Linear so i can rip through all social mentions from X/Github/Reddits/any other feed you like. It’s open-source so you can copy it and use it for yourself. Here’s a lil video  [ https://substack.com/redirect/a5bd4f51-5e9b-427a-8072-268135529c1a?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ]I recorded of it in action. Built with Factory’s Droid, Gemini did the initial design pass (which was ok-ish), but trusty Sonnet polished the F out of it. (My first time open-sourcing something I’m proud of!)
Google released Nano Banana Pro [ https://substack.com/redirect/996706c2-bfbb-4faf-8a54-73fe596f53c5?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ], i.e. image generation powered by Gemini 3 Pro. Massive upgrade over already amazing Nano Banana (on top of Gemini 2.5 Flash). It’s great at text rendering, uses reasoning to think about styles, visual layout and content inside an image.
Gemini 3 Pro had its charm for a week, but Claude took it back with Opus 4.5 [ https://substack.com/redirect/6fb0961d-7044-47a2-8219-2c9690bcfff1?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ]. It’s the best coding model in the world (crossing 80% on SWE-Bench Verified) and 3x cheaper than previous Opus models. Opus 4.5’s medium reasoning performs the same as Sonnet 4.5’s best results while using 76% fewer tokens. So, in real use, sometimes using Opus 4.5 might end up being cheaper.
I’m working on a web app for a board game. I tested both Gemini 3 Pro and Opus 4.5 on adding multiplayer room creation to it. First, both of these models gave an almost working solution, whereas older models just didn’t work. Opus was definitely better (and faster) than Gemini, but it still has a few familiar issues. It creates too much duplicate code and very easily falls into the “you’re absolutely right” trap when fixing bugs.
– Keshav
Anthropic also released many product updates with this: “no cutoff” in long conversations on the Claude app, Claude Code on Desktop, Claude in Chrome and Excel, new tools for avoiding the pitfalls of MCP [ https://substack.com/redirect/03a16c55-fd02-4a35-b498-520e814558db?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ] and more.
OpenAI also released two major features in ChatGPT: Group Chats [ https://substack.com/redirect/31981ae2-dcb0-4a1a-bed3-a890c7bdd707?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ] and Shopping Research [ https://substack.com/redirect/00cfc1b5-e18b-4727-bda3-98003f6e0db3?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ]. Group Chats have a limit of 20 people and still have that beta feel. They were earlier launched in a few countries but are now available globally. Shopping Research is a new mode like Deep Research, powered by a special version of GPT-5 Mini. It promises to find the product on the web for your query, but since it’s a small model, be wary of inaccuracies in details like pricing, sizes, etc. Both these features are available to free and paid users (except teams/enterprises).
Hardcoded permissions slowing you down? As systems grow, hardcoded authorization creates security gaps and dev bottlenecks. This 80-page ebook walks through the move to externalized authorization - with frameworks, code samples, and lessons from teams who’ve made the transition. Download free ebook. Factory skills [ https://substack.com/redirect/936776d7-8e0d-4dff-bd90-7ea4fd7dfc4a?j=eyJ1IjoicXd0ciJ9.0tEpqnOrcfuKYRYnEOurmlOZ_pVIUnS06konjQA-bCs ] are now live (links to my video)! Skills are essentially instructions + tools to run workflows super easily. I built this browser tool that opens sites, takes actions (clicks, fills in forms etc) and pulls all that back into my chat. I also recently added a DOM picker so i can select elements on a webpage and that code gets sent to the chat so i can identify specific elements to change (instead of saying no not that button, the other blue button)
