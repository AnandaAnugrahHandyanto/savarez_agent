>>
>> At the other end of the spectrum is Preplexity - with its default choice
>> of as much as possible providing one answer to the user intent and an
>> operating model of directly answering the question.
>>
>> Somewhere in the middle is the new Hybrid Google using a mix of the new
>> LLM power of Gemini AND the freshness and factuality of old PageRank Google
>> - to provide a mix of an answer, PPC ads and organic Search links.

>>

>> I'd also shared three Organic Search challenges:
>>
>> *O1.* Change in methodology (tech powering Search).
>> *O2*. Change to the UX (the purpose of Search).
>> *O3*. Change in User Behavior.
>>
>> *[*There was a fourth hidden one, but let me make it explicit: *O4*. Data
>> that provides visibility about what's going on, what does the *intelligent

>>
>> We don't know enough about O1 apart from surface-level stuff, *it's
>> using LLMs *or *it is using GenAI*. I'm going to leave that alone today

>>
>> AI Search presents a potential grave threat to your company's Organic
>> traffic > revenue > profits. It is optimal, for now, to worry and focus on
>> O2 and O3.
>> *O2: Change to the UX | The Switch from "Results" to "Answer."*
>>
>> If you pause and think about it, for 20 years, little has changed in
>> terms of *what *Search Engines provide.
>>
>> You and I type in keywords, and we are delivered a combination of three
>> to four ads and, the famous, *ten blue links.*
>>
>> Yes, the *one box *would often appear on the right side with a direct
>> answer. Sometimes, questions you can click on appear under the fist Organic
>> result. But. The underlying purpose of the Engine remained:
>>
>> *Here are the most relevant other places for your query, please go there.*
>>
>> In an underappreciated irony, *searching *for Users started after they
>> had done the search and received the results.
>>
>> This is not to imply that Search Engines have been twiddling their thumbs
>> for 20 years. No. Every single month, Google, Yandex, Seznam improved the
>> quality of the Search results with a focus on relevance. Their goal was for
>> us to click less back and forth.
>>
>> I'm simply saying, the foundational business was the same:* Provide
>> Search Results.*
>>
>> Companies pushing the edge on AI Search are in a different business: *Provide
>> an answer.*
>>
>> Not answer*s*. Answer. One.
>>
>> Their goal: Eliminate the need for *searching. *(The back and forth *ten
>> blue links - *AKA you figure it out).
>>
>> Perplexity
>>
>> is at the forefront of this *answer *model.
>>
>> The other day, *on LinkedIn*
>> ,
>> on my post on AI Search, someone shared a query they had just done: "*What
>> is the best way to get from Bristol to Clarkson's Farm*" As a non-native
>> to the UK, I did not understand what this was.
>>
>> This real query turns out to be a great way to illustrate challenge O2,
>> and the foundational shift in Search, from *results *to *answer.  *Old
>> Bing on the left, Perplexity on the right...
>> [All images are high resolution. Right-mouse click, choose Open Image in
>> a New Tab, review all the details.]
>>
>> How bad is the old *results* Search model!
>>
>> Especially contrasted with Perplexity's *answer *model!!
>>
>> They actually had so much space left after giving the answer, they
>> answered questions I would have *later*: When to Visit, What to Expect,
>> etc.
>>
>> Irony:  Perplexity has some custom LLMs, but it is mostly using other
>> people’s LLMs (!). Simply building a new User Experience is such a lovely
>> way to be valued at $3 billion dollars. :) I sincerely congratulate the
>> team at Perplexity.
>>
>> I’m getting side tracked.
>>
>> Back to the story.
>>
>> I'll plant a worry:
>>
>> What happens to your company if all Engines switch to AI Search's
>> *answer *model?
>>
>> The *answer *model does not always mean just one result from one
>> company. The UX can take different shape based on expressed intent.
>>
>> As an example, if someone did an *old school keywords *search without
>> clear intent, you might see *answers*, but in a different experience...
>> Searching for *best cross body bag *in the old model gives me what I
>> expected: Around 10 ads at the top (which I know is not the literal answer
>> to my query *best*), and, above the fold, five or seven more links to do
>> my *searching.*
>>
>> In the *answer *model, I get answers. Ranked. My *searching *could be
>> done on the results page, I can accept the first recommendation, or scroll
>> and read the others. This is the important part: There is one answer at the
>> top, and I could be done. [Even in scenarios of *zero-click searches, *common
>> on Google today, this query would not end on Google with *zero-clicks.*]
>>
>> If Search Engines are switching from *results *to *answer*…. What are
>> the chances that you’ll be *the **answer*?
>>
>> Worrying, no?
>>
>> We can’t “hack” with “10 Proven SEO Shortcuts,” get to the fourth Organic
>> link (for a few days/weeks) and pray some Users will come to us.
>>
>> What now?
>>
>> Action: I suppose we will just have to build really great products and
>> services that people love and talk about. That’ll result in the LLMs
>> choosing us as the *answer*.
>>
>> Action: For my clients, we are also solving for the *talk about *part
>> with specific tactics based on the *social built-in
>> *
>>  philosophy, and solving for *content *through the full lens of See AND
>> Think AND Do AND Care (vs. the common tactic of just the Do lens).
>>
>> Action: Of course, also wait and see how Paid Search options evolve and
>> leverage them (aggressively if we won't do the two actions above - both
>> hard).
>>
>> There are other search results page problems to deal with.
>>
>> In early iterations of AI Overviews, Google has put the AI Overviews
>> answer first, then a PPC ad. Often, it is AI Overview, then some Organic
>> answer, then, below the fold (!) PPC ads. [See TMAI #412.]
>>
>> I so, so, so, worry that numerous companies will lose lots of their PPC
>> traffic at the same time an *answer *model will cause them to
>> substantially lose their Organic traffic.
>>
>> We can't predict how fast this model will shift, only Google, Bing,
>> Seznam know. My belief is that it is coming, and coming soon.
>>
>> *Note*: For those of you who hate getting an *answer *and want to do all
>> the *searching *yourself because you'll *never trust Google*... I
>> believe you'll likely always have that choice, especially if you are not a
>> cohort of just 50k people. My prediction is that the trust model in place
>> today will change profoundly in the next five years, and shift to the
>> intelligent machine. If you are a business relying on most of the Users to
>> be *give me ten links, I'll help myself because I don't trust Google... *I
>> worry you'll likely be querying Chapter 11.
>> *O3. Change in User Behavior. | The Switch to True Intelligence.*
>>
>> If you pause and think about it, for 20 years little has changed about *how
>> *humans search.
>>
>> We type in two or five or six *keywords* into a box, the Search Engine
>> delivers a page full of links.
>>
>> The base assumption since Alta Vista, for me, for nearly all of us, has
>> been:
>>
>> *There’s a kind of smart thing at the other end.*
>>
>> We learned clues to make the two or five or six keywords better, to make
>> the smart thing give us relevant Results.
>>
>> [True story: In Verona last week, my wife gets on her phone to find us a
>> good restaurant, but after a few minutes nothing popped. I then tried and
>> got the perfect restaurant in 30 seconds. The difference? She typed in
>> “restaurants,” and I’d learned that Google Maps works better when you type
>> in “restaurant.” How insane? Oh, and she had just typed “res” and
>> “restaurants” was the first suggestion! I’d learned to ignore it. That, in
>> a nutshell, is how humans learned to work with *a kind of smart thing at
>> the other end*.*]*
>>
>> Search Engines *have *gotten better and better at *interpretability*: *What
>> might the human actually mean with these two or four or six words?*
>>
>> Outcome: *A.* Humans stayed rudimentary (keyword-land). *B.* Engines
>> improved at interpretation, delivering ever more relevant *ten blue
>> links.*
>>
>> With AI Search, something deep is changing regarding A and B.
>>
>> AI Search is being powered by something far, far, far more intelligent
>> than the “PageRank Tech.” It is intelligent LLMs for now (and in a few
>> years AGI).
>>
>> Realizing that, the small number of humans using AI Search are building a
>> new habit:
>>
>> Unshackling themselves from the *keyword keyword keyword keyword *yoke,
>> and asking more complex, multi-layered, *end point *questions.
>> *[Note:* I’m not implying Old Search lacks intelligence. It does. I'm
>> using the word *interpretability *to sharpen the difference with AI
>> Search, where the multi-modal *intelligence *at play is a whole new ball
>> game.*]*
>>
>> As the right side of the picture above goes from *interpretability *to
>> *intelligence*, my behavior as a human changes in fundamental ways. And,
>> it presents a small threat to you or HP or Michael Kors or Nespresso
>> today, and a significant threat ahead.
>>
>> Let me illustrate using an example.
>>
>> Instead of my typical *kw kw kw kw *query *best leather cross body bag*,
>> which would yield this very normal Old Search set of results with relevant *ten
>> blue links *telling me to go help myself…
>> I know that with AI Search there is an *intelligent **agent* at the
>> other end. So, I can throw at it the real problem I need solved, a present
>> for my daughter. I type something far more complex:
>>
>> *What’s the best cross-body bag for a Masters student at NYU, with an
>> understated sense of style?*
>>
>> I expect the *intelligent **agent *to understand the complex clues in
>> “Masters student,” in “NYU,” and in “an understated sense of style.”
>> [Ponder that last one.]
>>
>> It does, giving me an *answer *vs. *search results...*
>> BOOM!
>>
>> An answer. One.
>>
>> New user behavior, using new search, powered by *intelligence.*
>>
>> I'll plant a worry:
>>
>> If you were showing up in the *ten blue links *plus *four or ten PPC
>> ads... *What happens to your traffic > revenue > profits now?
>>
>> Bing CoPilot has also been on the innovation train over the last year
>> (and hence my default Search Engine). Their UX is on a slightly different
>> path compared to Perplexity. But, Bing is also firmly in the *answer *operating
>> model...
>> Unlike in the *results *operating model. I search, and then there is no
>> more *searching *by me. There's an answer, an *intelligent *answer.
>>
>> I would have never typed my more complex real question into Old Search.
>> Because something built for *interpretability* cannot understand what
>> I'm saying, and will fail me.
>>
>> See below, see above...
>> See what I mean? Compare it to how Bing or Perplexity handled my complex
>> question.
>>
>> Does this mean Google sucks? No, no, no. It is working exactly as
>> designed. It is built for *keyword keyword keyword keyword, *and uses
>> *interpretability *to provide *results *so that I can continue
>> *searching*. It is working exactly as designed for that human.
>>
>> This also explains why humanity's default for searching is *keyword
>> keyword keyword keyword.*
>>
>> It simply won't be true, slowly, from now on. Humans will understand
>> there's *true intelligence *at the other end and change their behavior -
>> which is good for them and a threat to your profits.
>>
>> *IMPORTANT*: If I use the *intelligent *version of Google vs. the
>> *interpretability *version of Google, I get search results similar to
>> Bing CoPilot. If I use Old Bing for my complex query, I get results very
>> similar to the sub-optimal ones as Old Google immediately above.
>>
>> In my experience, the *intelligent *version of Google (Gemini) is still
>> early in evolution. At the moment it often delivers results of inferior
>> quality...
>> It would be a stretch for anyone to consider the Bellroy as a *cross-body
>> bag, *it is a waist pouch. Also, the "eco conscious," "organized,"
>> "stylish" feel odd and irrelevant, when I was clear my priority was *an
>> understated sense of style*.
>>
>> Gemini is new. You can count on the sun to rise tomorrow, with just as
>> much certainty you can count on Gemini to become better and better and
>> better.
>>
>> When I use AI Search, there’s more expectation re the complexity stuffed
>> into my questions.
>>
>> With AI Search, there’s a mental reset.
>>
>> I really want you to get that.
>>
>> Stay with me.
>>
>> I anticipate humans will not go to Search and type more keywords to do
>> *searching. *They will go to Search, type in complex *end-point *questions,
>> and expect an *answer**.*
>>
>> The evolution will NOT be from *oh, I'm typing some keywords *to *let me
>> type more keywords.*
>>
>> So. No more *best running shoes for high arches*.
>>
>> Not even this version of that Search to keep on *searching*...
>> Because that only leads to this... More *searching*...
>> Instead of *how to pick, *realizing that they have an intelligent
>> *agent *at the other end, they can simply do an *end-point *search:
>>
>> *which is the best men's running shoe for beginners with high arches*
>>
>> This shift from *how *to *which* is key to internalize, because for the
>> user that leads to the end of *search *and to an *end-point...*
>> The above question's result and experience is even better on Perplexity.
>> Try it.
>>
>> Prediction to worry you: Today, we may trust word of mouth, or we may
>> trust ourselves to come to an answer after 8 minutes or 8 hours of
>> research. As *intelligence *improves (LLMs > AGI), I believe there will
>> be a *transfer of trust *(to the *intelligent agent*). Because, it is so
>> good already, and will be mind-blowingly so soon enough.
>>
>> Ponder that for the longer term, re implications.
>>
>> Back to today. In the short-term, this *transfer of trust *manifest
>> itself in even more *end-point *search queries, and an increased
>> preference for an *answer *vs. *results.*
>>
>> I cannot emphasize this enough:
>>
>> This is good for Users. I've been So Excited about the better that's
>> already here.
>>
>> This is also a profound challenge for companies - you see there are
>> 17,000 shoe companies, how do you get into the top result (Perplexity
>> model) or the top four (Bing CoPilot, Gemini, model)?
