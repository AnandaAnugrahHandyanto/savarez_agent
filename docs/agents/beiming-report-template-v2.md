# Beiming Report Template v2 / 北冥汇报模板 v2

> 定位：这里定义前线 CLI 执行体向北冥与参谋部回报结果时的统一格式。
> 它不是战役说明书，也不是执行协议。
> 它只回答：一次任务完成后，最低必须汇报哪些物理证据、状态字段与路径写法。

## 1. 强制总则

未来所有 CLI 端任务完成回报，必须至少包含以下 11 个顶层字段，且不得改名、缺省或重排：
- `Session ID`
- `模式`
- `阶段`
- `worktree`
- `任务名称`
- `任务结论`
- `法典状态`
- `关键证据`
- `关键文件清单`
- `Git 状态`
- `下一步唯一建议`

同时必须遵守以下硬规则：
- 若执行体运行时可直接读取会话标识，则必须填写真实 `Session ID`
- 若执行体运行时无法读取会话标识，则必须明确写为：`Session ID: 未暴露（需北冥通过 /status 补充）`
- 不得编造 Session ID，不得留空不解释
- 仓库内文件一律使用仓库相对路径
- 不得混用 `.hermes/hermes-agent/...` 这类仓库内投影式路径
- Skill 或系统文件若在仓库外，可使用绝对路径，但必须明确标注为“仓库外”

## 2. 稳定输出格式

```text
【会话信息】
- Session ID: <真实值 | 未暴露（需北冥通过 /status 补充）>
- 模式: <必须始终输出真实值>
- 阶段: <必须始终输出真实值>
- worktree: <必须始终输出真实值；未使用独立 worktree 时应如实写“主仓（非独立 worktree）”或等价真实描述>

【任务名称】
- 任务名称: <必须始终输出真实值>

【任务结论】
- 任务结论: <已完成 / 可收口 / 未完成 / 打回 / APPROVE_CANDIDATE / REJECT_HARD / FAKE_WIN>

【法典状态】
- 红线是否被削弱: <必须始终输出真实判断>
- 是否触发 M6: <必须始终输出真实判断>
- 是否需要北冥额外审批: <必须始终输出真实判断>
- 没有北冥签字前是否保持未合并: <必须始终输出真实判断>

【关键证据】
- 结案报告: <仓库内相对路径 | 仓库外绝对路径并标注仓库外 | 未产出>
- Status Ledger: <仓库内相对路径>
- Verification Chain: <仓库内相对路径>
- Tech Debt Tracker: <仓库内相对路径>
- 测试结果: <真实结果 | 未执行>

【关键文件清单】
- 代码文件: <仓库内相对路径列表 | 无>
- 测试文件: <仓库内相对路径列表 | 无>
- 文档文件: <仓库内相对路径列表 | 无>

【Git 状态】
- 是否已 git add: <必须始终输出真实状态>
- 是否已 commit: <必须始终输出真实状态>
- 是否已 merge: <必须始终输出真实状态>
- 是否已 push: <必须始终输出真实状态>
- 当前分支: <真实值 | 未暴露（运行时未提供 Git 分支信息）>
- 当前候选提交（如有）: <真实值 | 无 | 未暴露（运行时未提供候选提交信息）>
- 说明: <必须始终输出真实说明>

【下一步唯一建议】
- <只能给 1 条，不得给选择题>
```

## 3. “未暴露”使用边界

以下字段必须始终输出，但仅在“运行时确实无法读取”的前提下允许写“未暴露”：
- `Session ID`：且只能写固定句式 `未暴露（需北冥通过 /status 补充）`
- `Git 状态 / 当前分支`：可写 `未暴露（运行时未提供 Git 分支信息）`
- `Git 状态 / 当前候选提交（如有）`：可写 `未暴露（运行时未提供候选提交信息）`

除上述情况外，其余字段不应写“未暴露”，而应按事实写：
- 有真实值就写真实值
- 没有该对象就写 `无`
- 本轮未运行验证就写 `未执行`
- 本轮尚未形成报告就写 `未产出`
- 未使用独立 worktree 时就写真实执行形态，而不是写“未暴露”

## 4. 拦截原则

- 若执行体可读取会话标识却未填写 `Session ID` → 至少 `WARN`
- 若执行体无法读取会话标识，但未明确写出“`Session ID: 未暴露（需北冥通过 /status 补充）`” → 至少 `WARN`
- 若执行体编造 Session ID 或留空不解释 → `REJECT_HARD`
- 缺 `worktree` / `模式` / `阶段` → 至少 `WARN`
- 若把仓库内文件写成 `.hermes/hermes-agent/...` 等投影式路径 → 至少 `WARN`
- 若把“未暴露”滥用于本应写真实值、`无`、`未执行`、`未产出` 的字段 → 至少 `WARN`
- 主线收编类任务缺 `Git 状态` → `REJECT_HARD`
- 缺 `下一步唯一建议` → 至少 `WARN`
- 给多个下一步选项而不是唯一建议 → 主线收编模式下可升级为 `REJECT_HARD`

## 5. 使用边界

- 这里不替代具体任务合同
- 这里不替代 `safe-refactor-loop` 的状态机
- 这里不自动定义任务是否通过
- 它只定义：前线回报给北冥时，最低必须带回哪些字段、路径写法与兜底口径
