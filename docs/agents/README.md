# Agent Docs README / 二级导航索引

## 1. 本目录职责
- 这里只做 Agent 二级导航，不替代代码、规格、迁移文档。
- 根 `AGENTS.md` 是总地图；这里是按任务下钻的第二层入口。
- 本目录固定包含 4 份文档，不扩展为资料仓。

| 文件 | 只负责什么 | 不负责什么 |
|---|---|---|
| `README.md` | 二级导航、阅读顺序、分流 | 具体实现细节 |
| `architecture.md` | 系统结构、关键入口、依赖链 | 操作步骤、交付纪律 |
| `workflows.md` | 任务型入口、常见修改路径 | 风险总则、系统总图 |
| `guardrails.md` | 风险边界、验证纪律、交接纪律 | 结构说明、功能教程 |

## 2. 推荐阅读顺序
1. 先读根 `AGENTS.md`，确认本轮任务属于哪一类。
2. 再读本文，确定该下钻到哪份二级文档。
3. 然后只进入当前任务所需文档：
   - 想知道系统结构 → `architecture.md`
   - 想知道这类改动从哪里入手 → `workflows.md`
   - 想知道哪些不能碰、怎么验证、如何交接 → `guardrails.md`

## 3. 按任务分流
### 3.1 理解系统结构
进入：`architecture.md`

适用问题：
- `AIAgent` 主循环从哪里进？
- 工具 schema / 分发链路在哪里？
- CLI、gateway、ACP、tests 的入口分别在哪？
- 改某个目录前，应该先读哪几个文件？

### 3.2 做任务改动
进入：`workflows.md`

适用问题：
- 新增 Tool
- 新增 slash command
- 新增配置项或 `.env` 变量
- 调整测试入口
- 需要按批次 0 骨架推进任务

### 3.3 做风险控制与交付
进入：`guardrails.md`

适用问题：
- 哪些路径必须 profile-safe
- 哪些改动会破坏 prompt caching
- 测试为什么不能写 `~/.hermes`
- 修改完成后要怎样给出验证证据
- 交接给下一个 Agent 时必须写什么

## 4. 与仓库其他文档的关系
- `docs/exec-plans/`：执行现场文档；记录合同、台账、验证链、验收报告，不替代本目录。
- `docs/specs/`：需求/方案规格；当任务绑定具体规格时读取。
- `docs/migration/`：迁移、兼容、切换说明；当任务涉及升级或迁移时读取。

## 5. 进入前的最小判断
开始任务前，至少先回答：
- 我是在**读结构**、**做改动**，还是**控风险/交付**？
- 我的任务合同是什么？
- 我要提交的证明是什么？
- 当前现场停点和下一手入口是什么？

如果答不出，先回到根 `AGENTS.md`，不要直接进代码海里游泳。

## 6. 本目录不做的事
- 不复制旧版 470 行 Development Guide。
- 不引入 Hook、自动触发器、批次 1 之外能力。
- 不新增计划外目录或“以后再填”的空壳文档。
